# 安全策略

感谢你帮助改进 `鹦鹉剪贴板（Parrot Clipboard）` 的安全性。

## 支持范围

当前优先维护的公开版本基线为：

- `main` 分支最新代码
- `docs/v1.4` 对应的当前开源版本基线

较早期版本文档与历史分支默认不承诺持续提供安全修复。

## 如何报告安全问题

如果你发现了可能导致以下风险的问题，请不要直接公开披露完整利用细节：

- 任意代码执行
- 权限绕过
- 本地敏感数据泄露
- 更新链路被劫持
- 文件系统越权访问

建议的披露顺序：

1. 优先使用 GitHub 的私密安全报告能力（Security Advisory / private vulnerability report）
2. 如果仓库尚未启用该功能，请先通过 GitHub Discussion 或 Issue 简要联系维护者，说明“存在安全问题需要私下沟通”
3. 在获得私下沟通渠道前，不要在公开 Issue 中直接贴出可复现攻击步骤、PoC、敏感路径或利用脚本

## 报告内容建议

为了帮助快速定位，请尽量提供：

- 影响版本或提交哈希
- 受影响平台（macOS / Windows / Linux）
- 风险级别与影响范围
- 最小复现步骤
- 可能的修复建议或缓解方式

## 响应目标

当前项目为个人维护仓库，不承诺企业级 SLA，但会尽量按以下节奏处理：

- 7 天内完成首次响应
- 14 天内确认问题有效性与影响范围
- 对高风险问题优先安排修复与发布说明
