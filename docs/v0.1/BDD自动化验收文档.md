# v0.1 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档以**行为驱动开发（BDD）**格式描述 **v0.1 MVP 版本**的验收测试场景，与《用户故事文档》中的验收标准（AC）一一对应。

v0.1 BDD 验收采用**手动验收 + 关键路径自动化**的混合策略：
- 核心用户流程（复制→调出→选择→粘贴）以 Playwright E2E 形式描述用例（但 v0.1 暂不自动运行，作为 v0.2 E2E 基础）
- 单元级验收通过《自动化测试文档》中的测试用例覆盖

### 1.2 BDD 格式规范

使用 Gherkin 语法描述场景：

```gherkin
功能（Feature）：功能模块名称
  场景（Scenario）：具体场景描述
    假设（Given）：前置条件
    当（When）：用户执行的操作
    那么（Then）：期望的系统行为
    并且（And）：补充期望行为
```

## 2. 功能：粘贴板自动监听（Feature: Clipboard Monitoring）

### 场景 BDD-01-01：复制文本后自动记录

```gherkin
功能：粘贴板自动监听

  场景：复制普通文本后应自动添加记录
    假设 应用正在 macOS 后台运行
    当 用户在任意应用中复制文本 "Hello, 粘贴板记录工具！"
    那么 在 500ms 内，主面板的历史列表中应出现包含该文本的卡片
    并且 该卡片显示在列表的最左侧（最新位置）
    并且 卡片中显示的时间戳在当前时刻的 2 秒以内
```

### 场景 BDD-01-02：连续复制多条内容均被记录

```gherkin
  场景：连续复制多条不同内容，每条均被记录
    假设 应用正在后台运行
    当 用户依次复制 "内容 A"、"内容 B"、"内容 C"
    那么 历史列表中应有 3 条记录
    并且 "内容 C" 在最左侧
    并且 "内容 A" 在最右侧
```

### 场景 BDD-01-03：重复复制相同内容不产生重复记录

```gherkin
  场景：复制与最新记录相同的内容时不新增
    假设 历史列表最新记录的内容为 "重复内容"
    当 用户再次复制 "重复内容"
    那么 历史列表的记录数量应保持不变
    并且 "重复内容" 仍然只有一条记录

  场景：复制与非最新记录相同的内容时复用并置顶
    假设 历史列表从左到右依次为 "内容 C"、"内容 B"、"内容 A"
    当 用户再次复制 "内容 A"
    那么 历史列表的记录数量应保持不变
    并且 历史列表从左到右依次为 "内容 A"、"内容 C"、"内容 B"
```

### 场景 BDD-01-04：记录数量超过上限时自动清理最旧记录

```gherkin
  场景：记录数超过 20 条时自动移除最旧记录
    假设 历史列表已有 20 条记录（最旧记录内容为 "最旧内容"）
    当 用户复制新内容 "第 21 条内容"
    那么 历史列表仍然只有 20 条记录
    并且 "第 21 条内容" 出现在最左侧
    并且 "最旧内容" 从列表中消失
```

## 3. 功能：快捷键调出面板（Feature: Shortcut Key Panel)

### 场景 BDD-02-01：默认快捷键调出面板

```gherkin
功能：快捷键调出主面板

  场景：按下 Shift+Command+V 后面板从底部弹出
    假设 应用正在后台运行
    并且 主面板当前不可见
    当 用户按下 Shift + Command + V
    那么 主面板应在 100ms 内从屏幕底部出现
    并且 面板宽度占满屏幕宽度
    并且 最新的一条记录处于选中（高亮）状态
    并且 面板获得键盘焦点
```

### 场景 BDD-02-02：面板已显示时再次按下快捷键收起面板

```gherkin
  场景：面板可见时再次按快捷键应收起面板
    假设 主面板当前可见
    当 用户按下 Shift + Command + V
    那么 主面板应在 300ms 内收起不可见
```

### 场景 BDD-02-03：无历史记录时调出面板显示空状态

```gherkin
  场景：无历史记录时调出面板展示空状态提示
    假设 历史记录列表为空
    当 用户按下 Shift + Command + V
    那么 主面板应弹出
    并且 卡片区域显示「还没有复制记录」的空状态提示
```

## 4. 功能：历史记录浏览（Feature: Record Browsing)

### 场景 BDD-03-01：方向键切换选中记录

```gherkin
功能：历史记录浏览与卡片切换

  场景：按 → 键选中下一条记录
    假设 面板已展示至少 3 条记录
    并且 第 1 条记录（最左/最新）处于选中状态
    当 用户按下 →
    那么 第 2 条记录应处于选中（高亮）状态
    并且 第 1 条记录失去高亮
    并且 切换响应时间 < 50ms

  场景：按 ← 键选中上一条记录
    假设 面板已展示至少 3 条记录
    并且 第 2 条记录处于选中状态
    当 用户按下 ←
    那么 第 1 条记录应处于选中状态

  场景：在第 1 条按 ← 时不越界
    假设 面板已展示至少 2 条记录
    并且 第 1 条记录（最左）处于选中状态
    当 用户按下 ←
    那么 选中状态不变，仍然是第 1 条记录

  场景：在最后一条按 → 时不越界
    假设 面板已展示 3 条记录
    并且 第 3 条记录（最右/最旧）处于选中状态
    当 用户按下 →
    那么 选中状态不变，仍然是第 3 条记录
```

## 5. 功能：粘贴操作（Feature: Paste Operation)

### 场景 BDD-04-01：Enter 键粘贴并关闭面板

```gherkin
功能：粘贴选中记录内容

  场景：按 Enter 将选中记录内容粘贴到当前应用
    假设 面板已展示若干记录
    并且 第 1 条记录（内容为 "粘贴测试内容"）处于选中状态
    并且 用户在调出面板前的活动应用为文本编辑器（光标在文本中）
    当 用户按下 Enter
    那么 "粘贴测试内容" 应出现在文本编辑器的光标位置
    并且 面板应在按键后 200ms 内自动关闭
    并且 历史列表总数保持不变
    并且 刚刚粘贴的记录移动到列表最左侧

  场景：粘贴操作响应时间达标
    假设 面板已展示记录，且有记录处于选中状态
    当 用户按下 Enter
    那么 从按键到内容出现在目标应用的总耗时 < 200ms
```

## 6. 功能：记录删除（Feature: Record Deletion)

### 场景 BDD-05-01：Delete 键删除选中记录

```gherkin
功能：删除历史记录

  场景：按 Delete 键删除选中记录，右侧记录自动成为选中项
    假设 面板展示 3 条记录
    并且 第 1 条记录处于选中状态
    当 用户按下 Delete
    那么 第 1 条记录从列表中消失
    并且 原来的第 2 条记录变为选中状态（自动聚焦）
    并且 删除响应时间 < 50ms

  场景：按 Backspace 键也能删除选中记录
    假设 面板展示至少 1 条记录，且有记录处于选中状态
    当 用户按下 Backspace
    那么 选中记录从列表中消失

  场景：删除最后一条记录后显示空状态
    假设 面板仅剩 1 条记录，且该记录处于选中状态
    当 用户按下 Delete
    那么 列表变为空
    并且 空状态视图出现

  场景：删除位于最右侧的记录，左侧记录成为选中项
    假设 面板展示 3 条记录
    并且 第 3 条记录（最右）处于选中状态
    当 用户按下 Delete
    那么 第 3 条记录消失
    并且 第 2 条记录（现在变为最右）成为选中状态
```

## 7. 功能：关闭面板（Feature: Close Panel)

### 场景 BDD-06-01：Esc 键关闭面板

```gherkin
功能：关闭主面板

  场景：按 Esc 键收起面板，不执行粘贴
    假设 主面板当前可见，有记录处于选中状态
    当 用户按下 Esc
    那么 面板应在 300ms 内收起不可见
    并且 系统粘贴板内容不发生变化
    并且 目标应用未收到 Cmd+V 模拟事件

  场景：点击面板外区域关闭面板
    假设 主面板当前可见
    当 用户点击面板区域以外的屏幕区域
    那么 面板自动收起
    并且 行为与按 Esc 键一致
```

## 8. 非功能需求验收场景（Performance BDD)

### 场景 BDD-NFR-01：快捷键响应时间

```gherkin
功能：性能指标验收

  场景：快捷键响应时间 < 100ms
    假设 应用在后台运行，历史列表有 20 条记录
    当 用户按下调出快捷键
    那么 从按键到面板可见的时间应 < 100ms

  场景：内存占用在空闲时 < 80MB
    假设 应用在后台运行，面板未显示，未有用户操作
    那么 应用进程的内存占用（RSS）应 < 80MB

  场景：卡片切换响应时间 < 50ms
    假设 面板已展示 20 条记录
    当 用户按下方向键切换选中记录
    那么 高亮状态变化应在 50ms 内完成
```

## 9. 验收通过标准（DoD - Definition of Done）

| 类别 | 通过标准 |
|------|----------|
| 功能完整性 | 所有 BDD 场景（BDD-01 ~ BDD-06）人工验收通过 |
| 性能指标 | 快捷键响应 < 100ms，内存 < 80MB，切换响应 < 50ms |
| 核心流程 | 复制 → 调出 → 选择 → 粘贴 全流程无阻断执行 |
| 稳定性 | 连续运行 2 小时无崩溃 |
| 单元测试 | 后端覆盖率 ≥ 80%，前端覆盖率 ≥ 75%，所有测试全绿 |
| 平台限制 | 仅在 macOS 13+ 设备上验收（v0.1 单平台） |

## 10. 手动验收检查清单

在每次 v0.1 版本发布前，逐项完成以下手动验证：

### 粘贴板监听

- [ ] 在 Safari 中复制文本，面板中出现对应记录
- [ ] 在 Terminal 中复制命令，面板中出现对应记录
- [ ] 在 VSCode 中复制代码，面板中出现对应记录
- [ ] 复制同一内容两次，列表不新增重复条目
- [ ] 复制第 21 条内容后，列表仍为 20 条，最旧一条消失

### 快捷键与面板

- [ ] 按 Shift+Command+V，面板从底部弹出，响应迅速
- [ ] 面板宽度占满屏幕，位于底部
- [ ] 面板置顶，覆盖其他应用窗口
- [ ] 最新记录默认高亮
- [ ] 再次按快捷键，面板收起

### 键盘操作

- [ ] 按 → 切换至下一条，高亮移动
- [ ] 按 ← 切换至上一条，高亮移动
- [ ] 到达边界时按方向键无越界
- [ ] 按 Enter，内容粘贴到之前的活动应用，面板自动关闭
- [ ] 按 Delete，选中记录消失，焦点移至下一条
- [ ] 按 Backspace，行为同 Delete
- [ ] 按 Esc，面板收起，不执行粘贴

### 边界情况

- [ ] 历史为空时调出面板，显示空状态提示
- [ ] 删除最后一条记录，显示空状态
- [ ] 应用重启后，历史记录清空（v0.1 无持久化）
