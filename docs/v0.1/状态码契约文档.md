# v0.1 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.1 MVP 版本**中所有 IPC Command 可能返回的错误状态码、错误消息格式及前端处理规范。

### 1.2 错误传递机制

Tauri IPC 错误传递流程：

```
后端 Rust 函数返回 Err(AppError)
    │ serde::Serialize 序列化为 JSON
    ▼
Tauri IPC 层将 Err 包装为 rejected Promise
    │
    ▼
前端 invoke() 调用 catch 块捕获
    │ 解析 error 对象
    ▼
M12/M13 错误处理层 → Toast 提示 / 静默处理
```

## 2. 错误对象格式

### 2.1 后端错误结构体

```rust
/// 统一应用错误类型
/// 实现 serde::Serialize 以通过 IPC 传输
#[derive(Debug, thiserror::Error, serde::Serialize)]
#[serde(tag = "code", content = "message")]
pub enum AppError {
    #[error("Record not found: {0}")]
    RecordNotFound(u64),

    #[error("Invalid parameter: {0}")]
    InvalidParam(String),

    #[error("Clipboard write error: {0}")]
    ClipboardWriteError(String),

    #[error("Key simulation error: {0}")]
    KeySimulationError(String),

    #[error("Window manager error: {0}")]
    WindowError(String),

    #[error("Internal error: {0}")]
    Internal(String),
}
```

### 2.2 前端接收到的错误 JSON 格式

```json
{
  "code": "RECORD_NOT_FOUND",
  "message": "Record with id 42 not found"
}
```

### 2.3 前端 TypeScript 错误类型

```typescript
// src/api/types.ts

/** IPC 错误类型 */
interface IpcError {
  code: ErrorCode;
  message: string;
}

/** 所有可能的错误码 */
type ErrorCode =
  | 'RECORD_NOT_FOUND'
  | 'INVALID_PARAM'
  | 'CLIPBOARD_WRITE_ERROR'
  | 'KEY_SIM_ERROR'
  | 'WINDOW_ERROR'
  | 'INTERNAL';
```

## 3. 错误码详细说明

### SC-001：RECORD_NOT_FOUND（记录不存在）

| 属性 | 值 |
|------|-----|
| 错误码 | `RECORD_NOT_FOUND` |
| HTTP 语义对应 | 404 Not Found |
| 触发 Command | `delete_record`、`paste_record` |
| 触发条件 | 传入的 `id` 在内存存储中不存在 |

**错误消息格式**：

```
"Record with id {id} not found"
```

**示例**：

```json
{ "code": "RECORD_NOT_FOUND", "message": "Record with id 7 not found" }
```

**前端处理规范**：

| 场景 | 处理方式 |
|------|----------|
| `delete_record` 返回此错误 | 静默处理（记录可能已因超限被自动移除），同步移除前端列表中对应项 |
| `paste_record` 返回此错误 | 显示 Toast 提示「记录已不存在，请重新选择」|

---

### SC-002：INVALID_PARAM（参数非法）

| 属性 | 值 |
|------|-----|
| 错误码 | `INVALID_PARAM` |
| HTTP 语义对应 | 400 Bad Request |
| 触发 Command | `get_records`、`paste_record` |
| 触发条件 | 参数值不在合法范围或格式不符 |

**常见触发情况**：

| Command | 触发条件 | 错误消息 |
|---------|----------|----------|
| `get_records` | `limit <= 0` | `"limit must be > 0"` |
| `get_records` | `limit > 1000` | `"limit must be <= 1000"` |
| `paste_record` | `mode` 不是 `"original"` | `"Unsupported paste mode: {mode}"` |

**示例**：

```json
{ "code": "INVALID_PARAM", "message": "Unsupported paste mode: plain_text" }
```

**前端处理规范**：

此类错误属于开发时编程错误（应通过 TypeScript 类型系统提前规避），生产中出现时：
- 控制台打印错误日志
- 不向用户展示（属于应用内部错误）

---

### SC-003：CLIPBOARD_WRITE_ERROR（粘贴板写入失败）

| 属性 | 值 |
|------|-----|
| 错误码 | `CLIPBOARD_WRITE_ERROR` |
| HTTP 语义对应 | 503 Service Unavailable |
| 触发 Command | `paste_record` |
| 触发条件 | 调用 `arboard` 写入系统粘贴板时发生 OS 级错误 |

**错误消息格式**：

```
"Failed to write to clipboard: {os_error}"
```

**示例**：

```json
{ "code": "CLIPBOARD_WRITE_ERROR", "message": "Failed to write to clipboard: Permission denied" }
```

**前端处理规范**：

- 显示 Toast 提示「粘贴失败，请重试」
- 确保面板不被隐藏（避免用户无感知粘贴失败）

---

### SC-004：KEY_SIM_ERROR（按键模拟失败）

| 属性 | 值 |
|------|-----|
| 错误码 | `KEY_SIM_ERROR` |
| HTTP 语义对应 | 503 Service Unavailable |
| 触发 Command | `paste_record` |
| 触发条件 | macOS CGEvent 按键模拟失败（常见于辅助功能权限未授予） |

**错误消息格式**：

```
"Failed to simulate paste key: {reason}"
```

**常见原因**：

| 原因 | 消息内容 |
|------|----------|
| 未授予辅助功能权限 | `"Accessibility permission not granted"` |
| CGEvent 创建失败 | `"CGEvent creation failed"` |

**前端处理规范**：

- 显示醒目 Toast 提示「自动粘贴失败，请手动按 Cmd+V 粘贴」
- 内容已写入粘贴板，用户可手动粘贴

---

### SC-005：WINDOW_ERROR（窗口管理失败）

| 属性 | 值 |
|------|-----|
| 错误码 | `WINDOW_ERROR` |
| HTTP 语义对应 | 500 Internal Server Error |
| 触发 Command | `hide_panel` |
| 触发条件 | Tauri 窗口操作失败 |

**前端处理规范**：

- 控制台打印错误日志
- 面板可能仍然可见，不影响主要功能

---

### SC-006：INTERNAL（内部错误）

| 属性 | 值 |
|------|-----|
| 错误码 | `INTERNAL` |
| HTTP 语义对应 | 500 Internal Server Error |
| 触发 Command | 所有 Command |
| 触发条件 | 未预期的运行时错误（锁毒死、panic recover 等） |

**前端处理规范**：

- 显示 Toast 提示「发生未知错误，请重启应用」
- 打印完整错误日志到控制台

## 5. 前端统一错误处理工具

```typescript
// src/api/errorHandler.ts

import type { IpcError } from './types';

/**
 * 解析 invoke 抛出的错误对象为 IpcError
 * Tauri 将 Err 序列化为字符串或对象，需统一处理
 */
export function parseIpcError(error: unknown): IpcError {
  if (typeof error === 'object' && error !== null && 'code' in error) {
    return error as IpcError;
  }
  if (typeof error === 'string') {
    try {
      return JSON.parse(error) as IpcError;
    } catch {
      return { code: 'INTERNAL', message: error };
    }
  }
  return { code: 'INTERNAL', message: String(error) };
}

/**
 * 根据错误码决定用户提示文案
 */
export function getErrorMessage(error: IpcError): string | null {
  switch (error.code) {
    case 'RECORD_NOT_FOUND':
      return '记录已不存在，请重新选择';
    case 'CLIPBOARD_WRITE_ERROR':
      return '粘贴失败，请重试';
    case 'KEY_SIM_ERROR':
      return '自动粘贴失败，请手动按 Cmd+V 粘贴';
    case 'WINDOW_ERROR':
      return null;  // 静默处理
    case 'INVALID_PARAM':
      return null;  // 开发时错误，不对外展示
    case 'INTERNAL':
      return '发生未知错误，请重启应用';
    default:
      return '操作失败，请重试';
  }
}
```
