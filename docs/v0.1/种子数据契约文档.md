# v0.1 种子数据契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.1 MVP 版本**的种子数据（Seed Data）规范，用于：

- **开发环境**：初始化本地开发时的测试数据，便于 UI 开发和调试
- **测试环境**：单元测试和集成测试中使用的标准测试夹具（Test Fixture）
- **验收演示**：手动验收时快速生成演示数据

### 1.2 v0.1 存储特性

v0.1 使用**纯内存存储**，不持久化。种子数据通过以下方式注入：

1. **测试中**：直接向 `InMemoryStore` 插入预设记录
2. **开发模式**：应用启动时通过特殊标志加载种子数据
3. **无配置文件**：v0.1 没有 `config.json`，不需要配置种子数据

### 2.2 SEED-002：单元测试夹具（Test Fixtures）

用于 Rust 单元测试的标准夹具工厂函数。

```rust
// src-tauri/src/clipboard/test_fixtures.rs

/// 创建一条标准文本测试记录
pub fn make_text_record(id: u64, text: &str, ago_ms: i64) -> ClipboardRecord {
    ClipboardRecord {
        id,
        content_type: "text".to_string(),
        text_content: text.to_string(),
        created_at: now_ms() - ago_ms,
    }
}

/// 预设的测试记录常量
pub mod fixtures {
    /// 普通短文本记录
    pub const SHORT_TEXT: &str = "Hello World";

    /// 中等长度文本记录
    pub const MEDIUM_TEXT: &str = "The quick brown fox jumps over the lazy dog.";

    /// 超长文本记录（超过 3 行展示限制）
    pub const LONG_TEXT: &str = "第一行内容：这是一段较长的测试文本\n\
                                  第二行内容：用于验证文本截断功能\n\
                                  第三行内容：此处是第三行\n\
                                  第四行内容：这一行应该被截断不显示";

    /// 包含特殊字符的文本
    pub const SPECIAL_CHARS: &str = "Hello < World > & \"Quotes\" 'Apostrophes'";

    /// 包含换行符的文本
    pub const MULTILINE: &str = "Line 1\nLine 2\nLine 3";

    /// 纯数字文本
    pub const NUMERIC: &str = "1234567890";

    /// 中文文本
    pub const CHINESE: &str = "你好，世界！这是中文测试文本。";

    /// 空格文本（边界情况）
    pub const WHITESPACE_ONLY: &str = "   ";

    /// 单字符文本（最小有效文本）
    pub const SINGLE_CHAR: &str = "A";
}

/// 创建包含 N 条记录的标准测试存储
pub fn make_store_with_n_records(n: usize) -> InMemoryStore {
    let mut store = InMemoryStore::new(20);
    for i in 0..n {
        store.insert(
            format!("测试记录 {}", i + 1),
            (i as i64) * 60_000,
        );
    }
    store
}

/// 创建满载（20条）的测试存储
pub fn make_full_store() -> InMemoryStore {
    make_store_with_n_records(20)
}

/// 创建仅 1 条记录的测试存储
pub fn make_single_record_store() -> InMemoryStore {
    make_store_with_n_records(1)
}

/// 创建空存储
pub fn make_empty_store() -> InMemoryStore {
    InMemoryStore::new(20)
}
```

## 3. 开发模式种子数据加载

### 3.1 后端加载逻辑

```rust
// src-tauri/src/lib.rs

#[cfg(debug_assertions)]
fn load_seed_data_if_dev(store: &Arc<RwLock<InMemoryStore>>) {
    let env_flag = std::env::var("CLIPBOARD_SEED").unwrap_or_default();
    if env_flag != "1" {
        return;
    }

    let mut store = store.write().unwrap();
    for seed in get_dev_seed_records() {
        store.insert(seed.text_content, seed.created_at);
    }

    tracing::info!("Dev seed data loaded: {} records", store.count());
}
```

### 3.2 启动命令

```bash
# 正常启动（无种子数据）
pnpm tauri dev

# 带种子数据启动（开发调试 UI 时使用）
CLIPBOARD_SEED=1 pnpm tauri dev
```

## 4. 种子数据覆盖的测试场景

| 种子数据 | 覆盖的测试场景 |
|----------|---------------|
| SHORT_TEXT（短文本） | 卡片基础渲染、字符数显示 |
| LONG_TEXT（长文本） | 文本截断（> 3 行）+ 省略号 |
| CHINESE（中文） | 字符计数（中文字符）、字体渲染 |
| SPECIAL_CHARS（特殊字符） | XSS 防护（HTML 转义）、特殊字符展示 |
| MULTILINE（多行） | 换行符保留、行数计算 |
| NUMERIC（纯数字） | 纯数字文本识别 |
| WHITESPACE_ONLY（空格） | 边界：仅空格内容是否被记录（应跳过） |
| SINGLE_CHAR（单字符） | 最小有效文本 |
| 5条记录列表 | 方向键切换、边界测试 |
| 20条记录列表 | 满载滚动、数量上限验证 |
| 空列表 | 空状态视图展示 |

## 5. 数据清理约定

| 场景 | 清理方式 |
|------|----------|
| 应用重启 | 内存自动清空（v0.1 无持久化） |
| 单元测试结束 | 每个测试用 `make_empty_store()` 或独立实例，互不干扰 |
| 前端测试结束 | Vitest 的 `afterEach(() => { vi.clearAllMocks(); })` |
| 开发模式重启 | 重新执行 `load_seed_data_if_dev`，数据重置为初始种子数据 |
