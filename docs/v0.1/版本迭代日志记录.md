### 2.5 2026-03-06 `worktree`

**提交信息**：`fix(paste): 修复粘贴后重复新增记录并改为复用原记录置顶`

**问题背景**：
- 当用户粘贴历史记录时，应用会先把目标文本写回系统粘贴板，监听恢复后又把这次写入当成新的复制内容捕获。
- 如果该文本原本就在列表中但不是最新项，会生成一条新的重复记录，导致卡片数量错误增加。

**修复内容**：
- 后端监听改为“全历史去重”：命中已有文本时复用原记录并移动到最左侧，不再创建新 ID。
- 粘贴流程在写入系统粘贴板后同步监听快照，避免恢复监听后把自写入内容重新采集。
- 粘贴成功后由后端显式将对应记录置顶，并通过 `paste_record` 返回最新记录数据给前端同步。
- 前端 Store 改为按“记录身份”维护选中项，保证记录被置顶后仍保持该记录为选中状态。

**验证结果**：
- `pnpm run test` 通过（33 tests）
- `pnpm run type-check` 通过
- `pnpm run lint` 通过
- `cargo test --manifest-path src-tauri/Cargo.toml` 通过（19 tests）
- `cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings` 通过

### 2.6 2026-03-06 `worktree`

**提交信息**：`fix(window): 修复 macOS 下主面板被 Dock 遮挡`

**问题背景**：
- 主面板当前虽然配置了 `alwaysOnTop`，但在 macOS 下只会提升到 `NSFloatingWindowLevel`，仍低于 Dock。
- 同时窗口贴底逻辑基于整块屏幕高度计算，导致底部 Dock 存在时，面板会落入被系统 UI 占用的区域。

**修复内容**：
- 主面板定位改为基于显示器 `work_area` 计算宽度、起点和底部位置，自动避开 Dock 与菜单栏占用区域。
- 窗口显示前重新设置 `alwaysOnTop`，并在 macOS 下开启 `visibleOnAllWorkspaces`。
- macOS 下将原生窗口层级提升到 `NSStatusWindowLevel`，同时追加“全屏辅助窗口”行为，确保面板高于 Dock 且更稳定地出现在最前。
- 为面板工作区定位补充单元测试，覆盖底部 Dock、侧边 Dock 与可用高度不足三类场景。

**验证结果**：
- `cargo test --manifest-path src-tauri/Cargo.toml` 通过（22 tests）
- `cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings` 通过
