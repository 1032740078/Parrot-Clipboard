### 2.3 2026-03-06 `63de182`

**提交信息**：`fix(monitor): 增加文本变化兜底检测以提升粘贴板监听稳定性`

**问题背景**：
- 部分环境下仅依赖 `NSPasteboard.changeCount` 可能导致复制文本后未及时入库。

**修复内容**：
- 在监听服务增加“双重触发条件”：
  - `change_count` 变化
  - 文本内容变化（兜底）
- 仅当两者都未变化时跳过本轮处理。

**验证结果**：
- 后端 `cargo test` 通过
- 后端 `cargo clippy -D warnings` 通过

---

### 2.4 2026-03-06 `worktree`

**提交信息**：`feat(logging): 增加前后端统一日志模块并补齐关键链路埋点`

**主要变更**：
- 后端新增 `logging.rs`：
  - 接入 `tracing + tracing-subscriber + tracing-appender`
  - 日志输出到控制台 + 文件（按天滚动）
  - 初始化时自动创建日志目录
  - 增加 panic hook，捕获 `location` 与 `backtrace`
- 后端 IPC 新增：
  - `write_client_log`（接收前端日志并统一落盘）
  - `get_log_directory`（用于获取当前日志目录）
- 后端关键链路接入结构化日志：
  - `clipboard/monitor`、`paste`、`shortcut`、`window`
  - `ipc/commands`、`ipc/events`
- 前端新增 `src/api/logger.ts`：
  - 支持 `debug/info/warn/error`
  - 错误标准化与上下文序列化
  - Tauri 环境自动上报后端日志
- 前端关键路径补齐日志：
  - IPC 命令调用失败日志
  - 事件订阅/注销异常日志
  - 键盘交互日志与失败兜底
  - 主面板初始化成功/失败日志
  - 全局 `error` / `unhandledrejection` 捕获
- 新增《日志模块说明文档》：`docs/v0.1/日志模块说明文档.md`

**验证结果**：
- `pnpm run lint` 通过
- `pnpm run type-check` 通过
- `pnpm run test` 通过（30 tests）
- `cargo test --manifest-path src-tauri/Cargo.toml` 通过（16 tests）
- `cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings` 通过
