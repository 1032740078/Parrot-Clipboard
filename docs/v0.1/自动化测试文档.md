# v0.1 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档描述 **v0.1 MVP 版本**的自动化测试策略与测试用例清单，覆盖后端单元测试和前端组件测试。

v0.1 不包含 E2E 测试（E2E 测试从 v0.2 开始引入，需要完整系统集成）。

### 1.2 测试架构分层

```
┌─────────────────────────────────────────┐
│  暂不覆盖（v0.2 引入）                   │   E2E 层
│  Playwright + tauri-driver              │
└─────────────────────────────────────────┘
              ▼
┌─────────────────────────────────────────┐
│  集成测试（少量）                         │   集成层
│  cargo test（#[tokio::test]）            │   跨模块链路验证
└─────────────────────────────────────────┘
              ▼
┌──────────────────┬──────────────────────┐
│  后端单元测试     │  前端单元测试          │   单元层
│  cargo test      │  Vitest + RTL         │
└──────────────────┴──────────────────────┘
```

### 1.3 覆盖率目标

| 层次 | 工具 | 覆盖率目标 |
|------|------|-----------|
| 后端核心业务逻辑 | `cargo llvm-cov` | ≥ 80% 行覆盖率 |
| 前端核心组件与 Store | Vitest + `@vitest/coverage-v8` | ≥ 75% 分支覆盖率 |

### 2.2 InMemoryStore 测试（数据层）

**测试文件**：`src-tauri/src/clipboard/storage_tests.rs`

#### UT-STORE-001：插入单条记录

```rust
#[test]
fn test_insert_single_record() {
    let mut store = InMemoryStore::new(20);
    let id = store.insert("Hello World".to_string(), 1000);

    assert_eq!(id, 1);
    assert_eq!(store.count(), 1);

    let record = store.get_by_id(1).unwrap();
    assert_eq!(record.text_content, "Hello World");
    assert_eq!(record.created_at, 1000);
    assert_eq!(record.content_type, "text");
}
```

#### UT-STORE-002：最新记录排在索引 0

```rust
#[test]
fn test_records_ordered_by_newest_first() {
    let mut store = InMemoryStore::new(20);
    store.insert("First".to_string(), 1000);
    store.insert("Second".to_string(), 2000);

    let records = store.get_recent(10);
    assert_eq!(records[0].text_content, "Second");  // 最新在前
    assert_eq!(records[1].text_content, "First");
}
```

#### UT-STORE-003：超过上限时淘汰最旧记录

```rust
#[test]
fn test_evicts_oldest_when_over_limit() {
    let mut store = InMemoryStore::new(3);
    store.insert("A".to_string(), 1000);
    store.insert("B".to_string(), 2000);
    store.insert("C".to_string(), 3000);
    store.insert("D".to_string(), 4000);  // 触发淘汰

    assert_eq!(store.count(), 3);
    let records = store.get_recent(10);
    let texts: Vec<&str> = records.iter().map(|r| r.text_content.as_str()).collect();
    assert!(texts.contains(&"D"));
    assert!(texts.contains(&"C"));
    assert!(texts.contains(&"B"));
    assert!(!texts.contains(&"A"));  // A 被淘汰
}
```

#### UT-STORE-004：删除存在的记录

```rust
#[test]
fn test_delete_existing_record() {
    let mut store = InMemoryStore::new(20);
    let id = store.insert("To Delete".to_string(), 1000);

    let deleted = store.delete(id);

    assert!(deleted);
    assert_eq!(store.count(), 0);
    assert!(store.get_by_id(id).is_none());
}
```

#### UT-STORE-005：删除不存在的记录返回 false

```rust
#[test]

    let mut store = InMemoryStore::new(20);

```

#### UT-STORE-006：get_recent 按 limit 限制返回数量

```rust
#[test]
fn test_get_recent_respects_limit() {
    let mut store = InMemoryStore::new(20);

    let records = store.get_recent(5);
```

#### UT-HIST-003：超限时产生 RecordEvicted 事件

```rust
#[test]
fn test_overflow_produces_evicted_event() {
    let mut history = ClipboardHistory::new(2);
    history.add_record("A".to_string(), 1000);
    history.add_record("B".to_string(), 2000);
    let events = history.add_record("C".to_string(), 3000);

    assert_eq!(events.len(), 2);
    assert!(events.iter().any(|e| matches!(e, ClipboardDomainEvent::RecordAdded { .. })));
    assert!(events.iter().any(|e| matches!(e, ClipboardDomainEvent::RecordEvicted { .. })));
```

```rust
        record.id
    } else {
        panic!("Expected RecordAdded");
    };

    let removed_id = history.remove_record(id);
```

```rust
async fn test_copy_text_adds_record_to_store() {
    let mut mock_clipboard = MockPlatformClipboardMock::new();
    // 第一次 change_count 返回 0，第二次返回 1（模拟变化）
    mock_clipboard
        .expect_change_count()
        .returning_st({

            move || count.fetch_add(1, std::sync::atomic::Ordering::SeqCst)

    mock_clipboard
        .expect_read_text()
        .returning(|| Ok(Some("Integration Test".to_string())));

    let store = Arc::new(RwLock::new(InMemoryStore::new(20)));
    // 调用 monitor 的单次 poll，验证 store 中出现记录
    // ...
```

## 6. 测试命名规范

| 规范 | 说明 |
|------|------|
| 测试文件命名 | 与被测文件同名，加 `.test.ts(x)` 后缀 |
| 测试用例命名 | 中文描述，格式：「[条件] 应 [期望行为]」|
| Mock 变量命名 | `mock_` 前缀（Rust）或 `Mock` 后缀（TS） |
| 测试分组 | 使用 `describe` / `mod tests` 按模块分组 |
