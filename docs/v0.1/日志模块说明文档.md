# 日志模块说明文档

## 目标

为前端（React）与后端（Tauri/Rust）提供统一日志链路，确保线上出现问题时可通过日志快速定位：

1. 发生了什么（操作轨迹）。
2. 在哪里失败（模块、命令、事件）。
3. 为什么失败（错误信息、上下文）。

## 日志落盘位置

后端启动时会自动解析并创建应用日志目录，日志文件按天滚动写入：

- 文件名：`clipboard-manager.log.YYYY-MM-DD`
- 路径来源：Tauri `app_log_dir()`

可通过 IPC 命令 `get_log_directory` 获取当前运行实例的日志目录。

## 级别与来源

统一使用四级：

1. `debug`：细粒度行为（如窗口显示/隐藏、监控启停）。
2. `info`：关键业务节点（如记录捕获、粘贴完成、删除完成）。
3. `warn`：可恢复异常或不合理输入（如非法参数、不支持的粘贴模式）。
4. `error`：失败分支（如 IPC 调用失败、事件分发失败、未捕获异常）。

日志来源分为：

1. 后端结构化日志（`tracing`）。
2. 前端日志（控制台 + 通过 `write_client_log` 转发到后端统一落盘）。
3. panic 捕获日志（包含 `location` 与 `backtrace`）。

## 已接入关键链路

### 后端

1. 应用启动/初始化（`src-tauri/src/lib.rs`）。
2. 剪贴板监听服务（`src-tauri/src/clipboard/monitor.rs`）。
3. 粘贴服务（`src-tauri/src/paste/mod.rs`）。
4. 全局快捷键（`src-tauri/src/shortcut/mod.rs`）。
5. 窗口管理（`src-tauri/src/window/mod.rs`）。
6. IPC 命令入口与事件分发（`src-tauri/src/ipc/commands.rs`、`src-tauri/src/ipc/events.rs`）。

### 前端

1. IPC 调用失败日志（`src/api/commands.ts`）。
2. 事件订阅与事件处理异常日志（`src/api/events.ts`、`src/hooks/useClipboardEvents.ts`）。
3. 键盘交互链路日志（`src/hooks/useKeyboard.ts`）。
4. 主面板初始化日志（`src/components/MainPanel/index.tsx`）。
5. 全局异常兜底（`src/main.tsx`，`error` + `unhandledrejection`）。

## 安全约束

为降低敏感信息风险，日志默认记录元信息（如 `record_id`、`text_length`），不主动记录完整剪贴板文本。

## 排障建议

1. 先按时间窗口过滤 `error` 级别，定位第一处失败点。
2. 回溯同一时间段的 `info/debug`，还原操作顺序。
3. 如果是前端异常，优先搜索目标字段：`frontend log`。
4. 如果是崩溃问题，优先搜索目标字段：`application panic captured`。
