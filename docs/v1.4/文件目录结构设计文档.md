# v1.4 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

本文档只描述 `v1.4` 为支持类型扩展、搜索筛选、类型色卡片、文本预览工作区与多屏定位建议采用的文件目录结构。

### 1.2 设计原则

- 搜索、筛选、类型展示、预览工作区要有清晰模块边界
- 语义分类与底层载荷处理分离，避免继续堆在已有 `monitor / query` 模块中
- 多屏定位与来源应用图标解析优先归入平台 / 窗口能力层

```text

```

### 3.2 推荐新增辅助文件

| 文件 | 说明 |
|------|------|
| `src/components/MainPanel/AdaptiveSearchBox.tsx` | 自适应宽度搜索框 |
| `src/components/MainPanel/TypeFilterRail.tsx` | 左侧类型快捷筛选栏 |
| `src/components/MainPanel/RecordCardShell.tsx` | 统一卡片头部、类型色与来源应用图标 |
| `src/components/preview/TextPreviewWorkspace.tsx` | 文本预览正文工作区 |
| `src/components/preview/PreviewSearchToolbar.tsx` | 文本查找替换工具条 |
| `src/hooks/useDisplayAffinity.ts` | 预览窗口多屏定位协同 |
| `src/stores/useSearchStore.ts` | 搜索关键字、结果数、类型筛选等运行态 |

### 3.3 推荐约束

- 卡片类型色与卡片头部布局规则集中到 `contentTypeTheme.ts`，不要分散在每类卡片里
- 文本预览工作区与普通预览渲染拆分，避免 `PreviewWindow.tsx` 继续膨胀
- 搜索状态要么统一进 `useClipboardStore`，要么单独建 `useSearchStore`，不能一半在 store 一半在组件局部

```text
runtime/
  icons/
    apps/
    meta/
  previews/
  logs/
```

### 6.2 约束说明

- 图标缓存与预览临时资源分目录存放
- 运行时缓存都必须可安全清理
- 图标缓存失败不影响主功能

---

## 7. 命名规范

- 语义类型统一使用 `contentType / semantic_type`
- 原始载荷统一使用 `payloadType / payload_type`
- 显示器定位统一使用 `displayAffinity / display_context`
- 搜索与筛选相关模块优先使用 `search / filter / query` 命名，不混用 `find` 表述

---

## 8. 与 v1.3 的主要差异

- `v1.3` 的重点文件仍集中在主面板预览与右键菜单，`v1.4` 需要新增搜索、筛选、分类和多屏定位模块
- `v1.3` 的预览窗口更多是单组件承载，`v1.4` 建议把文本预览工作区拆出独立目录
- `v1.3` 尚无来源应用图标专门模块，`v1.4` 建议显式建立图标解析与缓存模块
