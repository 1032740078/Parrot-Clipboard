# v1.4 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档只覆盖 `v1.4` 的类型扩展、搜索筛选、全窗口毛玻璃视觉统一、文本预览查找替换与多屏定位的自动化测试策略。

### 1.2 测试架构分层

- **后端单元测试**：验证语义分类、搜索查询、窗口定位与迁移逻辑
- **前端单元测试**：验证搜索框、类型栏、卡片壳层、文本预览工具条与状态 store
- **集成测试**：验证主面板搜索筛选链路、预览工作区链路、来源应用图标降级与多屏定位协同
- **BDD / E2E**：验证真实用户路径与关键验收场景
- **视觉回归测试**：验证毛玻璃风格、类型色系统、卡片布局与关于页精简结果

### 1.3 覆盖率目标

- 前端代码覆盖率继续满足项目既有门槛：`lines / branches / functions / statements >= 75%`
- `v1.4` 新增搜索、筛选、预览工作区与多屏定位相关模块建议覆盖率不低于 `85%`
- 后端新增分类、搜索和窗口定位模块的关键分支应有专项测试

## 2. 后端单元测试（Rust / cargo test）

### 2.1 语义分类回归

重点验证：

- URL 文本被识别为 `link`
- 单个图片文件可识别为 `image`
- 文件扩展名与 MIME 可识别为 `video / audio / document`
- 无法可靠识别时回退到 `text / files`
- 旧记录迁移后能正确补齐 `payload_type / semantic_type`
- `semantic_type = image / video / audio / document` 且 `payload_type = files` 时，`plain_text` 粘贴输出路径列表

### 2.2 搜索查询回归

重点验证：

- `search_records` 能按 `query + type_filter` 返回稳定排序结果
- 搜索为空结果时返回空数组，而不是错误
- `normalized_search_text` 可处理大小写、空白和 URL 场景

### 2.3 窗口定位回归

重点验证：

- 主面板在副屏时，预览窗口落在同一副屏
- 显示器上下文为空或失效时回退到安全默认位置
- 不同工作区尺寸下的位置不会越界

### 2.4 文本更新回归

重点验证：

- 文本查找替换保存后会重建搜索索引
- 替换失败时返回结构化错误，不污染原记录

## 3. 前端单元测试（Vitest + RTL）

### 3.1 搜索与筛选交互

建议覆盖：

- 搜索框输入后实时刷新结果
- 点击清空 icon 后恢复当前筛选下的完整列表
- 搜索关键字与左侧类型筛选叠加时结果正确
- 搜索框动态宽度变化符合预期

### 3.2 类型栏与卡片布局

建议覆盖：

- 左侧类型栏渲染完整的 8 类入口
- 当前类型激活态样式正确
- 卡片标题栏高度升级后的结构稳定
- 标题栏右侧来源应用图标存在与缺失时都能正确渲染

### 3.3 类型色与辅助信息

建议覆盖：

- 各语义类型映射到稳定标题色与背景色
- 时间信息位于底部辅助行且与统计信息同色
- 宽屏下底部快捷提示居中并具备最大宽度类名或样式

### 3.4 文本预览工作区

建议覆盖：

- 代码片段启用代码高亮模式
- 搜索关键字后显示匹配数与当前命中
- 单次替换与全部替换行为正确
- 搜索无结果时显示轻量提示

## 4. 集成测试

### 4.1 主面板搜索链路

- 初始化历史记录后输入关键字
- 验证结果集、选中态和空结果反馈
- 清空后恢复默认结果集

### 4.2 搜索与类型筛选协同链路

- 先筛选类型，再输入搜索词
- 先搜索，再切换类型
- 验证二者组合结果不冲突

### 4.3 预览窗口链路

- 从搜索结果中打开预览
- 文本记录执行查找与替换
- 保存后同步刷新主面板摘要与搜索结果

### 4.4 文件路径纯文本粘贴链路

- 选中 `image / video / audio / document` 且底层来自单文件或文件集合的记录
- 触发 `Shift+Enter`
- 验证最终走 `payload_type = files` 的路径列表纯文本粘贴

### 4.5 来源应用图标与降级链路

- 图标存在时渲染真实图标
- 图标缺失时渲染默认占位，不影响卡片布局

### 4.6 多屏定位协同链路

- 模拟主面板位于副屏并打开预览
- 验证预览窗口定位请求携带正确显示器上下文
- 验证失败时的降级提示与安全位置

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- 继续使用 `Playwright` 或项目当前 E2E 工具
- 对玻璃视觉与类型色补充截图断言
- 对多屏定位使用窗口上下文 mock 或平台抽象夹具

### 5.2 必测主链路

- 打开主面板 -> 搜索 -> 类型筛选 -> 打开预览
- 文本预览 -> 查找 -> 替换 -> 保存
- 单个图片文件 / 视频 / 音频 / 文稿记录 -> `Shift+Enter` -> 输出路径列表文本
- 副屏打开主面板 -> 打开预览 -> 预览跟随副屏
- 托盘打开关于页 -> 页面信息精简且无冗余块

## 6. 测试数据与夹具规范

### 6.1 记录夹具

- 统一从 `SEED-1401 ~ SEED-1405` 组织数据
- 语义类型、来源应用、URL、代码片段、多媒体文件名都需覆盖

### 6.2 能力夹具

- 显示器上下文使用固定 mock 数据
- 图标缓存成功与失败都需有独立夹具

### 6.3 脱敏要求

- 不得使用真实代码仓库私链、真实本地路径、真实会议视频或音频链接
- 来源应用名称可使用通用代称或公开应用名称
