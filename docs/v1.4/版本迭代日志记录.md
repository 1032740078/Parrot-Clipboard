# v1.4 版本迭代日志记录

## 1. 迭代总览

`v1.4` 当前阶段先完成文档基线建设，锁定以下 6 个版本目标：

- 主面板顶部移除无效说明文案，替换为横向居中的动态搜索框
- 主面板左侧新增竖向类型快捷筛选栏，并支持 `全部 / 文本 / 图片 / 文件 / 超链接 / 视频 / 音频 / 文稿`
- 剪贴板记录类型从三类扩展到七类，支持自动识别与模糊搜索
- 文本预览升级为支持代码高亮、搜索、替换的轻量编辑器
- 主面板、预览、设置、关于页统一升级为新版 macOS 风格毛玻璃半透明 UI
- 预览窗口根据激活主面板所在屏幕定位，不再固定跳回主屏

### 2.15 2026-03-10 `v1.4-source-app-icon-rendering`

**变更类型**：`fix(source-app-icon)`

**主要变更**：

- `ipc/commands.rs`、`platform/macos.rs`、`platform/mod.rs`：新增 `get_source_app_icon` IPC，基于记录中的 `source_app` 按需解析 macOS 应用真实图标，并返回 PNG 字节
- `SourceAppIcon.tsx`、`api/commands.ts`：卡片右上角优先显示真实来源应用图标，解析失败时回退为首字母徽标，同时增加前端缓存避免重复请求
- `SourceAppIcon.test.tsx`、`commands.test.ts`：补充图标获取、失败回退、缓存命中测试
- `platform/macos.rs`：新增 `Finder` 图标解析单测，验证真实 macOS 应用图标路径可稳定返回 PNG 数据
- 实机检查 `~/Library/Application Support/com.robin.clipboard/clipboard.db`，确认新复制记录已写入 `source_app`，例如 `webstorm`、`WeChat`

**问题描述**：

- 卡片标题右侧此前只显示来源应用名首字母，并不是真实软件图标
- 用户测试时会误以为“来源应用没有采集到”或“图标没有刷新”
- 实际数据库中的新记录已经存在 `source_app`，问题出在前端展示层而不是采集链路

**修复方案**：

- 保持数据库继续只存 `source_app` 显示名，不额外扩展表结构
- 在卡片渲染时按需向 Rust 请求对应应用图标，并在前端按应用名缓存结果
- 当应用图标无法解析时，继续使用首字母徽标兜底，避免列表出现空白占位

**验证结果**：

- `pnpm run type-check` ✅
- `pnpm run test`（44 files, 225 tests）✅
- `cargo test --manifest-path src-tauri/Cargo.toml`（175 passed）✅

- 脏数据记录将显示为 `"(未知文件) 等 N 项"`，保证应用可用性
