# v1.4 版本迭代日志记录

## 1. 迭代总览

`v1.4` 当前阶段先完成文档基线建设，锁定以下 6 个版本目标：

- 主面板顶部移除无效说明文案，替换为横向居中的动态搜索框
- 主面板左侧新增竖向类型快捷筛选栏，并支持 `全部 / 文本 / 图片 / 文件 / 超链接 / 视频 / 音频 / 文稿`
- 剪贴板记录类型从三类扩展到七类，支持自动识别与模糊搜索
- 文本预览升级为支持代码高亮、搜索、替换的轻量编辑器
- 主面板、预览、设置、关于页统一升级为新版 macOS 风格毛玻璃半透明 UI
- 预览窗口根据激活主面板所在屏幕定位，不再固定跳回主屏

## 2. 本次变更明细

### 2.13 2026-03-10 `v1.4-hotfix-primary-name`

**变更类型**：`fix(db)`

**主要变更**：

- `row_mapper.rs`：修复 `build_files_meta` 函数，当 `file_count > 0` 但 `primary_name` 为 NULL 时使用降级值 `"(未知文件)"` 而非报错

**问题描述**：

- 数据库中存在 `file_count > 0` 但 `file_items` 表无对应记录的脏数据
- 导致 `primary_name` 子查询返回 NULL，触发 `"missing primary_name for files record"` 错误
- 应用无法启动，主面板初始化失败

**修复方案**：

- 采用防御性编程，使用 `unwrap_or_else` 提供降级值，避免因数据不一致导致应用崩溃
- 脏数据记录将显示为 `"(未知文件) 等 N 项"`，保证应用可用性

**验证结果**：

- `cargo build` ✅
- `cargo test` ✅
