# v1.4 数据库设计文档

## 1. 文档概述

### 1.1 版本范围

本文档只描述 `v1.4` 为支持类型扩展、模糊搜索、多屏定位配套字段与来源应用图标展示所需的数据设计结论。

### 1.2 设计原则

- 保持“原始载荷类型”和“展示语义类型”分离，避免影响既有粘贴主链路
- 搜索优先复用已有可索引文本字段，但要补齐新类型所需索引信息
- 应用图标以**缓存资源 + 运行态解析**为主，不把大体积图标数据直接落入业务表
- 多屏定位属于运行态能力，不写入数据库

---

## 2. 存储布局总览

### 2.1 版本结论

`v1.4` **建议新增一次 SQLite schema 迁移**，并升级配置版本号以支持搜索与显示偏好最小集合。

### 2.2 本版数据设计结论

- 继续保留已有记录主表 `clipboard_items`
- 继续保留图片资产与文件明细表
- 在记录主表中补充语义分类、搜索归一化与来源应用标识字段
- 新增轻量图标缓存目录与元数据文件，但不新增图标业务表

---

## 3. SQLite 业务模型设计

### 3.1 `clipboard_items` 建议新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `payload_type` | `TEXT NOT NULL` | 原始载荷类型，取值固定为 `text / image / files` |
| `semantic_type` | `TEXT NOT NULL` | 展示语义类型，取值为 `text / image / files / link / video / audio / document` |
| `detected_mime_type` | `TEXT NULL` | 从系统载荷或文件扩展名推导出的 MIME |
| `source_app_id` | `TEXT NULL` | 来源应用的 bundle id / process name / app id |
| `normalized_search_text` | `TEXT NOT NULL DEFAULT ''` | 用于模糊搜索的归一化文本 |
| `primary_uri` | `TEXT NULL` | 超链接或媒体主地址 |

### 3.2 与旧字段的关系

- 旧字段 `content_type` 建议迁移为 `payload_type`
- 对外展示和筛选统一基于 `semantic_type`
- 既有 `search_text` 可以保留为原始索引文本；`normalized_search_text` 用于大小写、空白、URL 归一化后的搜索

### 3.3 约束建议

- `payload_type` 使用 `CHECK(payload_type IN ('text', 'image', 'files'))`
- `semantic_type` 使用 `CHECK(semantic_type IN ('text', 'image', 'files', 'link', 'video', 'audio', 'document'))`
- `primary_uri` 仅在 `semantic_type` 属于 `link / video / audio` 时允许非空

---

## 4. 关联表与资产设计

### 4.1 文件明细表沿用策略

- 文件记录仍继续使用现有文件明细表承载路径清单
- `semantic_type = document / video / audio / files` 的差异主要来自扩展名、MIME 和路径特征推导，不需要拆分多张文件表

### 4.2 图标缓存设计

建议在运行时目录新增：

```text
runtime/
  icons/
    apps/
      {platform}/{icon_cache_key}.png
    meta/
      {icon_cache_key}.json
```

说明：

- 图标文件作为缓存资源存在磁盘，不进入 SQLite 主表
- `meta` 文件保存 `source_app_id`、生成时间、尺寸与失效信息

---

## 5. 配置存储设计（`config.json`）

### 5.1 `v1.4` 结构结论

`v1.4` 建议新增轻量 `search` 与 `appearance` 分组，但只开放最小必要配置：

- `search.case_sensitive = false`
- `search.preserve_query_on_hide = true`
- `appearance.prefer_glass_effect = true`
- `appearance.show_source_app_icon = true`

### 5.2 不进入配置的内容

以下能力在本版中仍作为默认内建体验，不进入配置：

- 类型颜色自定义
- 搜索框动态宽度阈值
- 预览窗口多屏定位策略选择
- 关于页文案模块开关

---

## 6. 查询与索引约束

### 6.1 搜索查询策略

- 主面板搜索应基于 `normalized_search_text LIKE ?`
- 类型筛选应基于 `semantic_type = ?`
- 默认排序继续沿用 `last_used_at DESC, id DESC`

### 6.2 推荐索引

| 索引名 | 说明 |
|--------|------|
| `idx_clipboard_items_semantic_type_last_used_at` | 支持类型筛选 + 默认排序 |
| `idx_clipboard_items_normalized_search_text` | 支持模糊搜索 |
| `idx_clipboard_items_source_app_id` | 支持后续来源应用图标缓存和分析 |

### 6.3 详情读取约束

- 搜索结果列表阶段只返回摘要数据
- 预览窗口打开后再按需读取详情
- 文本替换如落库，应复用更新链路并重建 `normalized_search_text`
- `semantic_type = video / audio / document / files` 且 `payload_type = files` 的记录，在 `plain_text` 模式下统一输出路径列表，不新增额外持久化字段

---

## 7. 运行态数据设计

以下状态只在内存中存在，不应落库：

| 数据 | 说明 |
|------|------|
| `searchBoxLayoutState` | 搜索框当前宽度与动效状态 |
| `typeFilterSelection` | 左侧筛选当前高亮态 |
| `previewReplaceSession` | 查找替换输入值、当前命中索引、是否展开 |
| `displayAffinity` | 当前主面板与预览窗口屏幕映射关系 |
| `glassThemeToken` | 当前窗口的玻璃视觉令牌 |

---

## 8. 性能与可靠性目标

- 记录入库时完成语义分类与搜索归一化，避免每次打开面板再实时重算
- 搜索应优先命中本地索引字段，不依赖额外远程服务
- 图标缓存丢失时允许降级为默认占位，不得阻塞列表渲染
- 多屏信息读取失败时不应影响记录查询和预览详情读取

---

## 9. 与 v1.3 的主要差异

- `v1.3` 可以不改 SQLite 模型，`v1.4` 因为新增语义分类与搜索能力，建议显式引入 schema 迁移
- `v1.3` 只有三类内容，`v1.4` 需要把底层 `payload_type` 与展示 `semantic_type` 分离建模
- `v1.3` 搜索主要靠前端摘要字段，`v1.4` 应把搜索索引提升为数据库一等字段
