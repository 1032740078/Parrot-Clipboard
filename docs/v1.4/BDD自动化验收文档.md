# v1.4 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档只覆盖 `v1.4` 的主面板信息架构重构、类型扩展、搜索筛选、全窗口毛玻璃视觉统一、文本预览增强与多屏定位的 BDD 自动化验收场景。

### 1.2 BDD 编写规范

- 场景命名采用“用户目标 + 可验证结果”方式
- 场景编号统一使用 `BDD-41-xx`
- `Given / When / Then` 优先围绕用户可观察行为编写，而不是内部实现细节

---

## 2. Feature：主面板搜索与类型筛选（Panel Search & Filter）

### 场景 BDD-41-01：顶部口号移除后，主面板居中显示搜索框

- **Given** 用户打开主面板
- **When** 主面板完成首屏渲染
- **Then** 原“最近记录支持托盘 设置与关于页的发布版基础能力”文案不存在
- **And** 搜索框位于主面板顶部中心区域

### 场景 BDD-41-02：输入关键字后模糊搜索所有卡片

- **Given** 历史记录中存在包含 `meeting` 的文本、文稿和链接记录
- **When** 用户在搜索框输入 `meeting`
- **Then** 列表只显示命中记录
- **And** 命中范围覆盖不同类型卡片

### 场景 BDD-41-03：点击左侧类型栏后按类型过滤结果

- **Given** 列表中同时存在文本、图片、视频、音频和文稿记录
- **When** 用户点击左侧 `视频`
- **Then** 列表只显示 `视频` 类型记录
- **And** 当前类型入口处于激活态

### 场景 BDD-41-04：搜索与类型筛选可组合使用

- **Given** 用户已输入搜索词 `meeting`
- **When** 用户点击左侧 `文稿`
- **Then** 列表只显示同时满足 `meeting + 文稿` 的记录
- **And** 清空搜索词后恢复到当前 `文稿` 类型的完整列表

### 场景 BDD-41-05：搜索框支持清空按钮与动态宽度

- **Given** 用户已在搜索框输入较长关键字
- **When** 用户点击搜索框内清空按钮
- **Then** 搜索关键字被清空
- **And** 搜索框宽度回退到默认最小宽度

## 3. Feature：类型扩展与卡片信息层级（Semantic Types & Card Layout）

### 场景 BDD-42-01：新增类型被自动识别并展示正确类型

- **Given** 用户复制了超链接、单个图片文件、视频文件、音频文件和文稿文件
- **When** 这些记录进入历史列表
- **Then** 它们分别显示为 `超链接 / 图片 / 视频 / 音频 / 文稿` 类型

### 场景 BDD-42-02：不同类型卡片具有稳定的标题色和背景色

- **Given** 列表中存在多种语义类型卡片
- **When** 用户浏览主面板
- **Then** 各类型卡片显示稳定且可区分的标题色与背景色
- **And** 类型色不影响文字可读性

### 场景 BDD-42-03：卡片标题栏右侧显示来源应用图标

- **Given** 某条记录带有可解析的来源应用信息
- **When** 该卡片渲染到列表中
- **Then** 标题栏右侧显示来源应用图标
- **And** 图标缺失时显示统一降级占位

### 场景 BDD-42-04：时间信息下沉到底部辅助信息行

- **Given** 列表中存在文本或文件卡片
- **When** 用户查看卡片辅助信息
- **Then** 时间位于卡片底部，与字符数或文件数同一行居中显示
- **And** 时间颜色与辅助信息颜色一致

### 场景 BDD-42-05：单个图片文件、视频、音频、文稿记录按 `Shift+Enter` 时输出路径列表

- **Given** 某条记录在界面上显示为 `图片`、`视频`、`音频` 或 `文稿`
- **And** 该记录底层 `payload_type = files`
- **When** 用户按下 `Shift+Enter`
- **Then** 系统执行纯文本粘贴
- **And** 输出内容为换行分隔的文件路径列表，而不是图片或媒体内容本体

## 4. Feature：全窗口毛玻璃与关于页精简（Glass UI & About Simplification）

### 场景 BDD-43-01：主面板采用半透明毛玻璃风格

- **Given** 用户打开主面板
- **When** 主面板渲染完成
- **Then** 面板具备半透明背景、模糊层和细边框

### 场景 BDD-43-02：预览窗口、设置页、关于页共享玻璃视觉语言

- **Given** 用户先后打开预览窗口、设置页和关于页
- **When** 各窗口完成渲染
- **Then** 三者都体现统一的玻璃视觉风格
- **And** 内容区域保持可读

### 场景 BDD-43-03：关于入口从主面板移除，关于页内容精简

- **Given** 用户打开主面板
- **When** 用户查看右侧系统操作区
- **Then** 不存在关于按钮入口
- **And** 从托盘打开关于页时，仅显示精简后的必要信息块

### 场景 BDD-43-04：底部快捷提示在宽屏下居中且限宽

- **Given** 用户在宽屏环境中打开主面板
- **When** 查看底部快捷提示区域
- **Then** 提示整体居中
- **And** 最大宽度受控，不会分散到整行两端

## 5. Feature：文本预览工作区与多屏定位（Preview Workspace & Multi-Display）

### 场景 BDD-44-01：文本预览支持代码高亮

- **Given** 用户打开一条代码片段记录的预览
- **When** 文本预览加载完成
- **Then** 正文以代码高亮方式展示

### 场景 BDD-44-02：文本预览支持搜索与替换

- **Given** 用户在文本预览中输入查找关键字
- **When** 用户执行单次替换或全部替换
- **Then** 匹配内容被正确替换
- **And** 预览中显示匹配数或替换结果反馈

### 场景 BDD-44-03：副屏打开主面板时，预览窗口跟随副屏显示

- **Given** 用户在副屏激活主面板
- **When** 用户打开当前记录的预览窗口
- **Then** 预览窗口出现在副屏，而不是主屏

### 场景 BDD-44-04：多屏定位失败时安全降级

- **Given** 当前环境返回无效的显示器上下文
- **When** 用户打开预览窗口
- **Then** 预览窗口仍能打开
- **And** 窗口落在安全默认位置并给出可接受反馈

## 6. 非功能需求验收场景（Performance / Reliability / Readability）

### 场景 BDD-NFR-41：搜索输入不会明显阻塞主面板交互

- 连续输入关键字时，面板仍保持可滚动与可选中

### 场景 BDD-NFR-42：玻璃视觉升级不降低文本与代码可读性

- 在浅色和深色主题下，主面板与预览窗口的文本对比度都应满足阅读需求

### 场景 BDD-NFR-43：类型扩展不破坏既有粘贴链路

- 新增语义类型记录仍能基于底层 `payload_type` 正确执行既有粘贴语义
