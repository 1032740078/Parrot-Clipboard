# v1.4 冲刺任务文档

## 1. 冲刺目标

在保持既有托盘、独立预览、粘贴与删除主链路稳定的前提下，完成 `v1.4` 的 6 个核心目标：

- 主面板顶部改为搜索优先，左侧加入类型筛选栏
- 类型从三类扩展到七类，并支持自动识别
- 支持模糊搜索全部卡片内容
- 文本预览升级为支持代码高亮、查找、替换的轻量编辑器
- 预览窗口跟随主面板所在屏幕打开
- 主面板、预览、设置、关于页统一升级为新版 macOS 风格毛玻璃 UI

## 3. 各批次任务详情

### 3.1 第 0 批：文档基线

**目标**：锁定范围、统一类型语义、测试与契约口径。

**任务**：

- 完成 `docs/v1.4` 全套版本文档
- 更新 `docs/v1.4/版本迭代日志记录.md`
- 更新 `docs/项目决策记录文档.md`

### 3.2 第 1 批：搜索与筛选

**目标**：把主面板改造成搜索优先的信息检索面板。

**任务**：

- 扩展 `useUIStore` 增加搜索词、筛选类型、搜索结果状态
- 实现顶部居中搜索框、动态宽度与清空按钮
- 实现左侧类型筛选栏
- 重排主面板顶部与底部布局

**当前进展（2026-03-10）**：

- 已完成 `TASK-14-01-1`、`TASK-14-01-2`、`TASK-14-01-3`
- 主面板已经移除顶部说明文案与关于入口，改为居中搜索框 + 左侧类型筛选栏的新布局
- 搜索会话已统一为 `query + type_filter`，并将键盘操作、列表渲染、搜索结果状态对齐到同一份过滤结果
- 已通过前端相关单测、类型检查与 `lint`

### 3.3 第 2 批：分类与存储

**目标**：把七类记录识别与检索链路打通。

**任务**：

- 新增 URL、视频、音频、文稿分类规则
- 明确单个图片文件也归入 `image` 语义类型，而不是普通 `files`
- 锁定 `video / audio / document` 在 `payload_type = files` 下的 `Shift+Enter` 路径列表粘贴语义
- 扩展数据库 `content_type` 与搜索索引
- 扩展摘要 / 详情 DTO
- 实现 `search_records`

**当前进展（2026-03-10）**：

- 已在前后端引入 `payload_type` 与扩展后的七类 `content_type` 分离建模，当前口径为 `text / image / files` 对应底层载荷，`text / image / files / link / video / audio / document` 对应展示与筛选语义
- Rust 侧已完成 schema v3 迁移、摘要 / 详情 DTO 扩展、SQLite 行映射与仓储查询能力，唯一约束已调整为 `payload_type + content_hash`
- `runtime_repository` 已接入 URL 文本识别、单文件图片 / 视频 / 音频 / 文稿识别，以及按 `payload_type` 进行去重、裁剪和纯文本粘贴语义判断
- IPC 已新增 `search_records(query, type_filter, limit)`，前端 `commands`、类型定义与 adapter 兼容逻辑已同步补齐
- 主面板、右键菜单和纯文本粘贴能力判断已切换为优先读取 `payload_type`
- 全量自动化验证已通过：`pnpm run lint` + `pnpm run type-check` + `pnpm run test:coverage`（42 文件 208 用例全通过）+ `cargo test`（147 通过）
- 修复了测试夹具中 `payload_type` 缺失、`settings.ts` IPC 参数命名不一致（`launchAtLogin` → `launch_at_login`）等问题
- **第 2 批已完成，代码已提交**

## 4. 风险与缓解

| 风险 | 说明 | 缓解策略 |
|------|------|----------|
| 搜索与主列表状态冲突 | 搜索词、筛选与实时事件刷新容易互相覆盖 | 搜索会话以 `query + category` 为唯一真源 |
| 类型识别误判 | 文件扩展名和 MIME 不完整时易误分类 | 引入明确优先级与 `files` 回退规则 |
| 多屏定位不稳定 | 不同平台显示器 API 差异大 | 先获取主面板当前屏幕，失败则回退默认居中 |
| 来源应用图标解析失败 | 平台差异或权限限制导致图标缺失 | 使用占位图标与异步缓存策略 |
| 毛玻璃风格影响可读性 | 半透明过重会让文本发灰 | 对比度优先，模糊与透明度可适当收敛 |
| 文本替换破坏保存语义 | 编辑器升级后保存时机变复杂 | 继续复用现有 `update_text_record` 主链路 |
