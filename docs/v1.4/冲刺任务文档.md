# v1.4 冲刺任务文档

## 1. 冲刺目标

在保持既有托盘、独立预览、粘贴与删除主链路稳定的前提下，完成 `v1.4` 的 6 个核心目标：

- 主面板顶部改为搜索优先，左侧加入类型筛选栏
- 类型从三类扩展到七类，并支持自动识别
- 支持模糊搜索全部卡片内容
- 文本预览升级为支持代码高亮、查找、替换的轻量编辑器
- 预览窗口跟随主面板所在屏幕打开
- 主面板、预览、设置、关于页统一升级为新版 macOS 风格毛玻璃 UI

## 3. 各批次任务详情

### 3.1 第 0 批：文档基线

**目标**：锁定范围、统一类型语义、测试与契约口径。

**任务**：

- 完成 `docs/v1.4` 全套版本文档
- 更新 `docs/v1.4/版本迭代日志记录.md`
- 更新 `docs/项目决策记录文档.md`

### 3.2 第 1 批：搜索与筛选

**目标**：把主面板改造成搜索优先的信息检索面板。

**任务**：

- 扩展 `useUIStore` 增加搜索词、筛选类型、搜索结果状态
- 实现顶部居中搜索框、动态宽度与清空按钮
- 实现左侧类型筛选栏
- 重排主面板顶部与底部布局

### 3.3 第 2 批：分类与存储

**目标**：把七类记录识别与检索链路打通。

**任务**：

- 新增 URL、视频、音频、文稿分类规则
- 明确单个图片文件也归入 `image` 语义类型，而不是普通 `files`
- 锁定 `video / audio / document` 在 `payload_type = files` 下的 `Shift+Enter` 路径列表粘贴语义
- 扩展数据库 `content_type` 与搜索索引
- 扩展摘要 / 详情 DTO
- 实现 `search_records`

### 3.4 第 3 批：视觉与卡片升级

**目标**：统一新版 macOS 风格视觉语言，并提升卡片信息密度。

**任务**：

- 为七类卡片定义类型配色
- 标题栏高度统一增加 20px
- 接入来源应用图标
- 时间与辅助信息移到底部同一行居中
- 底部快捷提示居中并限制最大宽度

### 3.5 第 4 批：预览窗口升级

**目标**：让预览窗口兼顾编辑器能力和多屏体验。

**任务**：

- 为文本预览接入代码高亮
- 增加搜索、替换、全部替换
- 保存后同步更新卡片摘要与搜索结果
- 增加基于主面板屏幕的预览窗口定位逻辑

### 3.6 第 5 批：设置 / 关于与回归

**目标**：收尾界面统一性，完成整体验收。

**任务**：

- 设置页升级毛玻璃风格
- 关于页精简并升级毛玻璃风格
- 移除主面板关于入口
- 补齐自动化测试与 BDD 场景

## 4. 风险与缓解

| 风险 | 说明 | 缓解策略 |
|------|------|----------|
| 搜索与主列表状态冲突 | 搜索词、筛选与实时事件刷新容易互相覆盖 | 搜索会话以 `query + category` 为唯一真源 |
| 类型识别误判 | 文件扩展名和 MIME 不完整时易误分类 | 引入明确优先级与 `files` 回退规则 |
| 多屏定位不稳定 | 不同平台显示器 API 差异大 | 先获取主面板当前屏幕，失败则回退默认居中 |
| 来源应用图标解析失败 | 平台差异或权限限制导致图标缺失 | 使用占位图标与异步缓存策略 |
| 毛玻璃风格影响可读性 | 半透明过重会让文本发灰 | 对比度优先，模糊与透明度可适当收敛 |
| 文本替换破坏保存语义 | 编辑器升级后保存时机变复杂 | 继续复用现有 `update_text_record` 主链路 |

## 5. 冲刺完成后建议

1. 在真实双屏 macOS 环境下验证主屏 / 副屏预览定位
2. 对七类卡片做一次人工扫读评审，确认配色与信息层级稳定
3. 若 `v1.4` 搜索反馈良好，可在后续版本再评估收藏、标签和高级搜索
