# v1.4 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 **v1.4** 当前版本的新增与调整能力。

`v1.4` 默认继承当前版本已经具备的托盘入口、独立预览窗口、设置窗口、关于窗口、图片 OCR 纯文本粘贴、右键菜单与基础玻璃风样式；本版本只聚焦以下 6 个主题：

- 主面板信息架构重构：移除占空间的说明文案与关于入口，替换为**横向居中的搜索框**
- 主面板、预览窗口、设置窗口、关于窗口统一升级为**macOS 风格半透明毛玻璃 UI**
- 剪贴板类型从 `文本 / 图片 / 文件` 扩展为 `文本 / 图片 / 文件 / 超链接 / 视频 / 音频 / 文稿`，并支持**自动识别**
- 新增**模糊搜索 + 左侧竖向类型快捷筛选**
- 文本预览升级为**代码高亮 + 搜索 + 替换**的轻量编辑器体验
- 预览窗口支持**多屏跟随定位**，在哪块屏幕激活面板，就优先在哪块屏幕打开预览

以下能力 **不属于 `v1.4` 当前版本范围**：

- 云同步、账号体系、跨设备共享
- AI 摘要、AI 自动标签、OCR 新模型升级
- 收藏、置顶、标签树、批量管理
- 用户自定义主题编辑器、颜色面板、动画时长面板
- 视频 / 音频 / 文稿的深度编辑，仅提供预览与识别分类

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 高频检索用户 | 需要在大量历史记录中快速定位目标的用户 | 搜索、筛选和类型识别足够快且准确 |
| 内容确认用户 | 需要在粘贴前认真核对文本、链接、媒体与文稿内容的用户 | 预览足够清晰，文本预览具备搜索 / 替换能力 |
| 多屏办公用户 | 经常在主屏和副屏之间切换工作焦点的用户 | 预览窗口应跟随当前激活屏幕，而不是总跳回主屏 |
| 视觉敏感用户 | 注重桌面工具风格统一与层级质感的用户 | 主面板、预览、设置、关于页都更接近新版 macOS 工具窗口 |

## 3. 用户故事详细说明

### E01 - 主面板信息架构重构

#### US-14-01：移除顶部无效说明文案，替换为居中搜索框

**作为** 高频检索历史记录的用户  
**我希望** 主面板顶部不要再显示“最近记录支持托盘 设置与关于页的发布版基础能力”这类无效说明，而是换成一个横向居中的搜索框  
**以便** 我能把最宝贵的首屏空间留给真正高频使用的检索入口。

**验收要点**：
- 顶部说明文案完全移除，不保留占位空白
- 搜索框位于主面板顶部视觉中心
- 搜索框默认存在最小宽度，但输入变长时可以动态增长
- 搜索框内置清空图标按钮

#### US-14-02：左侧提供竖向类型快捷筛选栏

**作为** 需要快速缩小结果范围的用户  
**我希望** 在主面板左侧看到一条竖向菜单栏，支持 `全部 / 文本 / 图片 / 文件 / 超链接 / 视频 / 音频 / 文稿` 的快速切换  
**以便** 我不用每次都依赖关键词搜索，也能快速聚焦某类内容。

**验收要点**：
- 左侧菜单为稳定常驻区，不随卡片列表滚动
- 当前选中的筛选类型有明确高亮态
- 切换筛选不会清空搜索词
- “全部”与具体类型之间切换时，结果立即刷新

#### US-14-03：主面板不再提供关于入口

**作为** 希望主面板更专注的用户  
**我希望** 面板右侧不再出现关于按钮入口  
**以便** 主面板只承载浏览、搜索、筛选和操作历史记录的核心任务。

**验收要点**：
- 主面板内移除关于按钮
- 关于页仍可从托盘或其他次级入口打开
- 移除后顶部布局重新平衡，不出现视觉塌陷

### E02 - 类型识别扩展

#### US-14-04：系统自动识别超链接

**作为** 经常复制网址的用户  
**我希望** 当剪贴板内容是链接时，系统能自动识别为“超链接”卡片，而不是普通文本  
**以便** 我能更快区分普通文本和可跳转的 URL。

**验收要点**：
- 对 `http / https / mailto` 等常见链接协议进行识别
- 超链接卡片展示域名或标题化摘要
- 超链接参与搜索、筛选与预览

#### US-14-05：系统自动识别单个图片、视频、音频与文稿文件

**作为** 经常复制媒体文件和文稿的用户  
**我希望** 系统能把单个图片、视频、音频、文稿文件从普通文件中区分出来  
**以便** 我在卡片列表里一眼知道复制的是哪种资源。

**验收要点**：
- 单个文件优先根据剪贴板文件路径、扩展名、MIME 或 UTType 推断其语义类型
- 单个图片文件应识别为“图片”，而不是普通“文件”
- 单个视频、音频、文稿文件分别识别为对应语义类型
- 混合文件集合无法稳定归类时回落为普通“文件”
- 新类型卡片拥有独立图标、配色和预览文案
- 当上述文件型记录底层 `payload_type = files` 时，用户按 `Shift+Enter` 仍按文件路径列表执行纯文本粘贴

### E03 - 搜索与筛选

#### US-14-06：支持模糊搜索全部卡片内容

**作为** 需要回忆式检索历史记录的用户  
**我希望** 输入一部分关键词、文件名、域名或片段文本时，也能模糊搜索到对应卡片  
**以便** 即使记不住完整内容，我也能尽快找到目标记录。

**验收要点**：
- 搜索范围覆盖文本正文、预览摘要、文件名、文件路径、链接 URL、来源应用名
- 支持大小写无关与部分字符模糊匹配
- 搜索时列表实时刷新，不要求手动回车
- 空搜索时恢复当前类型筛选下的默认列表

#### US-14-07：搜索与类型筛选可组合使用

**作为** 想快速缩小结果范围的用户  
**我希望** 搜索词和类型筛选能一起工作  
**以便** 我可以先锁定“文稿”再搜关键字，而不是在全量结果里翻找。

**验收要点**：
- 搜索条件和类型条件组合后结果正确
- 清空搜索词时保留当前类型筛选
- 切换类型时保留搜索词
- 无结果时显示明确的空状态说明

### E04 - 文本预览编辑器升级

#### US-14-08：文本预览支持代码高亮

**作为** 经常复制代码片段的用户  
**我希望** 文本预览能按代码样式高亮显示  
**以便** 我在预览窗口中更快辨认结构、关键字和格式。

**验收要点**：
- 对常见代码片段进行自动语言推断
- 推断失败时回退为纯文本高亮方案
- 保持既有可编辑语义，不因高亮而失去文本修改能力

#### US-14-09：文本预览支持搜索与替换

**作为** 需要在预览中快速校对和修改文本的用户  
**我希望** 在预览窗口里直接搜索、替换关键字  
**以便** 我不用把内容粘到外部编辑器再处理。

**验收要点**：
- 支持查找下一个、上一个、全部替换
- 替换后仍走已有文本保存链路
- 搜索结果高亮清晰，不影响代码高亮可读性

### E05 - 多窗口玻璃化与多屏定位

#### US-14-10：主面板、预览、设置、关于页统一半透明毛玻璃风格

**作为** 注重界面一致性的用户  
**我希望** 主面板、预览窗口、设置页和关于页都采用统一的毛玻璃半透明效果  
**以便** 工具整体更像一套精致的桌面应用，而不是多个割裂页面。

**验收要点**：
- 四类窗口共享统一的玻璃表面风格
- 保留足够的文本对比度和分区层级
- 风格接近新版 macOS 工具面板，而不是厚重的拟物窗口

#### US-14-11：预览窗口应跟随当前激活屏幕

**作为** 多屏办公用户  
**我希望** 在副屏打开主面板并进入预览时，预览窗口也出现在副屏  
**以便** 我不用视线和鼠标反复跳回主屏。

**验收要点**：
- 系统根据当前主面板所在屏幕决定预览窗口初始位置
- 主屏激活面板时，预览仍显示在主屏
- 副屏激活面板时，预览优先显示在副屏
- 屏幕信息不可用时回退为系统默认居中策略

### E06 - 卡片视觉系统升级

#### US-14-12：卡片按类型拥有独立配色

**作为** 需要高速扫读列表的用户  
**我希望** 文本、图片、文件、超链接、视频、音频、文稿卡片都拥有各自稳定的标题色和背景色  
**以便** 我在不阅读全部文案的情况下也能快速识别内容类型。

**验收要点**：
- 每种类型至少定义一组标题强调色与低透明背景色
- 配色在深色 / 浅色主题下都保持一致语义
- 颜色只负责增强识别，不牺牲文本可读性

#### US-14-13：卡片标题栏更高，并显示来源应用图标

**作为** 需要确认内容来源的用户  
**我希望** 卡片标题栏高度增加 20px，并在右侧显示复制来源应用的图标  
**以便** 我在卡片级别就知道这段内容来自哪个应用。

**验收要点**：
- 三类旧卡片和四类新卡片共用统一标题栏高度
- 标题栏右侧显示来源应用图标；无法获取时显示占位图标
- 图标不会挤压标题文本到不可读

#### US-14-14：卡片底部时间与字符统计同一行居中显示

**作为** 关注记录信息层级的用户  
**我希望** 时间信息移到底部，与“xxx 字符”等辅助信息同一行居中显示，颜色也保持一致  
**以便** 卡片信息层级更紧凑，标题区和正文区更聚焦。

**验收要点**：
- 时间不再出现在卡片顶部
- 底部信息行整体居中
- 时间颜色与字符统计色一致

#### US-14-15：底部快捷键提示在宽屏下居中且限制最大宽度

**作为** 在大屏幕上使用主面板的用户  
**我希望** 底部快捷键提示居中显示，并限制一个最大宽度  
**以便** 它不会在宽屏上摊得太散，影响阅读节奏。

**验收要点**：
- 快捷提示整体水平居中
- 设定最大宽度，超宽窗口不再过度拉伸
- 提示内容根据当前版本能力同步更新

## 5. 排除范围说明

- `v1.4` 的搜索只覆盖本地历史记录，不引入远程索引或云搜索
- `v1.4` 的类型识别以规则推断和本地元数据为主，不引入 AI 分类服务
- `v1.4` 的代码高亮、查找、替换只覆盖文本预览窗口，不扩展到主面板卡片内联编辑
- `v1.4` 会精简关于页信息架构，但不会在本版本中加入更新日志浏览器、许可证中心等复杂内容
