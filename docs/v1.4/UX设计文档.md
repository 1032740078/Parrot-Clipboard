# v1.4 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

本文档只描述 `v1.4` 的体验设计目标：搜索优先、类型可感知、预览更专业、多窗口统一玻璃化，并整体向新版 macOS 工具 UI 风格靠拢。

### 1.2 设计原则（`v1.4` 强调）

- **检索优先**：顶部最核心的位置给搜索，不再浪费在无效文案上
- **类型一眼可辨**：分类不只存在于数据层，也必须在视觉上快速可感知
- **窗口是一套系统**：主面板、预览、设置、关于页共享同一视觉语言
- **专业但克制**：文本预览要更像轻量编辑器，但仍保持剪贴板工具的轻巧感
- **多屏不打断**：用户在哪块屏幕工作，预览就应跟到哪块屏幕

## 2. 主面板 UX

### 2.1 信息架构目标

- 移除“最近记录支持托盘 设置与关于页的发布版基础能力”这类低价值文案
- 用搜索框接管主面板顶部视觉中心
- 左侧加一条固定的类型筛选栏，形成“筛选 + 搜索 + 结果”三段式结构

### 2.2 搜索框设计规则

- 默认最小宽度建议 `240px`
- 根据输入字符长度逐步扩展，最大宽度建议不超过主面板内容区的 `60%`
- 搜索框内部右侧放置清空图标按钮
- 搜索框始终横向居中，不因左右其他按钮有无而偏移

### 2.3 底部快捷提示设计规则

- 提示内容整体居中
- 最大宽度建议 `560px ~ 680px`
- 各快捷键之间保持均匀但紧凑的间距
- 在宽屏下不允许两端过度拉散

## 3. 类型识别 UX

### 3.1 类型命名与筛选项

- `全部`
- `文本`
- `图片`
- `文件`
- `超链接`
- `视频`
- `音频`
- `文稿`

### 3.2 类型视觉语言

建议配色：

| 类型 | 标题强调色 | 卡片背景色 |
|------|------------|------------|
| 文本 | 冰蓝色 | 低透明冷白蓝 |
| 图片 | 湖绿色 | 低透明青绿色 |
| 文件 | 石墨灰 | 低透明中性灰 |
| 超链接 | 天青色 | 低透明青蓝色 |
| 视频 | 珊瑚红 | 低透明暖红色 |
| 音频 | 琥珀橙 | 低透明暖橙色 |
| 文稿 | 纸米金 | 低透明暖米色 |

规则：

- 标题色用于标题文本、类型徽标、选中态局部强调
- 背景色只能轻微染色，不能压倒正文内容
- 深色与浅色主题都保持相同语义映射

## 4. 卡片 UX

### 4.1 卡片结构调整

- 标题栏统一增加 `20px`
- 标题栏右侧新增来源应用图标
- 时间移到底部，与 `xxx 字符 / 文件数 / 时长` 同一行
- 底部信息行整体居中，颜色统一为辅助信息色

### 4.2 来源应用图标规则

- 图标尺寸建议 `16px ~ 18px`
- 优先使用真实应用图标
- 获取失败时显示首字母占位图标
- 图标视觉权重低于标题文本，不应喧宾夺主

### 4.3 状态设计

- 默认态：通透、安静、便于扫读
- Hover 态：轻微提亮与边框增强
- Selected 态：更明确的边框与表面高亮
- Searching Hit 态：正文内高亮命中关键词

---

## 5. 预览窗口 UX

### 5.1 总体目标

- 文本预览像轻量编辑器
- 媒体和文稿预览像系统级内容查看器
- 整体外观像新版 macOS 浮动工具窗口

### 5.2 文本预览规则

- 代码类文本默认进入高亮视图
- 查找与替换应是可一眼发现的辅助工具，而不是隐藏功能
- 高亮、查找命中和替换后的反馈要清晰但不过度炫技

### 5.3 多屏定位规则

- 预览窗口默认跟随主面板激活屏幕
- 如果副屏空间不足，仍应优先在副屏内做自适应缩放，而不是直接跳主屏
- 只有在显示器信息不可用时才回退系统默认居中

---

## 6. 设置页与关于页 UX

### 6.1 设置页

- 保持工具感而不是传统厚重表单页
- 背景使用轻毛玻璃与细描边
- 分组卡片保持清晰边界，避免整页发灰

### 6.2 关于页

- 内容精简为必要信息
- 不再堆砌过多说明文字
- 视觉风格与设置页保持一致
- 从主面板移除入口后，关于页定位为“次级信息页”

---

## 7. 动效与可访问性

### 7.1 动效

- 搜索结果刷新使用轻微淡入 / 位移即可
- 窗口玻璃表面不做夸张闪烁或流光
- 输入框宽度变化要平滑，不应跳变

### 7.2 可访问性

- 毛玻璃下文字对比度必须始终可读
- 类型识别不能只靠颜色，还要有图标 / 文案
- 搜索框、清空按钮、筛选栏和替换工具都必须支持键盘访问

---

## 8. 平台特定 UX 规范

### 8.1 macOS

- 强化毛玻璃、饱和度和细边框，更贴近新版系统浮层
- 窗口阴影柔和，不要过黑

### 8.2 Windows

- 毛玻璃效果应更克制，避免脏灰感
- 类型颜色可略提升明度以维持可读性

### 8.3 Linux

- 模糊不可用时至少保留半透明、描边与层级阴影
- 所有布局不能依赖系统专属玻璃 API 才能成立
