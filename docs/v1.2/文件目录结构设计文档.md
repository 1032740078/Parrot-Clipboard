# v1.2 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

`v1.2` 不新增新的顶层业务模块，主要在既有主面板与设置页模块内完成交互收敛与展示修复。

### 1.2 设计原则

- 尽量在既有模块内完成修复，避免为单次体验收敛过度拆散目录
- 新增辅助逻辑优先以 `helper / hook / presenter` 形式落在当前领域目录内
- 测试文件与实现文件保持同领域分布，便于回归定位

---

## 3. 前端目录设计（`src/`）

### 3.1 `v1.2` 重点文件

| 文件 | v1.2 职责 |
|------|-----------|
| `src/components/MainPanel/CardList.tsx` | 计算可视区域起点、刷新快捷槽位、承接卡片点击事件 |
| `src/components/MainPanel/QuickSelectBadge.tsx` | 把绝对索引编号改为可视槽位编号 |
| `src/components/MainPanel/ImageCard.tsx` | 统一图片预览源解析与占位回退 |
| `src/components/MainPanel/TextCard.tsx` | 与图片 / 文件卡片对齐鼠标选中语义 |
| `src/components/MainPanel/FileCard.tsx` | 与其他卡片统一单击 / 双击交互 |
| `src/hooks/useKeyboard.ts` | 消费“当前可视槽位映射”，而不是固定前 9 条 |
| `src/components/SettingsWindowPlaceholder.tsx` | 设置页左侧导航、右侧内容与独立会话能力分组 |

### 3.2 推荐新增辅助文件

| 文件 | 说明 |
|------|------|
| `src/components/MainPanel/visibleQuickSlots.ts` | 抽取可视槽位计算逻辑 |
| `src/components/MainPanel/previewSource.ts` | 抽取图片预览源解析逻辑 |
| `src/hooks/useCardPointer.ts` | 抽取单击 / 双击与防重复提交逻辑 |
| `src/components/settings/CapabilitySection.tsx` | 可选：把会话能力独立分组拆成单独组件 |

### 3.3 推荐约束

- 鼠标交互逻辑不要分散到每张卡片里重复实现，优先抽到共享 hook 或 CardList 协调层
- 图片预览源解析逻辑不要直接散落在 JSX 中，避免后续回归难以定位
- 设置页“会话能力”分组应作为独立 section / component，而不是继续嵌回通用页说明块

## 4. 后端目录设计（`src-tauri/src/`）

### 4.1 `v1.2` 重点文件

| 文件 | v1.2 关系 |
|------|-----------|
| `src-tauri/src/ipc/commands.rs` | 不新增命令，但继续承接 `paste_record / get_record_detail / get_platform_capabilities` |
| `src-tauri/src/clipboard/query.rs` | 继续提供 `thumbnail_path / original_path` 所需的数据结构 |
| `src-tauri/src/ipc/events.rs` | 继续广播 `thumbnail_ready / thumbnail_failed` |

### 4.2 模块归属说明

- 鼠标单击 / 双击与可视槽位映射属于前端交互层，不应下沉到 Rust 层
- 图片预览修复若只涉及路径转换与渲染回退，也优先停留在前端适配层
- 后端只需继续保证图片详情与更新事件可靠输出

## 6. 运行时生成目录

### 6.1 推荐布局

`v1.2` 继续沿用既有布局：

```text
runtime/
├── clipboard.db
├── config.json
├── images/
│   ├── original/
│   └── thumbs/
└── logs/
```

### 6.2 约束说明

- 不新增新的图片预览缓存目录
- 图片卡片预览优先复用现有 `thumbs/` 与 `original/` 目录

---

## 7. 命名规范

- 与可视编号相关的概念统一使用 `visibleQuickSlot`、`visibleSlotWindow`、`visibleSlotStartIndex`
- 与图片预览相关的概念统一使用 `previewSource`、`thumbnailPath`、`originalPath`
- 设置页新分组建议使用 `capability` 或 `session-capability`，避免与 `general` 混淆

---

## 8. 与 v1.1 的主要差异

- `v1.1` 重点文件集中在键盘与滚动；`v1.2` 额外把鼠标与预览适配纳入主面板核心文件
- `v1.2` 推荐把“可视槽位”“预览源”“会话能力分组”提炼为可复用辅助模块，减少继续在单文件内堆叠分支
