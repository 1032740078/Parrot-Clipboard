# v1.2 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

`v1.2` 不新增业务错误体系，主要沿用 `v1.1` 状态码。本文件仅列出当前版本会直接涉及的状态码与处理约定。

### 1.2 错误对象格式

```ts
export interface AppErrorPayload {
  code: string;
  message: string;
  details?: Record<string, unknown>;
}
```

### 1.3 错误传递机制

- 后端 `Command` 失败时，统一返回标准错误对象
- 前端根据 `code` 映射用户可读提示
- 图片预览资源失效优先走**前端占位降级**，不作为必须弹错的 IPC 错误

## 2. 当前版本状态码清单

| 状态码 | 名称 | 适用场景 |
|--------|------|----------|
| `SC-001` | `RECORD_NOT_FOUND` | 双击卡片或可视槽位快贴命中已不存在的记录 |
| `SC-008` | `UNSUPPORTED_PLATFORM_FEATURE` | 在不支持的平台 / 会话下触发受限粘贴能力 |
| `SC-010` | `WINDOW_ERROR` | 双击粘贴成功后隐藏主面板失败 |
| `SC-013` | `CLIPBOARD_WRITE_ERROR` | 写入系统剪贴板失败 |
| `SC-014` | `KEY_SIM_ERROR` | 模拟粘贴按键失败 |
| `SC-017` | `PERMISSION_REQUIRED` | 缺少辅助功能权限，无法执行双击粘贴或快贴 |
| `SC-021` | `INTERNAL` | 其他未归类错误 |

## 3. 状态码详细说明

### SC-001：`RECORD_NOT_FOUND`

- **触发条件**：鼠标双击、`Enter`、`Command + 数字` 命中的记录已删除或当前槽位映射失效
- **前端处理**：提示“目标记录不存在，请重新选择”；保持主面板打开并重新收敛选中态

### SC-008：`UNSUPPORTED_PLATFORM_FEATURE`

- **触发条件**：当前平台或会话不支持当前触发的粘贴能力
- **前端处理**：提示“当前平台或会话暂不支持该操作”，并提供托盘 / 设置入口作为替代路径

### SC-010：`WINDOW_ERROR`

- **触发条件**：双击或快贴成功后，主面板隐藏、焦点恢复或窗口操作失败
- **前端处理**：提示“主面板窗口操作失败，请重试”

### SC-013：`CLIPBOARD_WRITE_ERROR`

- **触发条件**：粘贴前写入系统剪贴板失败
- **前端处理**：提示“写入系统粘贴板失败，请重试”

### SC-014：`KEY_SIM_ERROR`

- **触发条件**：双击粘贴或快捷快贴时，模拟按键失败
- **前端处理**：提示“模拟粘贴失败，请检查系统权限或焦点状态”

### SC-017：`PERMISSION_REQUIRED`

- **触发条件**：执行双击粘贴、`Enter` 粘贴、`Command + 数字` 快贴时缺少权限
- **前端处理**：保持主面板打开，提供权限引导入口

### SC-021：`INTERNAL`

- **触发条件**：非预期内部错误
- **前端处理**：提示通用错误，并写入日志

## 4. Command / Event 对应矩阵

| 接口 | 可能状态码 |
|------|------------|
| `paste_record` | `RECORD_NOT_FOUND`、`UNSUPPORTED_PLATFORM_FEATURE`、`CLIPBOARD_WRITE_ERROR`、`KEY_SIM_ERROR`、`PERMISSION_REQUIRED`、`INTERNAL` |
| `get_record_detail` | `RECORD_NOT_FOUND`、`INTERNAL` |
| `get_platform_capabilities` | `INTERNAL` |
| `clipboard:record-updated` | 正常情况下不携带错误码；广播失败仅记录日志 |

---

## 5. 前端统一处理约定

- 单击卡片只是选中，不应触发错误提示
- 双击粘贴失败时主面板保持可见，便于用户再次尝试或改用键盘操作
- 可视槽位越界（如当前仅 4 个槽位却按了 `6`）属于正常忽略，**不弹错**
- 图片预览路径不可用时只切换为占位态，不中断主流程、不弹全局错误
- 所有错误日志禁止带出完整敏感正文，只允许记录 `record_id`、触发源、错误码与平台上下文

---

## 6. 成功响应约定

- 单击选中成功：只更新高亮与当前选中项，不显示 Toast
- 双击粘贴成功：隐藏主面板、恢复目标应用焦点、不额外弹成功 Toast
- 图片预览修复成功：卡片自动刷新为可见预览，不打断用户当前浏览
- 切换设置分组成功：只刷新右侧内容区，不改动保存态与窗口框架
