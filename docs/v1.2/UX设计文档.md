# v1.2 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

`v1.2` UX 设计只覆盖当前版本的体验收敛目标：

- 鼠标单击选中、双击粘贴
- 图片卡片预览修复
- 可视区域快捷编号
- 设置页左右双栏布局
- 会话能力独立分组

### 1.2 设计原则（v1.2 强调）

- **键盘优先不变，鼠标体验补齐**：鼠标是增强，不是替代键盘主链路
- **可见即可信**：当前被选中的卡片、当前 `1~9` 对应的卡片、当前会话能力状态，都必须让用户一眼看懂
- **失败优雅降级**：图片预览失败时退为占位态，不让用户面对空白与破损图标
- **结构清晰胜过信息堆叠**：能力说明移出通用设置，避免普通配置流程被噪音打断

---

## 2. 主面板鼠标交互 UX

### 2.1 单击选中原则

- 单击卡片后立即高亮，视觉上与键盘左右切换完全一致
- 单击只改变“当前目标”，不直接触发粘贴，避免误操作
- 已选中卡片再次单击，不应出现明显闪烁或列表抖动

### 2.2 双击粘贴原则

- 双击属于“直接执行动作”，应明显快于“单击后再按 Enter”
- 双击后不额外弹成功 Toast，避免打断用户回到目标应用
- 双击失败时主面板保留在前，方便改用键盘或进入权限引导
- 双击过程中必须防止重复提交，避免一次双击造成两次粘贴

### 2.3 视觉反馈建议

| 动作 | 反馈 |
|------|------|
| 单击卡片 | 即时高亮、边框 / 阴影变化 |
| 双击开始 | 可短暂进入轻量 loading / disabled 态 |
| 双击成功 | 主面板隐藏，不额外提示 |
| 双击失败 | 展示轻量错误提示，保留当前选中态 |

---

## 3. 图片预览 UX

### 3.1 体验目标

- 用户在主面板中就能识别大多数图片记录
- 即便缩略图链路异常，也不出现“卡片里什么都没有”的不确定感

### 3.2 预览优先级

1. **缩略图**：首选，加载更快，占用更低
2. **原图回退**：缩略图不可用时的可用兜底
3. **占位态**：所有预览资源都不可用时的最终降级

### 3.3 设计要求

- 占位态需明确写出“正在生成预览”或“预览不可用”
- 不允许直接展示浏览器默认的破损图片图标
- 预览加载失败不应改变卡片高度，避免列表跳动
- 后续收到 `thumbnail_ready` 后，卡片应无感切换到真实预览

---

## 4. 可视区域快捷编号 UX

### 4.1 体验目标

- 用户滚到哪里，就能立刻用哪里的 `1~9`
- 编号语义要稳定、直觉、可预测

### 4.2 设计规则

- 当前视口左侧第一个可见卡片始终映射为 `1`
- 编号只服务于当前视口，不表达全局绝对顺序
- 卡片右上角的数字徽标与键盘快选 / 快贴使用同一套映射结果
- 横向滚动停止后，编号应尽快刷新，但避免在滚动过程中频繁抖动

### 4.3 文案建议

- 快捷键提示从“`1-9 快选 / ⌘+1-9 快贴`”补充为“当前可视区域 `1-9` 快选 / 快贴”
- 必要时可在帮助文案中补充：“编号随当前可视区域变化”

---

## 5. 设置页结构 UX

### 5.1 左右双栏布局

- 左侧导航承载结构认知，右侧承载当前任务内容
- 切换分组时不重绘整个窗口壳层，减少上下文丢失
- 当前分组标题、描述、保存入口统一位于右侧顶部，形成稳定操作节奏

### 5.2 会话能力独立分组

- `当前会话能力完整支持` 必须从通用页迁出，避免“正常情况也持续提示”造成视觉噪音
- 能力完整支持与能力受限统一收口到“会话能力”分组
- 其他分组只保留必要的局部禁用说明，例如“当前会话不支持全局快捷键”

---

## 6. 反馈设计

### 6.1 操作反馈矩阵

| 场景 | 反馈策略 |
|------|----------|
| 单击选中 | 无 Toast，仅高亮变化 |
| 双击粘贴成功 | 主面板隐藏，无成功 Toast |
| 双击粘贴失败 | 错误 Toast / 内联提示，面板保留 |
| 图片预览失败 | 卡片占位态，无全局错误弹窗 |
| 能力完整支持 | 独立分组展示成功摘要 |
| 能力受限 | 独立分组展示原因与替代路径 |

### 6.2 Toast 规范

- 只对“真正失败且需要用户知道”的场景提示
- 不对“单击选中”“编号刷新”“预览自动恢复”等正常行为提示
- 提示文案避免带出敏感正文内容

---

## 7. 可用性指标

- 单击选中反馈：`< 50ms`
- 双击粘贴触发到窗口隐藏：`< 150ms`（不含目标应用接受粘贴耗时）
- 可视槽位刷新：滚动稳定后 `100ms` 内完成
- 图片预览切换：可见卡片范围内资源解析 `150ms` 内完成首个结果

---

## 8. 平台特定 UX 规范

### 8.1 macOS

- `Command + 1~9` 继续作为主快贴语义
- 双击粘贴失败时优先给出辅助功能权限引导

### 8.2 Windows

- 当前版本不扩展新的数字快贴平台语义
- 鼠标单击 / 双击与设置页布局优化同样适用

### 8.3 Linux

- `Wayland` 下能力受限时，独立“会话能力”分组必须清晰展示降级原因
- 不因会话能力受限而影响主面板的浏览、选中与图片预览体验
