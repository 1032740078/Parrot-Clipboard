## 2. 本次变更明细

### 2.1 2026-03-09 `v1.2-visible-quick-slots`

**变更类型**：`feat(visible-quick-slots)`

**主要变更**：

- 更新 `src/components/MainPanel/CardList.tsx`，按当前视口计算可视槽位，并把槽位编号只渲染到当前可视卡片上
- 更新 `src/components/MainPanel/QuickSelectBadge.tsx` 及三类卡片组件，改为消费显式槽位编号，而不是绝对列表索引
- 更新 `src/components/MainPanel/index.tsx` 与 `src/hooks/useKeyboard.ts`，让 `1~9 / Command + 1~9` 统一基于当前可视槽位解析目标记录
- 更新 `src/__tests__/components/MainPanel.test.tsx` 与 `src/__tests__/components/MainPanel.virtualization.test.tsx`，补齐滚动后快选 / 快贴与槽位刷新测试

**产出结果**：

- 用户滚动到列表中后段时，仍可直接使用当前视口中的 `1~9` 进行快选与快贴，不再受绝对前 `9` 条限制
- 小列表与虚拟滚动场景都能稳定刷新槽位语义，键盘快捷操作与徽标展示保持一致
- 主面板底部快捷提示已明确为“可视 1~9”，降低用户对全局绝对顺序的误解

### 2.2 2026-03-09 `v1.2-image-preview-fallback`

**变更类型**：`fix(image-preview-fallback)`

**主要变更**：

- 更新 `src/components/MainPanel/ImageCard.tsx`，新增本地图片路径到 WebView 资源 URL 的转换逻辑
- 在图片卡片内实现 `thumbnail -> original -> placeholder` 回退链路，并在缩略图加载错误时自动尝试原图
- 更新 `src/components/MainPanel/CardList.tsx`，让图片缩略图状态变更时卡片可按需重挂载，确保预览状态无感刷新
- 更新 `src/__mocks__/@tauri-apps/api/core.ts` 与 `src/__tests__/components/ImageCard.test.tsx`，补齐图片预览相关测试基线

**产出结果**：

- 图片卡片不再直接把裸文件路径塞给 `<img>`，缩略图可在 Tauri WebView 中正常解析显示
- 当缩略图缺失、失效或异步生成失败时，已挂载卡片会按需回退读取原图详情，再不行才退到占位态
- 预览失败不再暴露浏览器破损图片图标，且不会阻塞主面板其他卡片渲染

**验证结果**：

- 已执行 `pnpm exec vitest run src/__tests__/components/ImageCard.test.tsx src/__tests__/hooks/useClipboardEvents.test.tsx src/__tests__/components/MainPanel.test.tsx src/__tests__/components/MainPanel.virtualization.test.tsx`
