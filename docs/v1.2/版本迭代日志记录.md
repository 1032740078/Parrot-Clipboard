## 2. 本次变更明细

### 2.0 2026-03-09 `v1.2-image-asset-protocol-hotfix`

**变更类型**：`fix(image-asset-protocol)`

**主要变更**：

- 更新 `src-tauri/tauri.conf.json`，开启 `app.security.assetProtocol`，并把 `$APPDATA/images/**` 与 `$APPLOCALDATA/images/**` 加入资源访问 scope
- 更新 `scripts/verify-release.mjs`，把 `assetProtocol` 与图片目录 scope 纳入发布前自动校验，避免后续配置回退
- 结合真实日志确认根因：截图记录虽然已经完成 `ThumbnailReady`，但 WebView 无法读取 `convertFileSrc` 生成的本地资源 URL，导致缩略图和原图双双加载失败

**产出结果**：

- 真实桌面环境下的截图 / 图片记录可通过 `asset://localhost/...` 正常访问应用图片目录中的缩略图与原图
- 图片预览链路从“前端路径转换”进一步收口到“前端路径转换 + Tauri 资源协议放行”的完整闭环
- 发布阶段会自动拦截 `assetProtocol` 缺失或 scope 缩小的配置回退

**验证结果**：

- 已执行 `pnpm run release:verify-config`
- 已执行 `pnpm run type-check`
- 已执行 `pnpm run lint`

## 3. 当前阶段结论

### 已锁定事项

- `v1.2` 主要修复的是**交互与展示闭环**，而不是新增业务能力
- 主面板鼠标单击 / 双击与键盘粘贴已经统一到共享执行链路，并带有前端防重入保护
- 图片卡片预览已经统一采用 `convertFileSrc + 原图懒加载回退` 策略，避免空白与破损图标
- 卡片 `1~9` 编号改为基于**当前可视区域**动态映射
- 主面板快捷提示、卡片徽标与键盘目标解析已经围绕同一套“可视槽位”模型收敛
- 设置页新增独立“会话能力”分组，用于承接 `当前会话能力完整支持` 与降级说明
- 设置页的常规配置分组只保留必要的上下文提示，不再重复堆叠完整能力摘要
- `v1.2` 前端覆盖率、Rust 单测与全量 `Playwright` / BDD 场景已经全部通过，当前基线可直接用于最终交付

### 下一步建议

1. 以 `v1.2` 当前全绿状态准备最终交付说明与手测清单
2. 若进入发布流程，优先核对 macOS 权限引导、Linux Wayland 能力提示与图片资源路径三条真实环境链路
3. 在稳定基线上评估后续搜索、标签等管理能力规划
