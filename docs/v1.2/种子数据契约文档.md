# v1.2 种子数据契约文档

## 1. 文档概述

### 1.1 版本范围

`v1.2` 种子数据只服务于当前版本的 5 类能力验证：鼠标交互、图片预览修复、可视槽位编号、设置页双栏布局、会话能力独立分组。

### 1.2 设计原则

- 必须能稳定复现“单击选中、双击粘贴”的交互路径
- 必须能覆盖图片预览的 `ready / failed / fallback` 三种状态
- 必须能构造“滚动后仍能使用 `1~9`”的长列表场景
- 严禁包含真实隐私内容

---

## 2. 标准种子数据集

### 2.1 SEED-1201：鼠标交互基线集

用途：验证单击选中、双击粘贴与粘贴成功后的置顶语义。

| 顺序 | 类型 | 标题示例 | 说明 |
|------|------|----------|------|
| 1 | `text` | 日报结论示例 | 单击后应高亮 |
| 2 | `image` | 截图示例 A | 双击后应直接粘贴 |
| 3 | `files` | 合同目录示例 | 验证非文本卡片双击粘贴 |
| 4 | `text` | 回复模板示例 | 验证双击后置顶 |
| 5-8 | 混合 | 备用记录 | 用于回归单击 / 双击切换 |

### 2.2 SEED-1202：图片预览回退集

用途：验证图片卡片在不同预览资源状态下的表现。

| 记录 ID | `thumbnail_state` | `thumbnail_path` | `original_path` | 预期 |
|---------|-------------------|------------------|-----------------|------|
| `IMG-1202-A` | `ready` | 有效路径 | 有效路径 | 直接显示缩略图 |
| `IMG-1202-B` | `failed` | 空 | 有效路径 | 回退原图预览 |
| `IMG-1202-C` | `ready` | 无效路径 | 有效路径 | 回退原图预览 |
| `IMG-1202-D` | `failed` | 空 | 无效路径 | 显示占位态 |
| `IMG-1202-E` | `pending` | 空 | 有效路径 | 初始显示“正在生成预览”，后续可切换到缩略图 |

### 2.3 SEED-1203：可视槽位长列表集

用途：验证横向滚动后，快捷编号与 `1~9 / Command + 1~9` 的目标同步变化。

建议内容：

- 记录总数：`18`、`24` 或 `40`
- 类型以文本为主，夹杂图片 / 文件卡片
- 记录标题需带可辨识编号，例如“记录 01”“记录 02”

建议验证窗口：

| 视口场景 | 左侧第一个可见卡片 | 预期槽位 `1` |
|----------|--------------------|---------------|
| `VIEW-A` | 绝对索引 `0` | 记录 01 |
| `VIEW-B` | 绝对索引 `4` | 记录 05 |
| `VIEW-C` | 绝对索引 `9` | 记录 10 |

### 2.4 SEED-1204：会话能力完整支持集

用途：验证“会话能力”独立分组在完整支持场景下的展示。

字段要求：

- `platform = macos | windows | linux`
- `session_type = native | x11`
- `clipboard_monitoring = supported`
- `global_shortcut = supported`
- `launch_at_login = supported`
- `tray = supported`
- `active_app_detection = supported`
- `reasons = []`

### 2.5 SEED-1205：会话能力受限集

用途：验证独立分组中的降级提示与替代路径。

字段要求：

- 推荐使用 `linux + wayland`
- 至少一项能力为 `degraded` 或 `unsupported`
- `reasons` 至少包含 `1` 条用户可见说明

---

## 3. 类型覆盖要求

### 3.1 历史记录覆盖

- 至少覆盖 `text / image / files` 三种记录
- 至少准备 `2` 条图片记录，覆盖成功预览与失败回退
- 长列表夹具必须能复现滚动后的 `1~9` 槽位重排

### 3.2 运行态覆盖

- `selection_source = mouse / keyboard / quick_slot`
- `thumbnail_state = pending / ready / failed`
- `capability_state = supported / degraded / unsupported`

---

## 4. 资产文件契约

### 4.1 推荐目录

```text
src/__tests__/fixtures/v1.2/
├── records.pointer.json
├── records.visible-slots.json
├── capabilities.full-support.json
├── capabilities.degraded.json
└── images/
    ├── valid-thumb-a.png
    ├── valid-original-a.png
    └── broken-path.marker
```

### 4.2 路径约束

- 有效图片路径与无效图片路径都应显式区分，避免测试误把 fixture 缺失当成功场景
- 不允许引用开发者本机绝对路径作为稳定 fixture

---

## 5. 加载约定

### 5.1 开发模式加载

- 鼠标交互与可视槽位验证优先加载 `SEED-1201`、`SEED-1203`
- 图片预览联调优先加载 `SEED-1202`
- 设置页能力页联调优先加载 `SEED-1204 / SEED-1205`

### 5.2 测试模式加载

- 单测允许按需拆分成更小的局部 fixture
- BDD / E2E 推荐一次性加载与场景对应的完整数据集，降低用例间状态偏差

---

## 6. 覆盖的测试场景

| 数据集 | 覆盖场景 |
|--------|----------|
| `SEED-1201` | 单击选中、双击粘贴、置顶回归 |
| `SEED-1202` | 缩略图成功、原图回退、占位态 |
| `SEED-1203` | 滚动后 `1~9` 快选 / 快贴 |
| `SEED-1204` | 会话能力完整支持独立分组展示 |
| `SEED-1205` | 会话能力受限分组展示与禁用说明 |

---

## 7. 数据清理约定

- 测试结束后需清理运行时生成的临时图片文件与缩略图缓存
- 不允许把失败场景中的破损资源文件长期留在正式运行目录
- 所有 fixture 命名需保持脱敏且可直接理解其意图
