# v1.2 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 **v1.2** 当前版本的新增与调整能力。

`v1.2` 默认继承 `v1.1` 已完成的主面板失焦自动隐藏、`Dock` 覆盖、固定高度、`Command + 数字` 快贴、自动横向滚动与虚拟滚动稳定性能力；本版本只聚焦以下 5 项体验收敛问题：

- 主面板卡片从“仅键盘高亮”升级为“**键盘 + 鼠标** 都可驱动选中”，支持**单击选中、双击粘贴**
- 修复图片卡片**预览图不显示**的问题，确保图片记录在正常数据下可展示可用预览
- 卡片 `1~9` 快捷编号改为依据**当前可视区域**动态映射，横向滚动到后续卡片后仍能使用快捷键快选 / 快贴
- 设置页分组导航统一调整为**左侧导航 + 右侧内容区**的双栏布局
- `当前会话能力完整支持` 从通用页提示块调整为**独立分组**，单独承接会话能力展示

以下能力 **不属于 `v1.2` 当前版本范围**：

- 全文搜索、标签、收藏、置顶、分组管理
- 新增 SQLite 业务表、数据库迁移、配置结构升级
- 自定义鼠标手势、拖拽排序、批量操作
- `Windows / Linux` 专属新快捷键方案
- 设置页新增新的业务配置项

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 键鼠混合用户 | 日常同时使用键盘与鼠标浏览历史记录的用户 | 点一下就选中，双击即可直接粘贴 |
| 图片记录用户 | 高频复制截图、设计稿、聊天图片的用户 | 图片卡片必须稳定展示预览，避免盲选 |
| 长列表历史用户 | 历史记录较多，常常左右滚动浏览的用户 | 滚到后面的卡片后仍可继续使用 `1~9` |
| 设置中心用户 | 需要查看配置与平台能力状态的用户 | 导航结构清晰，能力信息不打扰主配置流程 |

## 3. 用户故事详细说明

### E01 - 卡片鼠标交互补齐

#### US-01-01：单击卡片时选中并高亮

**作为** 使用鼠标浏览历史记录的用户  
**我希望** 单击任意卡片时，该卡片立即成为当前选中项并显示高亮  
**以便** 我不必先切回键盘，也能明确知道“当前操作对象”是谁。

**验收要点**：
- 单击卡片后，选中态与键盘左右切换产生的高亮使用同一套视觉样式
- 鼠标选中会同步更新全局 `selectedIndex / selectedRecordId`
- 单击只负责“选中”，**不能**直接触发粘贴
- 单击已经选中的卡片时，不应引发重复粘贴、副作用事件或列表跳动

#### US-01-02：双击卡片时直接粘贴记录内容

**作为** 熟悉桌面交互的用户  
**我希望** 双击卡片时直接执行粘贴  
**以便** 在鼠标主导场景下获得与 `Enter` / `Command + 数字` 一样高效的操作路径。

**验收要点**：
- 双击卡片时复用既有 `paste_record` 粘贴链路，不新开一套特殊后端逻辑
- 双击成功后主面板自动隐藏，并保持“粘贴成功置顶”语义不变
- 双击失败时保留主面板，给出可理解的错误提示
- 双击过程中不能重复触发两次粘贴请求

### E02 - 图片预览显示修复

#### US-02-01：图片记录在主面板中稳定展示预览

**作为** 依赖图片缩略图辨识记录内容的用户  
**我希望** 图片卡片能正确显示缩略图或合理回退到原图预览 / 占位态  
**以便** 我能在不打开详情的情况下识别目标图片。

**验收要点**：
- 当 `thumbnail_path` 可用时，卡片展示缩略图
- 当缩略图缺失但原图路径可读时，允许回退使用原图作为预览源
- 当预览资源不可用时，展示明确的占位态，不出现破损图片图标
- 缩略图异步生成完成后，卡片应能自动刷新到可见预览

### E03 - 可视区域快捷编号

#### US-03-01：当前视口左侧第一个可见卡片始终是 `1`

**作为** 在长列表中左右滚动浏览历史的用户  
**我希望** 卡片 `1~9` 编号跟着当前可视区域变化  
**以便** 我滚动到后面的记录后，仍能用熟悉的数字键快速操作当前看到的卡片。

**验收要点**：
- 以“当前视口左侧第一个可见卡片”为 `1` 号槽位起点
- 槽位 `1~9` 只分配给当前视口起点之后的最多 `9` 张卡片
- 横向滚动后，卡片右上角编号立即按新的可视槽位重算
- 快捷编号是**视口态**，不是持久化数据，不写入数据库

#### US-03-02：滚动到后续卡片后仍可使用数字快选与快贴

**作为** 已经把列表滚动到中后段的用户  
**我希望** 按下 `1~9` / `Command + 1~9` 时命中当前视口内的编号卡片  
**以便** 不需要先滚回前 9 条记录才能使用快捷操作。

**验收要点**：
- `1~9` 单按仍然只是快选，不直接粘贴
- `Command + 1~9` 仍然直接粘贴，但目标取自当前视口编号映射
- 当当前视口不足 `9` 张可操作卡片时，越界数字键静默忽略
- 键盘左右切换导致自动滚动后，快捷编号映射也必须同步刷新

### E04 - 设置页双栏布局

#### US-04-01：设置分组改为左侧导航 + 右侧内容

**作为** 需要频繁切换多个设置分组的用户  
**我希望** 设置窗口采用经典双栏布局  
**以便** 我能更快理解信息架构，并稳定定位当前所处分组。

**验收要点**：
- 左侧固定展示分组导航，右侧承载当前分组内容
- 切换分组时仅右侧内容区刷新，不重建整个窗口框架
- 当前分组应有明显高亮与标题说明
- 未保存拦截、保存按钮、Toast 等现有行为保持不变

### E05 - 会话能力独立分组

#### US-05-01：把 `当前会话能力完整支持` 独立成专门分组

**作为** 需要确认当前平台 / 会话支持情况的用户  
**我希望** 会话能力信息有单独分组，不混在通用设置里  
**以便** 我在查看能力状态时更聚焦，而在调整普通设置时更少被提示打扰。

**验收要点**：
- 设置页新增“会话能力”独立分组
- 当能力完整支持时，`当前会话能力完整支持` 只在该分组展示
- 当能力受限时，降级原因与替代路径也集中展示在该分组
- 其他分组仅保留必要的禁用态说明，不重复堆叠完整能力摘要

## 4. 用户故事优先级矩阵

| 用户故事 | 优先级 | 依赖关系 |
|----------|--------|----------|
| `US-01-01` 单击选中 | P0 | 依赖现有选中状态管理 |
| `US-01-02` 双击粘贴 | P0 | 依赖 `paste_record`、权限校验、隐藏主面板 |
| `US-02-01` 图片预览修复 | P0 | 依赖图片元数据与详情路径解析 |
| `US-03-01` 可视槽位编号 | P0 | 依赖列表滚动状态与可见区计算 |
| `US-03-02` 滚动后继续快选 / 快贴 | P0 | 依赖 `US-03-01` |
| `US-04-01` 设置页双栏布局 | P1 | 依赖现有设置分组结构 |
| `US-05-01` 会话能力独立分组 | P1 | 依赖 `get_platform_capabilities` |

## 5. 排除范围说明

- 不在 `v1.2` 中引入“鼠标中键粘贴”“右键菜单”“拖拽排序”

- 不在 `v1.2` 中扩展新的平台能力探测项目
