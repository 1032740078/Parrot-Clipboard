# v1.2 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档仅覆盖 `v1.2` 当前版本的自动化验收场景，不展开 `v1.1` 已完成能力的完整回归清单。

### 1.2 BDD 编写规范

- 使用 `Given / When / Then / And` 四段式表达
- 场景标题优先描述用户感知结果
- 每个场景只验证一个主结果，复杂链路拆成多个场景
- 若包含平台差异，标题中显式标记平台或会话条件

## 2. Feature：卡片鼠标交互（Pointer Interaction）

### 场景 BDD-21-01：单击卡片后当前高亮切换到被点击项

- **Given** 用户已经打开主面板
- **And** 主面板存在至少 `3` 张卡片
- **When** 用户单击第 `2` 张卡片
- **Then** 第 `2` 张卡片成为当前选中项
- **And** 该卡片使用与键盘选中一致的高亮样式
- **And** 不触发粘贴命令

### 场景 BDD-21-02：双击卡片后直接粘贴该记录

- **Given** 用户已经打开主面板
- **And** 当前存在一张可正常粘贴的记录卡片
- **When** 用户双击该卡片
- **Then** 系统调用 `paste_record`
- **And** 粘贴成功后主面板自动隐藏
- **And** 被粘贴记录移动到列表最前

### 场景 BDD-21-03：双击粘贴失败时主面板保持打开

- **Given** 用户已经打开主面板
- **And** 当前环境缺少完成粘贴所需权限
- **When** 用户双击卡片
- **Then** 主面板保持可见
- **And** 页面展示可理解的失败提示或引导入口

## 3. Feature：图片预览显示修复（Image Preview）

### 场景 BDD-22-01：缩略图可用时图片卡片显示真实预览

- **Given** 主面板中存在一张 `thumbnail_state=ready` 且 `thumbnail_path` 有效的图片卡片
- **When** 用户打开主面板
- **Then** 该卡片显示真实图片预览
- **And** 不显示“预览不可用”占位文案

### 场景 BDD-22-02：缩略图不可用时回退显示原图预览

- **Given** 主面板中存在一张缩略图缺失但 `original_path` 有效的图片卡片
- **When** 用户打开主面板
- **Then** 该卡片仍显示可见图片预览
- **And** 预览来源来自原图回退，而不是空白区域

### 场景 BDD-22-03：所有预览资源不可用时显示占位态

- **Given** 主面板中存在一张缩略图与原图路径都不可用的图片卡片
- **When** 用户打开主面板
- **Then** 卡片显示占位态
- **And** 不出现浏览器破损图片图标

## 4. Feature：可视区域快捷编号（Visible Quick Slots）

### 场景 BDD-23-01：初始视口内左侧第一个可见卡片显示为 `1`

- **Given** 主面板存在不少于 `12` 张卡片
- **And** 当前列表位于最左起点
- **When** 用户观察当前可视区域
- **Then** 左侧第一个可见卡片右上角编号为 `1`
- **And** 后续可视卡片依次显示 `2~n`

### 场景 BDD-23-02：横向滚动后快捷编号随视口更新

- **Given** 主面板存在不少于 `18` 张卡片
- **When** 用户把列表水平滚动到后半段
- **Then** 当前视口左侧第一个可见卡片重新显示为 `1`
- **And** 之前处于前半段的卡片不再保留旧编号语义

### 场景 BDD-23-03：滚动到后半段后 `Command + 数字` 仍能快贴当前可视卡片

- **Given** 主面板存在不少于 `18` 张卡片
- **And** 用户已把列表滚动到后半段
- **And** 当前视口中某张卡片显示编号 `3`
- **When** 用户按下 `Command + 3`
- **Then** 被粘贴的是当前视口中编号为 `3` 的卡片
- **And** 不是绝对顺序中的第 `3` 条记录

## 5. Feature：设置页双栏布局（Settings Split Layout）

### 场景 BDD-24-01：设置页采用左侧导航与右侧内容区

- **Given** 用户打开设置窗口
- **When** 页面完成加载
- **Then** 左侧展示分组导航
- **And** 右侧展示当前分组的标题、描述与内容区
- **And** 页面不再使用顶部横向分组 tabs 作为主结构

### 场景 BDD-24-02：切换设置分组时仅右侧内容变化

- **Given** 用户已打开设置窗口并停留在“通用”分组
- **When** 用户点击左侧“快捷键”分组
- **Then** 左侧导航保持可见且当前项高亮变化
- **And** 右侧内容区切换到快捷键相关内容

---

## 6. Feature：会话能力独立分组（Capability Section）

### 场景 BDD-25-01：当前会话能力完整支持时显示独立分组

- **Given** 当前平台能力快照全部为 `supported`
- **When** 用户打开设置窗口并进入“会话能力”分组
- **Then** 页面显示 `当前会话能力完整支持`
- **And** 该信息不再混在“通用设置”主体内容中

### 场景 BDD-25-02：当前会话能力受限时降级原因集中在独立分组中展示

- **Given** 当前会话存在至少一项 `degraded` 或 `unsupported` 能力
- **When** 用户进入“会话能力”分组
- **Then** 页面展示受限原因与替代路径说明
- **And** 其他设置分组仅保留必要禁用态提示，不重复堆叠完整说明

## 7. 非功能需求验收场景（Performance / Reliability）

### 场景 BDD-NFR-21：双击卡片不会触发重复粘贴请求

- **Given** 用户双击一张可粘贴卡片
- **When** 粘贴请求发出
- **Then** 前端与后端只收到一次实际粘贴调用

### 场景 BDD-NFR-22：图片预览解析失败不会阻塞主面板渲染

- **Given** 主面板中包含多张图片记录，其中一部分预览资源无效
- **When** 用户打开主面板
- **Then** 主面板其余卡片正常显示
- **And** 失效图片卡片平滑退化到占位态
