# v1.2 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

`v1.2` 自动化测试只覆盖当前版本新增的 5 个主题：

- 卡片鼠标单击选中、双击粘贴
- 图片卡片预览显示修复
- 基于当前可视区域的 `1~9` 快捷编号与快贴
- 设置页左侧导航 + 右侧内容布局
- `当前会话能力完整支持` 独立分组

`v1.1` 已稳定的失焦自动隐藏、`Dock` 覆盖、固定高度与虚拟滚动基础能力，不作为本版新增测试设计重点，只保留回归验证。

### 1.2 测试架构分层

| 层级 | 工具 | 目标 |
|------|------|------|
| 后端单元测试 | `cargo test` | 验证记录详情、图片缩略图状态与既有粘贴链路稳定性 |
| 前端单元测试 | `Vitest + RTL + jsdom` | 验证鼠标交互、图片预览回退、可视槽位映射、设置页布局 |
| 集成测试 | 前后端桥接测试 / Mock Tauri | 验证双击粘贴、缩略图更新事件、快贴目标映射 |
| BDD / E2E | `Playwright` | 从真实用户视角验证完整交互闭环 |

### 1.3 覆盖率目标

- 前端覆盖率继续满足仓库基线：`lines / branches / functions / statements >= 75%`
- `v1.2` 新增交互逻辑的关键分支覆盖率目标：`>= 90%`
- 鼠标双击粘贴、图片预览修复、可视槽位映射三条主链路必须同时具备**单测 + BDD**双保险

## 4. 集成测试

### 4.1 双击粘贴链路

- 打开主面板
- 单击卡片，确认选中态同步
- 双击同一卡片
- 调用 `paste_record`
- 粘贴成功后主面板隐藏、记录置顶、焦点回到目标应用

### 4.2 图片预览刷新链路

- 加载一条 `thumbnail_state=pending` 的图片记录
- 初始展示“正在生成预览”占位态
- 后端广播 `thumbnail_ready`
- 前端刷新为真实图片缩略图

### 4.3 可视槽位快贴链路

- 主面板加载 `18+` 条记录
- 水平滚动到后半段
- 重新计算当前可视槽位映射
- 按下 `Command + 1`
- 验证被粘贴的是当前视口中标记为 `1` 的卡片，而不是绝对索引 `0`

### 4.4 设置页结构链路

- 打开设置窗口
- 验证左侧展示 `通用 / 记录与存储 / 快捷键 / 隐私 / 会话能力`
- 切换到“会话能力”分组
- 根据不同 `PlatformCapabilities` 快照验证完整支持或降级说明

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- 保持 `Playwright` 作为 BDD 驱动层
- `tauriMock` 需要补充：
  - 双击卡片触发 `paste_record`
  - 图片详情路径与缩略图更新事件
  - 可视槽位映射所需的滚动位置夹具
  - 能力完整支持 / 能力受限两类快照夹具

### 5.2 必测主链路

1. 单击卡片后选中态正确切换
2. 双击卡片后直接粘贴且只触发一次请求
3. 图片缩略图可见时正常渲染
4. 图片缩略图不可用时原图回退生效
5. 横向滚动后 `Command + 数字` 命中当前视口卡片
6. 设置页显示左侧导航 + 独立“会话能力”分组

## 6. 测试数据与夹具规范

### 6.1 记录夹具

- 至少准备 `8` 条记录覆盖鼠标交互
- 至少准备 `18` 条记录覆盖滚动后的槽位重算
- 图片夹具必须覆盖 `ready / pending / failed` 三种状态

### 6.2 能力夹具

- 一套“完整支持”能力快照
- 一套“会话受限”能力快照
- `reasons` 字段必须可直接展示给用户

### 6.3 脱敏要求

- 记录正文使用脱敏文本，如“订单号示例”“测试图片”“演示路径”
- 不允许在截图、fixture、日志中包含真实密码、验证码、客户隐私
