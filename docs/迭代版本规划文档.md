# 粘贴板记录管理工具 - 迭代版本规划文档

## 1. 文档概述

### 1.1 目的
本文档基于项目愿景需求、可行性评审及模块拆分文档，对"粘贴板记录管理工具"的迭代开发路线进行系统性规划，明确各版本的交付目标、功能范围、涉及模块及验收标准，为开发团队提供清晰的迭代路线图。

### 1.2 版本概览

| 版本 | 代号 | 目标 | 预计周期 |
|------|------|------|----------|
| v0.1 | MVP | 单平台核心功能验证 | 2-3 周 |
| v0.2 | Alpha | 图片/文件支持 + 数据持久化 | 2-3 周 |
| v0.3 | Feature Complete | 完整交互 + 系统集成 | 2-3 周 |
| v0.5 | Beta | 三平台支持 + 设置界面 | 2-3 周 |
| v1.0 | Release | 性能优化 + 稳定性 + 完整文档 | 1-2 周 |
| v2.0 | Future | 高级功能（云同步/插件/搜索） | 待规划 |

### 1.3 技术栈
- **框架**：Tauri 2.0
- **前端**：React 18 + TypeScript + Tailwind CSS + Framer Motion
- **后端**：Rust
- **数据库**：SQLite（rusqlite）
- **状态管理**：Zustand

## 2. v0.1 - MVP 版本

### 2.1 版本目标
在单一平台（优先 macOS）上验证核心使用流程：复制 -> 调出面板 -> 选择 -> 粘贴。确认技术方案可行，建立项目基础架构。

### 2.2 功能范围

#### 纳入范围
- 文本内容的粘贴板监听与记录（仅纯文本）
- 全局快捷键调出/隐藏主面板（Shift+Command+V）
- 横向卡片列表展示最近 20 条文本记录
- 键盘操作：`←` / `→` 切换、`Enter` 粘贴、`Esc` 关闭
- 单条记录删除（`Delete` / `Backspace` 键）
- 内存存储（不持久化，重启后清空）
- 单平台支持：macOS

#### 排除范围
- 图片、文件类型记录
- 数据持久化
- 系统托盘
- 开机自启动
- 设置界面
- 纯文本粘贴模式
- Windows / Linux 支持

### 2.3 涉及模块

| 模块 | 状态 | 说明 |
|------|------|------|
| M09 配置管理 | 基础版 | 仅内存配置，暂不持久化 |
| M14 平台抽象 | macOS | 仅实现 macOS 适配 |
| M13 IPC 通信 | 核心接口 | 定义记录查询、粘贴、删除接口 |
| M01 粘贴板监听 | 文本版 | 仅监听文本内容 |
| M03 窗口管理 | 基础版 | 屏幕底部显示/隐藏 |
| M04 快捷键管理 | 基础版 | 注册调出面板快捷键 |
| M06 粘贴执行 | 文本版 | 仅文本粘贴，原格式 |
| M12 前端状态管理 | 基础版 | 记录列表、选中状态 |
| M05 主面板 UI | 基础版 | 文本卡片列表、键盘操作 |

### 2.4 验收标准

| 验收项 | 标准 |
|--------|------|
| 快捷键响应 | < 100ms |
| 界面渲染 | < 200ms |
| 记录切换 | < 50ms |
| 粘贴操作 | < 100ms |
| 内存占用 | < 80MB |
| 稳定性 | 连续运行 2 小时无崩溃 |
| 核心流程 | 复制 -> 调出 -> 选择 -> 粘贴全流程无阻断 |

### 2.5 技术任务清单

1. 初始化 Tauri 项目结构，配置前端 React + TypeScript
2. 实现 M14 macOS 平台抽象（粘贴板读取、按键模拟）
3. 实现 M01 文本粘贴板轮询监听（200ms 间隔）
4. 实现 M03 无边框窗口管理，定位到屏幕底部
5. 实现 M04 全局快捷键注册
6. 实现 M06 文本内容写回粘贴板 + 模拟粘贴
7. 定义 M13 IPC Command / Event 接口
8. 实现 M12 Zustand 状态管理（记录列表 + 选中态）
9. 实现 M05 主面板 UI（横向文本卡片 + 键盘事件）

- 纯文本模式粘贴（Shift + Enter）

- 设置界面

## 5. v0.5 - Beta 版本

### 5.1 版本目标
实现三平台支持（macOS + Windows + Linux），提供用户可配置的设置界面，产品达到公测状态。

### 5.2 功能范围

#### 新增功能
- Windows 平台完整支持（粘贴板监听、全局快捷键、自启动、托盘）
- Linux 平台支持（X11 优先，Wayland 降级方案）
- 设置界面（通用、记录、快捷键、隐私四个分组）
- 自定义快捷键（含冲突检测）
- 黑名单应用管理（不记录特定应用的复制内容）
- 主题切换（浅色 / 深色 / 跟随系统）
- 记录数量、存储空间上限配置
- 快捷键冲突检测与提示

#### Windows 平台适配
- 粘贴板监听：`AddClipboardFormatListener` 事件驱动
- 全局快捷键：`RegisterHotKey` API
- 自启动：注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- 按键模拟：`SendInput` API

#### Linux 平台适配
- 粘贴板监听：X11 `XFixesSelectSelectionInput`（Wayland 降级为托盘菜单触发）
- 全局快捷键：X11 `XGrabKey`（Wayland 无法注册，提示用户）
- 自启动：XDG autostart `.desktop` 文件

### 5.3 涉及模块

| 模块 | 状态 | 说明 |
|------|------|------|
| M14 平台抽象 | 三平台 | 补全 Windows、Linux 实现 |
| M01 粘贴板监听 | 三平台 | 适配 Windows 事件、Linux X11 |
| M04 快捷键管理 | 三平台 + 自定义 | 支持自定义 + 冲突检测 |
| M08 自启动管理 | 三平台 | Windows 注册表、Linux .desktop |
| M10 设置界面 | 实现 | 四组设置 + 黑名单管理 |
| M09 配置管理 | 完整版 | 支持所有配置项 |
| M03 窗口管理 | 三平台 | 适配 Windows/Linux 窗口行为 |

| Linux X11 核心流程 | 核心流程正常（Wayland 有降级提示） |

## 9. 风险与缓解措施

| 风险 | 影响版本 | 缓解措施 |
|------|----------|----------|
| macOS 辅助功能权限被拒绝 | v0.1+ | 提供清晰引导，权限被拒时禁用快捷键并提示 |

| 大量图片导致内存过高 | v0.2+ | 缩略图预览 + LRU 缓存 + 数量限制（50 条） |
| 数据库结构升级破坏旧数据 | v1.0 | 版本迁移机制 + 升级前自动备份 |
| 三平台 API 差异过大 | v0.5 | 提前 POC 验证，M14 平台抽象层隔离差异 |
