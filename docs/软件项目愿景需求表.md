# 粘贴板记录管理工具 - 软件项目愿景需求表

## 1. 项目概述

### 1.1 项目名称
粘贴板记录管理工具（Clipboard History Manager）

### 1.2 项目简介
一个轻量级、高性能的跨平台粘贴板历史记录管理工具，能够自动捕获并管理用户的复制历史，支持文本、图片、文件等多种内容类型，通过快捷键快速调出并提供直观的可视化界面进行管理和粘贴操作。

### 1.3 项目背景
在日常工作中，用户经常需要在多个应用程序之间复制粘贴内容，但系统原生的粘贴板只能保存最后一次复制的内容。当需要重复使用之前复制的内容时，用户不得不重新查找和复制，这大大降低了工作效率。本工具旨在解决这一痛点，提供一个便捷的粘贴板历史管理方案。

## 2. 项目目标

### 2.1 核心目标
- 提供无感知的粘贴板监听和记录功能
- 实现快速、流畅的历史记录查看和使用体验
- 支持多种内容类型的记录和预览
- 提供跨平台一致的用户体验

### 2.2 业务目标
- 提升用户的复制粘贴工作效率至少 50%
- 减少用户重复查找和复制内容的时间
- 提供比市面上同类工具更流畅的使用体验

## 3. 目标用户

### 3.1 主要用户群体
- 程序开发人员：频繁复制代码片段、命令行指令
- 内容创作者：需要管理大量文本、图片素材
- 办公人员：处理文档、表格、邮件等多种内容
- 设计师：管理设计素材、颜色代码、文件路径

### 3.2 用户特征
- 熟悉键盘快捷键操作
- 追求高效的工作流程
- 需要在多个应用程序间频繁切换
- 对工具的响应速度和稳定性有较高要求

## 4. 核心功能需求

### 4.1 自动监听与记录

**功能描述**：
- 系统启动后自动在后台运行
- 实时监听系统粘贴板的变化
- 自动捕获并保存每次复制操作的内容

**支持的内容类型**：
- 纯文本内容
- 富文本内容（带格式）
- 图片（单张或多张）
- 文件和目录路径（单个或多个）

**记录信息**：
- 复制内容本身
- 复制时间戳
- 内容类型标识
- 内容来源应用（可选）

### 4.2 开机自启动

**功能描述**：
- 支持设置开机自动启动
- 启动后自动最小化到系统托盘
- 静默启动，不干扰用户正常使用

**平台支持**：
- macOS：通过 Launch Agents 实现
- Windows：通过注册表或启动文件夹实现
- Linux：通过 systemd 或 autostart 实现

### 4.3 快捷键调出界面

**快捷键设置**：
- macOS：Shift + Command + V（默认）
- Windows：Shift + Ctrl + V（默认）
- Linux：Shift + Ctrl + V（默认）
- 支持用户自定义快捷键

**调出行为**：
- 快速响应，延迟小于 100ms
- 界面出现在当前活动窗口之上
- 自动聚焦到最近的一条记录

### 4.4 可视化界面

**布局设计**：
- 位置：屏幕底部
- 宽度：占满屏幕宽度
- 高度：自适应内容，建议 150-300px
- 背景：半透明或毛玻璃效果

**内容展示**：
- 横向卡片布局
- 支持左右滚动查看更多记录
- 每个卡片显示：
  - 文本：显示前几行内容，超长省略
  - 图片：显示缩略图预览
  - 文件：显示文件图标、名称和数量

**视觉反馈**：
- 当前选中的卡片高亮显示
- 鼠标悬停效果
- 平滑的切换动画

### 4.5 键盘操作

**基础操作**：
- `←` / `→`：切换选中的记录（左右方向键）
- `Enter`：粘贴选中的记录到当前光标位置
- `Shift + Enter`：以纯文本模式粘贴（去除所有格式）
- `Delete` / `Backspace`：删除选中的记录
- `Esc`：关闭粘贴板工具界面

**高级操作（可选）**：
- `Ctrl/Cmd + A`：全选所有记录
- `Ctrl/Cmd + C`：复制选中记录
- `Ctrl/Cmd + F`：搜索记录
- 数字键 1-9：快速选择前 9 条记录

### 4.6 纯文本模式粘贴

**功能描述**：
- 去除所有富文本格式
- 保留纯文本内容
- 适用场景：
  - 从网页复制到代码编辑器
  - 从 Word 复制到纯文本编辑器
  - 去除背景色、字体样式、字号等格式

**实现要求**：
- 智能识别文本内容
- 保留换行符和基本结构
- 去除 HTML 标签、样式信息

### 4.7 记录管理

**存储策略**：
- 默认保存最近 100 条记录（可配置）
- 超出限制自动删除最旧的记录
- 支持手动删除单条或多条记录
- 支持清空所有历史记录

**数据持久化**：
- 记录保存到本地数据库
- 应用重启后恢复历史记录
- 支持导出和导入记录

**隐私保护**：
- 支持设置敏感应用黑名单（不记录特定应用的复制内容）
- 支持暂停监听功能
- 支持加密存储敏感内容

### 5.1 界面原型
参考文件：`docs/截原型图.jpg`

### 5.2 交互流程

**调出界面流程**：
1. 用户按下快捷键（Shift + Command + V）
2. 界面从屏幕底部弹出
3. 显示最近的复制记录，最新的记录默认选中
4. 用户可以通过方向键或鼠标选择记录

**粘贴流程**：
1. 用户选中目标记录
2. 按下 Enter 键
3. 内容自动粘贴到当前光标位置
4. 界面自动关闭

**删除流程**：
1. 用户选中要删除的记录
2. 按下 Delete 键
3. 记录从列表中移除
4. 自动选中下一条记录

### 5.3 视觉设计要求

**设计风格**：
- 简洁现代
- 符合各平台的设计规范
- 支持浅色和深色主题

**动画效果**：
- 界面弹出/收起动画（200-300ms）
- 卡片切换动画（100-150ms）
- 删除记录动画（150-200ms）

## 6. 技术要求

### 6.1 跨平台支持

**目标平台**：
- macOS 10.14+
- Windows 10/11
- Linux（主流发行版：Ubuntu, Fedora, Arch）

**技术选型建议**：
- Electron / Tauri：跨平台桌面应用框架
- React / Vue / Svelte：前端 UI 框架
- Node.js / Rust：后端逻辑处理

### 6.2 性能要求

**响应速度**：
- 快捷键响应时间：< 100ms
- 界面渲染时间：< 200ms
- 记录切换延迟：< 50ms
- 粘贴操作延迟：< 100ms

**资源占用**：
- 内存占用：< 100MB（空闲状态）
- CPU 占用：< 1%（空闲状态）
- 磁盘占用：< 50MB（应用本身）+ 记录数据

**稳定性**：
- 7x24 小时稳定运行
- 无内存泄漏
- 异常情况自动恢复

### 6.3 兼容性要求

**系统 API**：
- 粘贴板监听 API
- 全局快捷键注册 API
- 系统托盘 API
- 开机自启动 API

**内容格式支持**：
- 文本：UTF-8, UTF-16
- 图片：PNG, JPEG, GIF, BMP, WebP
- 文件：所有文件类型的路径

## 11. 风险评估

### 11.1 技术风险
- **风险**：不同平台的粘贴板 API 差异较大
- **应对**：提前调研各平台 API，设计统一的抽象层

### 11.2 性能风险
- **风险**：大量图片记录可能导致内存占用过高
- **应对**：实现智能缓存策略，限制图片大小和数量

### 11.3 兼容性风险
- **风险**：某些应用可能阻止粘贴板访问
- **应对**：提供降级方案，给出友好提示

### 11.4 用户体验风险
- **风险**：快捷键可能与其他应用冲突
- **应对**：支持自定义快捷键，提供冲突检测
