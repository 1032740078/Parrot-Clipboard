# 粘贴板记录管理工具 - 模块拆分文档

## 1. 文档概述

### 1.1 目的
本文档对"粘贴板记录管理工具"进行系统性的模块拆分，明确各模块的职责、边界、依赖关系和通信方式，为后续的开发实施提供清晰的架构指导。

### 1.2 拆分原则
- **单一职责**：每个模块只负责一个明确的功能领域
- **高内聚低耦合**：模块内部紧密相关，模块之间尽量减少依赖
- **跨平台抽象**：平台差异封装在底层，上层模块无需感知
- **可独立测试**：每个模块可以单独进行单元测试

### 1.3 技术栈参考
基于可行性评审推荐的技术栈：Tauri 2.0 + React 18 + TypeScript + Rust

## 3. 后端模块详细设计

### 3.1 M01 - 粘贴板监听模块

**职责**：实时监听系统粘贴板变化，捕获复制内容并分发事件。

**输入**：
- 系统粘贴板变更事件/轮询结果

**输出**：
- 结构化的粘贴板数据（`ClipboardEntry`）
- 变更通知事件

**核心数据结构**：
```rust
enum ClipboardContent {
    Text {
        plain: String,          // 纯文本内容
        rich: Option<String>,   // 富文本（HTML/RTF）
    },
    Images {
        items: Vec<ImageData>,  // 一张或多张图片
    },
    Files {
        paths: Vec<PathBuf>,    // 一个或多个文件/目录路径
    },
}

struct ClipboardEntry {
    id: u64,
    content: ClipboardContent,
    source_app: Option<String>, // 来源应用
    captured_at: i64,           // 捕获时间戳
}
```

**核心接口**：
```rust
trait ClipboardMonitor {
    fn start(&mut self) -> Result<()>;
    fn stop(&mut self) -> Result<()>;
    fn pause(&mut self);          // 暂停监听
    fn resume(&mut self);         // 恢复监听
    fn is_running(&self) -> bool;
    fn on_change(&self, callback: Box<dyn Fn(ClipboardEntry)>);
}
```

**实现要点**：
- macOS：轮询 `NSPasteboard.changeCount`，间隔 200ms
- Windows：注册 `AddClipboardFormatListener` 监听消息
- Linux：X11 使用 `XFixesSelectSelectionInput`，Wayland 提供降级方案
- 使用 `arboard` 库作为基础，按需扩展平台特性
- 黑名单应用过滤（不记录特定应用的复制内容）

**依赖模块**：M14（平台抽象）、M02（数据持久化）、M07（图片处理）

### 3.2 M02 - 数据持久化模块

**职责**：管理粘贴板记录的存储、查询、删除和清理。

**输入**：
- 粘贴板记录数据
- 查询/删除请求

**输出**：
- 记录列表
- 操作结果

**数据库表设计**：
```sql
-- 主记录表
CREATE TABLE clipboard_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT NOT NULL,      -- 'text' | 'images' | 'files'
    text_content TEXT,               -- 文本内容
    rich_content TEXT,               -- 富文本内容（HTML/RTF）
    source_app TEXT,                 -- 来源应用
    created_at INTEGER NOT NULL,     -- 时间戳
    is_favorite INTEGER DEFAULT 0    -- 是否收藏
);

-- 图片附件表（一条记录可对应多张图片）
CREATE TABLE clipboard_images (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    item_id INTEGER NOT NULL,        -- 关联 clipboard_items.id
    image_path TEXT NOT NULL,        -- 原图文件路径
    thumbnail_path TEXT,             -- 缩略图路径
    image_size INTEGER,              -- 文件大小（字节）
    image_width INTEGER,
    image_height INTEGER,
    image_format TEXT,               -- png/jpg/gif 等
    sort_order INTEGER DEFAULT 0,    -- 排序序号
    FOREIGN KEY (item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);

-- 文件附件表（一条记录可对应多个文件）
CREATE TABLE clipboard_files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    item_id INTEGER NOT NULL,        -- 关联 clipboard_items.id
    file_path TEXT NOT NULL,         -- 文件/目录路径
    file_name TEXT NOT NULL,         -- 文件名
    file_size INTEGER,               -- 文件大小
    is_directory INTEGER DEFAULT 0,  -- 是否为目录
    sort_order INTEGER DEFAULT 0,
    FOREIGN KEY (item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);

CREATE INDEX idx_items_created_at ON clipboard_items(created_at DESC);
CREATE INDEX idx_images_item_id ON clipboard_images(item_id);
CREATE INDEX idx_files_item_id ON clipboard_files(item_id);
```

**核心接口**：
```rust
trait ClipboardStorage {
    fn insert(&self, entry: &ClipboardEntry) -> Result<u64>;
    fn get_by_id(&self, id: u64) -> Result<Option<ClipboardEntry>>;
    fn get_recent(&self, limit: usize, offset: usize) -> Result<Vec<ClipboardEntry>>;
    fn delete(&self, id: u64) -> Result<()>;
    fn delete_batch(&self, ids: &[u64]) -> Result<()>;
    fn clear_all(&self) -> Result<()>;
    fn search(&self, keyword: &str) -> Result<Vec<ClipboardEntry>>;
    fn get_count(&self) -> Result<usize>;
    fn cleanup_old(&self, max_count: usize) -> Result<usize>;
}
```

**实现要点**：
- 使用 SQLite（`rusqlite`）存储元数据
- 图片原图存储在文件系统 `~/.clipboard-manager/images/original/`
- 缩略图存储在 `~/.clipboard-manager/images/thumbnails/`
- 支持数据库版本迁移
- 自动清理策略：按数量、按大小、按时间

**依赖模块**：M09（配置管理）

### 3.5 M06 - 粘贴执行模块

**职责**：将选中的记录内容写入系统粘贴板并模拟粘贴操作。

**输入**：
- 待粘贴的记录数据
- 粘贴模式（原格式 / 纯文本）

**输出**：
- 粘贴操作结果

**核心接口**：
```rust
trait PasteExecutor {
    fn paste(&self, entry: &ClipboardEntry, mode: PasteMode) -> Result<()>;
}

enum PasteMode {
    Original,   // 保持原格式
    PlainText,  // 纯文本模式（去除所有格式）
}
```

**执行流程**：
1. 暂停粘贴板监听（避免粘贴触发新记录）
2. 将内容写入系统粘贴板
3. 隐藏主面板窗口
4. 模拟 `Ctrl/Cmd + V` 按键
5. 恢复粘贴板监听

**实现要点**：
- 纯文本模式：提取纯文本，去除 HTML/RTF 格式
- 图片粘贴：将图片数据写入粘贴板
- 文件粘贴：将文件路径列表写入粘贴板
- 模拟按键需要平台适配

**依赖模块**：M01（粘贴板监听，暂停/恢复）、M03（窗口管理）、M14（平台抽象）

### 3.6 M07 - 图片处理模块

**职责**：处理图片的保存、缩略图生成和文件管理。

**输入**：
- 原始图片数据（来自粘贴板）

**输出**：
- 原图文件路径
- 缩略图文件路径

**核心接口**：
```rust
trait ImageProcessor {
    fn save_original(&self, data: &[u8], format: ImageFormat) -> Result<PathBuf>;
    fn generate_thumbnail(&self, original_path: &Path, max_size: u32) -> Result<PathBuf>;
    fn delete_image(&self, original_path: &Path, thumbnail_path: &Path) -> Result<()>;
    fn get_image_info(&self, path: &Path) -> Result<ImageInfo>;
    fn cleanup_orphaned(&self) -> Result<usize>;  // 清理无关联的图片文件
}

struct ImageInfo {
    width: u32,
    height: u32,
    size_bytes: u64,
    format: ImageFormat,
}
```

**实现要点**：
- 原图保存不做任何压缩或降质
- 缩略图规格：200x200，JPEG 格式，质量 85%
- 按日期分目录存储：`images/original/2026-03-05/`
- 使用 Rust `image` 库处理图片
- 缩略图在后台线程异步生成，不阻塞主流程
- 支持 PNG、JPEG、GIF、BMP、WebP 格式

**依赖模块**：M09（配置管理，获取存储路径和限制参数）

### 5.1 M13 - IPC 通信模块

**职责**：封装 Tauri IPC 调用，提供前后端通信的统一接口。

**通信方式**：

| 方向 | 方式 | 说明 |
|------|------|------|
| 前端 → 后端 | Tauri Command（invoke） | 前端主动调用后端功能 |
| 后端 → 前端 | Tauri Event（emit） | 后端推送事件通知前端 |

**Command 接口清单**：


## 9. 模块间通信协议

### 9.1 后端模块间通信
- 使用 Rust 的 trait 和依赖注入
- 通过 `Arc<Mutex<T>>` 或 `Arc<RwLock<T>>` 共享状态
- 事件通知使用 `tokio::sync::broadcast` 或 `mpsc` channel

### 9.2 前后端通信
- 使用 Tauri Command 进行同步调用
- 使用 Tauri Event 进行异步事件推送
- 数据序列化统一使用 JSON（serde）

### 9.3 前端模块间通信
- 通过 Zustand store 共享状态
- React 组件间通过 props 和 hooks 传递数据
