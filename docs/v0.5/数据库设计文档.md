# v0.5 数据库设计文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.5 Beta 版本** 的数据存储设计。`v0.5` 的重点是：

- 三平台支持
- 设置窗口及配置持久化
- 黑名单规则与主题 / 快捷键偏好

与 `v0.3` 相比，`v0.5` 的 **SQLite 历史业务模型保持稳定**，主要变化发生在 `config.json` 的结构升级与系统集成产物的扩展上。

### 1.2 设计原则

- **历史数据与配置分离**：历史记录继续进入 `SQLite`，用户偏好继续进入 `config.json`
- **配置可迁移**：从 `v0.3` 的扁平字段自动迁移到 `v0.5` 的分组化结构
- **黑名单规则属于配置，不属于业务历史**
- **运行态不滥持久化**：平台能力、当前窗口状态等仍只保留在内存

## 2. 存储布局总览

### 2.1 运行时目录

```text
runtime/
├── clipboard.db
├── config.json
├── images/
│   ├── original/
│   └── thumbs/
└── logs/
```

### 2.2 存储分工

| 介质 | 内容 | 说明 |
|------|------|------|
| `clipboard.db` | 历史记录主数据 | 沿用 `v0.3` 三表结构 |
| `images/original/` | 图片原图 | 再次粘贴图片时使用 |
| `images/thumbs/` | 缩略图 | 主面板图片卡片预览 |
| `config.json` | 设置中心配置 | 主题、容量、快捷键、黑名单规则 |
| 系统自启动产物 | 平台自启动配置 | macOS plist / Windows Registry / Linux `.desktop` |

## 3. SQLite 业务模型

### 3.1 版本结论

`v0.5` **不新增 SQLite 业务表**，继续复用 `v0.3` 的历史记录模型：

- `clipboard_items`
- `image_assets`
- `file_items`

### 3.2 主表 `clipboard_items`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 主键 |
| `content_type` | TEXT | `text` / `image` / `files` |
| `content_hash` | TEXT | 去重哈希 |
| `text_content` | TEXT | 文本正文 |
| `rich_content` | TEXT | 富文本内容 |
| `preview_text` | TEXT | 卡片摘要 |
| `search_text` | TEXT | 为后续搜索预留 |
| `source_app` | TEXT | 来源应用展示名 |
| `file_count` | INTEGER | 文件数 |
| `payload_bytes` | INTEGER | 原始负载大小估算 |
| `created_at` | INTEGER | 首次创建时间 |
| `last_used_at` | INTEGER | 最近被复制或再次粘贴时间 |

### 3.3 子表与索引

沿用 `v0.3`：

- `image_assets(record_id, original_path, thumbnail_path, mime_type, pixel_width, pixel_height, byte_size)`
- `file_items(record_id, path, display_name, entry_type, extension, sort_order)`
- 核心索引继续围绕 `content_hash`、`last_used_at DESC`、`record_id` 建立

### 3.4 为什么不把黑名单写入 SQLite

黑名单规则不进入 `SQLite`，原因如下：

- 它是“用户偏好”，不是“业务历史”
- 规则数量小，天然适合与主题、快捷键等一起进入配置快照
- 配置导入 / 迁移时，与其他设置一起处理更简单

## 4. 配置存储设计（`config.json`）

### 4.1 `v0.5` 结构

```json
{
  "config_version": 2,
  "general": {
    "theme": "system",
    "language": "zh-CN",
    "launch_at_login": true
  },
  "history": {
    "max_text_records": 200,
    "max_image_records": 50,
    "max_file_records": 100,
    "max_image_storage_mb": 512,
    "capture_images": true,
    "capture_files": true
  },
  "shortcut": {
    "toggle_panel": "Shift+Command+V"
  },
  "privacy": {
    "blacklist_rules": []
  }
}
```

### 4.2 配置字段说明

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `config_version` | number | `2` | 配置版本号 |
| `general.theme` | string | `system` | `light` / `dark` / `system` |
| `general.language` | string | `zh-CN` | 当前版本仅保留中文配置位 |
| `general.launch_at_login` | boolean | `true` | 是否登录启动 |
| `history.max_text_records` | number | `200` | 文本上限 |
| `history.max_image_records` | number | `50` | 图片上限 |
| `history.max_file_records` | number | `100` | 文件上限 |
| `history.max_image_storage_mb` | number | `512` | 图片资源总空间上限 |
| `history.capture_images` | boolean | `true` | 是否记录图片 |
| `history.capture_files` | boolean | `true` | 是否记录文件 |
| `shortcut.toggle_panel` | string | 平台默认值 | 调出主面板快捷键 |
| `privacy.blacklist_rules` | array | `[]` | 黑名单规则集合 |

### 4.3 黑名单规则结构

```json
{
  "id": "blr_wechat_macos",
  "app_name": "微信",
  "platform": "macos",
  "match_type": "bundle_id",
  "app_identifier": "com.tencent.xinWeChat",
  "enabled": true,
  "created_at": 1772812800000,
  "updated_at": 1772812800000
}
```

### 4.4 不落盘的运行态

以下信息仍只保留在内存：

- 当前监听状态 `running / paused`
- 当前主面板显隐状态
- 当前平台能力矩阵
- 当前会话是否为 `Wayland`
- 当前设置窗口是否已打开

## 5. 配置迁移设计

### 5.1 从 `v0.3` 到 `v0.5`

`v0.3` 配置：

```json
{
  "max_text_records": 200,
  "max_image_records": 50,
  "max_file_records": 100,
  "toggle_shortcut": "Shift+Command+V",
  "launch_at_login": true
}
```

迁移到 `v0.5` 时：

- 自动识别旧扁平结构
- 填充 `config_version = 2`
- 将字段映射到 `general / history / shortcut / privacy`
- 对新增字段回填默认值
- 原地回写为新结构

### 5.2 迁移规则

| 旧字段 | 新字段 |
|--------|--------|
| `max_text_records` | `history.max_text_records` |
| `max_image_records` | `history.max_image_records` |
| `max_file_records` | `history.max_file_records` |
| `toggle_shortcut` | `shortcut.toggle_panel` |
| `launch_at_login` | `general.launch_at_login` |

### 5.3 失败策略

- 解析失败：回退到默认配置并输出错误日志
- 回写失败：保留内存中已迁移配置，但设置页提示“持久化失败”
- 自启动同步失败：仅影响该配置项，不阻塞主面板与历史功能

## 6. 保留策略与清理边界

### 6.1 历史上限生效方式

- 文本、图片、文件仍按内容类型独立淘汰
- 图片额外受 `max_image_storage_mb` 约束
- 新上限低于当前占用时，保存配置后执行一次补偿清理

### 6.2 清理范围

| 操作 | 清理范围 |
|------|----------|
| 单条删除 | `clipboard_items` 及关联 `image_assets` / `file_items` / 图片文件 |
| 超限淘汰 | 同上 |
| 清空历史 | SQLite 全历史 + 图片资产文件 |
| 删除黑名单规则 | 仅改 `config.json`，不清理历史 |

## 7. 系统集成产物

| 平台 | 介质 | 说明 |
|------|------|------|
| macOS | Launch Agent plist | 登录启动配置 |
| Windows | `HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run` | 登录启动注册表项 |
| Linux | XDG autostart `.desktop` | 桌面环境启动项 |

这些产物不纳入 `clipboard.db`，也不纳入 `config.json` 正文，只由 `general.launch_at_login` 驱动生成或清理。

## 8. 性能与可靠性目标

| 项目 | 目标 |
|------|------|
| 首次读取设置快照 | `< 30ms` |
| 保存单个设置分组 | `< 50ms` |
| 快捷键冲突校验反馈 | `< 100ms` |
| 黑名单命中判定 | `< 5ms` |
| 配置迁移 | `< 20ms` |

---

## 9. 与 v0.3 的主要差异

| 项目 | v0.3 | v0.5 |
|------|------|------|
| SQLite Schema | 三表结构 | 不变 |
| `config.json` | 扁平结构 | 分组化、版本化结构 |
| 黑名单规则 | 无 | 落在 `privacy.blacklist_rules` |
| 主题配置 | 无 | `general.theme` |
| 存储空间上限 | 无 | `history.max_image_storage_mb` |

结论：`v0.5` 的数据设计重点不是扩表，而是 **让配置成为可以演进、可以迁移、可以承载隐私规则的一等模型**。
