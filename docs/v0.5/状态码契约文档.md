# v0.5 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.5 Beta 版本** 的 IPC 状态码与错误处理约定。

除 `v0.3` 的历史记录、托盘、自启动、监听控制错误外，`v0.5` 新增设置、快捷键冲突、黑名单规则与平台能力相关错误码。

### 1.2 错误对象格式

```typescript
interface IpcError {
  code: ErrorCode;
  message: string;
}
```

### 1.3 错误传递机制

- 后端统一返回结构化错误对象
- 前端先按 `code` 分类处理，再按 `message` 兜底展示
- 设置持久化、快捷键注册、平台能力探测类错误必须写日志

## 3. 错误码详细说明

### SC-001：`RECORD_NOT_FOUND`

适用于按 `id` 查询、删除、粘贴不存在的历史记录。

### SC-002：`INVALID_PARAM`

适用于数值越界、空字符串、非法枚举、无效确认令牌等基础参数错误。

### SC-003：`SETTINGS_VALIDATION_ERROR`

适用于：

- 主题值非法
- 上限值超出允许范围
- 黑名单规则字段缺失
- 快捷键格式无法解析

### SC-004：`SHORTCUT_CONFLICT`

适用于：

- 与系统保留快捷键冲突
- 与已注册全局快捷键冲突
- 当前环境明确定义为冲突态而不是单纯“不支持”

### SC-005：`SHORTCUT_REGISTER_ERROR`

适用于快捷键格式已通过校验，但在平台注册或替换时失败。

### SC-006：`BLACKLIST_RULE_DUPLICATE`

适用于同一平台 + 同一匹配类型 + 同一标识符重复创建规则。

### SC-007：`BLACKLIST_RULE_NOT_FOUND`

适用于更新或删除不存在的规则。

### SC-008：`UNSUPPORTED_PLATFORM_FEATURE`

适用于：

- Wayland 环境下全局快捷键不可用
- 当前平台不支持活动应用识别，无法启用黑名单功能
- 某平台不支持对应自启动实现

### SC-009：`AUTOSTART_ERROR`

系统自启动配置写入、删除或状态校验失败。

### SC-010：`WINDOW_ERROR`

主面板或设置窗口的打开、隐藏、激活失败。

### SC-011：`TRAY_ERROR`

托盘初始化或菜单状态同步失败。

### SC-012：`INTERNAL`

未分类异常兜底；前端统一展示“操作失败，请查看日志”。

## 5. 前端统一处理约定

- `SHORTCUT_CONFLICT`：在快捷键输入区内展示冲突原因，不弹全局错误 Toast
- `SETTINGS_VALIDATION_ERROR`：优先映射到字段级错误
- `UNSUPPORTED_PLATFORM_FEATURE`：展示灰色说明或禁用提示，不当作崩溃错误
- `AUTOSTART_ERROR` / `SHORTCUT_REGISTER_ERROR`：保留用户输入现场，并提示“未生效”
- `INTERNAL`：显示通用错误文案，同时建议用户打开日志目录排查

---

## 6. 成功响应约定

- 设置更新类接口保存成功后，统一返回最新 `SettingsSnapshot`
- 快捷键校验接口即使发现冲突，也优先返回结构化 `ShortcutValidationResult`；仅在无法继续时才抛错
- 平台能力查询应尽量返回 `degraded / unsupported` 结构化结果，而不是直接失败

这样可以让 `v0.5` 的 UI 更稳定地表达“当前不可用”与“真正失败”的差异。
