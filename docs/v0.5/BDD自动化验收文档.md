# v0.5 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.5 Beta 版本** 的 BDD 自动化验收场景，聚焦：

- 设置中心
- 快捷键自定义与冲突检测
- 黑名单隐私过滤
- 三平台能力与降级提示

### 1.2 BDD 编写规范

- 以用户可观察行为为中心，不描述底层实现细节
- 场景名必须能直接对应用户故事或验收标准
- 涉及平台差异时，必须显式说明平台与会话类型

---

## 2. Feature：设置中心（Settings Center）

### 场景 BDD-01-01：从托盘打开设置窗口并回显当前配置

**Given** 应用已在后台运行且托盘可用  
**When** 用户从托盘菜单点击“打开设置”  
**Then** 设置窗口被激活并显示当前设置快照  
**And** 当前主题、自启动开关、记录上限展示为最近一次保存的值

### 场景 BDD-01-02：关闭未保存设置时要求确认

**Given** 用户已在设置窗口修改主题但尚未保存  
**When** 用户直接关闭设置窗口  
**Then** 显示确认弹窗  
**And** 默认焦点落在“取消”按钮  
**And** 取消后保留当前未保存输入

### 场景 BDD-01-03：保存通用设置后主面板与托盘状态同步

**Given** 设置窗口已打开  
**When** 用户把主题改为“深色”并保存  
**Then** 设置窗口显示保存成功提示  
**And** 主面板切换为深色主题  
**And** 托盘相关状态保持一致

---

## 3. Feature：快捷键设置（Shortcut Settings）

### 场景 BDD-02-01：保存新的快捷键后立即生效

**Given** 当前平台支持全局快捷键  
**When** 用户录制并保存一个新的合法组合键  
**Then** 设置窗口提示“快捷键已更新”  
**And** 原快捷键不再触发主面板  
**And** 新快捷键可以打开主面板

### 场景 BDD-02-02：快捷键冲突时阻止保存

**Given** 当前平台支持全局快捷键  
**When** 用户录制一个与系统保留快捷键冲突的组合键  
**Then** 设置窗口显示冲突原因  
**And** 保存按钮不可点击  
**And** 当前已生效快捷键保持不变

### 场景 BDD-02-03：Wayland 下提示全局快捷键不可用

**Given** 当前环境为 `Linux Wayland`  
**When** 用户进入快捷键设置页  
**Then** 页面显示“当前会话不支持全局快捷键”提示  
**And** 录制与保存功能为禁用态  
**And** 页面提示可改用托盘入口打开主面板

## 4. Feature：隐私与黑名单（Privacy & Blacklist）

### 场景 BDD-03-01：新增黑名单规则后敏感应用复制不入库

**Given** 用户已在隐私设置页新增并启用“微信”黑名单规则  
**When** 用户在微信窗口复制一段文本  
**Then** 历史记录总数保持不变  
**And** 日志只记录“命中黑名单已跳过”，不记录原始文本

### 场景 BDD-03-02：停用黑名单规则后恢复正常采集

**Given** 已存在一条启用中的黑名单规则  
**When** 用户将该规则切换为停用  
**And** 在对应应用中再次复制文本  
**Then** 新复制内容重新进入历史记录

### 场景 BDD-03-03：重复添加同一规则会被拦截

**Given** 已存在平台和标识符相同的黑名单规则  
**When** 用户再次提交同一条规则  
**Then** 页面显示“规则已存在”错误  
**And** 列表数量不变

---

## 5. Feature：三平台能力与降级（Platform Capability）

### 场景 BDD-04-01：Windows 平台完整主流程可用

**Given** 当前平台夹具为 `Windows`  
**When** 用户复制文本、通过默认快捷键打开主面板并执行粘贴  
**Then** 主面板成功显示  
**And** 粘贴操作成功  
**And** 设置窗口中的自启动与快捷键项可正常使用

### 场景 BDD-04-02：Linux X11 平台完整主流程可用

**Given** 当前平台夹具为 `Linux X11`  
**When** 用户复制文本并通过默认快捷键打开主面板  
**Then** 主面板成功显示  
**And** 平台能力页显示快捷键与监听均为 `supported`

### 场景 BDD-04-03：Linux Wayland 提示降级而非静默失败

**Given** 当前平台夹具为 `Linux Wayland`  
**When** 用户打开设置窗口  
**Then** 通用页和快捷键页都能看到能力降级提示  
**And** 应用仍可通过托盘菜单打开主面板  
**And** 不显示误导性的“未知错误”

## 6. 非功能需求验收场景（Performance / Reliability）

### 场景 BDD-NFR-01：设置窗口首屏在预算时间内可交互

**Given** 应用已启动且存在标准设置快照  
**When** 用户打开设置窗口  
**Then** 窗口在 `300ms` 内进入可交互状态

### 场景 BDD-NFR-02：配置迁移后旧值不丢失

**Given** 用户目录中存在 `v0.3` 旧格式配置文件  
**When** 应用首次以 `v0.5` 启动  
**Then** 配置被自动迁移为新结构  
**And** 原有上限、自启动、快捷键值被正确保留

### 场景 BDD-NFR-03：主题切换不影响主面板可操作性

**Given** 用户已打开设置窗口和主面板  
**When** 用户切换主题并保存  
**Then** 主面板与设置窗口都切换到新主题  
**And** 用户仍可以继续用方向键和 `Enter` 操作主面板

---

## 7. 验收通过标准（DoD）

- 四类 Feature 的 P0 场景全部自动化通过
- 降级场景使用结构化提示，不出现静默失败
- 旧配置迁移与黑名单过滤至少各有一条端到端回归
