# v0.5 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

本文档描述 **v0.5 Beta 版本** 推荐采用的项目目录结构。

当前版本重点是：

- 新增设置窗口前端目录
- 新增设置 / 平台能力后端目录
- 为 Windows / Linux 平台适配预留清晰模块边界

### 1.2 设计原则

- **按领域分目录**，避免把设置逻辑散落在主面板目录中
- **平台差异集中到 `platform/`、`autostart/`、`shortcut/`**
- **设置窗口与主面板共享基础组件，但入口分离**
- **测试目录按功能域对齐**

## 6. 运行时生成目录

```text
runtime/
├── clipboard.db
├── config.json
├── logs/
├── images/original/
└── images/thumbs/
```

### 6.1 约束说明

- `config.json` 由 `settings/` + `config/` 统一读写
- `clipboard.db` 不保存设置与黑名单规则
- 自启动文件或注册表项不放入运行时目录，统一由平台模块管理

---

## 7. 命名规范

- 设置页分组组件统一使用 `Section` 后缀
- 与平台能力相关的类型统一使用 `Capability` 前缀或后缀
- 黑名单规则相关文件统一使用 `Blacklist` 前缀
- 平台实现文件统一使用 `macos.rs`、`windows.rs`、`linux.rs`

---

## 8. 与 v0.3 的主要差异

| 项目 | v0.3 | v0.5 |
|------|------|------|
| 前端窗口入口 | 仅主面板 | 主面板 + 设置窗口 |
| 前端状态 | 无 `SettingsStore` | 新增 `useSettingsStore.ts` |
| 后端目录 | 无 `settings/` | 新增设置领域模块 |
| 平台目录 | 以 macOS 为主 | 补齐 Windows / Linux 适配 |
| 测试目录 | 以主面板为中心 | 新增设置页与平台能力测试 |

目录结构的核心变化是：**把设置中心从“附属功能”升级为独立的一级实现面**。
