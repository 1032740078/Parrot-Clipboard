## 4. 风险与缓解

| 风险 | 说明 | 缓解方案 |
|------|------|----------|
| 平台 API 差异大 | Windows / Linux 与 macOS 实现差异明显 | 先统一能力矩阵，再按平台实现 |
| Wayland 不稳定 | 全局快捷键与活动应用识别能力受限 | 明确降级文案，不做假支持 |
| 配置迁移破坏旧用户 | 旧扁平配置升级为新结构有风险 | 单独做迁移单元与集成测试 |
| 设置同步不一致 | 主面板、托盘、设置页可能显示不同状态 | 引入统一 `settings-updated` 事件 |

---

## 5. 冲刺完成后建议

- `v0.5` 完成后优先进入跨平台真机回归，而不是继续扩搜索 / 收藏等新需求
- 若 Wayland 降级体验仍不足，可在后续版本继续增强替代入口与引导文案
- 待 `v0.5` 稳定后，再规划 `v1.0` 的性能、发布收敛与完整用户文档
