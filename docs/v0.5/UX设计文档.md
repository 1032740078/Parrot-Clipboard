# v0.5 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

`v0.5` 的 UX 重点不再是单纯打磨主面板，而是让用户可以 **放心配置、理解平台差异、清楚知道哪些能力当前可用**。

### 1.2 设计原则（v0.5 强调）

- **配置可理解**：每个设置项都要回答“改了会怎样”
- **能力可解释**：不支持的能力必须说明原因，而不是沉默失败
- **同步可感知**：设置成功后，托盘、主面板、设置窗口状态一致
- **高风险可撤销**：清空历史、关闭未保存设置必须二次确认

---

## 2. 设置窗口 UX 规范

### 2.1 信息架构

- 左侧只保留四个一级分组：通用、记录与存储、快捷键、隐私
- 每个分组顶部提供一句说明，降低配置理解成本
- 同类配置放在一起，避免“主题”和“快捷键”混排

### 2.2 保存策略

推荐采用 **分组内显式保存**：

- 用户修改当前分组后点击“保存本页”
- 成功后只提示当前分组保存成功
- 失败时保留表单现场，不自动回退输入

这样更适合桌面工具的低频设置场景，也方便定位哪个设置失败。

---

## 3. 主题体验

### 3.1 主题规则

| 模式 | 说明 |
|------|------|
| 浅色 | 设置窗口与主面板统一使用浅色底 |
| 深色 | 设置窗口与主面板统一使用深色底 |
| 跟随系统 | 根据系统主题自动切换 |

### 3.2 体验要求

- 切换后 `200ms` 内完成界面重绘
- 不允许出现大面积白屏 / 黑屏闪烁
- 主面板和设置窗口在同一时刻切换，不允许“一边深色一边浅色”

---

## 4. 快捷键设置 UX

### 4.1 录制体验

快捷键录制器应包含三种状态：

- 空闲态：展示当前快捷键
- 录制态：提示用户按下组合键
- 校验态：展示可保存、冲突、平台不支持等反馈

### 4.2 冲突反馈

| 场景 | UX 处理 |
|------|---------|
| 格式非法 | 字段下方显示错误说明 |
| 与系统快捷键冲突 | 阻止保存，提示改用其他组合 |
| Wayland 不支持全局快捷键 | 禁用保存按钮，并提示改用托盘入口 |
| 保存成功 | 成功提示 + 立即生效说明 |

### 4.3 恢复默认

- 提供“恢复平台默认值”按钮
- 默认值随平台变化：`macOS = Shift+Command+V`，`Windows / Linux = Shift+Ctrl+V`

---

## 5. 隐私设置 UX

### 5.1 黑名单规则展示

- 规则列表优先展示“应用名称”，其次展示“平台 + 标识符”
- 启用 / 停用用开关表达，删除使用危险操作按钮
- 若列表为空，显示“当前未配置黑名单应用”的空态提示

### 5.2 风险提示

- 对“黑名单命中后不会记录复制内容”做明确说明
- 对“删除规则不会恢复历史记录”做次级说明
- 对“清空历史”保持高风险强调样式

---

## 6. 平台能力降级体验

### 6.1 展示原则

- 降级信息应出现在与能力最相关的位置
- 不支持 ≠ 出错，应优先使用说明文案而非红色报错
- 若仍有替代路径，必须一并给出

### 6.2 典型场景

| 场景 | 建议反馈 |
|------|----------|
| Linux Wayland 无法注册全局快捷键 | 提示使用托盘菜单打开主面板 |
| 某平台不支持活动应用识别 | 隐私页提示黑名单能力受限 |
| 自启动注册失败 | 显示失败原因并保留当前开关状态为未生效 |

---

## 7. 反馈设计

### 7.1 操作反馈矩阵

| 操作 | 反馈 |
|------|------|
| 打开设置窗口 | 窗口激活并聚焦当前分组 |
| 保存通用设置 | Toast + 状态同步 |
| 保存记录策略 | Toast + 必要时触发补偿清理说明 |
| 快捷键冲突 | 字段内错误 + 保存按钮禁用 |
| 新增黑名单规则成功 | 列表即时新增 + 成功提示 |
| 删除黑名单规则 | 行移除 + 成功提示 |
| 清空历史成功 | 主面板空状态 + 成功提示 |

### 7.2 Toast 规范

- 成功提示应短句化，如“快捷键已更新”
- 失败提示先说动作，再说原因，如“保存失败：当前组合键已被占用”
- Toast 停留建议 `2.5s - 4s`

---

## 8. 可用性指标

| 指标 | 目标 |
|------|------|
| 设置窗口首屏可交互时间 | `< 300ms` |
| 分组切换反馈时间 | `< 80ms` |
| 快捷键冲突校验结果返回 | `< 100ms` |
| 主题切换完成时间 | `<= 200ms` |
| 黑名单规则保存 | `< 80ms` |

---

## 9. 平台特定 UX 规范

### 9.1 macOS

- 继续沿用简洁、轻量、顶部交通灯窗口习惯
- 主题跟随系统时优先匹配系统外观变化

### 9.2 Windows

- 默认快捷键提示必须使用 `Ctrl` 文案，不展示 `Command`
- 设置窗口按钮高度与间距适当放大，匹配 Windows 桌面点击习惯

### 9.3 Linux

- 显式展示当前会话类型：`X11` 或 `Wayland`
- 对 Wayland 的限制说明要更直接，避免用户误判为程序故障

`v0.5` 的 UX 成败关键，在于用户是否能快速理解：**哪些能力已经可用、哪些能力当前受限、改完设置后为什么会立刻生效或暂时无法生效**。
