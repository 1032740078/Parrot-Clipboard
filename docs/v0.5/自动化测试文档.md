# v0.5 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.5 Beta 版本** 的自动化测试策略，覆盖：

- 设置窗口与配置迁移
- 自定义快捷键与冲突检测
- 黑名单规则与隐私过滤
- Windows / Linux X11 / Wayland 平台能力矩阵

### 1.2 测试架构分层

| 层级 | 目标 | 主要工具 |
|------|------|----------|
| Rust 单元测试 | 校验配置迁移、黑名单匹配、快捷键校验、平台能力解析 | `cargo test` |
| 前端单元测试 | 校验设置页表单、主题同步、能力提示与交互状态 | `Vitest` + `RTL` |
| 后端集成测试 | 校验设置保存、自启动联动、平台适配 Mock 协同 | `cargo test` |
| BDD / E2E | 校验用户视角设置保存、冲突提示、黑名单过滤与降级说明 | `Playwright` + Tauri Mock |

### 1.3 覆盖率目标

| 范围 | 指标 |
|------|------|
| 前端 `lines / branches / functions / statements` | `>= 75%` |
| Rust 新增模块关键分支 | `>= 80%` |
| P0 用户故事自动化覆盖 | `100%` |

## 2. 后端单元测试（Rust / cargo test）

### 2.1 配置迁移

- `v0.3` 扁平结构迁移到 `v0.5` 分组结构
- 缺失字段自动补齐默认值
- 非法配置回退默认值并输出错误

### 2.2 快捷键校验

- 组合键标准化
- 非法格式识别
- 冲突结果映射到结构化状态
- Wayland 环境下返回不支持而不是静默失败

### 2.3 黑名单规则

- 规则创建 / 更新 / 删除
- 重复规则校验
- 活动应用命中黑名单后跳过采集
- 禁用规则不生效

### 2.4 平台能力解析

- macOS 返回完整支持
- Windows 返回完整支持
- Linux X11 返回完整支持
- Linux Wayland 返回降级 / 不支持矩阵

---

## 3. 前端单元测试（Vitest + RTL）

### 3.1 设置窗口交互测试

- 分组切换与初始回显
- 保存成功 / 失败反馈
- 关闭未保存设置时弹确认框

### 3.2 快捷键设置测试

- 录制器进入录制态
- 冲突态时保存按钮禁用
- 恢复默认值后表单回填平台默认快捷键

### 3.3 主题与能力提示测试

- `useThemeSync` 在 `light / dark / system` 下更新 DOM
- 能力提示条根据 `PlatformCapabilities` 正确显示文案
- Wayland 不支持全局快捷键时展示替代路径提示

### 3.4 隐私设置测试

- 黑名单规则列表渲染
- 新增、编辑、删除规则的表单交互
- 重复规则时错误信息正确展示

---

## 4. 后端集成测试

### 4.1 设置保存与自启动联动

- 保存 `launch_at_login=true/false` 后系统自启动状态同步变化
- 设置保存失败时不污染现有有效配置

### 4.2 历史策略联动

- 缩小记录上限后触发补偿清理
- 关闭图片采集后不再新增图片记录

### 4.3 隐私过滤联动

- 黑名单命中后不写入数据库
- 黑名单取消或停用后恢复正常采集

### 4.4 跨平台 Mock 联动

- Windows 自启动写注册表的 Mock 流程
- Linux `.desktop` 自启动文件的 Mock 流程
- Wayland 能力矩阵被正确返回给设置窗口

---

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- `Playwright`
- `Tauri Mock`
- 基于夹具切换平台能力矩阵，而不是在单机上真实切换操作系统

### 5.2 必测主链路

| 链路 | 说明 |
|------|------|
| 打开设置窗口并读取快照 | 确保多入口统一 |
| 保存主题与自启动 | 确保通用设置可落盘并同步 |
| 快捷键冲突阻止保存 | 确保风险可控 |
| 保存新的快捷键后立即生效 | 确保旧快捷键失效、新快捷键生效 |
| 新增黑名单规则后复制内容不入库 | 确保隐私链路闭环 |
| Linux Wayland 降级提示 | 确保用户知道替代路径 |

---

## 6. 测试数据与夹具规范

### 6.1 设置夹具

- 使用 `SEED-501` 作为标准设置快照
- 使用 `SEED-502` 作为旧配置迁移夹具

### 6.2 平台夹具

- 使用 `SEED-505` 模拟 `macOS / Windows / Linux X11 / Wayland`
- 通过注入平台能力快照替代真实系统切换

### 6.3 脱敏要求

- 黑名单应用名使用示例应用，不出现真实企业内部系统名称
- 文件路径使用夹具目录，不引用真实用户文件

---

## 7. CI 建议执行顺序

1. `pnpm run lint`
2. `pnpm run type-check`
3. `pnpm run test:coverage`
4. `pnpm run test:e2e`
5. `cargo test --manifest-path src-tauri/Cargo.toml`
6. `cargo fmt --manifest-path src-tauri/Cargo.toml --all -- --check`
7. `cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings`
