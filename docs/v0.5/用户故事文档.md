# v0.5 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 **v0.5 Beta 版本** 的用户故事。

`v0.5` 以 `v0.3` 已完成的 macOS 单平台基线为前提，当前版本只聚焦以下新增能力：

- 三平台支持：`macOS`、`Windows`、`Linux X11`
- `Wayland` 环境的能力探测与降级提示
- 独立设置窗口
- 自定义全局快捷键与冲突检测
- 主题切换与记录 / 存储策略配置
- 黑名单应用管理（不记录特定应用复制内容）

以下能力 **不属于 `v0.5` 当前版本范围**：

- 搜索栏、全文搜索与类型筛选
- 收藏、置顶、标签、分组管理
- 云同步、导入导出、插件能力
- 自动更新、账号体系、多端同步

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 跨平台办公用户 | 在 macOS、Windows、Linux 间切换工作的用户 | 三个平台的主流程一致、迁移成本低 |
| 偏好可配置用户 | 希望根据个人习惯调整主题、容量、快捷键 | 设置集中、可保存、修改后立即生效 |
| 隐私敏感用户 | 经常在 IM、密码管理器、远控工具中复制内容 | 可配置黑名单，敏感应用内容不入库 |
| 稳定性敏感用户 | 长期开机并依赖后台常驻工具 | 设置变更不破坏现有运行态，异常时有清晰提示 |

## 3. 用户故事详细说明

### E01 - 设置中心

#### US-01-01：从托盘或主面板打开设置窗口

**作为** 希望调整偏好的用户  
**我希望** 能从托盘菜单或主面板入口快速打开设置窗口  
**以便** 不离开当前桌面环境就完成配置修改。

**验收要点**：
- 设置窗口可以重复打开但只保留一个实例
- 再次触发打开时，已存在窗口应被激活并置前
- 打开设置窗口不会中断主面板当前历史记录状态

#### US-01-02：在设置窗口中按分组修改配置

**作为** 偏好可配置用户  
**我希望** 设置按“通用 / 记录与存储 / 快捷键 / 隐私”分组展示  
**以便** 我能快速定位需要修改的配置项。

**验收要点**：
- 左侧分组导航清晰可见
- 每个分组只展示当前版本支持的配置项
- 保存成功后有明确反馈，失败时保留编辑现场

#### US-01-03：设置修改后多入口状态保持一致

**作为** 长期后台运行的用户  
**我希望** 在设置窗口中修改配置后，托盘菜单、主面板与运行态立即同步  
**以便** 不会出现“设置已变更但其他入口仍显示旧状态”的问题。

**验收要点**：
- `launch_at_login` 在设置页、托盘勾选状态保持一致
- 主题切换后主面板与设置窗口一起更新
- 快捷键更新后旧快捷键立即失效，新快捷键立即生效

### E02 - 自定义快捷键

#### US-02-01：录制并保存新的全局快捷键

**作为** 有个人习惯的键盘用户  
**我希望** 在设置页直接录制新的调出面板快捷键  
**以便** 使用更符合自己习惯的按键组合。

**验收要点**：
- 不同平台展示各自默认快捷键
- 录制过程中显示当前捕获到的组合键
- 保存后立即重注册全局快捷键

#### US-02-02：快捷键冲突时阻止保存并解释原因

**作为** 不熟悉系统快捷键限制的用户  
**我希望** 当新快捷键与系统或应用已注册快捷键冲突时得到明确提示  
**以便** 避免保存一个不可用的组合键。

**验收要点**：
- 冲突状态有明显错误提示
- 不能保存处于冲突状态的快捷键
- Wayland 等不支持全局快捷键的环境需提示“能力不可用”，而不是无响应

### E03 - 隐私与黑名单

#### US-03-01：配置黑名单应用规则

**作为** 隐私敏感用户  
**我希望** 在隐私设置页添加、启用、停用、删除黑名单应用规则  
**以便** 控制哪些应用的复制内容不会被记录。

**验收要点**：
- 规则至少包含应用名称、平台标识、匹配标识符、启用状态
- 支持重复校验，避免同一应用被重复添加
- 已保存规则在重启后仍存在

#### US-03-02：黑名单应用中的复制内容不进入历史

**作为** 隐私敏感用户  
**我希望** 在黑名单应用中复制内容时，不会新增历史记录  
**以便** 避免敏感信息被工具持久化保存。

**验收要点**：
- 黑名单命中后不写入 SQLite
- 主面板历史数量保持不变
- 日志可记录“命中黑名单已跳过”，但不记录原始敏感内容

### E04 - 三平台与能力降级

#### US-04-01：在 Windows 上完成完整复制到粘贴流程

**作为** Windows 用户  
**我希望** 能像 macOS 一样完成复制、呼出主面板、选择、再次粘贴  
**以便** 工具在不同平台保持一致主流程。

**验收要点**：
- 粘贴板监听、全局快捷键、自启动、托盘都可用
- 默认快捷键为 `Shift + Ctrl + V`
- 设置窗口中的相关系统配置项可正常保存

#### US-04-02：在 Linux X11 上完成主流程，并在 Wayland 下看到清晰降级说明

**作为** Linux 用户  
**我希望** 在 X11 环境能完成完整主流程，在 Wayland 环境能得到明确限制说明  
**以便** 知道当前环境能否稳定使用全部功能。

**验收要点**：
- X11：监听、快捷键、自启动可正常工作
- Wayland：能力探测结果在设置页和托盘中可见
- 降级提示包含“当前不可用能力 + 建议替代操作”

#### US-04-03：系统能力异常时应用仍可继续浏览历史

**作为** 稳定性敏感用户  
**我希望** 即使快捷键、自启动或某平台能力不可用，应用仍可通过其他入口使用  
**以便** 单点能力失败不会导致整个工具不可用。

**验收要点**：
- 托盘或设置页可以继续打开主面板
- 设置失败只影响对应配置项，不影响历史浏览
- 错误有状态码、日志与用户可理解提示

### E05 - 主题与容量策略

#### US-05-01：调整记录数量上限与总存储上限

**作为** 偏好可配置用户  
**我希望** 可以设置文本、图片、文件的保留数量，以及图片总存储空间上限  
**以便** 更好控制本地资源占用。

**验收要点**：
- 变更后即时持久化到配置文件
- 新上限低于当前占用时，后台按规则执行淘汰
- 设置页明确展示每项配置影响范围

#### US-05-02：切换浅色 / 深色 / 跟随系统主题

**作为** 在不同环境工作的用户  
**我希望** 工具能支持浅色、深色和跟随系统主题  
**以便** 在不同桌面环境下保持舒适和一致的视觉体验。

**验收要点**：
- 主题切换对主面板与设置窗口同时生效
- 跟随系统时，系统主题变化能同步反映到应用
- 切换过程中不出现明显闪烁或布局跳动

## 5. 排除范围说明

以下能力虽在产品愿景中存在，但 **不纳入 `v0.5` 当前版本**：

- 主面板搜索态、全文检索与筛选器
- 收藏、固定置顶、标签、分组
- 导出 / 导入历史记录
- 云同步、账号体系、跨设备同步
- 自动更新与更新频道管理
- OCR、文本识别、AI 分类等高级能力

这样可以保证 `v0.5` 的重心保持在：**三平台可用 + 设置中心可用 + 隐私与快捷键可用**。
