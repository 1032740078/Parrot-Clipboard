# v1.3 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

本文档只描述 `v1.3` 的体验设计目标：空格聚焦预览、右键上下文菜单与更现代、更科技的主面板视觉风格。

### 1.2 设计原则（`v1.3` 强调）

- **先确认再执行**：预览优先于误操作，帮助用户在粘贴前快速确认完整内容
- **键鼠同权**：键盘与鼠标都能完成高频操作，不强迫用户迁移习惯
- **科技但不花哨**：视觉升级以层级、通透感、光感为主，不牺牲效率与可读性
- **局部反馈优先**：菜单失败、预览失败尽量在局部范围反馈，不打断整个主面板

---

## 2. 聚焦预览 UX

### 2.1 体验目标

- 让用户在 1 次按键内确认完整内容
- 保留主面板上下文，避免切窗打断操作流
- 关闭动作自然、可预期、低负担

### 2.2 设计规则

- 触发方式以 `Space` 为主，符合“查看 / 展开”直觉
- 预览层居中展示，宽度大于普通卡片，但保持“仍是一张卡片被放大”的心理模型
- 背景遮罩应弱化而不完全吞没主面板，让用户记得自己仍在历史列表场景中
- 预览关闭后回到原选中项，不能丢失上下文

### 2.3 内容呈现建议

- 文本：使用可滚动正文区，保留良好的行高与段落间距
- 图片：优先大图预览，必要时保留尺寸或来源信息
- 文件：使用更结构化的列表呈现文件名、数量和类型摘要

---

## 3. 右键菜单 UX

### 3.1 体验目标

- 让不熟悉快捷键的用户只用鼠标也能完成主要操作
- 保证菜单目标明确，避免“菜单作用错卡片”
- 让危险动作与普通动作在视觉上清晰分离

### 3.2 设计规则

- 右键时先更新卡片选中态，再打开菜单
- 菜单默认项顺序建议为：`预览完整内容`、`直接粘贴`、`纯文本粘贴`、分隔线、`删除记录`
- `删除记录` 使用危险色、独立分组或底部分区
- 菜单宽度应稳定，避免不同卡片上频繁跳动造成视觉干扰

### 3.3 反馈建议

- 菜单 hover 使用轻高亮 + 弱发光，而不是高饱和大面积底色
- 禁用项应可辨识但不过分抢眼
- 执行动作时可加入极轻量 loading 或点击态，避免用户怀疑未触发

---

## 4. 主面板视觉系统 UX

### 4.1 风格目标

`v1.3` 建议采用“**Glassmorphism + Tech Accent**”风格：

- 主色倾向深蓝黑、石墨灰或冷白背景
- 强调色建议使用青蓝、蓝紫、冰青等科技感色彩
- 层级通过半透明、模糊、弱描边、内发光和柔和阴影建立

### 4.2 面板设计要求

- 背景透明度建议控制在保证可读性的范围内
- 模糊存在感要有，但不能让文字边缘发虚
- 外层边框建议细而清晰，强化悬浮感
- 面板整体应呈现“精致工具面板”，而不是“大块毛玻璃”

### 4.3 卡片设计要求

- 默认态：低对比、稳定、可扫读
- Hover 态：轻微上浮、边框增强、投影略增
- Selected 态：明确高亮边框、强调色描边、局部光效
- Previewing 态：当前被预览卡片可带有关联态提示，帮助建立映射关系

---

## 5. 文案与反馈设计

### 5.1 快捷提示文案

建议在主面板底部明确出现：

- `空格 预览`
- `Enter 粘贴`
- `Shift + Enter 纯文本粘贴`
- `Delete 删除`

### 5.2 Toast 规范

- 预览失败：提示“预览内容加载失败，请稍后重试”
- 粘贴失败：提示“粘贴失败，请检查权限或当前目标应用”
- 删除成功可不强制 Toast，避免高频打扰；失败时必须提示

---

## 6. 可用性指标

- 用户能够在首次接触时理解 `空格 = 预览` 的主要语义
- 鼠标用户能够通过右键菜单完成至少 3 个主要操作
- 深色主题下科技风明显，但文本阅读不疲劳
- 视觉升级不应显著增加首次加载抖动或动画干扰

---

## 7. 可访问性与动效规范

### 7.1 可访问性

- 半透明层下文字对比度必须可读
- 危险动作不能仅靠颜色区分，应结合分组或图标语义
- 菜单层和预览层关闭方式必须简单直观

### 7.2 动效

- 卡片 hover 和预览打开使用短、轻、缓的动效
- 菜单打开建议使用轻量淡入 / 位移，不宜弹性过强
- 如系统偏好减少动态效果，应支持弱化过渡动画

---

## 8. 平台特定 UX 规范

### 8.1 macOS

- 更适合强化毛玻璃与通透感，但需注意 Tauri WebView 的性能上限
- 菜单和预览层可更贴近系统悬浮面板审美

### 8.2 Windows

- 注意阴影、边框和透明度的平衡，避免显得脏灰
- 菜单层建议更明确的边框与分隔线

### 8.3 Linux

- 不同桌面环境下模糊表现可能不一致，必须保证无模糊时仍然好看、清晰
- 若背景模糊不可用，至少保留半透明 + 边框 + 阴影的视觉层级
