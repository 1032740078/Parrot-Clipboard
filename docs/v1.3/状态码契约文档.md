# v1.3 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档只覆盖 `v1.3` 中与空格聚焦预览、右键上下文菜单、主面板视觉升级相关的状态码契约。

### 1.2 错误对象格式

前后端统一沿用 `IpcError`：

```ts
interface IpcError {
  code: ErrorCode;
  message: string;
}
```

### 1.3 错误传递机制

- 后端 `Tauri Command` 抛出结构化错误对象
- 前端 API 封装层负责日志记录与原始错误透传
- UI 层根据动作类型决定使用 Toast、局部错误态或静默忽略

## 2. 当前版本状态码清单

| 状态码编号 | 错误码 | 主要场景 |
|------------|--------|----------|
| SC-001 | `INVALID_PARAM` | 菜单或预览传入非法记录 ID / 模式 |
| SC-002 | `RECORD_NOT_FOUND` | 预览或菜单操作目标记录不存在 |
| SC-003 | `CLIPBOARD_WRITE_ERROR` | 菜单触发粘贴失败 |
| SC-004 | `KEY_SIM_ERROR` | 系统按键模拟失败，导致粘贴动作未完成 |
| SC-005 | `DB_ERROR` | 删除记录或读取详情时数据库操作失败 |
| SC-006 | `FILE_ACCESS_ERROR` | 图片或文件详情资源不可访问 |
| SC-007 | `IMAGE_PROCESS_ERROR` | 图片预览处理失败 |
| SC-008 | `WINDOW_ERROR` | 预览层或主面板显隐相关窗口操作失败 |
| SC-009 | `UNSUPPORTED_PLATFORM_FEATURE` | 当前平台不支持某些关联能力 |
| SC-010 | `INTERNAL` | 未归类内部错误 |

## 3. 状态码详细说明

### SC-001：`INVALID_PARAM`

- **触发场景**：调用 `paste_record` 时 `mode` 非法，或预览详情查询使用了无效 `id`
- **前端处理**：显示通用错误 Toast，并关闭执行中状态；不强制关闭主面板
- **备注**：`纯文本粘贴` 应在前端先做可用性判断，尽量避免触发此类错误

### SC-002：`RECORD_NOT_FOUND`

- **触发场景**：记录已被删除，但用户仍尝试打开预览或执行菜单动作
- **前端处理**：关闭对应菜单 / 预览层，刷新列表并提示“记录不存在或已删除”
- **备注**：若由事件先一步同步删除，可直接静默回收局部 UI 状态

### SC-003：`CLIPBOARD_WRITE_ERROR`

- **触发场景**：菜单触发 `直接粘贴` 或 `纯文本粘贴` 时写入系统剪贴板失败
- **前端处理**：保留主面板，弹出错误 Toast，不隐藏窗口
- **备注**：与键盘粘贴保持一致处理策略

### SC-004：`KEY_SIM_ERROR`

- **触发场景**：系统按键模拟失败，无法完成最终粘贴动作
- **前端处理**：保留当前界面，展示可理解错误文案
- **备注**：该错误不应改变当前选中项或菜单锚点信息

### SC-005：`DB_ERROR`

- **触发场景**：删除记录、读取详情或同步列表时数据库出现错误
- **前端处理**：展示错误 Toast；若为预览详情读取失败，则优先局部错误态
- **备注**：不因为详情读取失败而清空整个主面板

### SC-006：`FILE_ACCESS_ERROR`

- **触发场景**：文件型或图片型记录的原始文件不可访问
- **前端处理**：预览层进入降级展示，显示不可访问提示
- **备注**：不影响列表其他记录操作

### SC-007：`IMAGE_PROCESS_ERROR`

- **触发场景**：图片大图预览、缩略图转换或资源处理失败
- **前端处理**：显示图片预览失败占位态，并保留关闭能力
- **备注**：不允许出现浏览器破损图片图标直出

### SC-008：`WINDOW_ERROR`

- **触发场景**：隐藏面板、聚焦窗口、事件广播等窗口级动作失败
- **前端处理**：记录日志并提示用户稍后重试
- **备注**：主要影响粘贴成功后的关闭链路

### SC-009：`UNSUPPORTED_PLATFORM_FEATURE`

- **触发场景**：某些平台相关能力受限，导致部分行为无法完全落地
- **前端处理**：给出降级提示，但尽可能保留主流程可用性
- **备注**：`v1.3` 的预览与右键菜单本身应优先做到平台无关

### SC-010：`INTERNAL`

- **触发场景**：未归类错误或不可预期异常
- **前端处理**：统一错误 Toast + 日志采集
- **备注**：作为兜底错误，不承载具体业务分支

## 4. Command / Event 对应矩阵

| Command / Event | 可能关联的状态码 | `v1.3` 使用说明 |
|-----------------|------------------|-----------------|
| `get_record_detail` | `SC-001` `SC-002` `SC-005` `SC-006` `SC-007` `SC-010` | 打开预览时按需读取完整内容 |
| `paste_record` | `SC-001` `SC-003` `SC-004` `SC-009` `SC-010` | 菜单粘贴与键盘粘贴共用 |
| `delete_record` | `SC-001` `SC-002` `SC-005` `SC-010` | 菜单删除动作复用 |
| `hide_panel` | `SC-008` `SC-010` | 粘贴成功后维持既有隐藏逻辑 |
| `clipboard:record-deleted` | 无结构化错误 | 若命中预览 / 菜单目标，前端需主动关闭相应 UI |

---

## 5. 前端统一处理约定

- 预览详情失败优先展示**局部错误态**，不要直接把用户踢出主面板
- 菜单动作失败优先使用 Toast 反馈，并关闭执行中状态
- 删除类动作若后端返回 `RECORD_NOT_FOUND`，前端可视作“目标已不存在”，同步移除局部 UI
- 视觉升级不应引入新的错误码分类；样式问题应通过 CSS 回退策略而非 IPC 错误表达

---

## 6. 成功响应约定

- `预览打开`：属于纯前端状态变更，不额外定义后端成功码
- `菜单弹出`：属于纯前端状态变更，不额外定义后端成功码
- `直接粘贴 / 纯文本粘贴 / 删除记录`：继续沿用既有命令成功响应结构
- `视觉升级`：属于样式层默认行为，不额外定义 API 成功状态
