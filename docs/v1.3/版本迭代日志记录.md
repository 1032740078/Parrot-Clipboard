# v1.3 版本迭代日志记录

## 1. 迭代总览

`v1.3` 当前阶段先完成文档基线建设，锁定以下 3 个版本目标：

- 选中卡片后按空格，可在屏幕中心打开完整内容预览
- 卡片支持右键菜单，通过鼠标完成预览、粘贴、纯文本粘贴、删除等主要操作
- 主面板与卡片样式升级为半透明玻璃科技风，提升现代感与质感

### 2.29 2026-03-10 `v1.3-refine-image-ocr-layout`

**变更类型**：`fix(ocr-layout)`

**主要变更**：

- 更新 `src-tauri/src/ocr/mod.rs`，OCR 不再直接读取落盘后的缓存图，而是把剪贴板里的原始 RGBA 像素临时导出为无损 `bmp` 后再交给 `ocr-rs` sidecar，尽量避免 OCR 前再经历一次有感知的二次编码
- OCR 输出从 `text` 切换为 `json`，按识别框的 `top/left/width/height` 重新拼装文本，只在图片中本来分成不同行的位置插入换行；同一行内的文本块不再无脑换行
- 补充 `src-tauri/src/ocr/mod.rs` 单元测试，覆盖“同一行不换行”“多行才换行”“英文词组自动补空格”“空白块忽略”等重建规则

**产出结果**：

- 图片纯文本粘贴时，OCR 输入改为无损 `bmp` 临时文件，避免因为 OCR 前再次重编码而进一步影响识别
- OCR 识别出的文本会尽量贴近图片里的原始排版，不再把同一行的内容拆成很多换行
- 英文或数字词组在同一行存在明显间距时会自动补空格，减少 `Helloworld` 这类黏连结果

**验证结果**：

- 已通过 `cargo test --manifest-path src-tauri/Cargo.toml ocr::tests::`
- 已通过 `cargo test --manifest-path src-tauri/Cargo.toml paste::`
