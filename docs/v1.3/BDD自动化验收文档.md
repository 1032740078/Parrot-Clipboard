# v1.3 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.3` 当前版本的 BDD 自动化验收场景：空格聚焦预览、右键上下文菜单、主面板视觉升级。

### 1.2 BDD 编写规范

- 使用 `Given / When / Then` 描述用户视角下的真实场景
- 每个场景只验证 1 条主业务结论
- 视觉场景优先验证“可见层级 / 状态差异 / 可读性”，避免把 CSS 实现细节写死

---

## 2. Feature：空格聚焦预览（Focus Preview）

### 场景 BDD-31-01：选中卡片后按空格打开居中预览

- **Given** 主面板已打开，且存在一张已选中文本卡片
- **When** 用户按下空格键
- **Then** 屏幕中心出现该卡片的预览层
- **And** 主面板保持可见，但预览层获得视觉焦点

### 场景 BDD-31-02：预览层展示完整内容而不是列表摘要

- **Given** 当前选中的是一条长文本记录
- **When** 用户按下空格打开预览
- **Then** 预览层显示完整文本内容
- **And** 内容超出可视高度时允许滚动查看

### 场景 BDD-31-03：通过 `Esc`、空格或遮罩关闭预览

- **Given** 预览层已经打开
- **When** 用户按 `Esc`、再次按空格或点击遮罩
- **Then** 预览层关闭
- **And** 原选中卡片保持不变

---

## 3. Feature：右键上下文菜单（Context Menu）

### 场景 BDD-32-01：右键卡片时先选中再弹出菜单

- **Given** 主面板中存在多张卡片
- **When** 用户右键一张当前未选中的卡片
- **Then** 该卡片成为当前选中项
- **And** 菜单在鼠标附近弹出

### 场景 BDD-32-02：通过菜单执行“直接粘贴”

- **Given** 用户已经右键打开某张卡片的菜单
- **When** 用户点击 `直接粘贴`
- **Then** 系统执行原始内容粘贴
- **And** 成功后主面板按既有规则隐藏

### 场景 BDD-32-03：文本卡片支持“纯文本粘贴”

- **Given** 用户右键打开一张文本卡片的菜单
- **When** 用户点击 `纯文本粘贴`
- **Then** 系统以 `plain_text` 模式执行粘贴
- **And** 成功提示与键盘 `Shift + Enter` 保持一致

### 场景 BDD-32-04：非文本卡片不允许“纯文本粘贴”

- **Given** 用户右键打开一张图片卡片或文件卡片的菜单
- **When** 用户查看菜单项
- **Then** `纯文本粘贴` 显示为禁用态或不展示
- **And** 用户不能通过鼠标误触发该动作

### 场景 BDD-32-05：通过菜单删除记录

- **Given** 用户右键打开某张卡片的菜单
- **When** 用户点击 `删除记录`
- **Then** 该记录从列表中移除
- **And** 菜单自动关闭

## 4. Feature：主面板视觉升级（Glass Tech UI）

### 场景 BDD-33-01：主面板采用半透明科技风背景

- **Given** 主面板处于默认显示状态
- **When** 用户观察面板背景
- **Then** 面板具备半透明、弱模糊、边框与层次感
- **And** 文本内容仍清晰可读

### 场景 BDD-33-02：卡片在 hover 与 selected 状态下有明确差异

- **Given** 主面板中至少有两张卡片
- **When** 用户分别观察默认态、悬停态与选中态
- **Then** 三种状态存在清晰但统一的视觉差异
- **And** 选中态不只依赖单一背景色变化

### 场景 BDD-33-03：预览层与菜单层级正确

- **Given** 用户打开预览层或右键菜单
- **When** 页面进入叠层状态
- **Then** 预览层、菜单层、Toast 层之间不存在错误遮挡
- **And** 点击行为命中符合当前最高层语义

## 5. 非功能需求验收场景（Performance / Reliability）

### 场景 BDD-NFR-31：打开预览不会阻塞主面板基本渲染

- **Given** 主面板已有可见卡片列表
- **When** 用户按空格打开预览
- **Then** 主面板列表不会出现整屏冻结或闪烁
- **And** 预览详情失败时只影响预览层自身

### 场景 BDD-NFR-32：连续右键不同卡片时菜单目标始终正确

- **Given** 主面板存在连续多张卡片
- **When** 用户连续右键不同卡片
- **Then** 同一时刻仅有一个菜单存在
- **And** 菜单动作始终作用于最新右键的卡片

### 场景 BDD-NFR-33：视觉升级不降低可读性

- **Given** 主面板切换浅色与深色主题
- **When** 用户浏览文本、图片、文件卡片
- **Then** 关键文案对比度满足可读标准
- **And** 半透明背景不会掩盖主要信息

## 7. 通过标准

- 所有 `P0` 需求至少对应 1 条自动化 BDD 场景
- 所有关键鼠标链路与键盘链路场景全部通过
- 视觉相关场景至少具备可见性 / 层级 / 可读性断言之一
- 不允许因为 `v1.3` 新增能力导致 `v1.2` 关键主链路回归失败
