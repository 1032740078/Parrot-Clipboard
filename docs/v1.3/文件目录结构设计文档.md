# v1.3 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

本文档只描述 `v1.3` 为支持空格聚焦预览、右键上下文菜单、主面板视觉升级所关注的目录结构与文件归属建议。

### 1.2 设计原则

- 优先沿用现有 `MainPanel` 目录边界，不把新交互堆到 `index.tsx` 单文件中
- 新能力优先按“组件 / hook / store / 测试”拆分，避免隐式耦合
- 视觉令牌与业务行为解耦，样式收敛到单独模块或统一样式层

---

## 3. 前端目录设计（`src/`）

### 3.1 `v1.3` 重点文件

| 文件 / 目录 | 作用 |
|-------------|------|
| `src/components/MainPanel/index.tsx` | 主面板容器，组织列表、预览层、菜单层与交互入口 |
| `src/components/MainPanel/CardList.tsx` | 卡片列表渲染、右键事件分发 |
| `src/components/MainPanel/TextCard.tsx` | 文本卡片视觉升级与预览态映射 |
| `src/components/MainPanel/ImageCard.tsx` | 图片卡片大图预览与视觉升级 |
| `src/components/MainPanel/FileCard.tsx` | 文件卡片完整信息预览与视觉升级 |
| `src/hooks/useKeyboard.ts` | 新增空格预览快捷行为 |
| `src/stores/useUIStore.ts` | 新增预览层与菜单层运行态 |
| `src/index.css` | 半透明、玻璃拟态、光效与层级样式 |

### 3.2 推荐新增辅助文件

```text
src/components/MainPanel/
├── CardContextMenu.tsx      # 卡片右键菜单
├── PreviewOverlay.tsx       # 居中预览层
├── PanelSurface.tsx         # 统一面板玻璃外壳（可选）
└── visualTokens.ts          # 视觉令牌、层级与状态映射（可选）

src/hooks/
├── useCardContextMenu.ts    # 菜单打开、定位、关闭、动作分发
└── usePreviewOverlay.ts     # 预览层打开、详情加载、关闭
```

### 3.3 推荐约束

- `index.tsx` 只负责主面板编排，不直接内联全部菜单与预览实现
- 菜单动作解析优先收敛到 hook 或 service，不在多个组件中重复写 `switch`
- 视觉状态类名建议通过辅助函数或令牌映射输出，避免散落硬编码

## 4. 后端目录设计（`src-tauri/src/`）

### 4.1 `v1.3` 重点文件

| 文件 / 目录 | 作用 |
|-------------|------|
| `src-tauri/src/ipc/commands.rs` | 继续暴露既有 `get_record_detail / paste_record / delete_record` |
| `src-tauri/src/clipboard/` | 继续提供记录摘要与详情数据 |
| `src-tauri/src/paste/` | 继续承接原始 / 纯文本粘贴 |

### 4.2 模块归属说明

- `v1.3` 不建议为右键菜单或预览新建后端模块
- 所有新增能力都应建立在既有命令面之上
- 若后续需要新增“详情缓存”或“预览专用 DTO”，再评估是否扩展 `clipboard/` 或 `ipc/`

- `v1.3` 不要求新增专门后端测试目录

---

## 6. 运行时生成目录

### 6.1 推荐布局

`v1.3` 不新增新的运行时目录，继续沿用历史图片、日志与数据库目录。

### 6.2 约束说明

- 视觉升级不应依赖额外下载的本地 UI 资源包
- 预览层优先使用现有详情与图片资产，不新增缓存目录

---

## 7. 命名规范

- 组件文件使用 `PascalCase.tsx`，如 `PreviewOverlay.tsx`
- Hook 使用 `useXxx.ts`，如 `usePreviewOverlay.ts`
- 视觉令牌文件建议使用含义清晰命名，如 `visualTokens.ts`
- 测试文件命名为 `*.test.tsx` / `*.test.ts`

---

## 8. 与 v1.2 的主要差异

- `v1.2` 更偏向列表卡片层面的交互补齐，`v1.3` 开始把“叠层预览”和“菜单层”从主列表中独立出来
- `v1.3` 建议新增更明确的视觉样式载体文件，降低玻璃拟态样式分散在多个组件中的维护成本
- 后端目录结构在 `v1.3` 基本保持不变，变更重心仍在前端模块编排
