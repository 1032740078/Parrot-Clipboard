# v1.3 冲刺任务文档

## 1. 冲刺目标

在不扩张后端命令面和数据库结构的前提下，完成 `v1.3` 的 3 个核心目标：

- 支持选中卡片按空格进行居中完整内容预览
- 支持卡片右键菜单，通过鼠标完成主要操作
- 完成主面板与卡片的半透明科技风视觉升级

## 3. 各批次任务详情

### 3.1 第 0 批：文档基线

**目标**：锁定范围、统一语言、契约与测试口径。

**任务**：
- 完成 `docs/v1.3` 全套设计文档
- 更新 `docs/项目决策记录文档.md`
- 新增 `docs/v1.3/版本迭代日志记录.md`

### 3.2 第 1 批：空格聚焦预览

**目标**：建立“先确认再执行”的完整预览闭环。

**任务**：
- 扩展 `useUIStore`，新增预览层运行态
- 在 `useKeyboard.ts` 中接入 `Space` 打开 / 关闭预览逻辑
- 新增 `PreviewOverlay` 组件，支持完整内容、局部加载与错误态
- 复用 `get_record_detail` 完成详情按需读取

### 3.3 第 2 批：右键上下文菜单

**目标**：补齐纯鼠标操作路径。

**任务**：
- 扩展卡片右键事件，右键时先切换选中项
- 新增 `CardContextMenu` 组件与定位逻辑
- 菜单动作统一复用 `paste_record`、`delete_record`
- 补齐文本 / 图片 / 文件卡片动作可用性判断

### 3.4 第 3 批：视觉升级

**目标**：让主面板从功能面板升级为更现代的桌面浮层。

**任务**：
- 升级主面板背景为半透明玻璃质感
- 增强卡片边框、阴影、强调色与层级
- 统一菜单、预览层、Toast 的层叠视觉语言
- 完成深 / 浅主题下的对比度与可读性调整

### 3.5 第 4 批：测试与回归

**目标**：保证 `v1.3` 新能力上线后不破坏 `v1.2` 主链路。

**任务**：
- 增加预览层、右键菜单、视觉状态单元测试
- 增加主面板集成测试与 BDD 场景
- 增加视觉回归测试或截图基线
- 完成 `lint / type-check / test / cargo test` 回归

## 4. 风险与缓解

| 风险 | 说明 | 缓解策略 |
|------|------|----------|
| 预览层与键盘事件冲突 | 空格可能误触多次打开 / 关闭 | 增加预览打开态判断与事件短路 |
| 菜单目标错位 | 连续右键不同卡片时菜单状态不同步 | 菜单状态以 `record_id + anchor` 为单一真源 |
| 视觉升级影响可读性 | 半透明与模糊过重 | 以对比度和信息优先级为第一准则 |
| 菜单和预览叠层混乱 | 多层浮层遮挡关系错误 | 预先定义清晰 `z-index` 层级策略 |
| 回归旧快捷键能力 | 新交互影响 `Enter`、`Delete`、数字快选 | 补齐集成测试与 BDD 自动化 |

---

## 5. 冲刺完成后建议

1. 先通过真实桌面环境验证深 / 浅主题和不同壁纸背景下的可读性
2. 若 `v1.3` 体验稳定，可在后续版本评估开放“外观偏好设置”
3. 若用户对预览满意度高，再评估是否扩展详情独立窗口或固定预览模式
