# v1.3 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档只覆盖 `v1.3` 的空格聚焦预览、右键上下文菜单与主面板视觉升级的自动化测试策略。

### 1.2 测试架构分层

- **后端单元测试**：确认既有详情、粘贴、删除链路对 `v1.3` 入口仍稳定可用
- **前端单元测试**：验证键盘事件、菜单状态、预览层组件与视觉状态
- **集成测试**：验证主面板、store、API mock 与组件之间的完整交互链路
- **BDD / E2E**：验证面向用户的真实操作路径与主要验收场景
- **视觉回归测试**：验证半透明与卡片状态风格不会被样式回归破坏

### 1.3 覆盖率目标

- 前端代码覆盖率继续满足项目既有门槛：`lines / branches / functions / statements >= 75%`
- `v1.3` 新增组件、hook、store 分支覆盖率建议不低于 `85%`

---

## 2. 后端单元测试（Rust / cargo test）

### 2.1 详情查询回归

重点验证：

- `get_record_detail` 对文本、图片、文件三类记录都能返回完整详情
- 图片详情在原图可读、缩略图失败等情况下仍能稳定返回可消费字段
- 记录不存在时返回结构化错误

### 2.2 粘贴与删除回归

重点验证：

- `paste_record(original)` 与 `paste_record(plain_text)` 维持既有行为
- `delete_record` 在普通记录与不存在记录上的返回符合预期
- `v1.3` 不新增后端命令，因此后端测试以回归为主

---

## 3. 前端单元测试（Vitest + RTL）

### 3.1 键盘预览交互

建议覆盖：

- 选中卡片后按空格打开预览
- 无选中卡片时按空格静默忽略
- 预览打开后再次按空格或按 `Esc` 关闭
- 预览打开期间不触发粘贴逻辑

### 3.2 右键菜单交互

建议覆盖：

- 右键卡片后先切换选中态，再打开菜单
- 文本卡片显示 `纯文本粘贴` 可用态
- 图片 / 文件卡片对 `纯文本粘贴` 进入禁用或隐藏态
- 点击菜单外区域后关闭菜单

### 3.3 预览层组件

建议覆盖：

- 文本完整内容渲染
- 图片大图预览与失败占位
- 文件列表完整渲染
- 详情加载失败时显示局部错误态

### 3.4 视觉状态与样式令牌

建议覆盖：

- 主面板根节点具备半透明 / 模糊 / 边框类名或样式令牌
- 卡片在默认、hover、selected、previewing 状态下 class 变化正确
- 菜单与预览层的层级 class 正确，避免相互遮挡

---

## 4. 集成测试

### 4.1 空格预览链路

- 使用主面板真实组件树
- 通过键盘选择记录后按空格打开预览
- mock `get_record_detail` 验证详情拉取与关闭行为

### 4.2 右键菜单动作链路

- 右键打开菜单后点击 `直接粘贴`
- 右键打开菜单后点击 `纯文本粘贴`
- 右键打开菜单后点击 `删除记录`
- 验证成功 / 失败时的 Toast、面板与菜单状态

### 4.3 预览与事件协同链路

- 预览打开后收到 `clipboard:record-deleted`，预览自动关闭
- 菜单打开后收到目标记录删除事件，菜单自动关闭
- 收到 `clipboard:record-updated` 后，预览头部摘要可同步刷新

### 4.4 视觉回归链路

- 针对 `SEED-1303` 进行截图回归
- 核对深色主题下半透明背景与高亮边框是否稳定
- 核对浅色主题下文字对比度与卡片层次是否可读

---

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- 继续使用 `Playwright` 或项目当前 E2E 工具
- 对关键视觉状态补充截图断言或视觉快照
- 对菜单位置与预览层展示使用可访问性查询 + 样式断言联合校验

### 5.2 必测主链路

- 键盘选中 -> 空格预览 -> 关闭预览
- 右键卡片 -> 菜单粘贴 -> 面板隐藏
- 右键文本卡片 -> 纯文本粘贴 -> 成功反馈
- 右键卡片 -> 删除记录 -> 列表更新
- 深 / 浅主题下视觉样式与可读性

---

## 6. 测试数据与夹具规范

### 6.1 记录夹具

- 统一从 `src/__tests__/fixtures/clipboardRecords.ts` 扩展 `v1.3` 所需数据工厂
- 长文本、图片失败、文件多项列表都应有稳定夹具

### 6.2 能力夹具

- 不需要为 `v1.3` 新增平台能力类型
- 若视觉效果涉及不同平台表现差异，可通过 theme / userAgent mock 进行最小模拟

### 6.3 脱敏要求

- 图片、文件名、文本内容均使用脱敏示例
- 不得使用真实账号、路径、聊天记录或隐私截图

---

## 7. CI 建议执行顺序

1. `pnpm run lint`
2. `pnpm run type-check`
3. `pnpm run test`
4. 视觉回归 / BDD 自动化
5. `cargo test --manifest-path src-tauri/Cargo.toml`

---

## 8. 通过标准（Definition of Done）

- `v1.3` 新增交互均有单元测试与至少 1 条集成测试覆盖
- 关键 BDD 场景全部通过
- 视觉回归在深 / 浅主题下均无明显异常
- 不因新增预览与菜单逻辑破坏 `v1.2` 的快选、快贴、双击粘贴能力
