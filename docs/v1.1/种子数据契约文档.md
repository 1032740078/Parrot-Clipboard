# v1.1 种子数据契约文档

## 1. 文档概述

### 1.1 版本范围

`v1.1` 种子数据只服务于当前版本的 4 类能力验证：自动隐藏、`Dock` 覆盖、快速粘贴、自动横向滚动。

### 1.2 设计原则

- 数据必须可复现同一组序号映射结果
- 需要同时提供“短列表”“长列表”“权限缺失”“多显示器布局”四类夹具
- 严禁包含真实隐私内容

---

## 2. 标准种子数据集

### 2.1 SEED-1101：主面板快速粘贴基线集

用途：验证 `1~9` 快选与 `Command + 1~9` 快贴的顺序一致性。

| 顺序 | 类型 | 标题示例 | 说明 |
|------|------|----------|------|
| 1 | `text` | 最近日报结论 | 对应 `Command + 1` |
| 2 | `text` | 客户地址示例 | 对应 `Command + 2` |
| 3 | `image` | 截图示例 A | 对应 `Command + 3` |
| 4 | `files` | 合同目录示例 | 对应 `Command + 4` |
| 5 | `text` | SQL 片段示例 | 对应 `Command + 5` |
| 6 | `text` | 工单编号示例 | 对应 `Command + 6` |
| 7 | `image` | 截图示例 B | 对应 `Command + 7` |
| 8 | `files` | 项目路径示例 | 对应 `Command + 8` |
| 9 | `text` | 回复模板示例 | 对应 `Command + 9` |
| 10-12 | 混合 | 备用记录 | 用于越界与滚动组合场景 |

### 2.2 SEED-1102：权限缺失快贴集

用途：验证 `macOS` 权限缺失时，`Command + 数字` 只提示、不执行。

字段要求：

- `permission.platform = macos`
- `permission.accessibility = missing`
- 至少包含 `3` 条记录

### 2.3 SEED-1103：自动横向滚动长列表集

用途：验证左右切换、数字快选和虚拟滚动下的“选中项可见”。

建议内容：

- 记录总数：`40` 或 `60`
- 类型以文本为主，夹杂少量图片 / 文件记录
- 卡片标题长度要有长短差异，避免只测理想尺寸

### 2.4 SEED-1104：Dock 覆盖布局夹具集

用途：验证主面板 frame 计算不再依赖 `work_area`。

| 夹具 ID | 布局说明 |
|---------|----------|
| `LAYOUT-1104-A` | 底部 `Dock` |
| `LAYOUT-1104-B` | 左侧 `Dock` |
| `LAYOUT-1104-C` | 右侧 `Dock` |
| `LAYOUT-1104-D` | 双显示器 + 主屏底部 `Dock` |
| `LAYOUT-1104-E` | 双显示器混合缩放 |

### 2.5 SEED-1105：失焦自动隐藏事件集

用途：验证显隐同步事件与运行态收敛。

字段建议：

- `initial_panel_visible = true`
- `blur_reason = focus_lost`
- `expected_panel_visible = false`
- `expected_restore_focus = true`

---

## 3. 类型覆盖要求

### 3.1 历史记录覆盖

- 文本、图片、文件三种记录至少各覆盖一次
- 前 9 条记录必须顺序固定、主键稳定，避免快贴序号在不同环境下漂移

### 3.2 运行态覆盖

- 面板可见 / 不可见
- 权限已授权 / 未授权
- `Dock` 底部 / 左侧 / 右侧
- 小列表 / 长列表 / 虚拟滚动

---

## 4. 资产文件契约

### 4.1 推荐目录

- `src/__tests__/fixtures/clipboardRecords.ts`
- `src/__tests__/fixtures/layouts/`
- `e2e/fixtures/v1.1/`

### 4.2 路径约束

- 图片路径使用仓库内可重复访问的脱敏样例文件
- 文件记录路径使用虚拟演示路径，不依赖真实用户目录

---

## 5. 加载约定

### 5.1 开发模式加载

- 前端单测优先直接复用 `fixtures/clipboardRecords.ts`
- `Dock` 布局夹具通过函数参数注入，不要求真实系统环境

### 5.2 测试模式加载

- BDD 场景按场景名组合加载 `SEED-1101 ~ SEED-1105`
- 不同场景之间必须重置运行态，避免 `panel_visible` 与 `selectedIndex` 串场

---

## 6. 覆盖的测试场景

| 种子集 | 覆盖场景 |
|--------|----------|
| `SEED-1101` | 快选 / 快贴序号一致性 |
| `SEED-1102` | 权限缺失快贴被阻止 |
| `SEED-1103` | 自动横向滚动与虚拟滚动 |
| `SEED-1104` | `Dock` 覆盖与多显示器定位 |
| `SEED-1105` | 失焦自动隐藏与显隐同步 |

---

## 7. 数据清理约定

- 所有 `v1.1` 种子数据仅用于开发、测试和演示
- 测试结束后应重置内存态，避免快贴序号被前序测试污染
- 如写入图片 / 临时文件资产，必须在测试结束后清理
