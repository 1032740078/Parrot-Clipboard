# v1.1 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 **v1.1** 当前版本的新增与调整能力。

`v1.1` 默认继承 `v1.0` 已完成的发布稳定性、权限引导、关于页、更新检查、虚拟滚动、图片缓存与孤立文件清理能力；本版本只聚焦以下 4 项交互与窗口体验问题：

- 主面板在 **失去焦点时自动隐藏**，恢复“呼出即用、离开即收”的桌面工具体验
- 不再通过 **缩短主面板高度** 来避让 `macOS Dock`，改为允许主面板直接覆盖 `Dock`，并调整为可完整展示卡片的固定高度 `336px`，卡片高度同步提升到 `192px`
- 在主面板可见时，支持 **`Command + 1~9` 直接粘贴对应序号记录**
- 使用左右方向键切换记录时，若目标卡片超出当前可见区域，列表需要 **自动横向滚动**，确保选中项始终可见

以下能力 **不属于 `v1.1` 当前版本范围**：

- 全文搜索、标签、收藏、置顶、分组
- 云同步、导入导出、账号体系、插件系统
- `Windows / Linux` 专属的 `Ctrl + 数字` 快贴方案
- 新增业务表、设置页信息架构重构、记录模型重构

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 高速粘贴用户 | 依赖键盘完成频繁复制 / 粘贴的用户 | 呼出快、切换快、一步直贴 |
| 多显示器 macOS 用户 | 使用外接显示器并保留 Dock 的用户 | 面板不被裁剪、位置稳定 |
| 重度历史用户 | 历史记录多，需要靠左右键快速浏览的用户 | 选中项永远可见、不会迷失 |
| 权限受限用户 | 尚未完成辅助功能授权的用户 | 失败有提示、不会无反馈 |

## 3. 用户故事详细说明

### E01 - 主面板显隐一致性

#### US-01-01：主面板失去焦点后自动隐藏

**作为** 高频使用快捷键呼出主面板的用户  
**我希望** 当我点击其他应用、切换窗口或把焦点移出主面板时，主面板自动收起  
**以便** 不需要再额外按一次 `Esc`，并且不会长期挡住工作界面。

**验收要点**：
- 主面板失去焦点后应在 `100ms` 内触发隐藏
- 自动隐藏后，焦点应回到用户原本操作的前台应用
- 自动隐藏与 `Esc` 隐藏共用同一套状态收敛路径
- 自动隐藏不能误触发清空历史、删除记录、权限弹窗等副作用

#### US-01-02：重新呼出时状态一致

**作为** 刚刚因失焦而收起主面板的用户  
**我希望** 再次通过全局快捷键呼出时，主面板状态保持可预测  
**以便** 不会出现前端还认为可见、但原生窗口已经隐藏的错位问题。

**验收要点**：
- 原生窗口显隐状态必须是最终真相来源
- 前端状态必须在显隐变化后及时同步
- 主面板再次呼出后仍能正确响应左右切换、数字快选与 `Enter` 粘贴

### E02 - Dock 覆盖与尺寸恢复

#### US-02-01：主面板恢复原始高度并覆盖底部 Dock

**作为** 使用底部 `Dock` 的 `macOS` 用户  
**我希望** 主面板恢复原始高度并直接盖在 `Dock` 上层  
**以便** 卡片底部内容不再因为“避让 Dock”而被裁掉。

**验收要点**：
- 主面板高度调整为新的设计基线 `336px`，卡片高度同步提升到 `192px`
- 主面板底边对齐物理屏幕底部，而不是 `visibleFrame / work_area` 的底边
- `Dock` 位于底部时，面板底部内容完整可见，不出现裁切
- 不允许再通过缩小主面板高度规避 `Dock`

#### US-02-02：侧边 Dock 与多显示器场景表现一致

**作为** 使用左侧 / 右侧 `Dock` 或多显示器的 `macOS` 用户  
**我希望** 主面板仍能覆盖 `Dock` 并稳定贴合当前目标屏幕底部  
**以便** 不会因为不同桌面布局再次出现尺寸异常或显示错位。

**验收要点**：
- `Dock` 位于左 / 右侧时，主面板宽度也以物理显示区域为准
- 混合缩放、多显示器来回切换时，主面板仍显示在当前目标显示器
- 面板阴影、圆角和点击区域不能被截断

### E03 - 快速粘贴捷径

#### US-03-01：`Command + 数字` 一步直贴前 9 条记录

**作为** 熟悉键盘操作的 `macOS` 用户  
**我希望** 在主面板打开时按下 `Command + 1~9` 直接粘贴对应记录  
**以便** 不必先数字选中、再按 `Enter`，减少一次确认动作。

**验收要点**：
- `Command + 1~9` 直接触发对应序号记录的粘贴
- `1~9` 单独按下仍保留“仅选中不粘贴”的现有行为
- 粘贴成功后主面板自动隐藏，并将记录置顶
- 当记录数量不足时，超出范围的组合键静默忽略

#### US-03-02：权限缺失时快速粘贴可降级提示

**作为** 尚未完成辅助功能授权的 `macOS` 用户  
**我希望** 使用 `Command + 数字` 时得到明确提示而不是无反应  
**以便** 知道为什么无法完成直接粘贴。

**验收要点**：
- 权限缺失时阻止快速粘贴实际执行
- 保持主面板可见，并给出明确提示或引导入口
- 失败提示不暴露敏感记录正文

### E04 - 选中项可见性保障

#### US-04-01：左右切换时自动滚动到选中卡片

**作为** 依赖左右方向键浏览大量历史的用户  
**我希望** 当选中项超出可见区域时，列表自动滚动让它出现在视口内  
**以便** 我始终知道当前将要粘贴的是哪一条记录。

**验收要点**：
- `ArrowLeft / ArrowRight` 切换后，选中卡片必须保持在可见区域内
- 小列表和虚拟滚动列表都要使用一致的“确保选中项可见”策略
- 自动滚动不能引发闪烁、白屏或重复跳动

#### US-04-02：快速粘贴与自动滚动行为兼容

**作为** 同时使用左右切换、数字快选和 `Command + 数字` 的用户  
**我希望** 列表滚动、选中高亮与粘贴动作保持一致  
**以便** 不会出现“高亮的是 A，实际粘贴的是 B”的认知错位。

**验收要点**：
- 数字快选后若目标超出可见区，也要自动滚动到位
- `Command + 数字` 的序号定义必须与 UI 上的可见顺序一致
- 删除、置顶、粘贴成功后，新的选中项和滚动位置仍然稳定

## 5. 排除范围说明

- `v1.1` 不引入新的设置分组，不提供“自定义快速粘贴修饰键”能力
- `v1.1` 不修改历史记录的存储模型，不增加数据库表或迁移版本
- `v1.1` 不新增搜索、过滤、标签、收藏等信息管理能力
- `v1.1` 不改变 `Enter` / `Shift+Enter` 的既有粘贴语义
