### 2.2 2026-03-08 `v1.1-macos-dock-window-level-hotfix`

**变更类型**：`fix(window-level)`

**主要变更**：
- 更新 `src-tauri/src/window/mod.rs`，`macOS` 主面板不再调用 Tauri `set_always_on_top(true)`，避免异步回写把窗口层级降回 `floating`
- 更新 `src-tauri/src/window/mod.rs`，改为通过 CoreGraphics `CGWindowLevelForKey` 动态获取 `popup menu` 窗口层级，并在显示前后重复应用
- 更新 `src-tauri/src/window/mod.rs`，新增 `popup menu` 层级高于 `Dock` 的 Rust 回归断言

**产出结果**：
- 主面板在 `macOS` 下会优先使用更高的原生窗口层级，而不是依赖 Tauri 的通用 `always on top` 语义
- 主面板显示流程中不再容易被异步层级回写降回 `floating`，覆盖 `Dock` 的稳定性更高
- 本次热修保持既有 `220px` 高度、物理屏幕贴底定位和失焦自动隐藏逻辑不变

**验证结果**：
- `cargo test --manifest-path src-tauri/Cargo.toml` ✅

- `pnpm run lint` ✅
- `pnpm run test:coverage` ✅
