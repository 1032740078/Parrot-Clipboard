# v1.1 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

`v1.1` 自动化测试只覆盖当前版本新增的 4 个主题：

- 主面板失焦自动隐藏
- `macOS Dock` 覆盖与主面板高度调优
- `Command + 1~9` 快速粘贴
- 左右切换超出可见区时自动横向滚动

`v1.0` 已稳定的数据库迁移、关于页、更新检查、图片清理等能力，不作为本版新增测试设计重点，只保留回归验证。

### 1.2 测试架构分层

| 层级 | 工具 | 目标 |
|------|------|------|
| 后端单元测试 | `cargo test` | 验证窗口定位、显隐状态、原生事件收敛 |
| 前端单元测试 | `Vitest + RTL + jsdom` | 验证键盘交互、滚动同步、UI 状态收敛 |
| 集成测试 | Rust 集成 / 前后端桥接测试 | 验证显隐链路和粘贴链路联动 |
| BDD / E2E | `Playwright` | 从真实用户视角验证完整桌面交互 |

### 1.3 覆盖率目标

- 前端覆盖率继续满足仓库基线：`lines / branches / functions / statements >= 75%`
- `v1.1` 新增逻辑的关键分支覆盖率目标：`>= 90%`
- 关键窗口行为必须同时具备 **单元测试 + BDD 场景** 双保险

## 4. 后端集成测试

### 4.1 主面板显隐链路

- 呼出主面板 → 主面板获得焦点 → 失去焦点 → 原生窗口隐藏 → 运行态 `panel_visible=false`
- `Esc` 隐藏与失焦隐藏都能让运行态和前端事件同步一致

### 4.2 `Dock` 覆盖链路

- 多显示器场景下选择目标屏幕后，面板 frame 应以物理显示区域为准
- 同时验证底部 / 左侧 / 右侧 `Dock` 三类布局

### 4.3 快贴链路

- 主面板可见时按 `Command + 3`
- 解析目标记录 `id`
- 调用 `paste_record`
- 粘贴成功后主面板隐藏、记录置顶、焦点回到目标应用

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- 保持 `Playwright` 作为 BDD 驱动层
- `tauriMock` 需要补充“面板显隐变化事件”“窗口失焦模拟”“多显示器布局夹具”
- `macOS` 相关原生能力在 E2E 中采用可控 mock，避免 CI 对系统桌面环境过度依赖

### 5.2 必测主链路

1. 主面板失焦自动隐藏
2. 主面板覆盖底部 `Dock`，底部内容完整可见
3. `Command + 数字` 成功直贴
4. 权限缺失时快贴失败并出现提示
5. 左右切换超出可见区时自动滚动
6. 虚拟滚动、大列表与快贴组合使用仍稳定

---

## 6. 测试数据与夹具规范

### 6.1 记录夹具

- 至少准备 `12` 条记录，覆盖文本、图片、文件三种类型
- 前 `9` 条记录必须固定顺序，供 `Command + 1~9` 映射验证
- 长列表夹具建议使用 `40` 或 `60` 条记录，验证虚拟滚动与自动滚动组合场景

### 6.2 布局夹具

- `DockBottomFixture`
- `DockLeftFixture`
- `DockRightFixture`
- `MixedScaleDisplaysFixture`

### 6.3 脱敏要求

- 记录正文使用脱敏文本，如“订单号示例”“测试地址”“演示截图”
- 不允许在日志、fixture、截图中包含真实密码、验证码、客户隐私

3. `pnpm run test`
4. `pnpm run test:e2e`
5. `cargo test --manifest-path src-tauri/Cargo.toml`
