# v1.1 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

`v1.1` UX 只关注主面板的桌面工具体验修正，不重做视觉风格，也不扩展新页面信息架构。

### 1.2 设计原则（v1.1 强调）

1. **离开即收起**：主面板是临时工作层，不应长期停留在桌面前景
2. **内容完整优先**：不能再用缩小高度换取 `Dock` 规避
3. **一步完成**：高频场景优先减少键击次数
4. **所见即所贴**：用户看到哪个高亮项，就应该粘贴哪个目标项

---

## 2. 主面板显隐 UX

### 2.1 自动隐藏原则

- 用户一旦把焦点切到其他应用，主面板应自动消失
- 自动隐藏应当“像系统工具一样自然”，不出现额外确认或 Toast
- 隐藏后工作焦点回到用户原本的目标应用

### 2.2 一致性约束

- 主面板显示 / 隐藏不能依赖浏览器窗口 `focus` 的偶然行为
- 前端可见状态、后端运行态、原生窗口实际状态必须一致

---

## 3. Dock 覆盖 UX

### 3.1 设计结论

- 主面板回到原始高度 `220px`
- 主面板允许直接覆盖 `Dock`
- `Dock` 的存在不再影响主面板内容布局高度

### 3.2 用户感知目标

- 卡片底部按钮、时间、辅助信息完整可见
- 面板阴影和边缘不被系统栏裁断
- 用户不需要理解“为什么本次主面板比之前矮”这种异常现象

---

## 4. 快速粘贴 UX

### 4.1 交互规则

- `1~9`：快选
- `Command + 1~9`：快贴
- `Enter`：粘贴当前选中项
- `Shift + Enter`：文本记录纯文本粘贴

### 4.2 反馈原则

- 快贴成功：无 Toast，直接隐藏主面板
- 快贴失败：保留主面板，提供明确错误提示
- 权限缺失：提示原因并提供权限引导入口

### 4.3 提示文案建议

- `1~9 快选`
- `⌘+1~9 快贴`
- `Esc 关闭`

---

## 5. 自动滚动 UX

### 5.1 体验目标

- 左右切换时，用户始终看得到当前高亮项
- 自动滚动要“跟得上”，但不能跳得过猛
- 普通列表与虚拟滚动列表的感受应一致

### 5.2 设计要求

- 选中项刚好出界时，只滚动最小必要距离
- 避免每次切换都重新居中，减少晕动感
- 数字快选跳转远端项时也必须自动滚动到位

---

## 6. 反馈设计

### 6.1 操作反馈矩阵

| 行为 | 成功反馈 | 失败反馈 |
|------|----------|----------|
| 失焦自动隐藏 | 无反馈，直接隐藏 | 不暴露给用户，记录日志 |
| `Command + 数字` 快贴 | 直接隐藏主面板 | Toast + 保持面板 |
| 左右切换 | 高亮移动 + 自动滚动 | 不应出现失败提示 |
| 权限缺失快贴 | 打开或提示权限引导 | 同左 |

### 6.2 Toast 规范

- 成功型高频动作不弹 Toast，避免噪音
- 失败型动作必须可读、可操作、可重试
- Toast 不得包含敏感正文

---

## 7. 可用性指标

| 指标 | 目标 |
|------|------|
| 自动隐藏体感延迟 | 几乎不可察觉，目标 `<= 100ms` |
| 快贴完成总耗时 | 文本常规场景 `<= 180ms` |
| 自动滚动稳定性 | 不出现明显白屏、跳动、错选 |
| 内容完整度 | 主面板底部内容 `100%` 可见 |

---

## 8. 平台特定 UX 规范

### 8.1 macOS

- `Command + 1~9` 是本版核心增强能力
- 主面板覆盖 `Dock` 是既定策略，不再规避
- 权限不足时要清楚提示“可浏览，不可粘贴”

### 8.2 Windows

- 继续沿用 `v1.0` 交互模式
- 本版不引入 `Ctrl + 数字` 快贴，避免和应用内快捷键冲突

### 8.3 Linux

- 继续沿用 `v1.0` 交互模式
- Wayland / X11 的平台差异不在本版 UX 调整范围内
