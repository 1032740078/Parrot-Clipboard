# v1.1 冲刺任务文档

> 更新时间：2026-03-08
> 当前状态：进行中（3/6）
> 版本目标：修正主面板桌面交互体验，使“呼出 → 选择 → 快贴 / 粘贴 → 收起”链路更自然

## 1. 冲刺目标

- 完成主面板失焦自动隐藏
- 彻底移除“缩短主面板高度避让 `Dock`”方案，恢复 `220px` 原始高度
- 支持 `Command + 1~9` 直接粘贴
- 确保左右切换与数字快选命中不可见卡片时自动横向滚动

## 3. 各批次任务详情

### 3.1 第 0 批：文档基线

- 产出 `v1.1` 用户故事、DDD、数据库、测试、BDD、API、状态码、种子数据、目录设计、页面地图、UX、冲刺文档
- 更新项目决策记录，明确“覆盖 `Dock` 而非规避 `Dock`”

### 3.2 第 1 批：窗口与显隐收敛

- 原生窗口失焦时自动隐藏主面板
- 建立面板显隐变化事件，收敛前后端状态一致性
- 移除“窗口 focus 自动显示主面板”的错误依赖

### 3.3 第 2 批：交互效率增强

- 实现 `Command + 1~9` 快贴
- 保留 `1~9` 仅快选的既有语义
- 更新主面板底部快捷键提示

### 3.4 第 3 批：滚动与回归

- 小列表与虚拟滚动列表统一采用“选中项确保可见”策略
- 补齐前端单测、Rust 单测、BDD 自动化场景
- 验证 `Dock` 位置、多显示器和权限缺失组合场景

---

## 4. 风险与缓解

| 风险 | 说明 | 缓解方案 |
|------|------|----------|
| 面板显隐状态错位 | 后端已隐藏，前端仍认为可见 | 引入显隐变化事件，原生窗口状态作为真相源 |
| 覆盖 Dock 后层级不足 | 面板仍可能被系统 UI 压住 | 复用并校验高于 `Dock` 的原生层级策略 |
| 快贴和快选语义冲突 | `1~9` 与 `Command+1~9` 混淆 | 在键盘层显式区分修饰键，并补齐回归测试 |
| 自动滚动引发跳动 | 高频切换时滚动过度 | 仅滚动最小必要距离，不做强制居中 |

---

## 5. 冲刺完成后建议

2. 手测覆盖单屏、双屏、混合缩放
3. 若 `v1.1` 验收通过，再评估是否进入下一版本的搜索与管理能力规划
