# v1.1 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

`v1.1` 不重做目录结构，只在 `v1.0` 基础上对主面板交互、窗口行为和自动化测试相关目录做增量设计。

### 1.2 设计原则

- 尽量复用现有模块边界，不为小范围交互问题新开大模块
- 平台相关逻辑收敛在 `src-tauri/src/window/`
- 前端键盘行为与视口滚动留在主面板相关目录内解决

---

## 2. 顶层目录结构（v1.1 关注范围）

```text
粘贴板记录管理工具/
├── docs/
│   ├── v1.0/
│   └── v1.1/
├── src/
├── src-tauri/
├── e2e/
└── public/
```

---

## 3. 前端目录设计（`src/`）

### 3.1 `v1.1` 重点文件

| 文件 | 职责 |
|------|------|
| `src/App.tsx` | 主面板显隐同步入口；去除错误的“窗口 focus 即自动显示”依赖 |
| `src/hooks/useKeyboard.ts` | 区分 `1~9` 快选与 `Command + 1~9` 快贴 |
| `src/components/MainPanel/CardList.tsx` | 统一小列表与虚拟列表的自动横向滚动策略 |
| `src/components/MainPanel/index.tsx` | 更新页脚提示文案与主面板交互说明 |
| `src/stores/useUIStore.ts` | 收敛主面板可见状态 |
| `src/stores/useSystemStore.ts` | 同步运行态 `panelVisible` |
| `src/api/events.ts` | 新增面板显隐事件订阅封装 |

### 3.2 推荐约束

- 不新增独立“快贴页面”或“滚动服务目录”
- 自动滚动逻辑优先收敛在 `CardList.tsx`
- 键盘语义判断优先收敛在 `useKeyboard.ts`

---

## 4. 后端目录设计（`src-tauri/src/`）

### 4.1 `v1.1` 重点文件

| 文件 | 职责 |
|------|------|
| `src-tauri/src/window/mod.rs` | 处理主面板显示、隐藏、焦点恢复、原生窗口事件 |
| `src-tauri/src/window/position.rs` | 从 `work_area` 逻辑切换到物理显示 frame 逻辑 |
| `src-tauri/src/ipc/events.rs` | 新增主面板显隐变化事件广播 |
| `src-tauri/src/state.rs` | 保存运行态 `panel_visible` 与可选隐藏原因 |
| `src-tauri/src/paste/mod.rs` | 复用现有粘贴链路，无需新增快贴专属模块 |

### 4.2 模块归属说明

- `Dock` 覆盖是窗口问题，因此归 `window/`
- 快贴本质是触发入口变化，不拆新 `shortcut/quick_paste.rs`
- 若后续面板事件增多，再考虑在 `window/` 下拆分 `panel_events.rs`

## 6. 运行时生成目录

### 6.1 推荐布局

`v1.1` 沿用 `v1.0`：

- 数据库目录
- 图片资产目录
- 缩略图目录
- 日志目录

### 6.2 约束说明

- 自动隐藏、快贴、滚动位置都不应产生新的运行时持久化目录
- 不新增“窗口状态缓存文件”或“快贴映射缓存文件”

---

## 7. 命名规范

- 事件命名沿用 `system:*` 前缀，例如 `system:panel-visibility-changed`
- 测试用例命名推荐采用 `UT-FE-*` / `UT-BE-*` / `BDD-*`
- 文档命名延续 `docs/v1.x/中文文档名.md`

---

## 8. 与 v1.0 的主要差异

- `v1.0` 重点扩展在关于页、诊断、更新、清理；`v1.1` 重点回到主面板核心交互
- `v1.1` 目录设计强调“在原有模块上修正”，而不是继续横向扩展新模块
