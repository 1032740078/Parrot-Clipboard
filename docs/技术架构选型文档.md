## 3. 跨平台框架选型

### 3.1 候选方案对比

| 维度 | Tauri 2.0 | Electron | Flutter Desktop |
|------|-----------|----------|-----------------|
| 安装包体积 | ~5-10MB | ~100-150MB | ~30-50MB |
| 运行内存（空闲） | ~40-60MB | ~150-200MB | ~80-120MB |
| 启动速度 | 快 | 较慢 | 中等 |
| 后端语言 | Rust | Node.js | Dart |
| 前端技术 | Web (任意框架) | Web (任意框架) | Flutter Widget |
| 系统 API 集成 | 优秀 | 良好 | 一般 |
| 安全性 | 高 | 中 | 中 |
| 生态成熟度 | 成长中 | 非常成熟 | 桌面端一般 |
| 学习曲线 | 较陡（需要 Rust） | 平缓 | 中等 |

### 3.2 选型结论：Tauri 2.0

**理由**：

1. **性能契合需求**：需求要求快捷键响应 < 100ms、内存 < 100MB，Tauri 的轻量级架构天然满足
2. **Rust 后端的安全性**：系统级 API（粘贴板监听、键盘模拟）用 Rust 实现，内存安全有保障
3. **前端无缝复用 Web 技术**：UI 层仍使用 React + TypeScript，前端开发体验不受影响
4. **使用系统 WebView**：不内嵌 Chromium，体积大幅缩减
5. **官方插件完善**：`tauri-plugin-global-shortcut`、`tauri-plugin-autostart` 已覆盖核心系统需求

**风险与应对**：

| 风险 | 应对措施 |
|------|----------|
| Rust 学习成本高 | 后端模块按依赖顺序分批开发，优先实现基础无状态模块 |
| 生态不如 Electron 成熟 | 优先使用 Tauri 官方插件，第三方库选 crates.io 高星项目 |
| WebView 渲染差异（各平台） | 统一使用 Tailwind CSS 管控样式，避免依赖浏览器特性 |

## 4. 前端技术选型

### 4.1 UI 框架：React 18

- **选型理由**：成熟稳定，Hooks 模型与 Zustand 配合流畅；`react-window` 虚拟滚动可应对大量卡片渲染
- **关键特性使用**：`useEffect` 监听后端事件、`useMemo` / `useCallback` 优化渲染、`Suspense` 懒加载设置界面

### 4.2 样式方案：Tailwind CSS 3

- **选型理由**：原子类避免 CSS 文件膨胀；`dark:` 前缀天然支持深色模式；响应式写法统一
- **约定**：禁止内联 `style` 属性处理颜色/间距，统一走 Tailwind 类名；自定义 Token 在 `tailwind.config.ts` 中集中声明

### 4.3 动画：Framer Motion 11

- **选型理由**：声明式动画 API，面板弹出/收起（200-300ms）、卡片切换（100-150ms）可用 `AnimatePresence` + `motion.div` 一行实现
- **性能约束**：动画只允许操作 `transform` 和 `opacity`，禁止触发布局重排属性（如 `height`、`top`）

### 4.4 状态管理：Zustand 5

- **选型理由**：轻量（< 3KB），无 Provider 包裹，store 可按模块拆分，与 TypeScript 配合无额外类型成本
- **Store 划分**：
  - `useClipboardStore`：记录列表、选中索引、加载状态
  - `useUIStore`：面板可见性、搜索关键词
  - `useConfigStore`：应用配置项

### 4.5 虚拟滚动：react-window

- **选型理由**：横向卡片列表可能包含 100~200 条记录，全量渲染 DOM 节点会导致卡顿；`FixedSizeList`（水平模式）仅渲染可视区卡片
- **图片懒加载**：配合 `IntersectionObserver`，图片卡片离屏后卸载 `src`

## 5. 后端技术选型

### 5.1 语言：Rust 1.75+

- **选型理由**：Tauri 后端原生语言；内存安全无 GC 停顿；与 macOS/Windows/Linux 系统 API 直接 FFI
- **异步运行时**：Tokio（Tauri 内置），用于后台粘贴板监听线程、缩略图生成任务

### 5.2 粘贴板操作：arboard 3

- **选型理由**：跨平台统一 API，支持文本/图片/文件路径；活跃维护；Tauri 社区首选
- **限制说明**：Linux Wayland 下 `arboard` 能力受限，需降级到 X11 或提供托盘菜单替代

### 5.3 全局快捷键：tauri-plugin-global-shortcut 2

- **选型理由**：Tauri 官方插件，与应用生命周期绑定，支持运行时注册/注销；跨平台统一 API

### 5.4 开机自启动：tauri-plugin-autostart 2

- **选型理由**：Tauri 官方插件，封装 macOS Launch Agent、Windows 注册表、Linux XDG Autostart 三种实现

### 5.5 图片处理：image 0.25

- **选型理由**：Rust 生态最成熟的图片库；支持 PNG/JPEG/GIF/BMP/WebP；提供高质量缩放（Lanczos3）

### 5.6 数据库：SQLite + rusqlite 0.31

- **选型理由**：无需额外进程；文件级存储，方便备份和迁移；`rusqlite` 提供 Rust 安全绑定；支持 WAL 模式优化写性能
- **迁移方案**：使用 `rusqlite_migration` crate 管理 Schema 版本，确保版本升级数据不丢失

## 6. 数据存储架构

### 6.1 存储布局

```
~/.clipboard-manager/
├── database.db              # SQLite 主数据库（WAL 模式）
├── database.db-wal          # WAL 日志文件
├── config.json              # 应用配置
└── images/
    ├── original/            # 原图（按日期分目录）
    │   ├── 2026-03-05/
    │   │   ├── img_001.png
    │   │   └── img_002.jpg
    └── thumbnails/          # 缩略图（200x200, JPEG 85%）
        └── 2026-03-05/
            ├── img_001_thumb.jpg
            └── img_002_thumb.jpg
```

### 6.2 数据库表结构（最终版）

```sql
-- 粘贴板记录主表
CREATE TABLE clipboard_items (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT    NOT NULL,          -- 'text' | 'images' | 'files'
    text_content TEXT,                      -- 纯文本内容
    rich_content TEXT,                      -- 富文本（HTML/RTF），可为空
    source_app   TEXT,                      -- 来源应用名称
    created_at   INTEGER NOT NULL,          -- Unix 时间戳（毫秒）
    is_favorite  INTEGER NOT NULL DEFAULT 0
);

-- 图片附件表（一条记录对应多张图片）
CREATE TABLE clipboard_images (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    item_id        INTEGER NOT NULL,
    image_path     TEXT    NOT NULL,
    thumbnail_path TEXT,
    image_size     INTEGER,
    image_width    INTEGER,
    image_height   INTEGER,
    image_format   TEXT,
    sort_order     INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY (item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);

-- 文件附件表（一条记录对应多个文件）
CREATE TABLE clipboard_files (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    item_id      INTEGER NOT NULL,
    file_path    TEXT    NOT NULL,
    file_name    TEXT    NOT NULL,
    file_size    INTEGER,
    is_directory INTEGER NOT NULL DEFAULT 0,
    sort_order   INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY (item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);

-- Schema 版本表（迁移管理）
CREATE TABLE schema_version (
    version INTEGER NOT NULL
);

CREATE INDEX idx_items_created_at    ON clipboard_items(created_at DESC);
CREATE INDEX idx_images_item_id      ON clipboard_images(item_id);
CREATE INDEX idx_files_item_id       ON clipboard_files(item_id);
```

### 6.3 存储策略

| 内容类型 | 存储方式 | 大小限制 |
|----------|----------|----------|
| 文本 | 直接写入 SQLite | 单条 < 1MB |
| 富文本（HTML/RTF） | 直接写入 SQLite | 单条 < 2MB |
| 图片原图 | 文件系统，DB 存路径 | 单张默认 < 10MB（可配置） |
| 图片缩略图 | 文件系统，DB 存路径 | 200x200，JPEG 85%，约 20KB |
| 文件路径 | 直接写入 SQLite | — |

## 7. 跨平台适配策略

### 7.1 平台差异汇总

| 能力 | macOS | Windows | Linux (X11) | Linux (Wayland) |
|------|-------|---------|-------------|-----------------|
| 粘贴板监听 | 轮询 `NSPasteboard.changeCount`（200ms） | `WM_CLIPBOARDUPDATE` 事件 | `XFixesSelectSelectionInput` | 降级：托盘菜单触发 |
| 全局快捷键 | 需辅助功能权限 | `RegisterHotKey` 原生支持 | `XGrabKey` | 降级：托盘菜单触发 |
| 键盘模拟（粘贴） | `CGEventCreateKeyboardEvent` | `SendInput` | `XSendEvent` | `ydotool`（需 root） |
| 开机自启动 | Launch Agent plist | 注册表 Run 键 | XDG autostart | XDG autostart |
| 系统托盘 | NSStatusBar | Shell_NotifyIcon | libappindicator | libappindicator |
| 毛玻璃背景 | `vibrancy` 效果 | Acrylic/Mica 效果 | 不支持，回退半透明 | 不支持，回退半透明 |

### 7.2 平台抽象实现原则

- 所有平台差异封装在 `src-tauri/src/platform/` 目录下
- 上层业务模块只调用 `PlatformClipboard`、`PlatformKeySimulator`、`PlatformAppInfo` 三个 trait
- 条件编译通过 `#[cfg(target_os = "...")]` 实现，禁止在业务层出现平台判断分支

## 8. Tauri IPC 通信规范

### 8.1 通信方向约定

| 方向 | 机制 | 使用场景 |
|------|------|----------|
| 前端 → 后端 | `invoke()` Command | 主动请求：获取记录、执行粘贴、更新配置 |
| 后端 → 前端 | `emit()` Event | 被动推送：新记录到达、监听状态变更 |

### 8.2 Command 接口清单

```typescript
// 记录管理
invoke('get_records',    { limit: number, offset: number }): Promise<ClipboardRecord[]>
invoke('delete_record',  { id: number }):                   Promise<void>
invoke('delete_records', { ids: number[] }):                Promise<void>
invoke('clear_all_records'):                                Promise<void>
invoke('search_records', { keyword: string }):              Promise<ClipboardRecord[]>

// 粘贴操作
invoke('paste_record',   { id: number, mode: 'original' | 'plain_text' }): Promise<void>

// 配置管理
invoke('get_config'):                            Promise<AppConfig>
invoke('update_config', { config: AppConfig }): Promise<void>
invoke('reset_config'):                         Promise<void>

// 监听控制
invoke('pause_monitoring'):              Promise<void>
invoke('resume_monitoring'):             Promise<void>
invoke('get_monitoring_status'):         Promise<boolean>

// 窗口控制
invoke('hide_panel'): Promise<void>

// 自启动
invoke('get_autostart_status'):              Promise<boolean>
invoke('set_autostart', { enabled: boolean }): Promise<void>
```

### 8.3 Event 清单

```typescript
listen('clipboard:new-record',        (record: ClipboardRecord) => void)
listen('clipboard:record-deleted',    (id: number)              => void)
listen('monitoring:status-changed',   (isRunning: boolean)      => void)
listen('config:changed',              (config: AppConfig)        => void)
```

### 8.4 数据传输格式

- 序列化统一使用 JSON（Rust 端使用 `serde_json`）
- 时间戳统一使用 Unix 毫秒整数
- 枚举值使用 snake_case 字符串（如 `"plain_text"`）

**示例**（M02 数据持久化）：

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use rusqlite::Connection;

    fn setup_db() -> SqliteStorage {
        let conn = Connection::open_in_memory().unwrap();
        SqliteStorage::new(conn)
    }

    #[test]
    fn test_insert_and_get_recent() {
        let storage = setup_db();
        let entry = make_text_entry("hello");
        let id = storage.insert(&entry).unwrap();

        let records = storage.get_recent(10, 0).unwrap();
        assert_eq!(records.len(), 1);
        assert_eq!(records[0].id, id);
    }

    #[test]
    fn test_cleanup_old_by_count() {
        let storage = setup_db();
        for i in 0..110 {
            storage.insert(&make_text_entry(&format!("item {}", i))).unwrap();
        }
        let removed = storage.cleanup_old(100).unwrap();
        assert_eq!(removed, 10);
        assert_eq!(storage.get_count().unwrap(), 100);
    }
}
```

### 9.4 集成测试（Rust 模块间）

**关键集成场景**：

```
场景 1: 文本复制到存储链路
  M01（捕获文本） → M02（存储） → IPC（get_records）→ 前端收到正确记录

场景 2: 图片复制完整链路
  M01（捕获图片） → M07（生成缩略图） → M02（存储路径） → 前端 ImageCard 显示缩略图

场景 3: 超限自动清理
  M02 insert 第 201 条记录 → cleanup_old 触发 → 保留最新 200 条 → 旧图片文件被删除

场景 4: 粘贴执行
  前端 invoke paste_record → M06 暂停 M01 → 写入粘贴板 → 隐藏窗口 → 模拟 Cmd+V → 恢复 M01
```

## 10. 自动化验收（Acceptance Testing）

### 10.1 验收策略概述

验收测试基于**需求规格**（`软件项目愿景需求表.md` §4-6）进行映射，采用 **BDD（行为驱动开发）** 风格编写，以"用户视角"描述预期行为，与 E2E 测试工具（Playwright）结合执行。

### 10.2 验收测试用例（BDD 格式）

#### 功能：自动监听与记录（需求 §4.1）

```
场景：复制文本内容后应自动记录
  假设 应用正在后台运行
  当 用户在任意应用中复制一段文字 "Hello World"
  那么 主面板展示的最新记录应包含 "Hello World"
  并且 记录的时间戳应在 2 秒内

场景：复制图片后应自动记录
  假设 应用正在后台运行
  当 用户复制一张图片
  那么 主面板展示的最新记录应为图片卡片
  并且 缩略图应在 1 秒内生成并展示

场景：黑名单应用的复制内容不被记录
  假设 "1Password" 已加入黑名单
  当 用户在 "1Password" 中复制内容
  那么 主面板记录数量应保持不变
```

#### 功能：快捷键调出界面（需求 §4.3）

```
场景：默认快捷键调出面板
  假设 应用正在后台运行且面板不可见
  当 用户按下 "Shift + Command + V"（macOS）
  那么 面板应从屏幕底部弹出
  并且 响应时间应 < 100ms
  并且 最新的一条记录应默认选中

场景：Esc 关闭面板
  假设 面板当前可见
  当 用户按下 "Esc"
  那么 面板应在 300ms 内收起
```

#### 功能：键盘操作（需求 §4.5）

```
场景：方向键切换选中记录
  假设 面板已展示至少 3 条记录，第 1 条被选中
  当 用户按下 "→"
  那么 第 2 条记录应被高亮选中
  当 用户继续按下 "→"
  那么 第 3 条记录应被高亮选中
  当 用户按下 "←"
  那么 第 2 条记录应重新被选中

场景：Enter 粘贴并关闭面板
  假设 面板已展示记录，第 1 条被选中
  当 用户按下 "Enter"
  那么 第 1 条记录的内容应被写入系统粘贴板
  并且 面板应自动关闭

场景：Shift+Enter 以纯文本模式粘贴
  假设 面板已展示一条富文本记录（含 HTML 格式），该记录被选中
  当 用户按下 "Shift + Enter"
  那么 系统粘贴板中的内容应为纯文本（无 HTML 标签）

场景：Delete 删除选中记录
  假设 面板已展示至少 2 条记录，第 1 条被选中
  当 用户按下 "Delete"
  那么 第 1 条记录应从列表中消失
  并且 第 2 条记录应自动成为选中项
  并且 数据库中该记录应不存在
```

#### 功能：记录管理（需求 §4.7）

```
场景：记录数量超限时自动清理最旧记录
  假设 文本记录数量上限设置为 100
  当 第 101 条文本被复制
  那么 总文本记录数量应保持为 100
  并且 最旧的一条记录应被删除

场景：应用重启后历史记录恢复
  假设 用户已有 10 条历史记录
  当 应用被关闭并重新启动
  那么 主面板应展示 10 条历史记录
  并且 内容与重启前一致
```

#### 非功能需求：性能（需求 §6.2）

```
场景：快捷键响应时间
  当 用户按下调出快捷键
  那么 面板从不可见到可见的时间应 < 100ms

场景：内存占用（空闲状态）
  假设 应用在后台运行，面板未展示，无用户操作
  那么 进程内存占用应 < 100MB

场景：CPU 占用（空闲状态）
  假设 应用在后台运行，无复制操作
  那么 CPU 占用率应 < 1%
```

### 10.3 验收通过标准（Definition of Done）

| 类别 | 通过标准 |
|------|----------|
| 功能完整性 | 所有 BDD 场景测试通过，E2E 用例全部绿灯 |
| 性能指标 | 快捷键响应 < 100ms，内存 < 100MB，CPU < 1% |
| 稳定性 | 连续运行 24 小时无崩溃，无内存增长趋势 |
| 跨平台 | macOS、Windows、Linux(X11) 三平台验收全部通过 |
| 数据安全 | 随机写入 500 条记录后重启，数据零丢失 |
| 代码质量 | 单元测试覆盖率：后端 ≥ 80%，前端 ≥ 75% |

## 11. CI/CD 流水线

### 11.1 流水线架构（GitHub Actions）

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  # 阶段 1：代码质量检查（并行）
  lint-frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pnpm install
      - run: pnpm run lint          # ESLint + TypeScript 类型检查
      - run: pnpm run format:check  # Prettier 格式检查

  lint-backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: cargo clippy -- -D warnings  # Rust 静态分析
      - run: cargo fmt --check            # Rust 格式检查

  # 阶段 2：单元测试（并行）
  test-frontend:
    runs-on: ubuntu-latest
    needs: [lint-frontend]
    steps:
      - uses: actions/checkout@v4
      - run: pnpm install
      - run: pnpm run test:coverage  # Vitest 覆盖率报告
      - uses: actions/upload-artifact@v4
        with:
          name: frontend-coverage
          path: coverage/

  test-backend:
    runs-on: ubuntu-latest
    needs: [lint-backend]
    steps:
      - uses: actions/checkout@v4
      - run: cargo test --all-features
      - run: cargo llvm-cov --all-features --lcov --output-path lcov.info
      - uses: actions/upload-artifact@v4
        with:
          name: backend-coverage
          path: lcov.info

  # 阶段 3：构建（三平台并行）
  build:
    needs: [test-frontend, test-backend]
    strategy:
      matrix:
        os: [macos-latest, windows-latest, ubuntu-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - run: pnpm install
      - run: pnpm tauri build

  # 阶段 4：E2E 测试（三平台，仅 main 分支）
  e2e:
    needs: [build]
    if: github.ref == 'refs/heads/main'
    strategy:
      matrix:
        os: [macos-latest, windows-latest, ubuntu-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - run: pnpm install
      - run: pnpm run test:e2e  # Playwright + tauri-driver
```

### 11.2 质量门禁

| 检查项 | 阻塞 PR 合并 |
|--------|-------------|
| ESLint / Clippy 无 error | 是 |
| TypeScript 类型检查通过 | 是 |
| 后端单元测试全绿 | 是 |
| 前端单元测试全绿 | 是 |
| 前端覆盖率 ≥ 75% | 是 |
| 后端覆盖率 ≥ 80% | 是 |
| 三平台构建成功 | 是（main 分支） |
| E2E 测试全绿 | 是（main 分支） |

### 11.3 发布流程

```
develop 分支
    │ PR + Code Review
    ▼
main 分支
    │ 触发 CI 全量流水线
    │ E2E 验收通过
    ▼
打 release tag（如 v1.0.0）
    │ 触发 release workflow
    ▼
三平台构建产物（.dmg / .exe / .AppImage）
    │
    ▼
GitHub Releases 自动发布
```

## 13. 技术债务与后续演进

### 13.1 已知技术限制

| 限制 | 影响 | 计划版本 |
|------|------|----------|
| Linux Wayland 全局快捷键不支持 | Wayland 用户无法使用快捷键 | v1.1 调研 `xdg-desktop-portal` |
| 搜索功能暂未实现 | 记录多时查找不便 | v1.1 实现全文搜索（SQLite FTS5） |
| 云同步未支持 | 多设备无法共享历史 | v2.0 |
| 插件系统未设计 | 扩展能力有限 | v2.0 |

### 13.2 可优化方向

- **SQLite WAL 模式**：已在初始化时启用，减少写阻塞
- **图片缩略图缓存**：可引入内存 LRU 缓存（最多 100 张），减少磁盘读取
- **Tauri Splashscreen**：首次加载时展示启动画面，改善冷启动体验
- **增量更新**：使用 `tauri-plugin-updater` 实现差量升级，降低更新包大小

## 14. 总结

### 14.1 技术选型决策汇总

| 决策点 | 结论 | 核心理由 |
|--------|------|----------|
| 跨平台框架 | Tauri 2.0 | 性能最优，满足内存/响应时间需求 |
| 前端框架 | React 18 + TypeScript | 成熟生态，Zustand + react-window 搭配高效 |
| 后端语言 | Rust | 内存安全，系统 API 集成能力强 |
| 数据库 | SQLite (rusqlite) | 无服务进程，文件级存储，查询灵活 |
| 状态管理 | Zustand 5 | 轻量，TypeScript 友好，无 Provider 样板 |
| 单元测试（后端） | cargo test + mockall | Rust 原生支持，mock 方案成熟 |
| 单元测试（前端） | Vitest + RTL | 快速，与 Vite 生态一致 |
| E2E / 验收测试 | Playwright + tauri-driver | Tauri 官方支持，跨平台覆盖 |
| CI/CD | GitHub Actions | 免费三平台矩阵构建，生态插件丰富 |

### 14.2 风险等级汇总

| 风险 | 等级 | 已有应对措施 |
|------|------|-------------|
| Linux Wayland 支持受限 | 高 | 托盘菜单降级方案，文档说明 |
| macOS 辅助功能权限被拒 | 中 | 引导流程，检测后提示 |
| 快捷键与第三方应用冲突 | 中 | 支持自定义快捷键，冲突检测 |
| 大量图片导致内存增长 | 中 | LRU 缓存 + 数量/大小上限配置 |
| Rust 学习成本 | 中 | 按依赖顺序分批开发，基础模块先行 |
