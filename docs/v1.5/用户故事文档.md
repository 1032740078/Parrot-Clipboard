# v1.5 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.5` 当前版本的新增与调整能力。

`v1.5` 默认继承当前版本已经具备的托盘入口、搜索与类型筛选、文本预览编辑器、图片预览、右键菜单、多屏定位与玻璃风格；本版本只聚焦以下 4 个主题：

- 新增 3 组系统反馈音效：复制成功入库、粘贴成功执行、显示预览内容
- 预览窗口从“单次打开”升级为“打开后跟随当前选中卡片自动切换内容”
- 预览窗口新增 `音频 / 视频 / PDF / DOC / DOCX / EXCEL / PPT / 超链接` 预览能力
- 音效资源从 `docs/音效/` 复制到项目运行资源目录，并统一改为英文命名后再参与打包与播放

以下能力 **不属于 `v1.5` 当前版本范围**：

- 云同步、账号体系、跨设备共享
- 音频 / 视频剪辑编辑、波形编辑、字幕编辑
- Office 文稿像素级排版还原，仅要求稳定显示主要内容
- 内置浏览器级网页完全渲染，不追求站点脚本完整执行
- 用户自定义音效、音量面板、音效开关页

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 高频复制用户 | 一天内频繁复制与粘贴多类内容的用户 | 每次关键动作都有明确但不过度打扰的反馈 |
| 预览确认用户 | 粘贴前需要快速确认记录内容的用户 | 打开预览后切换选中卡片时，预览应同步切换 |
| 媒体内容用户 | 经常复制音频、视频与多媒体素材的用户 | 预览窗口可直接播放，不必跳出到外部应用 |
| 文稿核对用户 | 经常复制 PDF、Word、Excel、PPT 等文稿的用户 | 能在当前工具内快速查看主要内容与结构 |
| 链接整理用户 | 经常复制网页链接、文档链接或媒体链接的用户 | 预览能看到链接标题、摘要与目标地址 |

## 3. 用户故事详细说明

### E01 - 音效反馈升级

#### US-15-01：新增复制内容后播放复制音效

**作为** 高频复制用户  
**我希望** 当系统监听到新的复制内容并成功写入历史记录后，立即播放一段轻量复制音效  
**以便** 我不用盯着面板，也能知道复制已被工具成功捕获。

**验收要点**：
- 只有“新记录成功入库”才播放复制音效
- 同一次复制去重命中或被过滤时不重复播放
- 音效资源使用项目内英文文件名，不直接引用 `docs/音效/` 原始路径
- 音效播放失败时不得影响复制入库主流程

#### US-15-02：执行粘贴后播放粘贴音效

**作为** 高频粘贴用户  
**我希望** 当我从面板或快捷键执行一次有效粘贴后，能听到粘贴音效  
**以便** 我能确认粘贴动作已经完成。

**验收要点**：
- 仅当粘贴链路成功完成时播放音效
- 原样粘贴与纯文本粘贴共用同一组粘贴完成音效
- 粘贴失败、权限不足或目标记录不存在时不播放成功音效

#### US-15-03：显示预览内容时播放预览音效

**作为** 需要先看再粘贴的用户  
**我希望** 当预览窗口第一次显示目标内容时播放预览音效  
**以便** 我能感知自己已经进入预览模式。

**验收要点**：
- 预览窗口首次显示有效内容时播放音效
- 已打开的预览窗口在跟随选中卡片切换内容时，默认不重复播放“首次打开”音效
- 预览加载失败或内容为空时不播放成功音效

### E02 - 预览联动升级

#### US-15-04：预览窗口显示时，切换选中卡片自动切换预览内容

**作为** 预览确认用户  
**我希望** 当预览窗口保持显示时，我在主面板上下切换卡片，预览窗口自动切换到对应记录  
**以便** 我可以连续浏览内容，而不用每次都重新打开预览。

**验收要点**：
- 预览窗口显示状态下，键盘上下切换与鼠标点击切换都能驱动预览目标更新
- 更新过程复用同一个预览窗口，不闪烁、不重复创建新窗口
- 当前选中记录被删除或失效时，预览窗口给出清晰降级或自动关闭

#### US-15-05：预览窗口关闭后，切换卡片不应再次自动打开预览

**作为** 希望掌控界面节奏的用户  
**我希望** 关闭预览窗口后，只切换选中卡片而不再自动弹出预览  
**以便** 我能只浏览列表，不被额外窗口持续打扰。

**验收要点**：
- 自动联动只在预览窗口当前可见时生效
- 关闭预览后，选中变化只更新主面板，不触发自动打开
- 再次按空格或执行显式预览命令后才恢复联动会话

### E03 - 媒体预览升级

#### US-15-06：音频记录支持直接播放与暂停

**作为** 媒体内容用户  
**我希望** 当我预览音频记录时，可以在预览窗口内直接播放、暂停和拖动进度  
**以便** 我快速判断是否是我要粘贴或管理的音频内容。

**验收要点**：
- 单个音频文件或被识别为 `audio` 的记录支持播放控件
- 显示基础元信息，如文件名、时长、MIME 或扩展名
- 音频源不可用时显示“无法预览”的降级提示

#### US-15-07：视频记录支持直接播放与定位

**作为** 媒体内容用户  
**我希望** 当我预览视频记录时，可以在预览窗口内直接播放、暂停、拖动进度和查看首帧  
**以便** 我能迅速确认视频内容而不必切到外部播放器。

**验收要点**：
- 单个视频文件或被识别为 `video` 的记录支持视频播放器控件
- 支持封面图或首帧展示
- 视频源不可用时显示清晰降级态，不影响预览窗口整体可用性

### E04 - 文稿与链接预览升级

#### US-15-08：PDF 文稿支持正文预览

**作为** 文稿核对用户  
**我希望** 当我预览 PDF 文稿时，能在预览窗口中看到文稿正文页面  
**以便** 我在当前工具里完成初步核对。

**验收要点**：
- PDF 预览至少支持分页滚动查看正文
- 需要显示页数或当前文稿基础信息
- 大文件加载时提供清晰加载状态

#### US-15-09：DOC / DOCX / EXCEL / PPT 文稿支持显示主要内容

**作为** 文稿核对用户  
**我希望** 当我预览 Word、Excel、PPT 等文稿时，预览窗口能显示主要内容、结构或摘要  
**以便** 我在不打开原生办公软件的情况下快速确认文稿是否正确。

**验收要点**：
- `DOC / DOCX` 至少能显示正文内容或结构化文本
- `XLS / XLSX` 至少能显示工作表名和表格主要内容片段
- `PPT / PPTX` 至少能显示页码、标题和页面主要文本
- 无法做高保真排版还原时，允许回退为“结构化内容预览”

#### US-15-10：超链接记录支持显示链接内容摘要

**作为** 链接整理用户  
**我希望** 当我预览超链接记录时，能看到标题、摘要、来源站点与目标地址  
**以便** 我在粘贴前知道这个链接具体指向什么内容。

**验收要点**：
- 预览包含规范化 URL、页面标题、摘要与站点信息
- 可提供“在默认浏览器打开”入口
- 链接抓取失败时至少显示 URL 本身与失败说明

### E05 - 稳定性与降级体验

#### US-15-11：预览内容尚未生成时显示明确加载态

**作为** 使用多媒体与文稿预览的用户  
**我希望** 当预览资源还在准备中时看到明确加载态，而不是空白窗口  
**以便** 我能理解系统仍在处理，而不是功能失效。

**验收要点**：
- 文稿解析、链接抓取、封面提取等异步流程统一显示“处理中”状态
- 加载完成后自动刷新当前预览内容
- 失败时显示原因或通用降级提示

#### US-15-12：不支持或损坏的内容提供安全降级

**作为** 希望工具稳定的用户  
**我希望** 即使遇到损坏文件、受保护文稿或不可访问链接，预览窗口也能安全降级  
**以便** 应用不会崩溃，也不会阻断我继续浏览其他记录。

**验收要点**：
- 预览失败只影响当前记录，不影响主面板与其他记录
- 降级态至少展示文件名、路径、URL 或错误说明中的一种
- 不支持格式时明确提示“不支持当前格式预览”

## 5. 排除范围说明

以下内容明确排除在 `v1.5` 当前版本之外：

- 音效设置页、音量调节与自定义音效上传
- 视频倍速、字幕轨、音频波形编辑、播放列表
- 文稿批注、局部编辑、另存为、导出 PDF
- 网页完整沙箱浏览、登录态网页还原与跨站脚本执行
- 新增新的底层 `payload_type`，当前版本仍保持 `text / image / files` 三类载荷
