# v1.5 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档只覆盖 `v1.5` 的音效反馈、预览自动联动、音频 / 视频预览、文稿预览与超链接预览的自动化测试策略。

### 1.2 测试架构分层

- **后端单元测试**：验证音效触发策略、预览资源准备、文稿 / 链接解析与状态流转
- **前端单元测试**：验证预览联动、各类预览渲染组件、降级态与局部交互
- **集成测试**：验证主面板选中变化与独立预览窗口联动、详情刷新与缓存协同
- **BDD / E2E**：验证真实用户路径与关键验收场景
- **回归测试**：验证 `v1.4` 已有文本 / 图片 / 文件列表预览不被破坏

### 1.3 覆盖率目标

- 前端代码覆盖率继续满足项目既有门槛：`lines / branches / functions / statements >= 75%`
- `v1.5` 新增预览组件、联动逻辑与预览资源适配模块建议覆盖率不低于 `85%`
- 后端新增音效、预览资源准备与预览状态模块的关键分支应具备专项测试

## 2. 后端单元测试（Rust / cargo test）

### 2.1 音效触发策略回归

重点验证：

- 新记录成功入库时发布 `copy_captured` 音效指令
- 去重命中、过滤命中或捕获失败时不发布复制成功音效
- 粘贴成功时发布 `paste_completed` 音效指令
- 预览内容首次显示成功时发布 `preview_revealed` 音效指令
- 音效播放失败只记录错误，不中断原业务链路

### 2.2 预览状态与详情回归

重点验证：

- `get_record_detail` 扩展字段后仍兼容文本、图片与文件列表记录
- `content_type = audio / video / document / link` 时可返回正确的 `preview_renderer`
- `preview_status` 从 `pending -> ready / failed / unsupported` 的状态流转正确
- 记录删除时同步清理 `preview_assets` 与缓存目录索引

### 2.3 文稿与链接预览准备回归

重点验证：

- PDF 文稿能返回可读取的预览资源引用或页索引
- `DOCX / XLSX / PPTX` 能生成结构化内容预览
- `DOC / XLS / PPT` 无法稳定解析时返回 `unsupported` 或明确失败状态
- 链接抓取超时、失败与成功场景都返回结构化结果，而不是未分类异常

### 2.4 媒体元数据与降级回归

重点验证：

- 音频时长、视频时长、MIME 与封面等元信息映射正确
- 媒体源缺失、文件被删除或编码不支持时进入降级态
- 降级不影响同一批次其他记录的读取与预览

## 3. 前端单元测试（Vitest + RTL）

### 3.1 预览联动交互

建议覆盖：

- 预览窗口可见时，主面板切换选中卡片会自动更新预览目标
- 预览窗口关闭后，继续切换选中卡片不会再次自动打开预览
- 快速上下切换时，只渲染最后一次目标记录结果
- 切换到被删除或不可访问记录时显示降级或关闭逻辑正确

### 3.2 音频 / 视频预览组件

建议覆盖：

- `audio` 渲染器显示播放 / 暂停控件与基础元信息
- `video` 渲染器显示视频播放器、封面或 loading 态
- 媒体源无效时显示统一 fallback 组件
- 切换预览目标时旧媒体状态被正确重置

### 3.3 文稿 / 链接预览组件

建议覆盖：

- `pdf` 渲染器显示文稿内容容器与页数信息
- `document` 渲染器可显示正文片段、工作表或幻灯片摘要
- `link` 渲染器可显示 URL、标题、摘要、站点信息与外部打开入口
- `pending / failed / unsupported` 三种状态渲染正确

### 3.4 既有预览回归

建议覆盖：

- 文本预览自动保存仍然可用
- 图片预览缩放与拖拽不受新增分支影响
- 文件列表预览在无预览资源时仍能正确显示路径信息

## 4. 集成测试

### 4.1 复制 / 粘贴音效主链路

- 模拟新记录成功入库
- 验证音效请求被触发且主面板列表刷新成功
- 模拟粘贴成功
- 验证粘贴成功音效与隐藏面板逻辑并存

### 4.2 预览联动链路

- 打开预览窗口
- 在主面板中连续切换不同类型记录
- 验证独立预览窗口在同一实例中更新内容
- 验证预览关闭后联动停止

### 4.3 媒体预览链路

- 打开音频记录预览并验证可播放
- 切换到视频记录并验证播放器更新
- 切换到失效媒体源并验证 fallback

### 4.4 文稿与链接预览链路

- 打开 PDF 记录并验证正文内容显示
- 打开 `DOCX / XLSX / PPTX` 并验证结构化内容渲染
- 打开链接记录并验证标题、摘要与 URL 展示
- 验证文稿 / 链接解析失败时的降级态与错误提示

### 4.5 预览资源更新链路

- 先返回 `preview_status = pending`
- 后续派发资源更新事件
- 验证当前预览窗口自动刷新而不需要重新打开

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- 继续使用项目当前 E2E 工具链，优先 `Playwright`
- 对音效播放可采用 mock 播放器或事件观测，不依赖真实声卡输出断言
- 对文稿与链接预览补充截图断言或关键 DOM 断言

### 5.2 必测主链路

- 复制任意内容成功入库 -> 播放复制音效
- 粘贴当前选中记录成功 -> 播放粘贴音效
- 打开预览 -> 播放预览音效 -> 上下切换记录 -> 预览跟随切换
- 打开音频 / 视频记录 -> 播放器可见且可操作
- 打开 PDF / Office / 链接记录 -> 主要内容可读或出现明确降级态

## 6. 测试数据与夹具规范

### 6.1 记录夹具

- 统一从 `SEED-1501 ~ SEED-1506` 组织数据
- 需要覆盖 `audio / video / document / link` 以及回归用 `text / image / files`
- 文稿样本应同时覆盖 `pdf / docx / xlsx / pptx`，并预留 `doc / xls / ppt` 降级样本

### 6.2 能力夹具

- 音效服务需要成功与失败两类 mock
- 链接抓取需要成功、超时、不可访问三类 mock
- 文稿解析需要成功、部分成功、失败三类 mock
- 媒体源需要有效文件、损坏文件与缺失文件三类 mock

### 6.3 脱敏要求

- 不得使用真实隐私文档、真实本地下载目录、真实受保护链接
- 链接使用公开可替代的演示地址
- 音效文件可直接复用项目内测试资源，不引用 `docs/音效/` 原路径做断言

## 8. 通过标准（Definition of Done）

- `v1.5` 每个 `P0` 用户故事至少具备 1 条单元测试和 1 条集成或 BDD 场景覆盖
- 新增音效、预览联动、音频 / 视频 / 文稿 / 链接预览全部具备成功场景与失败场景覆盖
- `v1.4` 的文本、图片、文件列表预览主链路无阻断回归
- 文稿与链接预览失败时均能稳定进入安全降级态，而不是白屏或崩溃
