# v1.5 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档只覆盖 `v1.5` 的音效反馈、预览自动联动、音频 / 视频预览、文稿预览与超链接预览的 BDD 自动化验收场景。

### 1.2 BDD 编写规范

- 场景命名采用“用户目标 + 可验证结果”方式
- 场景编号统一使用 `BDD-51-xx`
- `Given / When / Then` 优先围绕用户可观察行为编写，而不是内部实现细节

---

## 2. Feature：关键动作音效反馈（Sound Feedback）

### 场景 BDD-51-01：新复制内容成功入库后播放复制音效

- **Given** 系统正在监听剪贴板变化
- **When** 用户复制一条新的可入库内容
- **Then** 该记录进入历史列表
- **And** 系统播放复制成功音效

### 场景 BDD-51-02：粘贴成功后播放粘贴音效

- **Given** 用户已在主面板选中一条可粘贴记录
- **When** 用户执行一次成功的粘贴操作
- **Then** 目标内容被写回并完成粘贴
- **And** 系统播放粘贴成功音效

### 场景 BDD-51-03：预览窗口首次显示内容时播放预览音效

- **Given** 用户当前未打开预览窗口
- **When** 用户显式打开某条记录的预览
- **Then** 预览窗口显示该记录内容
- **And** 系统播放预览成功音效

---

## 3. Feature：预览窗口自动联动（Preview Auto-Follow）

### 场景 BDD-51-04：预览窗口显示时，切换选中卡片自动切换预览内容

- **Given** 用户已经打开预览窗口
- **And** 主面板当前存在多条不同类型记录
- **When** 用户使用键盘或鼠标切换到另一张卡片
- **Then** 预览窗口在同一实例中更新为新卡片内容
- **And** 旧内容不再残留为当前显示主体

### 场景 BDD-51-05：关闭预览窗口后，切换卡片不会再次自动打开预览

- **Given** 用户已手动关闭预览窗口
- **When** 用户继续在主面板中切换选中卡片
- **Then** 主面板选中状态正常变化
- **And** 不会自动再次弹出预览窗口

### 场景 BDD-51-06：预览联动遇到失效记录时进入安全降级

- **Given** 预览窗口当前可见
- **And** 当前选中目标记录在后台已被删除或资源不可访问
- **When** 预览窗口尝试切换到该记录
- **Then** 预览窗口显示明确错误或自动关闭
- **And** 主面板仍保持可继续操作

---

## 4. Feature：音频与视频预览（Media Preview）

### 场景 BDD-51-07：音频记录可在预览窗口中直接播放

- **Given** 用户选中一条 `audio` 类型记录
- **When** 用户打开预览窗口
- **Then** 预览窗口显示音频播放器
- **And** 用户可以播放、暂停与拖动进度

### 场景 BDD-51-08：视频记录可在预览窗口中直接播放

- **Given** 用户选中一条 `video` 类型记录
- **When** 用户打开预览窗口
- **Then** 预览窗口显示视频播放器或首帧封面
- **And** 用户可以播放、暂停与拖动进度

### 场景 BDD-51-09：媒体源不可用时显示降级提示

- **Given** 用户选中一条音频或视频记录
- **And** 其源文件已经缺失或不可访问
- **When** 用户打开预览窗口
- **Then** 预览窗口显示“无法预览”或等价降级提示
- **And** 不会导致窗口白屏或崩溃

---

## 5. Feature：文稿与超链接预览（Document & Link Preview）

### 场景 BDD-51-10：PDF 文稿可显示正文页面

- **Given** 用户选中一条 `PDF` 文稿记录
- **When** 用户打开预览窗口
- **Then** 预览窗口显示 PDF 正文内容
- **And** 用户可以滚动查看文稿页面

### 场景 BDD-51-11：Office 文稿可显示主要内容

- **Given** 用户选中一条 `DOCX`、`XLSX` 或 `PPTX` 文稿记录
- **When** 用户打开预览窗口
- **Then** 预览窗口显示正文、表格或幻灯片主要内容
- **And** 若无法高保真渲染，则至少显示结构化内容预览

### 场景 BDD-51-12：超链接记录可显示标题与摘要

- **Given** 用户选中一条 `link` 类型记录
- **When** 用户打开预览窗口
- **Then** 预览窗口显示规范化 URL、标题、摘要与站点信息
- **And** 提供在默认浏览器打开的入口

### 场景 BDD-51-13：文稿或链接解析失败时显示明确降级态

- **Given** 用户选中一条文稿或链接记录
- **And** 其解析过程失败、超时或不被支持
- **When** 用户打开预览窗口
- **Then** 预览窗口显示失败说明、原始文件名或 URL
- **And** 用户仍可关闭窗口或切换到其他记录

---

## 6. 非功能需求验收场景（Performance / Reliability / Readability）

### 场景 BDD-NFR-51：预览联动不会因快速切换而明显卡顿

- 用户连续切换多条记录时，预览窗口应保持稳定，仅最终停留在最后一次目标记录上

### 场景 BDD-NFR-52：音效反馈不会阻塞复制、粘贴与预览主流程

- 任一音效播放失败时，复制、粘贴和预览主链路仍能继续完成

### 场景 BDD-NFR-53：文稿与链接预览失败时不影响既有文本、图片、文件列表预览

- 新增预览类型的失败不应破坏 `v1.4` 已有预览主链路

## 8. 通过标准

- 所有 `P0` 用户故事至少对应 1 条通过的 BDD 场景
- 音效、预览联动、音频 / 视频 / 文稿 / 链接预览的关键场景全部通过
- 文稿与链接解析失败场景必须验证降级能力，而不是只验证成功链路
- 无阻断级回归影响既有文本、图片、文件列表预览与粘贴主流程
