# v1.5 版本迭代日志记录

## 2026-03-15

### 已完成

- 完成主面板复制语义修正：当主面板激活时按下 `Cmd/Ctrl + C`，应用会直接把当前选中卡片写回系统剪贴板，不再落回 WebView 默认复制空内容，同时该记录会立即提升到列表首位并驱动卡片区滚动回到最前
- 完成重复复制反馈修正：相同内容再次复制虽然仍然按去重策略复用原记录，但后台监听与主面板显式复制都会继续播放复制成功音效，避免“复制成功却无声反馈”的体验断层
- 完成打包版粘贴音效时序修正：前端原生补播改为在 `hidePanel` 之前先触发，避免面板隐藏后 WebView 调度被打断，导致补发粘贴音效没有真正送达 Tauri 原生层
- 完成粘贴音效兜底修正：Tauri 前端恢复对原生 `play_sound_effect` 的补充调用，Rust 音效层新增短时间重复播放抑制，保证粘贴成功时即使主链路偶发未命中也能补播，同时避免复制/粘贴双重触发导致的重复响铃
- 完成打包版音效最终修正：复制提示音改为在 Rust 剪贴板监听捕获新记录时直接触发，粘贴提示音改为在 Rust `paste_record` 成功后直接触发；Tauri 前端不再重复请求音效播放，彻底绕开隐藏窗口 / WebView 音频策略导致的无声问题
- 完成打包版媒体预览宿主收敛：独立预览窗口改为不使用毛玻璃 `backdrop-filter` 宿主承载媒体与 iframe，并新增未知渲染器降级提示与关键状态日志，避免打包后音频 / 视频 / 网址预览只剩标题栏或黑色内容区时无从定位
- 完成打包版音效链路重构：复制 / 粘贴提示音新增 Tauri 原生播放命令，macOS 打包版改为通过内置 MP3 资源落地到临时目录并调用 `afplay` 播放，前端 Howler 退化为非 Tauri / 原生失败时的兜底方案，修复打包后复制粘贴无声的问题
- 完成预览窗口按键回归修正：独立预览窗口忽略按住空格打开预览后产生的重复 `Space` 按键，避免媒体 / 网址预览刚打开就被同一次长按空格立刻关闭，表现为黑窗或只剩标题栏
- 完成预览运行时日志补强：预览详情 ready 后会记录 `content_type`、原始 / 解析渲染器、预览状态与核心资源来源，帮助继续定位打包版媒体 / 网页预览问题
- 完成打包版预览窗口黑屏修正：独立预览窗口改为非透明窗体，并为预览入口补上宿主背景；同时在前端为缺少 `preview_renderer` 的媒体 / 链接详情补充按 `content_type` 的渲染兜底，修复打包后音频 / 视频 / 网址预览出现整窗黑屏或只剩标题栏的问题
- 完成打包版音效回归修正：复制 / 粘贴提示音改为内联 MP3 资源并优先走 HTML5 Audio 播放，修复构建产物中音效资源已打包但实际无声的问题
- 完成图片完整预览质量修正：独立预览窗口与历史覆盖层预览统一只使用原图资源，不再在完整预览中回退到缩略图，避免放大查看时看到压缩后的预览图
- 完成网址预览极简收口：链接预览头部进一步压缩为单行地址栏样式，仅保留站点、地址与打开按钮
- 完成网址预览头部收紧：进一步压缩链接预览信息栏尺寸，去掉标题与地址重复展示，首屏只保留一份核心 URL 信息
- 完成网址预览体验重构：链接记录改为在预览窗口内直接内嵌网页页面，默认优先展示实际页面内容，不再以站点封面、简介和抓取摘要作为主视觉
