# v1.5 数据库设计文档

## 1. 文档概述

### 1.1 版本范围

本文档只描述 `v1.5` 为支持音效反馈、预览联动、音频 / 视频 / 文稿 / 链接预览所需的数据设计结论。

### 1.2 设计原则

- 保持 `payload_type` 与 `content_type` 分离，预览能力不能反向改变既有粘贴语义
- 音效文件作为项目资源打包，不进入数据库
- 可异步准备的预览内容统一抽象为“预览资源”，通过独立状态管理而不是塞进主表大字段
- 运行态联动状态保留在内存，不做落库

---

## 2. 存储布局总览

### 2.1 版本结论

`v1.5` **建议新增一次 SQLite schema 迁移**，把当前 schema 从 `v3` 升级到 `v4`，以支持预览状态与预览资源索引。

### 2.2 本版数据设计结论

- 继续保留已有 `clipboard_items`、`image_assets`、`file_items`
- 在 `clipboard_items` 中补充与预览相关的轻量字段
- 新增通用预览资源表 `preview_assets`
- 链接摘要、文稿抽取结果、视频封面等重资源落缓存目录，数据库只保留索引与状态

---

## 3. SQLite 业务模型设计

### 3.1 `clipboard_items` 建议新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `primary_uri` | `TEXT NULL` | 当 `content_type = link` 时保存规范化 URL；媒体远端地址场景也可复用 |
| `preview_renderer` | `TEXT NOT NULL DEFAULT 'summary'` | 当前记录推荐使用的预览渲染器，取值建议为 `text / image / audio / video / pdf / document / link / file_list / summary` |
| `preview_status` | `TEXT NOT NULL DEFAULT 'pending'` | 预览资源状态，取值建议为 `pending / ready / failed / unsupported` |
| `preview_error_code` | `TEXT NULL` | 最近一次预览准备失败的错误码 |
| `preview_updated_at` | `INTEGER NULL` | 最近一次预览状态刷新时间 |

### 3.2 与旧字段的关系

- 继续沿用 `payload_type` 作为底层粘贴语义真源
- `content_type` 继续表示展示 / 检索语义
- `preview_renderer` 是 `content_type` 的运行时展示派生，不替代 `content_type`
- `preview_status` 反映当前记录是否具备可直接展示的预览资源

### 3.3 约束建议

- `preview_renderer` 使用 `CHECK(preview_renderer IN ('text', 'image', 'audio', 'video', 'pdf', 'document', 'link', 'file_list', 'summary'))`
- `preview_status` 使用 `CHECK(preview_status IN ('pending', 'ready', 'failed', 'unsupported'))`
- `primary_uri` 仅在 `content_type = link` 或存在远端资源映射时允许非空

---

## 4. 关联表与资产设计

### 4.1 新增 `preview_assets` 表

建议新增表：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `INTEGER PRIMARY KEY` | 主键 |
| `item_id` | `INTEGER NOT NULL` | 关联 `clipboard_items.id` |
| `asset_role` | `TEXT NOT NULL` | 资源角色，如 `document_html`、`document_text`、`pdf_pages`、`link_summary`、`video_poster` |
| `storage_path` | `TEXT NULL` | 缓存文件路径 |
| `text_content` | `TEXT NULL` | 小体积可直接读取的抽取文本 |
| `mime_type` | `TEXT NULL` | 资源 MIME |
| `byte_size` | `INTEGER NOT NULL DEFAULT 0` | 资源大小 |
| `generator` | `TEXT NULL` | 资源生成器标识 |
| `status` | `TEXT NOT NULL DEFAULT 'ready'` | 当前资源状态 |
| `created_at` | `INTEGER NOT NULL` | 创建时间 |
| `updated_at` | `INTEGER NOT NULL` | 更新时间 |

### 4.2 资源角色建议

| `asset_role` | 说明 |
|--------------|------|
| `document_html` | Word / Excel / PPT 抽取后的 HTML 预览 |
| `document_text` | 文稿纯文本或结构化文本摘要 |
| `pdf_pages` | PDF 预览页缓存或页索引 |
| `link_summary` | 链接标题、摘要、站点信息 |
| `link_cover` | 链接主图缓存 |
| `video_poster` | 视频封面 |
| `audio_meta` | 音频扩展元信息，如时长或波形摘要 |

### 4.3 与现有表的关系

- `image_assets` 继续只负责图片原图与缩略图
- `file_items` 继续负责文件路径清单
- `preview_assets` 只保存为预览而生成的派生资源，不承接原始文件清单

## 5. 配置存储设计（`config.json`）

### 5.1 `v1.5` 结构结论

本版本默认**不开放新的用户设置项**，保持最小配置原则。

可选的内部结构预留：

- `preview.max_prepare_ms`
- `preview.max_document_preview_pages`
- `preview.max_link_summary_chars`

### 5.2 不进入配置的内容

以下内容在本版中仍作为内建默认体验，不开放设置：

- 音效文件路径选择
- 复制 / 粘贴 / 预览音效开关
- 预览联动开关
- 文稿预览保真度模式切换

---

## 6. 查询与索引约束

### 6.1 查询策略

- 摘要列表继续从 `clipboard_items` 读取
- 打开预览后，通过详情查询联动读取 `preview_assets`
- `preview_status != 'ready'` 时，前端允许先显示 loading / fallback，再等待资源更新事件

### 6.2 推荐索引

| 索引名 | 说明 |
|--------|------|
| `idx_clipboard_items_preview_status_last_used_at` | 支持预览状态过滤与默认排序 |
| `idx_clipboard_items_primary_uri` | 支持链接记录定位与去重辅助 |
| `idx_preview_assets_item_id_asset_role` | 支持按记录和资源角色快速读取 |
| `idx_preview_assets_status_updated_at` | 支持后台扫描失败与重试 |

### 6.3 详情读取约束

- 同一条记录可存在多条 `preview_assets`
- `text_content` 仅适合保存小体积摘要或结构化文本，不建议存超大正文
- 超大文稿页缓存与主图缓存优先走磁盘文件，数据库只保留索引

---

## 7. 运行态数据设计

以下状态只在内存中存在，不应落库：

| 数据 | 说明 |
|------|------|
| `previewSession.visible` | 预览窗口当前显隐 |
| `previewSession.followSelection` | 当前会话是否允许联动 |
| `previewTargetRecordId` | 当前预览目标记录 |
| `soundPlaybackState` | 单次音效播放的瞬时状态 |
| `mediaPlaybackState` | 音频 / 视频当前播放位置、暂停态、静音态 |

---

## 8. 运行时生成目录

### 8.1 推荐布局

```text
app_data/
  preview/
    documents/
    links/
    posters/
    temp/
  images/
  logs/
```

### 8.2 说明

- `documents/`：文稿 HTML、文本片段或页索引缓存
- `links/`：链接摘要与主图缓存
- `posters/`：视频封面与可选媒体附属图
- 音效文件不放运行时缓存，统一放项目静态资源目录并参与应用打包

---

## 9. 性能与可靠性目标

- 预览打开时优先秒开主框架，重资源解析允许异步进行
- 同一记录的预览资源可复用，不重复解析
- 预览资源失败不应污染原始记录，也不应影响历史列表读取
- 删除记录时应同时清理其 `preview_assets` 与关联缓存文件

---

## 10. 与 v1.4 的主要差异

- `v1.4` 主要依赖既有详情表结构，`v1.5` 需要显式引入预览状态与预览资源索引
- `v1.4` 的文件型记录主要展示路径列表，`v1.5` 要为 `audio / video / document / link` 提供真正的预览资产模型
- `v1.4` 没有音效资源的数据边界，`v1.5` 明确音效只属于项目资源，不进数据库
