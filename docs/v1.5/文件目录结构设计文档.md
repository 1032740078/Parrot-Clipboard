# v1.5 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.5` 为支持音效反馈、预览联动、多媒体 / 文稿 / 超链接预览所建议采用的目录结构设计。

### 1.2 设计原则

- 在既有结构上做增量扩展，不重构已稳定的 `clipboard / window / components` 主目录
- 音效资源统一进入项目静态资源目录，并改为英文命名
- 预览渲染与预览资源准备分层：前端负责渲染器，后端负责资源准备与缓存
- 文档中允许使用“推荐新增 / 建议归属”的表达，作为后续开发落地参考

## 3. 前端目录设计（`src/`）

### 3.1 `v1.5` 重点文件

| 文件 / 目录 | 说明 |
|-------------|------|
| `src/components/PreviewWindow.tsx` | 继续作为独立预览窗口入口 |
| `src/components/MainPanel/PreviewOverlay.tsx` | 主面板完整预览浮层继续复用既有能力 |
| `src/hooks/useRecordPreviewDetail.ts` | 继续承接详情读取与缓存 |
| `src/hooks/useKeyboard.ts` | 需接入预览联动触发逻辑 |
| `src/api/types.ts` | 扩展 `audio_detail / video_detail / document_detail / link_detail` |
| `src/api/events.ts` | 扩展预览资源状态事件订阅 |

### 3.2 推荐新增辅助文件

```text
src/
  assets/
    sounds/
      copy-notification.mp3
      paste-notification.mp3
      preview-open.mp3
  components/
    preview/
      AudioPreview.tsx
      VideoPreview.tsx
      PdfPreview.tsx
      DocumentPreview.tsx
      LinkPreview.tsx
      PreviewFallback.tsx
      rendererResolver.ts
      vidstack/
        mediaPlayer.tsx
  hooks/
    usePreviewFollow.ts
    usePreparedPreview.ts
  lib/
    audio/
      howlerPlayer.ts
```

### 3.3 推荐约束

- 音效文件统一使用英文 kebab-case 命名
- 预览具体渲染组件放在 `src/components/preview/`，不要继续把所有分支堆进单个窗口文件
- 预览跟随逻辑与具体渲染器逻辑分离，避免 `useKeyboard.ts` 直接耦合全部预览实现细节
- `Vidstack` 作为音频 / 视频预览的统一播放器壳层，项目内只封装一层适配，不在业务组件中直接散落播放器初始化逻辑
- `Howler.js` 只用于短音效播放，不承担内容级音频预览播放器职责

## 4. 后端目录设计（`src-tauri/src/`）

### 4.1 `v1.5` 重点文件

| 文件 / 目录 | 说明 |
|-------------|------|
| `src-tauri/src/ipc/commands.rs` | 扩展预览准备命令与详情返回结构 |
| `src-tauri/src/ipc/events.rs` | 增加预览资源更新事件常量 |
| `src-tauri/src/window/preview_window.rs` | 保持窗口复用与目标切换能力 |
| `src-tauri/src/persistence/sqlite.rs` | 扩展预览资源读写 |
| `src-tauri/src/persistence/migrations.rs` | 新增 schema `v4` 迁移 |

### 4.2 模块归属说明

```text
src-tauri/src/
  preview/
    mod.rs
    media.rs
    document.rs
    link.rs
    cache.rs
  sound/
    mod.rs
    player.rs
    catalog.rs
```

说明：

- `preview/` 负责不同内容类型的预览资源准备、状态更新与缓存读写
- `sound/` 负责英文音效资源目录、音效键映射与播放实现
- 不建议把文稿解析、链接抓取与音效播放继续分散到 `window/` 或 `clipboard/` 模块中

## 6. 运行时生成目录

### 6.1 推荐布局

```text
app_data/
  preview/
    documents/
    links/
    posters/
    temp/
```

### 6.2 约束说明

- 文稿与链接预览缓存属于运行时目录，不提交到仓库
- 音效资源属于项目静态资源，不进入运行时缓存目录
- 删除记录时应尝试清理对应缓存文件，避免长期堆积

---

## 7. 命名规范

- 音效文件：英文 kebab-case，如 `copy-notification.mp3`
- 预览资源角色：英文 snake_case，如 `document_html`、`link_summary`
- 预览渲染器组件：`PascalCase.tsx`
- 后端预览模块：`snake_case.rs`
- 事件名继续沿用 `kebab-case` 风格，如 `clipboard:preview-asset-updated`

---

## 8. 与 v1.4 的主要差异

- `v1.4` 没有独立音效资源目录，`v1.5` 需要显式新增 `src/assets/sounds/`
- `v1.4` 的预览逻辑主要聚焦 `PreviewWindow.tsx` 单文件分支，`v1.5` 推荐拆出 `src/components/preview/` 子目录
- `v1.4` 后端没有独立 `preview/` 与 `sound/` 模块，`v1.5` 需要把多媒体 / 文稿 / 链接预览与音效播放能力模块化
- `v1.5` 明确引入开源库策略：媒体预览优先 `Vidstack`，短音效优先 `Howler.js`
