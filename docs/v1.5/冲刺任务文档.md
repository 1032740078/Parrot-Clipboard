# v1.5 冲刺任务文档

## 1. 冲刺目标

在保持 `v1.4` 搜索、筛选、文本编辑器、图片预览、文件列表预览与多屏定位稳定的前提下，完成 `v1.5` 的 5 个核心目标：

- 新增复制、粘贴、显示预览三类音效反馈
- 把音效资源复制到项目内并统一改为英文命名
- 让预览窗口在已打开状态下跟随当前选中卡片自动切换内容
- 为音频与视频记录补齐可直接播放的预览体验
- 为 PDF、Office 文稿和超链接记录补齐内容级预览与降级策略

**任务**：

### 3.2 第 1 批：音效反馈

**目标**：让复制、粘贴、显示预览三条主链路具备清晰反馈。

**任务**：

- 把 `docs/音效/` 中的文件复制到项目资源目录并改为英文命名
- 基于 `Howler.js` 新增音效资源索引与播放服务
- 在复制入库成功、粘贴成功、预览首次显示成功后触发音效
- 确保音效失败不会阻断业务主流程

**当前进展（2026-03-14）**：

- 已完成 `TASK-15-01-1`、`TASK-15-01-2`
- 3 个音效素材已复制到 `src/assets/sounds/`，并统一改名为英文文件名
- 已新增前端 `SoundEffectService`，在复制成功、粘贴成功、预览首次显示成功时触发音效

**目标**：把独立预览窗口升级为连续浏览工作区。

**任务**：

**当前进展（2026-03-14）**：

## 4. 风险与缓解

| 风险 | 说明 | 缓解策略 |
|------|------|----------|
| 音效播放过于频繁 | 快速操作时可能造成听觉打扰 | 只在关键成功节点播放，联动切换默认不重复播放打开音效 |
| 预览联动抖动 | 快速切换记录会触发频繁刷新 | 允许轻量节流，只保留最后一次目标记录 |
| 媒体源不可访问 | 文件可能已被移动、删除或权限受限 | 进入统一 fallback，并展示文件名、路径或错误说明 |
| 播放器库不等于万能解码器 | 统一播放器方案不能突破底层 WebView 编解码限制，且还要考虑与当前前端栈的兼容性 | 首版先采用原生媒体元素稳定落地，对不支持编码显式降级，不把兼容性承诺写得过满 |
| 文稿高保真预览难度高 | Office 与 PDF 解析复杂度较高 | 采用“PDF 优先 + Office 结构化内容 + 旧格式降级”策略 |
| 链接预览不稳定 | 远端站点抓取受网络、权限与 CSP 影响 | 默认做摘要型预览，失败时保留 URL 与外部打开入口 |
| 新预览破坏旧能力 | 预览分支过多可能影响文本 / 图片 / 文件列表 | 维持渲染器分层与专项回归测试 |

3. 对链接预览抓取失败率做一次统计，决定后续版本是否需要缓存重试或更强抓取策略
