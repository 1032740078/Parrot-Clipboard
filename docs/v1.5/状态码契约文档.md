# v1.5 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.5` 为支持音效反馈、预览自动联动、多媒体 / 文稿 / 超链接预览所涉及的状态码契约。

### 1.2 错误对象格式

```ts
interface IpcError {
  code: ErrorCode;
  message: string;
  details?: Record<string, unknown>;
}
```

### 1.3 错误传递机制

- Tauri Command 失败时统一返回结构化 `IpcError`
- 预览类错误优先允许进入局部降级，不应默认中断主面板主流程
- 音效播放失败只用于记录日志与调试，不应默认弹出阻断式错误反馈

## 3. 状态码详细说明

### SC-001：`INVALID_PARAM`

- 适用场景：`recordId <= 0`、预览准备命令缺失必要参数、无效 `preview_renderer`
- 前端处理：阻止继续请求，并提示用户修正输入

### SC-002：`RECORD_NOT_FOUND`

- 适用场景：预览联动过程中目标记录已被删除，或详情查询时记录不存在
- 前端处理：关闭当前局部预览或刷新列表后重新选择

### SC-003：`CLIPBOARD_WRITE_ERROR`

- 适用场景：粘贴或文本保存后的写回失败
- 前端处理：保留当前窗口状态并提示失败

### SC-004：`KEY_SIM_ERROR`

- 适用场景：系统模拟粘贴按键失败
- 前端处理：提示系统权限或环境问题

### SC-005：`DB_ERROR`

- 适用场景：读取预览资源、更新预览状态或清理缓存索引时数据库异常
- 前端处理：进入局部错误态并保留主面板可用

### SC-006：`FILE_ACCESS_ERROR`

- 适用场景：预览文件、文稿原始路径、封面文件或缓存文件不可访问
- 前端处理：回退到文件信息或 URL 信息，不阻断其他记录

### SC-007：`IMAGE_PROCESS_ERROR`

- 适用场景：图片缩略图或与图片有关的派生资源处理异常
- 前端处理：显示错误占位或文本降级

### SC-008：`WINDOW_ERROR`

- 适用场景：预览窗口打开失败、关闭失败或联动切换失败
- 前端处理：优先尝试一次安全降级，再提示用户

### SC-009：`UNSUPPORTED_PLATFORM_FEATURE`

- 适用场景：当前平台缺失某种媒体解码、文稿解析或音效播放能力
- 前端处理：应用降级样式或安全默认行为

### SC-010：`INTERNAL`

- 适用场景：未被更细状态码覆盖的异常
- 前端处理：提示通用错误并记录日志

### SC-015：`SOUND_PLAY_FAILED`

- 适用场景：英文音效资源找不到、解码失败或播放设备不可用
- 前端处理：不弹阻断提示，必要时只打日志

### SC-016：`PREVIEW_ASSET_NOT_READY`

- 适用场景：文稿抽取、链接抓取、封面生成尚未完成
- 前端处理：显示 loading / preparing 状态，并等待更新事件

### SC-017：`PREVIEW_SOURCE_MISSING`

- 适用场景：原始文件已被删除、URL 为空、缓存文件丢失
- 前端处理：显示源缺失提示与降级信息

### SC-018：`PREVIEW_RENDER_FAILED`

- 适用场景：预览资源已经存在，但当前渲染器无法正常展示
- 前端处理：切换到通用 fallback 视图

### SC-019：`DOCUMENT_PARSE_FAILED`

- 适用场景：PDF 或 Office 文稿抽取正文、结构化内容失败
- 前端处理：回退到文件元信息、文件列表或“无法预览”提示

### SC-020：`MEDIA_DECODE_FAILED`

- 适用场景：音频 / 视频编码不支持或媒体解码失败
- 前端处理：显示“不支持当前媒体编码预览”之类提示

### SC-021：`LINK_FETCH_FAILED`

- 适用场景：网页抓取超时、被拒绝或摘要提取失败
- 前端处理：显示原始 URL 与失败说明，同时允许外部打开

### SC-022：`PREVIEW_UNSUPPORTED_TYPE`

- 适用场景：当前内容类型在 `v1.5` 未纳入预览范围
- 前端处理：显示通用说明，不继续尝试细分渲染器

## 5. 前端统一处理约定

- `SC-016` 优先在预览区内联展示，不需要全局 Toast
- `SC-017 ~ SC-022` 优先走预览降级，而不是关闭整个主面板
- `SC-015` 默认为静默处理，仅在诊断或开发模式下增强提示
- `SC-008` 若发生在联动切换过程中，允许保留窗口并展示错误态，而不是直接销毁窗口

---

## 6. 成功响应约定

- `prepare_record_preview` 成功时，即使资源仍在后台准备，也返回结构化 `preview_status`
- `get_record_detail` 成功时允许返回 `preview_status = pending` 与局部空预览字段
- 预览资源准备成功后，应通过事件或重新拉取详情的方式让前端感知状态变化
- 音效成功播放不要求前端额外等待确认，业务成功以原主链路结果为准
