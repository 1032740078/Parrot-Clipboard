# v0.3 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.3 Feature Complete 版本** 的自动化测试策略，覆盖：
- 系统托盘与监听控制
- 开机自启动（macOS Launch Agent）
- 主面板数字键快选、动效、删除与显隐
- 多显示器定位
- 清空历史与系统集成错误处理

### 1.2 测试架构分层

| 层级 | 目标 | 主要工具 |
|------|------|----------|
| Rust 单元测试 | 校验托盘、自启动、窗口定位、清空历史的核心规则 | `cargo test` |
| 前端单元测试 | 校验键盘交互、状态更新、动效降级与 UI 反馈 | `Vitest` + `RTL` |
| 后端集成测试 | 校验配置、数据库、文件系统、Launch Agent 的协同 | `cargo test` |
| BDD / E2E | 校验用户视角主链路 | `Playwright` + Tauri Mock |

### 1.3 覆盖率目标

| 范围 | 指标 |
|------|------|
| 前端 `lines / branches / functions / statements` | `>= 75%` |
| Rust 新增模块关键分支 | `>= 80%` |
| P0 用户故事自动化覆盖 | `100%` |

## 2. 后端单元测试（Rust / cargo test）

### 2.1 自启动管理

建议覆盖：
- `UT-BE-AUTOSTART-001`：开启自启动时生成合法 Launch Agent plist
- `UT-BE-AUTOSTART-002`：关闭自启动时删除 plist 或标记为禁用
- `UT-BE-AUTOSTART-003`：重复启用 / 禁用操作具备幂等性
- `UT-BE-AUTOSTART-004`：配置与系统状态不一致时可进行弱修复

### 2.2 托盘与监听控制

建议覆盖：
- `UT-BE-TRAY-001`：根据 `monitoring` 状态生成正确菜单文案
- `UT-BE-TRAY-002`：暂停监听后 `is_monitoring=false`
- `UT-BE-TRAY-003`：恢复监听后新复制内容允许重新入库
- `UT-BE-TRAY-004`：托盘初始化失败时返回 `TRAY_ERROR`

### 2.3 清空历史与资源清理

建议覆盖：
- `UT-BE-CLEAR-001`：清空历史会删除三张业务表数据
- `UT-BE-CLEAR-002`：清空历史会删除图片原图与缩略图
- `UT-BE-CLEAR-003`：图片文件删除失败时数据库仍保持已清空状态并记日志

### 2.4 多显示器定位

建议覆盖：
- `UT-BE-WINDOW-001`：根据目标显示器 `work_area` 计算底部面板位置
- `UT-BE-WINDOW-002`：Dock 在左 / 右 / 下方时定位仍正确
- `UT-BE-WINDOW-003`：单显示器与多显示器共用同一计算逻辑

## 3. 前端单元测试（Vitest + RTL）

### 3.1 键盘交互测试

建议覆盖：
- `UT-FE-KB-001`：按 `1-9` 选中对应卡片
- `UT-FE-KB-002`：超出记录数量的数字键不改变选中项
- `UT-FE-KB-003`：数字键快选后按 `Enter` 能粘贴当前选中记录
- `UT-FE-KB-004`：`Delete` 删除后焦点自动移动到下一条合理记录

### 3.2 状态与 UI 测试

建议覆盖：
- `UT-FE-STATE-001`：收到 `system:monitoring-changed` 后 UI 状态同步更新
- `UT-FE-STATE-002`：收到 `clipboard:history-cleared` 后列表清空并显示空状态
- `UT-FE-STATE-003`：监听暂停态下展示弱提示，不影响历史浏览

### 3.3 动效与降级测试

建议覆盖：
- `UT-FE-MOTION-001`：主面板挂载时带有进入动画配置
- `UT-FE-MOTION-002`：删除记录时存在离场动画 key
- `UT-FE-MOTION-003`：`prefers-reduced-motion` 场景下降级为无动画或弱动画

## 4. 后端集成测试

### 4.1 配置与自启动联动

建议覆盖：
- `IT-BE-CONFIG-001`：`launch_at_login=true` 时写入配置并成功注册 Launch Agent
- `IT-BE-CONFIG-002`：旧配置升级时自动补齐 `launch_at_login`

### 4.2 历史清空联动

建议覆盖：
- `IT-BE-CLEAR-001`：清空历史后摘要列表为空
- `IT-BE-CLEAR-002`：清空后重新复制内容可正常重新入库

### 4.3 窗口与显示器定位

建议覆盖：
- `IT-BE-WINDOW-001`：主面板显示到当前活动显示器底部
- `IT-BE-WINDOW-002`：从托盘与快捷键打开主面板，定位结果一致

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- `Playwright`：驱动前端视图与键盘事件
- `Tauri Mock Runtime`：模拟 `invoke`、事件订阅、托盘动作回放
- `临时目录夹具`：模拟 `clipboard.db`、图片资源目录与 Launch Agent 文件

### 5.2 必测主链路

| 链路 | 说明 |
|------|------|
| 托盘显示 / 隐藏面板 | 保证无快捷键场景可操作 |
| 托盘暂停 / 恢复监听 | 保证敏感复制不入库 |
| 托盘清空历史 | 保证确认后数据与资源同步清空 |
| 托盘开机自启动勾选切换 | 保证状态、配置、系统文件一致 |
| 数字键快选 + `Enter` 粘贴 | 保证高频操作路径更快 |

## 6. 测试数据与夹具规范

### 6.1 历史记录夹具

- 至少准备 `12` 条混合记录，用于覆盖数字键 `1-9`
- 至少准备 `2` 条图片记录，验证清空历史后的图片资源删除
- 至少准备 `1` 组多文件记录，验证清空后的 `file_items` 联动删除

### 6.2 系统集成夹具

- `launch-agent-enabled.plist`
- `launch-agent-disabled.plist`（不存在即可）
- `display-layouts.json`（单屏 / 双屏 / 三屏）
- `tray-runtime-state.json`（运行 / 暂停 / 初始化失败）

### 6.3 脱敏要求

- 所有路径均使用 `/mock/` 或临时目录
- 所有截图、图片、文件名不得包含真实姓名、公司名、项目代号

## 7. CI 建议执行顺序

```bash
pnpm run lint
pnpm run type-check
pnpm run test:coverage
cargo fmt --manifest-path src-tauri/Cargo.toml --all -- --check
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings
cargo test --manifest-path src-tauri/Cargo.toml
pnpm run test:e2e
```

说明：
- `test:e2e` 放在最后，避免影响前两层反馈速度
- 涉及 Launch Agent 的测试必须使用临时目录隔离，不得写入用户真实 `~/Library/LaunchAgents`

---

## 8. 通过标准（Definition of Done）

- 所有 P0 用户故事至少有 1 条自动化测试覆盖
- 托盘、监听、自启动、多显示器、清空历史均有 Rust 层验证
- 主面板数字键快选、删除与空状态切换有前端测试覆盖
- BDD / E2E 跑通至少 5 条主链路
- 自动化测试不依赖真实用户桌面环境或真实启动项
