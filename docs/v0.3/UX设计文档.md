# v0.3 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

`v0.3` 的 UX 重点从“能用”提升为“像系统工具一样顺手”：
- 托盘常驻
- 状态清晰
- 快捷操作更快
- 多屏场景不迷路
- 动效更自然但可降级

### 1.2 设计原则（v0.3 强调）

- **后台感**：常驻但不打扰，托盘是第一入口
- **快进快出**：主面板仍是一眼可扫、一键可走
- **状态透明**：监听暂停、自启动开启、清空历史等状态必须可见
- **多屏就近**：在哪里工作，就在哪里看到面板

---

## 2. 系统托盘 UX 规范

### 2.1 托盘图标

- 正常态：清晰、低干扰、适配深浅色菜单栏
- 暂停态：弱化或增加暂停标识
- 异常态：尽量避免常驻错误红点，仅在日志中记录并用一次性提示告知

### 2.2 托盘菜单排序

推荐顺序：
1. 显示 / 隐藏主面板
2. 暂停 / 恢复监听
3. 开机自启动（勾选）
4. 清空历史
5. 打开日志目录
6. 退出应用

排序原则：高频在上，危险项在下。

---

## 3. 主面板视觉与信息层级

### 3.1 信息层级

1. 当前选中的卡片
2. 卡片类型与内容摘要
3. 监听暂停提示
4. 底部快捷键提示

### 3.2 数字键快选提示

- 仅前 `9` 条记录显示数字槽位
- 数字标记放在卡片角落，不与内容争抢主视觉
- 主面板底栏必须出现 `1-9 快选` 提示

---

## 4. 动效设计

### 4.1 面板弹出 / 收起

| 动作 | 时长 | 说明 |
|------|------|------|
| 打开主面板 | `220ms` | 自下向上轻微上浮 |
| 关闭主面板 | `180ms` | 向下收回并淡出 |

### 4.2 卡片切换

| 动作 | 时长 | 说明 |
|------|------|------|
| 键盘左右切换 | `120ms` | 仅做轻量高亮过渡 |
| 数字键快选 | `100ms` | 直接跳转，不做复杂滚动动画 |

### 4.3 删除动效

| 动作 | 时长 | 说明 |
|------|------|------|
| 删除当前卡片 | `160ms` | 缩小 + 淡出，随后焦点切换 |

### 4.4 动效降级

- 检测 `prefers-reduced-motion`
- 降级后保留状态变化，不保留位移与缩放

---

## 5. 多显示器体验

### 5.1 定位规则

- 面板永远优先出现在当前活动显示器底部
- 基于 `work_area` 而不是物理屏幕尺寸
- 避让 Dock 与菜单栏

### 5.2 预期感受

- 用户无需跨屏寻找面板
- 即便使用副屏工作，面板也像“就地弹出”

---

## 6. 反馈设计

### 6.1 操作反馈矩阵

| 操作 | 反馈 |
|------|------|
| 暂停监听 | 托盘图标变化 + 主面板弱提示 |
| 恢复监听 | 托盘图标恢复 + 弱提示消失 |
| 开机自启动切换成功 | 短 Toast 或菜单勾选变化 |
| 清空历史成功 | 主面板切为空状态 + 成功提示 |
| 清空历史失败 | 错误 Toast + 列表保持原状 |

### 6.2 Toast 规范

- 信息型：用于纯文本模式、自启动切换成功
- 错误型：用于清空历史失败、自启动失败、托盘不可用
- 不建议对“暂停监听”使用强打断 Toast，优先用弱提示条

---

## 7. 错误与降级体验

### 7.1 托盘不可用

- 记录日志
- 应用继续保留快捷键路径
- 不弹持续性错误，只提示一次

### 7.2 自启动切换失败

- 不修改 UI 为成功态
- 引导用户查看日志目录

### 7.3 清空历史失败

- 不提前清空前端本地列表
- 保持当前可见历史，避免“看起来删了其实没删”的错觉

---

## 8. 可用性指标

| 指标 | 目标 |
|------|------|
| 托盘菜单响应 | `< 100ms` |
| 主面板显示动效总时长 | `<= 220ms` |
| 数字键快选可感知延迟 | `< 50ms` |
| 多显示器定位成功率 | `100%`（自动化夹具覆盖范围内） |

---

## 9. macOS 平台特定 UX 规范

- 托盘图标建议使用 Template Image，适配深浅菜单栏
- 自启动采用符合 macOS 习惯的登录项语义，不使用侵入式弹窗
- 主面板在所有 Space 与全屏辅助场景下保持可见一致性
