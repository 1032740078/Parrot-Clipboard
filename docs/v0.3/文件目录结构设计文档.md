# v0.3 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

`v0.3` 的目录设计在 `v0.2` 基础上，重点补充：
- 托盘模块
- 自启动模块
- 运行态状态管理
- 多显示器定位与系统集成测试夹具

### 1.2 设计原则

- 系统集成能力优先落在 `src-tauri/src/`，避免前端承担平台差异
- 前端仍以主面板为中心，不提前引入设置页目录
- 测试夹具按“历史数据”与“系统集成状态”分层
- 新增目录命名保持职责单一，避免把托盘 / 自启动逻辑继续堆在 `lib.rs`

## 6. 运行时生成目录

```text
~/Library/Application Support/clipboard-record-manager/
├── clipboard.db
├── config.json
├── images/
│   ├── original/
│   └── thumbs/
└── logs/

~/Library/LaunchAgents/
└── com.robin.clipboard-record-manager.plist
```

### 6.1 约束说明

- `clipboard.db` 仅存历史数据，不存托盘和运行态
- `config.json` 存持久化偏好，如 `launch_at_login`
- `LaunchAgents` 仅在 macOS 真机运行时写入；测试必须改写到临时目录

---

## 7. 命名规范

- Rust 模块使用 `snake_case`
- 托盘、自启动、系统状态相关类型以 `Tray` / `Autostart` / `System` 前缀命名
- 前端 Store 使用 `useXxxStore.ts`
- 系统事件文件统一归到 `hooks/useSystemEvents.ts`

---

## 8. 与 v0.2 的主要差异

| 项目 | v0.2 | v0.3 |
|------|------|------|
| 前端状态 | `ClipboardStore` + `UIStore` | 新增 `SystemStore` |
| 后端模块 | 无托盘 / 自启动目录 | 新增 `tray/`、`autostart/` |
| 测试夹具 | 仅历史数据 | 新增系统集成状态与显示器布局夹具 |

目录设计的核心意图是：**把平台相关代码收敛到后端，把主面板继续保持轻量。**
