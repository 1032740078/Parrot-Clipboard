# v0.3 种子数据契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.3 Feature Complete 版本** 的种子数据与测试夹具契约，用于：
- 本地开发演示托盘与主面板协同行为
- 后端集成测试（清空历史、自启动、监听控制）
- 前端数字键快选与空状态测试
- BDD 自动化验收准备

### 1.2 设计原则

- 延续 `v0.2` 的三种历史记录覆盖
- 新增系统集成状态夹具：监听状态、自启动状态、多显示器布局
- 所有系统文件（如 Launch Agent）必须使用测试专用临时目录
- 所有图片、文件、路径、日志目录均需脱敏

---

## 2. 标准种子数据集

### 2.1 SEED-301：快速选择演示集（开发演示用）

**用途**：覆盖主面板 `1-9` 快选、混合卡片浏览与删除后的重排。

**建议文件**：`fixtures/seeds/v0.3/demo-quick-select.json`

**建议内容**：
- 12 条混合记录（至少 6 文本、3 图片、3 文件）
- 最近 9 条必须都可见，便于数字键直接映射

**示例结构**：

```json
{

  "records": [
    {
      "content_type": "text",

      "source_app": "飞书",
      "created_at": 1772839200000
    },
    {
      "content_type": "image",
      "preview_text": "状态栏图标草图",
      "source_app": "Preview",
      "created_at": 1772839260000,
      "image_asset": {
        "original_fixture": "images/tray-icon-spec.png",
        "thumbnail_fixture": "images/tray-icon-spec-thumb.png",
        "mime_type": "image/png",
        "pixel_width": 1024,

    }

```

### 2.2 SEED-302：清空历史回归集（后端 / E2E）

**用途**：验证 `clear_history` 对数据库与图片资源的双向清理。

**建议目录**：`src-tauri/tests/fixtures/v0.3/clear-history/`

**最小内容**：
- 4 条文本记录
- 2 条图片记录（含原图与缩略图）
- 2 条文件记录
- 清理前后的期望快照文件

### 2.3 SEED-303：系统集成状态夹具集

**用途**：验证托盘与自启动状态切换。

**建议文件**：
- `config-enabled.json`
- `config-disabled.json`
- `launch-agent-enabled.plist`
- `runtime-monitoring-paused.json`

**示例结构**：

```json
{

    "max_file_records": 100,
    "toggle_shortcut": "Shift+Command+V",

    "monitoring": false,
    "tray_state": "paused"
  }
}
```

### 2.4 SEED-304：多显示器布局夹具集

**用途**：验证主面板定位逻辑。

**建议文件**：`src-tauri/tests/fixtures/v0.3/display-layouts.json`

**覆盖布局**：
- 单显示器
- 左右双屏
- 上下双屏
- 右屏为活动屏幕且 Dock 位于右侧

## 3. 类型覆盖要求

### 3.1 历史记录覆盖

- 文本：短文本、长文本、多行文本、富文本降级文本
- 图片：`pending` / `ready` / `failed` 三种缩略图状态
- 文件：单文件、多文件、包含目录

### 3.2 系统状态覆盖

- 监听运行态
- 监听暂停态
- 开机自启动开启态
- 开机自启动关闭态
- 托盘初始化失败降级态

### 3.3 显示器布局覆盖

- 当前活动显示器为主屏
- 当前活动显示器为副屏
- 不同 `work_area` 尺寸与 Dock 位置

---

## 4. 资产文件契约

### 4.1 图片资产目录

```text
fixtures/seeds/v0.3/images/
├── tray-icon-spec.png
├── tray-icon-spec-thumb.png
├── dual-screen-mock.png
├── dual-screen-mock-thumb.png
└── broken-thumb.placeholder
```

### 4.2 Launch Agent 夹具目录

```text
fixtures/seeds/v0.3/launch-agents/
└── com.robin.clipboard-record-manager.plist
```

### 4.3 路径约束

- Launch Agent 路径统一映射到测试临时目录，例如 `/tmp/mock-launch-agents/...`
- 文件记录路径统一使用 `/mock/workspace/...`
- 不允许使用开发者真实桌面、下载目录或用户名路径

---

## 5. 加载约定

### 5.1 开发模式加载

建议使用环境变量：

```bash
CLIPBOARD_SEED_FILE=fixtures/seeds/v0.3/demo-quick-select.json
CLIPBOARD_SEED_RESET=1
CLIPBOARD_SYSTEM_FIXTURE=fixtures/seeds/v0.3/config-enabled.json
```

### 5.2 测试模式加载

- 后端集成测试使用临时目录复制 `db`、`images`、`launch-agents` 夹具
- 前端单测直接导入 TypeScript Fixture，不依赖真实文件系统
- E2E 通过预置脚本创建假的 Launch Agent 根目录与日志目录

---

## 6. 覆盖的测试场景

| 场景 | 对应种子 |
|------|----------|
| 数字键 1-9 快选 | `SEED-301` |
| 删除重排 | `SEED-301` |
| 清空历史 | `SEED-302` |
| 自启动开启 / 关闭 | `SEED-303` |
| 监听暂停 / 恢复 | `SEED-303` |
| 多显示器定位 | `SEED-304` |

---

## 7. 数据清理约定

- 测试结束后删除临时 `clipboard.db`、临时图片目录、临时 Launch Agent 目录
- 所有 E2E 遗留文件必须由 `teardown` 清理
- 不得向真实 `~/Library/LaunchAgents` 写入测试文件
