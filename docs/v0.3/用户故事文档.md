# v0.3 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 **v0.3 Feature Complete 版本** 的用户故事，聚焦在 `v0.2` 已完成的多类型记录、持久化与再次粘贴能力之上，补齐当前版本新增的系统集成与交互完善能力。

**v0.3 核心目标**：
- 引入系统托盘，提供后台常驻入口与高频操作菜单
- 支持 macOS 开机自启动（Launch Agent）
- 支持暂停 / 恢复监听、清空历史等运行期控制能力
- 优化主面板交互：数字键 `1-9` 快速选择、删除与显隐动画、多显示器定位
- 让单平台 macOS 版本达到可长期使用的“准生产可用”状态

**不在 v0.3 范围内**：
- 独立设置界面
- 自定义快捷键
- 搜索 / 筛选界面
- Windows / Linux 支持

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 高频复制办公用户 | 在聊天、文档、浏览器、终端间频繁复制内容 | 快速呼出、快速选择、不中断当前工作流 |
| 常驻后台工具用户 | 希望应用安静运行在系统后台 | 通过托盘完成最常见的控制动作 |
| 稳定性敏感用户 | 长期开机、对监听状态和开机行为敏感 | 启动稳定、状态清晰、可控可恢复 |

## 3. 用户故事详细说明

### E01 - 系统托盘与运行控制

#### US-01-01：通过系统托盘快速打开或隐藏主面板

**作为** 常驻后台工具用户，**我希望** 可以通过系统托盘显示或隐藏主面板，**以便** 在快捷键不可用或不方便使用时仍能调出工具。

**验收标准**：
- 应用启动后系统托盘中可见图标
- 点击托盘菜单“显示主面板”后，主面板在当前活动显示器底部打开
- 当主面板已经显示时，菜单文案切换为“隐藏主面板”并可成功关闭
- 托盘操作不应破坏当前前台应用焦点恢复逻辑

#### US-01-02：暂停监听以避免记录敏感内容

**作为** 稳定性敏感用户，**我希望** 可以从托盘暂停粘贴板监听，**以便** 在复制敏感信息时不被系统记录。

**验收标准**：
- 点击托盘菜单“暂停监听”后，后续复制内容不会进入历史记录
- 托盘图标与菜单文案同步切换为暂停态
- 再次点击“恢复监听”后，后续新的复制内容可以重新被捕获
- 暂停 / 恢复过程中不会删除既有历史记录

#### US-01-03：清空全部历史记录

**作为** 高频复制办公用户，**我希望** 可以在托盘里一键清空历史记录，**以便** 在切换工作场景时快速重置历史。

**验收标准**：
- 从托盘触发“清空历史”时必须先经过确认步骤
- 确认后清空 `clipboard_items`、`image_assets`、`file_items` 与关联图片资源文件
- 清空后主面板进入空状态，托盘仍保持可用
- 取消清空时，历史数据与图片资源均不得变化

### E02 - 开机自启动集成

#### US-02-01：登录系统后自动启动应用

**作为** 常驻后台工具用户，**我希望** 登录系统后应用自动启动，**以便** 无需手动打开就能开始记录复制历史。

**验收标准**：
- 当“开机自启动”开启时，应用首次运行或切换为开启后，会生成有效的 macOS Launch Agent 配置
- 用户重新登录 macOS 后，应用能自动启动并进入后台监听状态
- 自启动不依赖主面板可见；后台常驻即可工作

#### US-02-02：可以关闭开机自启动

**作为** 稳定性敏感用户，**我希望** 可以关闭登录启动，**以便** 自主控制系统资源占用与启动项数量。

**验收标准**：
- 从托盘切换“开机自启动”后，应用会新增或删除对应 Launch Agent 配置
- 再次启动应用时可读取并展示当前自启动状态
- 自启动切换失败时，用户会收到清晰错误提示或日志记录

### E03 - 主面板交互增强

#### US-03-01：使用数字键快速选中前 9 条记录

**作为** 高频复制办公用户，**我希望** 在主面板打开时按 `1-9` 直接选中对应记录，**以便** 比连续方向键切换更快。

**验收标准**：
- 面板可见时按 `1` 选中第 1 条，按 `9` 选中第 9 条
- 当记录数少于对应数字时，不报错、不改变当前选中项
- 数字键只在主面板可见且非输入态下生效

#### US-03-02：面板与卡片动画自然流畅

**作为** 高频复制办公用户，**我希望** 主面板显示、隐藏、切换和删除都足够顺滑，**以便** 使用体验更像系统级工具而不是调试界面。

**验收标准**：
- 面板弹出 / 收起动画时长控制在 `200-300ms`
- 卡片切换动效不影响键盘连续操作
- 删除记录后有短促过渡动画，且焦点自动落到合理的下一项
- 开启系统“减少动态效果”时可降级为弱动画或无动画

#### US-03-03：在多显示器下显示到当前活动屏幕

**作为** 多屏办公用户，**我希望** 主面板出现在当前操作的显示器底部，**以便** 不需要跨屏寻找面板。

**验收标准**：
- 快捷键或托盘打开主面板时，窗口显示在当前活动显示器的可用工作区底部
- Dock 位于底部、左侧、右侧时均不会遮挡面板
- 面板宽度与定位基于当前显示器工作区计算，而不是整块虚拟桌面

### E04 - 状态一致性与可恢复性

#### US-04-01：托盘状态、监听状态与主面板状态保持一致

**作为** 稳定性敏感用户，**我希望** 托盘、日志和主面板对当前状态的表达一致，**以便** 我不会误判应用是否正在工作。

**验收标准**：
- 监听暂停时，托盘图标、托盘菜单、运行时状态接口返回值一致
- 历史被清空后，主面板和托盘均能反映“空历史”状态
- 粘贴完成、隐藏面板、再次打开面板的状态流转不互相污染

#### US-04-02：系统集成失败时应用仍可继续工作

**作为** 常驻后台工具用户，**我希望** 即便托盘或自启动注册失败，核心复制与粘贴流程仍可继续使用，**以便** 不因为辅助能力失败而完全不可用。

**验收标准**：
- 托盘初始化失败时，应用仍可通过快捷键打开主面板
- 自启动开关失败不会损坏 `clipboard.db` 或历史列表
- 所有系统集成失败都有日志可查，并返回明确状态码

## 5. 排除范围说明

当前版本明确不包含以下内容：
- 独立设置窗口及其多 Tab 配置页
- 搜索输入框、类型筛选、收藏、固定置顶
- Windows / Linux 平台适配
- 云同步、标签、插件扩展

这些能力分别留给 `v0.5+` 或后续版本处理，避免 `v0.3` 目标扩散。
