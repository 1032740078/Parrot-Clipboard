# v0.3 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.3 Feature Complete 版本** 的 IPC 状态码与错误处理约定。除 `v0.2` 的历史记录、粘贴、数据库错误外，新增系统托盘、自启动、监听控制相关错误码。

### 1.2 错误对象格式

```typescript
interface IpcError {
  code: ErrorCode;
  message: string;
}
```

### 1.3 错误传递机制

- 后端统一返回结构化错误对象
- 前端先按 `code` 分类处理，再按 `message` 兜底展示
- 系统集成错误必须写入日志，方便排查真实环境问题

## 2. 错误码清单

| 错误码 | 含义 |
|--------|------|
| `RECORD_NOT_FOUND` | 记录不存在 |
| `INVALID_PARAM` | 参数非法 |
| `CLIPBOARD_READ_ERROR` | 读取系统粘贴板失败 |
| `CLIPBOARD_WRITE_ERROR` | 写入系统粘贴板失败 |
| `KEY_SIM_ERROR` | 模拟按键失败 |
| `DB_ERROR` | 数据库读写失败 |
| `IMAGE_PROCESS_ERROR` | 图片处理失败 |
| `FILE_ACCESS_ERROR` | 文件系统访问失败 |
| `WINDOW_ERROR` | 窗口显示 / 隐藏 / 定位失败 |
| `MONITOR_CONTROL_ERROR` | 监听状态切换失败 |
| `AUTOSTART_ERROR` | 开机自启动注册或注销失败 |
| `TRAY_ERROR` | 托盘初始化或菜单操作失败 |
| `INTERNAL` | 未分类内部错误 |

## 3. 错误码详细说明

### SC-001：`RECORD_NOT_FOUND`

- **触发场景**：删除、读取、粘贴的目标记录不存在
- **典型命令**：`get_record_detail`、`delete_record`、`paste_record`
- **前端提示**：`记录不存在，列表已刷新`

### SC-002：`INVALID_PARAM`

- **触发场景**：`limit=0`、`limit>500`、非文本触发 `plain_text`、缺少 `confirm_token`
- **典型命令**：`get_records`、`paste_record`、`clear_history`
- **前端提示**：`请求参数无效，请重试`

### SC-003：`CLIPBOARD_READ_ERROR`

- **触发场景**：读取系统粘贴板失败
- **前端提示**：通常不直出，记录日志即可

### SC-004：`CLIPBOARD_WRITE_ERROR`

- **触发场景**：再次粘贴前写回系统粘贴板失败
- **前端提示**：`写入系统剪贴板失败`

### SC-005：`KEY_SIM_ERROR`

- **触发场景**：模拟 `Cmd+V` 失败
- **前端提示**：`模拟粘贴失败，请重试`

### SC-006：`DB_ERROR`

- **触发场景**：查询摘要、删除记录、清空历史、迁移失败
- **前端提示**：`本地数据异常，请稍后重试`

### SC-007：`IMAGE_PROCESS_ERROR`

- **触发场景**：缩略图生成失败、图片原图读取失败
- **前端提示**：图片卡片降级占位，不一定弹 Toast

### SC-008：`FILE_ACCESS_ERROR`

- **触发场景**：删除图片资源失败、读取 Launch Agent 文件失败、访问日志目录失败
- **前端提示**：`文件访问失败，请检查权限`

### SC-009：`WINDOW_ERROR`

- **触发场景**：主面板显示、隐藏、多显示器定位失败
- **前端提示**：`主面板显示失败，请重试`

### SC-010：`MONITOR_CONTROL_ERROR`

- **触发场景**：暂停 / 恢复监听失败
- **典型命令**：`set_monitoring`
- **前端提示**：`监听状态切换失败`

### SC-011：`AUTOSTART_ERROR`

- **触发场景**：写入或删除 Launch Agent plist 失败
- **典型命令**：`set_launch_at_login`
- **前端提示**：`开机自启动设置失败`

### SC-012：`TRAY_ERROR`

- **触发场景**：托盘初始化失败、菜单项更新失败
- **前端提示**：不阻塞主流程，记录日志并提示 `托盘不可用`

### SC-013：`INTERNAL`

- **触发场景**：未预期错误
- **前端提示**：`发生未知错误，请查看日志`

## 4. Command 与错误码对应矩阵

| Command | 可能错误码 |
|---------|------------|
| `get_records` | `INVALID_PARAM`、`DB_ERROR` |
| `get_record_detail` | `RECORD_NOT_FOUND`、`DB_ERROR` |
| `delete_record` | `RECORD_NOT_FOUND`、`DB_ERROR`、`FILE_ACCESS_ERROR` |
| `paste_record` | `RECORD_NOT_FOUND`、`INVALID_PARAM`、`CLIPBOARD_WRITE_ERROR`、`KEY_SIM_ERROR` |
| `hide_panel` | `WINDOW_ERROR` |
| `set_monitoring` | `MONITOR_CONTROL_ERROR` |
| `clear_history` | `INVALID_PARAM`、`DB_ERROR`、`FILE_ACCESS_ERROR` |
| `get_launch_at_login_status` | `AUTOSTART_ERROR`、`FILE_ACCESS_ERROR` |
| `set_launch_at_login` | `AUTOSTART_ERROR`、`FILE_ACCESS_ERROR` |
| `get_runtime_status` | `TRAY_ERROR`、`WINDOW_ERROR` |
| `get_log_directory` | `FILE_ACCESS_ERROR` |

---

## 5. 前端统一处理约定

- `TRAY_ERROR`：降级提示，不阻塞主面板使用
- `AUTOSTART_ERROR`：展示一次性错误提示，并建议查看日志目录
- `MONITOR_CONTROL_ERROR`：保持 UI 回滚到切换前状态
- `DB_ERROR`：清空历史失败时不得先把前端列表本地清空，必须等待后端成功结果

---

## 6. 成功响应约定

- `set_monitoring` 成功后返回最新 `MonitoringStatus`
- `set_launch_at_login` 成功后返回最新 `LaunchAtLoginStatus`
- `clear_history` 成功后返回删除统计，并同时发出 `clipboard:history-cleared`
