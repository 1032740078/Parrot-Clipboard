### 2.2 2026-03-07 `v0.3-config-launch-at-login`

**变更类型**：`feat(config)`

**主要变更**：
- 更新 `src-tauri/src/config/schema.rs`，为 `AppConfig` 增加 `launch_at_login` 字段，默认值为 `true`
- 更新 `src-tauri/src/config/mod.rs`，新增 `ConfigStore`，统一管理配置快照读取与 `launch_at_login` 的写回
- 补齐旧配置升级写回测试和新字段持久化测试，保证历史配置文件兼容 `v0.3`

**产出结果**：
- `config.json` 现在可稳定表达 `launch_at_login` 偏好，并和 v0.3 文档契约保持一致
- 后续 `TASK-01-2` 的 Launch Agent 服务可以直接复用配置存储能力，无需再临时拼接文件写回逻辑

**验证结果**：
- `cargo test --manifest-path src-tauri/Cargo.toml config::` ✅

### 2.3 2026-03-07 `v0.3-launch-agent-service`

**变更类型**：`feat(autostart)`

**主要变更**：
- 新增 `src-tauri/src/autostart/mod.rs`，实现 Launch Agent plist 的生成、查询、删除、幂等切换与弱修复
- 更新 `src-tauri/src/lib.rs`，在应用启动时注入 `LaunchAgentService`，并根据配置执行 `reconcile`
- 更新 `src-tauri/src/state.rs` 与 `src-tauri/src/error.rs`，为后续运行态、自启动切换和错误提示提供状态与错误分类基础

**产出结果**：
- `launch_at_login` 不再只是配置字段，而是已经有对应的 macOS Launch Agent 服务层承接
- 应用启动时可以自动修复“配置为开启但 plist 缺失”或“配置为关闭但 plist 残留”的常见不一致情况

**验证结果**：
- `cargo test --manifest-path src-tauri/Cargo.toml autostart::` ✅
- `cargo test --manifest-path src-tauri/Cargo.toml config::` ✅
