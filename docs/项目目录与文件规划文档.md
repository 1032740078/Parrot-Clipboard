# 粘贴板记录管理工具 - 项目目录与文件规划文档

## 1. 文档概述

### 1.1 目的

本文档基于《技术架构选型文档》《模块拆分文档》《模块界限文档》，规划项目完整的目录结构与文件职责，为开发团队提供统一的文件组织规范，明确每个文件的所属模块、职责边界和依赖关系。

### 1.2 文档依赖

| 前置文档 | 说明 |
|----------|------|
| `docs/技术架构选型文档.md` | 技术栈、分层架构、IPC 接口清单 |
| `docs/模块拆分文档.md` | 14 个模块的职责与接口定义 |
| `docs/模块界限文档.md` | 模块权责边界与数据所有权 |

### 1.3 约定

- 所有路径均以项目根目录（`/`）为基准
- `[模块编号]` 标注该文件归属的模块，对应《模块拆分文档》中的编号
- `(运行时)` 标注该目录/文件在运行时生成，不纳入版本控制

## 4. 后端目录结构（`src-tauri/`）

```

├── src/
│   ├── main.rs                 # 应用入口，初始化 Tauri Builder
│   ├── lib.rs                  # 库入口，注册所有 Tauri Command 和插件
│   ├── error.rs                # 全局错误类型 AppError，实现 serde::Serialize
│   ├── ipc/                    # [M13] IPC 通信模块（后端侧）
│   │   ├── mod.rs              # 模块导出
│   │   ├── commands.rs         # 所有 #[tauri::command] 函数定义
│   │   └── events.rs           # emit() 事件发送工具函数
│   ├── clipboard/              # [M01] 粘贴板监听模块
│   │   ├── mod.rs              # ClipboardMonitor trait 定义 + 模块导出
│   │   ├── monitor.rs          # 监听实现（轮询/事件循环，arboard）
│   │   └── filter.rs           # 黑名单过滤逻辑（读取 M09 配置）

│   │   ├── mod.rs              # ConfigManager trait 定义 + 模块导出
│   │   ├── schema.rs           # AppConfig 结构体定义（含默认值）
```

### 4.2 关键文件职责说明

| 文件 | 模块 | 职责 |
|------|------|------|
| `src/main.rs` | — | 创建 Tauri 应用、注册插件、启动事件循环 |
| `src/lib.rs` | — | 调用 `tauri::Builder`，注册全部 `invoke_handler` Command |
| `src/error.rs` | — | 定义 `AppError` 枚举，实现 `serde::Serialize` 供 IPC 传输 |
| `src/ipc/commands.rs` | M13 | `#[tauri::command]` 函数，负责参数解析、调用业务模块、返回结果 |

| `src/clipboard/monitor.rs` | M01 | 后台线程：轮询 `arboard` 或注册系统事件，检测粘贴板变化 |
| `src/clipboard/filter.rs` | M01 | 读取 M09 黑名单配置，过滤不应记录的应用 |
| `src/storage/sqlite.rs` | M02 | SQLite CRUD，WAL 模式，按数量/大小自动清理 |
| `src/storage/migration.rs` | M02 | Schema 版本升级，保证数据不丢失 |
| `src/paste/text_strip.rs` | M06 | HTML 标签剥离（纯文本粘贴模式） |
| `src/image/processor.rs` | M07 | 原图保存 + 200x200 JPEG 85% 缩略图异步生成 |
| `src/config/schema.rs` | M09 | `AppConfig` 结构体，全部配置项及默认值 |
| `src/platform/mod.rs` | M14 | 三个跨平台 trait 定义，条件编译入口 |
| `tauri.conf.json` | — | 窗口无边框/透明/置顶配置，插件权限声明 |

## 12. 文件命名规范

### 12.1 前端（TypeScript / React）

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| React 组件文件 | PascalCase | `TextCard.tsx`、`MainPanel/index.tsx` |
| Hook 文件 | camelCase，以 `use` 开头 | `useKeyboard.ts` |
| Store 文件 | camelCase，以 `use` 开头 | `useClipboardStore.ts` |
| API 封装文件 | camelCase | `commands.ts`、`events.ts` |
| 类型定义文件 | camelCase | `clipboard.ts`、`config.ts` |
| 测试文件 | 与被测文件同名，加 `.test` | `MainPanel.test.tsx` |

### 12.2 后端（Rust）

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 模块目录 | snake_case | `clipboard/`、`storage/` |
| 源文件 | snake_case | `monitor.rs`、`migration.rs` |
| 模块入口 | 固定为 `mod.rs` | `clipboard/mod.rs` |
| struct 名 | PascalCase | `ClipboardEntry`、`AppConfig` |
| trait 名 | PascalCase | `ClipboardMonitor`、`ClipboardStorage` |
| 函数/变量 | snake_case | `insert_record()`、`max_count` |
| 常量 | SCREAMING_SNAKE_CASE | `DEFAULT_MAX_RECORDS` |

## 13. 新增文件决策指引

开发过程中新增文件时，参考以下决策树：

```
需要新增文件？
│
├── 是前端代码？
│   ├── 是 UI 组件（有 JSX）→ 放 src/components/<所属模块>/
│   ├── 是状态管理 → 放 src/stores/
│   ├── 是 IPC 调用封装 → 放 src/api/
│   ├── 是复用 Hook → 放 src/hooks/
│   ├── 是类型定义 → 放 src/types/
│   └── 是测试文件 → 放 src/__tests__/<对应目录>/
│
└── 是后端代码？
    ├── 属于某个业务模块 → 放 src-tauri/src/<模块目录>/
    ├── 是跨平台适配代码 → 只允许放 src-tauri/src/platform/
    ├── 是 IPC Command/Event → 放 src-tauri/src/ipc/
    ├── 是全局类型/错误 → 放 src-tauri/src/error.rs 或 lib.rs
    └── 是图标/静态资源 → 放 src-tauri/icons/
```
