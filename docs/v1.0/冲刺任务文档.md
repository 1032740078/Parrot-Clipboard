## 1. 冲刺目标

在 `v0.5` 已具备三平台主流程与设置中心的基础上，完成 `v1.0` 的正式发布收口，重点达成以下目标：

- 建立稳定可恢复的数据库迁移与损坏恢复能力

- 完成 `macOS` 权限引导闭环
- 通过虚拟滚动、缓存治理和清理任务提升长时运行稳定性
- 完成正式发布前自动化与打包验收

## 4. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 数据库迁移脚本错误 | 启动失败或数据丢失 | 先备份后迁移，迁移前后都加自动化校验 |
| 权限检测依赖平台私有能力 | `macOS` 体验不稳定 | 将检测与引导分层，失败时回退为手工说明 |
| 虚拟滚动破坏键盘体验 | 主面板效率下降 | 先覆盖自动化用例，再接入视图层优化 |
| 更新检查依赖外部网络 | 误报失败或阻塞 UI | 接口异步化，失败仅反馈不阻断主流程 |
| 清理任务误删有效图片 | 数据损坏 | 先做数据库引用扫描，再执行删除，保留日志证据 |

---

## 5. 冲刺完成后建议

1. 补写 `v1.0` 用户手册、FAQ 与三平台安装说明。
2. 在正式发布后一周内重点观察日志与用户反馈，确认恢复链路稳定性。
3. `v2.0` 再进入搜索、收藏、标签、同步等新业务功能规划。
