# v1.0 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.0 Release` 当前版本的 BDD 自动化验收场景。

### 1.2 BDD 编写规范

- 只覆盖 `v1.0` 当前版本范围内的关键用户路径。
- 每个场景必须有清晰的 `Given / When / Then`。
- `P0` 场景默认进入 `Playwright` 自动化回归集。
- 性能与稳定性场景允许结合 mock、性能埋点或离线样例完成验收。

---

## 2. Feature：发布可靠性（Release Reliability）

### 场景 BDD-01-01：启动时自动迁移旧数据库

- **Given** 用户本地仍保留 `v0.5` 生成的旧版 `clipboard.db`
- **When** 用户首次启动 `v1.0`
- **Then** 应用会自动识别旧 `schema_version` 并执行迁移
- **And** 历史记录列表仍可正常展示与粘贴
- **And** 关于页可以看到迁移后的当前数据库版本

### 场景 BDD-01-02：损坏数据库会先备份后重建

- **Given** 本地 `clipboard.db` 已损坏且无法正常读取
- **When** 用户启动应用
- **Then** 应用会将损坏库重命名为 `corrupt-*` 备份文件
- **And** 自动重建新的数据库结构
- **And** 主面板与托盘仍可进入可工作状态

### 场景 BDD-01-03：清空历史后数据库与图片资源保持一致

- **Given** 用户历史中包含文本、图片、文件三类记录
- **When** 用户在主面板确认执行“清空历史”
- **Then** 数据库主表与子表都被清空
- **And** 关联的图片原图和缩略图文件一并删除
- **And** 配置文件与日志目录不会被误删

## 3. Feature：权限引导（Permission Guidance）

### 场景 BDD-02-01：macOS 权限缺失时展示辅助功能引导

- **Given** 用户运行在 `macOS` 且尚未授予辅助功能权限
- **When** 用户尝试使用快捷键调出面板或在设置中查看系统能力
- **Then** 界面应展示“需要辅助功能权限”的明确提示
- **And** 提供前往系统设置的操作入口
- **And** 说明授权完成后如何重试检测

### 场景 BDD-02-02：授权完成后重试可恢复可用状态

- **Given** 用户已按照引导完成系统授权
- **When** 用户点击“重新检测”或重新打开应用
- **Then** 权限状态更新为可用
- **And** 原先被禁用的快捷键或粘贴相关入口恢复可操作

### 场景 BDD-02-03：受限环境下仍可浏览历史

- **Given** 当前会话存在部分能力降级
- **When** 用户打开主面板
- **Then** 记录列表仍可浏览、选择与删除
- **And** 仅受限能力显示禁用态与原因说明

## 4. Feature：关于与更新（About & Update）

### 场景 BDD-03-01：关于页展示版本、平台与日志目录

- **Given** 用户打开“关于”页面
- **When** 页面加载完成
- **Then** 页面展示 `app_version`、`platform`、`session_type`、`schema_version`
- **And** 提供日志目录路径或打开入口
- **And** 展示开源协议或许可证入口

### 场景 BDD-03-02：检测到新版本时提供明确反馈

- **Given** 更新源返回一个高于当前版本的稳定版本
- **When** 用户点击“检查更新”
- **Then** 页面显示“发现新版本”结果
- **And** 提供下载页或发布页入口
- **And** 不会关闭当前关于页或影响主面板使用

### 场景 BDD-03-03：检查更新失败时不阻塞其他功能

- **Given** 当前网络不可用或更新源返回异常
- **When** 用户点击“检查更新”
- **Then** 页面显示可读的失败提示
- **And** 用户仍可继续查看版本信息、打开日志目录或关闭页面

## 5. Feature：性能与大历史可用性（Performance & Large History）

### 场景 BDD-04-01：大历史列表下虚拟滚动保持交互稳定

- **Given** 历史记录达到 `200+` 条且混合文本、图片、文件卡片
- **When** 用户滚动主面板并使用键盘切换记录
- **Then** 页面只渲染可见区域附近的卡片
- **And** 当前高亮记录不会因为滚动而丢失
- **And** `Enter`、`Delete`、数字键快选仍可正常执行

### 场景 BDD-04-02：高频图片使用下内存保持在预算范围内

- **Given** 用户反复浏览和粘贴多张图片记录
- **When** 应用持续运行一段时间
- **Then** 原图缓存会按 `LRU` 策略淘汰旧条目
- **And** 内存增长保持在预算范围内
- **And** 缩略图浏览不依赖原图常驻内存

---

## 6. Feature：诊断与清理（Diagnostics & Cleanup）

### 场景 BDD-05-01：孤立图片文件会被自动识别并清理

- **Given** 图片目录中存在数据库已无引用的原图或缩略图文件
- **When** 定时清理任务或手动诊断清理任务执行
- **Then** 系统只删除未被引用的文件
- **And** 在日志中记录清理数量和结果
- **And** 已被数据库引用的图片文件不会被误删

### 场景 BDD-05-02：前端与后端错误都能写入本地日志

- **Given** 前端发生调用异常，后端发生运行时错误
- **When** 用户随后打开日志目录
- **Then** 能看到同一轮会话的诊断日志文件
- **And** 日志中包含错误时间、级别和上下文
- **And** 不应记录敏感文本正文

## 7. 非功能需求验收场景（Performance / Reliability）

### 场景 BDD-NFR-01：主面板首屏在预算时间内可交互

- **Given** 应用已完成启动并存在最近历史记录
- **When** 用户调出主面板
- **Then** 首屏在 `< 200ms` 内可见且可交互

### 场景 BDD-NFR-02：空闲资源占用符合发布标准

- **Given** 应用在后台空闲运行
- **When** 连续观察一段时间
- **Then** 空闲 `CPU < 0.5%`
- **And** 空闲内存控制在目标预算内

### 场景 BDD-NFR-03：长时间运行无明显内存泄漏

- **Given** 应用持续运行 `24h`
- **When** 用户间歇执行复制、打开主面板、粘贴与清理操作
- **Then** 内存增长应小于 `5MB`
- **And** 不出现持续攀升且无法回落的趋势

---

## 8. 验收通过标准（DoD）

- `BDD-01-*` 到 `BDD-05-*` 的 `P0` 场景全部进入自动化回归。
- `BDD-NFR-*` 至少具备一套稳定的自动化或半自动性能测量方法。
- 任一场景失败都能回溯到对应日志、夹具或 mock 数据。
- `v1.0` 新增页面与新命令不允许出现“无反馈失败”路径。
