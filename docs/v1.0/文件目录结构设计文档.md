# v1.0 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.0 Release` 当前版本关注的目录结构设计。

### 1.2 设计原则

1. **在现有结构上增量演进**：尊重 `v0.5` 已形成的模块边界，不为文档而做大规模重构。
2. **按领域归位**：发布质量相关能力优先归拢到诊断、更新、权限或持久化边界内。
3. **前后端对称**：关于页、权限引导、诊断 API 与后端服务边界命名尽量一致。
4. **测试就近**：新增能力优先在相邻目录下补齐测试。

### 3.2 关键文件职责

| 路径 | 职责 |
|------|------|

| `src/api/settings.ts` | 设置相关命令封装 |
| `src/api/diagnostics.ts` | 关于页、权限、更新、清理相关命令封装 |

| `src/components/PermissionGuide/` | 权限说明、重试、打开系统设置 |
| `src/stores/useDiagnosticsStore.ts` | 诊断快照与更新结果共享状态 |

## 4. 后端目录设计（`src-tauri/src/`）

### 4.1 推荐结构

```text
src-tauri/src/
├── autostart/
├── clipboard/
├── config/
├── diagnostics/           # v1.0 推荐新增：诊断快照、清理统计聚合
├── image/
├── ipc/
├── paste/
├── persistence/
├── platform/
├── settings/
├── shortcut/
├── tray/
├── updater/               # v1.0 推荐新增：更新检查适配层
├── window/
├── error.rs
├── logging.rs
├── main.rs
└── state.rs
```

### 4.2 模块归属说明

| 模块 | 归属 |
|------|------|
| `clipboard/` | 剪贴板采集、查询、提升、删除、清空 |
| `persistence/` | `SQLite` 初始化、迁移、恢复、孤立资源扫描 |
| `image/` | 原图保存、缩略图生成、资源删除 |
| `logging.rs` | 本地日志初始化与 panic 捕获 |
| `diagnostics/` | 聚合版本信息、日志目录、迁移摘要、清理摘要 |
| `updater/` | 调用更新源、解析版本信息 |
| `platform/` | 平台能力、权限检测与会话差异 |
| `ipc/` | DTO 转换与命令 / 事件暴露 |

## 6. 运行时生成目录

### 6.1 推荐布局

```text
app_data/
├── clipboard.db
└── images/
    ├── original/
    └── thumbs/

app_log/
└── clipboard-manager.log

app_config/
└── config.json
```

### 6.2 约束说明

- `clipboard.db` 只承载业务历史数据。
- 图片目录只承载二进制资源，不持久化额外业务语义。
- 日志目录只承载诊断信息，不写用户配置。
- 损坏数据库备份文件与主库保持同目录，便于用户打包反馈。

---

## 7. 命名规范

- 前端组件目录采用 `PascalCase`。
- Hook 采用 `useXxx.ts`。
- Rust 模块采用 `snake_case`。
- 关于页、权限引导、诊断能力新增命名优先使用统一术语：`About`、`Permission`、`Diagnostics`、`Update`。

---

## 8. 与 v0.5 的主要差异

- `v0.5` 的前端重点是设置窗口；`v1.0` 增加关于页、权限引导与诊断状态的专属目录。
- `v1.0` 后端结构建议增加 `diagnostics/` 与 `updater/`，但不破坏既有 `clipboard/`、`settings/`、`persistence/` 边界。
- `v1.0` 更强调“代码目录”和“运行时目录”的双重可维护性。
