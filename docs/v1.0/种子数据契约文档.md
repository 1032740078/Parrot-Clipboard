# v1.0 种子数据契约文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.0 Release` 当前版本的种子数据、夹具与样例资产契约。

### 1.2 设计原则

1. **只服务当前版本**：只覆盖 `v1.0` 范围内的迁移、权限、更新、诊断、性能与清理场景。
2. **脱敏优先**：任何文本、日志、图片、文件路径都必须使用脱敏样例。
3. **可复用**：前端单测、后端测试、E2E 共享统一语义的数据集。
4. **可组合**：允许通过小型基础种子拼装出更复杂的验收场景。

### 2.5 SEED-1005：诊断与日志样例集

**用途**：验证日志目录、错误展示、问题反馈。

包含内容：

- `frontend-error.log`：前端调用错误样例
- `backend-error.log`：后端运行错误样例
- `panic.log`：panic 捕获样例
- `diagnostics-snapshot.json`：聚合诊断快照

### 2.6 SEED-1006：大历史性能集

**用途**：验证虚拟滚动、图片缓存与长列表交互。

组成规则：

- 文本记录 `200` 条
- 图片记录 `50` 条
- 文件记录 `40` 条
- 时间戳分布覆盖“刚刚 / 今天 / 昨天 / 更早”
- 至少包含 `5` 条超长文本、`5` 张大图、`5` 组多文件记录

## 3. 类型覆盖要求

### 3.1 历史记录覆盖

- `text / image / files` 三类必须全部覆盖
- 图片必须覆盖 `pending / ready / failed` 缩略图状态
- 文件必须覆盖 `file / directory` 两类条目

### 3.2 发布诊断覆盖

- 权限状态必须覆盖 `granted / missing / unsupported`
- 更新检查必须覆盖 `available / latest / failed`
- 迁移状态必须覆盖“无迁移 / 已迁移 / 损坏恢复后重建”

### 3.3 平台覆盖

- `macos native`
- `windows native`
- `linux x11`
- `linux wayland`

## 4. 资产文件契约

### 4.1 推荐目录

```text
src/__tests__/fixtures/v1.0/
├── diagnostics/
├── permissions/
├── release/
├── updates/
└── histories/

e2e/fixtures/v1.0/
├── app-state/
├── db/
└── logs/
```

### 4.2 路径约束

- 所有路径示例必须使用虚构用户或 `demo` 用户名。
- 不允许把真实发布地址、真实 CI Token、真实 Git 仓库私有链接写入样例。
- 不允许依赖本机绝对路径作为断言条件，E2E 中应以目录名或模式匹配为主。

---

## 5. 加载约定

### 5.1 开发模式加载

- 关于页与权限引导组件可通过本地 JSON fixture 直接渲染开发态预览。
- 虚拟滚动组件默认支持大历史夹具快速注入。

### 5.2 测试模式加载

- 前端单测通过 `fixture builder` 生成轻量对象。
- 后端集成测试使用临时目录复制数据库与图片夹具。
- E2E 使用 `Tauri Mock` 注入命令响应，避免依赖真实系统权限与网络。

---

## 6. 覆盖的测试场景

| 数据集 | 覆盖场景 |
|--------|----------|
| `SEED-1001` | 关于页渲染、日志目录展示、诊断摘要 |
| `SEED-1002` | 数据库迁移、损坏恢复、启动回归 |
| `SEED-1003` | 权限引导、按钮禁用、状态刷新 |
| `SEED-1004` | 检查更新三态反馈 |
| `SEED-1005` | 日志展示、问题反馈、错误回归 |
| `SEED-1006` | 虚拟滚动、缓存淘汰、长列表交互 |

---

## 7. 数据清理约定

- 每个后端测试必须在临时目录下运行并在结束后删除测试目录。
- E2E 生成的模拟日志与数据库文件在用例结束后必须清理。
- 不允许把临时生成的更新检查结果、损坏数据库副本、日志文件提交到正式仓库目录。
