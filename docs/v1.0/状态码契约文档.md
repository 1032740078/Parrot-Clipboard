# v1.0 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.0 Release` 当前版本的状态码与错误码契约。

### 1.2 错误对象格式

```json
{
  "code": "DB_ERROR",
  "message": "recover sqlite database failed"
}
```

字段说明：

- `code`：稳定错误码，供前端做分支处理
- `message`：面向日志与开发排查的可读信息

### 1.3 错误传递机制

- 后端命令失败时直接返回结构化错误对象。
- 前端统一通过错误处理器转为用户可理解文案。
- 对可降级能力，优先通过状态字段表达；只有真正无法继续执行时才返回错误码。

## 3. 错误码详细说明

### SC-001：`RECORD_NOT_FOUND`

- **触发场景**：读取详情、删除、粘贴一个已不存在的记录
- **前端处理**：提示记录已失效，并刷新列表

### SC-002：`INVALID_PARAM`

- **触发场景**：非法 `id`、空快捷键、无效枚举值
- **前端处理**：阻止提交并在表单处提示

### SC-003：`SETTINGS_VALIDATION_ERROR`

- **触发场景**：设置值超范围、黑名单规则字段缺失
- **前端处理**：表单内联提示 + Toast

### SC-004：`SHORTCUT_CONFLICT`

- **触发场景**：快捷键与系统或应用冲突
- **前端处理**：阻止保存，展示冲突原因

### SC-005：`SHORTCUT_REGISTER_ERROR`

- **触发场景**：系统注册全局快捷键失败
- **前端处理**：回退到旧值或提示用户重试

### SC-006：`BLACKLIST_RULE_DUPLICATE`

- **触发场景**：新增或编辑后与现有规则重复
- **前端处理**：表单错误提示，不落盘

### SC-007：`BLACKLIST_RULE_NOT_FOUND`

- **触发场景**：编辑或删除一个已不存在的规则
- **前端处理**：刷新规则列表并提示数据已变化

### SC-008：`UNSUPPORTED_PLATFORM_FEATURE`

- **触发场景**：当前平台不支持某个系统能力
- **前端处理**：显示降级说明，不弹出技术性报错

### SC-009：`AUTOSTART_ERROR`

- **触发场景**：注册表 / `.desktop` / 系统项写入失败
- **前端处理**：提示用户检查权限或手工配置

### SC-010：`WINDOW_ERROR`

- **触发场景**：主面板、设置窗口、关于页打开失败
- **前端处理**：Toast + 记录日志

### SC-011：`TRAY_ERROR`

- **触发场景**：托盘创建或菜单更新失败
- **前端处理**：提示托盘不可用，但应用继续运行

### SC-012：`CLIPBOARD_READ_ERROR`

- **触发场景**：读取文本、图片、文件失败
- **前端处理**：记录日志，必要时降级提示

### SC-013：`CLIPBOARD_WRITE_ERROR`

- **触发场景**：将记录写回系统剪贴板失败
- **前端处理**：提示粘贴失败，可建议重试

### SC-014：`KEY_SIM_ERROR`

- **触发场景**：模拟 `Cmd/Ctrl + V` 失败
- **前端处理**：提示权限或系统限制问题

### SC-015：`FILE_ACCESS_ERROR`

- **触发场景**：日志目录、图片目录、配置文件不可读写
- **前端处理**：提示路径权限问题，记录日志

### SC-016：`DB_ERROR`

- **触发场景**：数据库初始化、迁移、恢复、查询、清理失败
- **前端处理**：若为启动期失败则展示阻断性错误；若为后台清理失败则只提示诊断信息

### SC-017：`PERMISSION_REQUIRED`

- **触发场景**：执行需要系统权限的动作，但权限尚未授予
- **前端处理**：展示权限引导，不直接重试同一动作

### SC-018：`PERMISSION_GUIDE_ERROR`

- **触发场景**：打开系统设置入口失败或权限状态检测异常
- **前端处理**：展示替代文案与手工操作步骤

### SC-019：`UPDATE_CHECK_ERROR`

- **触发场景**：更新源超时、网络异常、版本解析失败
- **前端处理**：在关于页展示失败提示，不阻塞页面其他操作

### SC-020：`DIAGNOSTICS_ERROR`

- **触发场景**：读取诊断快照、日志目录、手动清理结果失败
- **前端处理**：提示诊断读取失败，并建议查看本地日志

### SC-021：`INTERNAL`

- **触发场景**：未归类异常
- **前端处理**：统一显示“发生未知错误”，并记录上下文日志

## 5. 前端统一处理约定

1. **阻断型错误**：`DB_ERROR`（启动失败）、`WINDOW_ERROR`（关键窗口无法打开）需要明确阻断提示。
2. **可重试错误**：`UPDATE_CHECK_ERROR`、`CLIPBOARD_WRITE_ERROR`、`FILE_ACCESS_ERROR` 提供重试或替代路径。
3. **降级型错误**：`UNSUPPORTED_PLATFORM_FEATURE`、`PERMISSION_REQUIRED` 优先展示说明和引导，不制造恐慌。
4. **表单型错误**：`SETTINGS_VALIDATION_ERROR`、`SHORTCUT_CONFLICT`、`BLACKLIST_RULE_DUPLICATE` 以内联反馈优先。

---

## 6. 成功响应约定

- 成功时直接返回业务对象。
- 对“检查更新”“清理孤立文件”这类动作型接口，即使没有发现新版本或没有可清理文件，也应返回成功态对象，而不是错误码。
- 成功但存在提醒时，通过响应字段或事件表达，不复用错误码通道。
