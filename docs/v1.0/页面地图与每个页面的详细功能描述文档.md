# v1.0 页面地图与每个页面的详细功能描述文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 `v1.0 Release` 当前版本页面与视图能力。

### 1.2 v1.0 界面清单

| 编号 | 页面 / 视图 | 类型 | 说明 |
|------|-------------|------|------|
| P01 | 主面板（Main Panel） | 主窗口 | 浏览历史、选择记录、执行粘贴 |
| P02 | 设置窗口（Settings Window） | 独立窗口 | 延续 `v0.5` 的设置中心 |
| P03 | 关于页面（About Window） | 独立窗口 | 版本信息、检查更新、日志目录、许可证 |
| P04 | 权限引导弹窗（Permission Guide） | 弹窗 / 模态 | `macOS` 权限缺失时的说明与重试 |
| P05 | 托盘菜单（Tray Menu） | 系统菜单 | 系统入口与快捷操作 |
| P06 | 通用确认弹窗（Confirm Dialog） | 模态 | 清空历史、放弃操作确认 |
| V09 | 权限状态提示条 | 视图组件 | 主面板或设置页中的权限提醒 |
| V10 | 更新结果卡片 | 视图组件 | 关于页中的更新结果反馈 |
| V11 | 诊断摘要卡片 | 视图组件 | 版本、日志、迁移、清理摘要 |

## 3. P01 主面板（Main Panel）

### 3.1 基本信息

| 项目 | 说明 |
|------|------|
| 打开方式 | 全局快捷键、托盘菜单、窗口重新聚焦 |
| 主要用户目标 | 快速浏览历史并再次粘贴 |
| 保留能力 | 文本 / 图片 / 文件卡片、键盘导航、删除、清空历史 |

### 3.2 v1.0 变化点

- 列表渲染升级为虚拟滚动，提升长历史场景性能。
- 图片卡片继续使用缩略图，但原图读取改为有限缓存策略。
- 在权限受限或平台降级时增加明确的状态提示条。
- 增加通往“关于页面”的轻量入口。

### 3.3 区域说明

| 区域 | 功能 |
|------|------|
| 顶部状态区 | 显示监听状态、权限提醒、平台能力提示 |
| 卡片列表区 | 渲染历史记录列表，支持键盘与滚动选择 |
| 底部操作区 | 设置入口、关于入口、清空历史入口 |

### 3.4 状态变体

- **正常态**：记录列表可操作，状态区仅展示轻量信息。
- **空态**：展示空状态文案与快捷键提示。
- **权限受限态**：展示 `V09 权限状态提示条`，引导用户进入 `P04`。
- **大列表态**：仅渲染可见卡片，不改变原有键盘操作习惯。

## 4. P02 设置窗口（Settings Window）

### 4.1 基本信息

| 项目 | 说明 |
|------|------|
| 打开方式 | 托盘菜单、主面板入口、命令调用 |
| 窗口策略 | 单实例窗口，重复打开时激活已有窗口 |
| 分组 | 通用 / 记录与存储 / 快捷键 / 隐私 |

### 4.2 布局结构

- 左侧为分组导航
- 右侧为分组内容区
- 顶部为标题与全局保存状态
- 底部可保留轻量说明或关闭入口

### 4.3 v1.0 保留原则

- 不在 `v1.0` 中继续扩展复杂设置分组
- 与发布质量相关的信息不强行塞进设置窗口主流程
- 关于、更新、日志信息优先放到 `P03 关于页面`

---

## 5. P03 关于页面（About Window）

### 5.1 功能定位

`v1.0` 新增页面，用于承接“版本说明 + 检查更新 + 诊断入口”。

### 5.2 页面组成

| 模块 | 内容 |
|------|------|
| 版本信息区 | `app_version`、平台、会话类型、数据库版本、配置版本 |
| 更新检查区 | 检查更新按钮、加载态、新版本提示、失败提示 |
| 诊断区 | 日志目录、最近恢复摘要、最近清理摘要 |
| 法务区 | 开源协议、许可证、项目主页入口 |

### 5.3 状态变体

- **默认态**：展示版本与诊断概览。
- **检查中**：更新检查按钮进入 loading 态。
- **有更新**：展示版本差异与下载入口。
- **检查失败**：展示失败原因，不影响其他模块。

## 6. P04 权限引导弹窗（Permission Guide）

### 6.1 触发条件

- 当前平台为 `macOS`
- 辅助功能权限缺失
- 用户触发了依赖权限的功能，或主动打开系统能力说明

### 6.2 内容组成

| 模块 | 内容 |
|------|------|
| 为什么需要权限 | 解释权限用途，避免用户误解 |
| 授权步骤 | 图文或条目式步骤说明 |
| 操作按钮 | 打开系统设置、重新检测、稍后处理 |
| 风险说明 | 未授权时哪些能力会受影响 |

### 6.3 行为规格

- 关闭弹窗不应导致页面卡死。
- 未授权时相关按钮禁用或给出说明。
- 授权成功后支持立即重试检测。

## 7. P05 托盘菜单（Tray Menu）

### 7.1 菜单项说明

| 菜单项 | 说明 |
|--------|------|
| 打开主面板 | 调出主面板 |
| 打开设置 | 激活设置窗口 |
| 关于 | 打开关于页面 |
| 暂停 / 恢复监听 | 切换监听状态 |
| 清空历史 | 触发确认弹窗 |
| 退出 | 退出应用 |

### 7.2 v1.0 变化点

- 新增“关于”入口。
- 当权限缺失或能力降级时，可在菜单中展示只读说明或引导入口。
- 托盘继续作为受限环境下的重要兜底入口。

---

## 8. P06 通用确认弹窗（Confirm Dialog）

### 8.1 使用场景

- 清空历史
- 关闭含未保存状态的设置页
- 风险性诊断操作（如手动清理孤立文件）

### 8.2 交互要求

- 明确区分“取消”和“确认”按钮层级
- 风险操作需要写明影响范围
- 执行中显示 loading 态，避免重复点击

---

## 9. 视图组件说明

### 9.1 V09 权限状态提示条

- 位置：主面板顶部或设置相关区域
- 目的：提示用户当前权限缺失并提供进入 `P04` 的入口

### 9.2 V10 更新结果卡片

- 位置：关于页更新检查区
- 目的：呈现“发现新版本 / 已是最新 / 检查失败”三态

### 9.3 V11 诊断摘要卡片

- 位置：关于页诊断区
- 目的：集中展示日志目录、数据库版本、最近恢复 / 清理摘要

## 11. 可访问性要求

- 所有按钮、切换项、卡片操作都需要清晰的可读标签。
- 权限引导和关于页中的关键信息需支持键盘访问。
- 加载态、失败态与成功态必须通过文字而非仅颜色区分。
- 虚拟滚动列表需要确保焦点与高亮记录一致，不因回收节点造成键盘迷失。
