## 3. 核心功能可行性分析

### 3.1 自动监听与记录

#### 3.1.1 功能描述
实时监听系统粘贴板变化，自动捕获并保存复制内容（文本、图片、文件）。

#### 3.1.2 技术实现方案

**macOS**：
- API：`NSPasteboard`
- 实现方式：轮询 `changeCount` 检测变化
- 代码示例（Rust）：
```rust
use cocoa::appkit::NSPasteboard;
use cocoa::base::id;

fn monitor_clipboard() {
    let pasteboard: id = unsafe { NSPasteboard::generalPasteboard() };
    let mut last_count = unsafe { pasteboard.changeCount() };

    loop {
        let current_count = unsafe { pasteboard.changeCount() };
        if current_count != last_count {
            // 粘贴板内容已变化
            handle_clipboard_change();
            last_count = current_count;
        }
        thread::sleep(Duration::from_millis(200));
    }
}
```

**Windows**：
- API：`AddClipboardFormatListener` 或 `SetClipboardViewer`
- 实现方式：注册窗口消息监听器
- 代码示例（Rust + winapi）：
```rust
use winapi::um::winuser::{AddClipboardFormatListener, GetClipboardData};

fn monitor_clipboard(hwnd: HWND) {
    unsafe {
        AddClipboardFormatListener(hwnd);
    }
    // 在窗口消息循环中处理 WM_CLIPBOARDUPDATE
}
```

**Linux**：
- API：X11 的 `XFixesSelectSelectionInput` 或 Wayland 的相应 API
- 实现方式：监听 selection 事件
- 注意：Linux 有多个粘贴板（PRIMARY, CLIPBOARD）

#### 3.1.3 内容类型支持

**文本**：
- macOS：`NSPasteboardTypeString`
- Windows：`CF_TEXT`, `CF_UNICODETEXT`
- Linux：`text/plain`, `UTF8_STRING`
- ✅ 可行性：高

**图片**：
- macOS：`NSPasteboardTypePNG`, `NSPasteboardTypeTIFF`
- Windows：`CF_DIB`, `CF_BITMAP`
- Linux：`image/png`, `image/jpeg`
- ✅ 可行性：高
- ⚠️ 注意：需要处理图片格式转换和存储策略

**文件路径**：
- macOS：`NSFilenamesPboardType`
- Windows：`CF_HDROP`
- Linux：`text/uri-list`
- ✅ 可行性：高

#### 3.1.4 技术难度
- 难度等级：⭐⭐⭐ (3/5)
- 主要挑战：
  1. 跨平台 API 差异较大，需要封装统一接口
  2. 不同格式的数据解析和转换
  3. 性能优化（避免频繁轮询）

#### 3.1.5 潜在风险
- ⚠️ 某些应用可能阻止粘贴板访问（如密码管理器）
- ⚠️ 大量图片可能导致内存占用过高
- ⚠️ Linux 下 Wayland 支持可能不完善

#### 3.1.6 解决方案
- 实现统一的粘贴板抽象层
- 图片存储到文件系统，数据库只保存路径
- 限制图片记录数量（如最近 50 条图片）
- 使用缩略图预览，原图按需加载
- 提供降级方案和友好提示
- 使用 `clipboard-rs` 或 `arboard` 等成熟库

#### 3.1.7 可行性结论
✅ **可行**，有成熟的库和方案，需要注意性能优化。

### 3.2 开机自启动

#### 3.2.1 功能描述
应用随系统启动自动运行，静默启动到系统托盘。

#### 3.2.2 技术实现方案

**macOS**：
- 方式：创建 Launch Agent plist 文件
- 路径：`~/Library/LaunchAgents/com.yourapp.plist`
- 实现：使用 `auto-launch` npm 包或 Tauri 的 `tauri-plugin-autostart`
- ✅ 可行性：高

**Windows**：
- 方式一：注册表 `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
- 方式二：启动文件夹 `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`
- 实现：使用 `auto-launch` 或 Windows API
- ✅ 可行性：高

**Linux**：
- 方式一：systemd service
- 方式二：XDG autostart（`~/.config/autostart/`）
- 实现：创建 `.desktop` 文件
- ✅ 可行性：高

#### 3.2.3 技术难度
- 难度等级：⭐⭐ (2/5)
- 主要挑战：不同平台的实现方式不同

#### 3.2.4 推荐方案
使用 `tauri-plugin-autostart` 或 `auto-launch` 库，已封装跨平台实现。

#### 3.2.5 可行性结论
✅ **可行**，有成熟的解决方案。

### 3.3 全局快捷键

#### 3.3.1 功能描述
注册全局快捷键（Shift + Command/Ctrl + V），在任何应用中都能调出界面。

#### 3.3.2 技术实现方案

**macOS**：
- API：`CGEventTapCreate` 或 `NSEvent.addGlobalMonitorForEvents`
- 权限：需要"辅助功能"权限
- 实现：监听全局键盘事件

**Windows**：
- API：`RegisterHotKey`
- 实现：注册系统级热键
- 无需特殊权限

**Linux**：
- API：X11 的 `XGrabKey` 或 Wayland 的相应 API
- 实现：监听全局键盘事件

#### 3.3.3 Tauri 实现
使用 `tauri-plugin-global-shortcut`：
```rust
use tauri_plugin_global_shortcut::GlobalShortcutExt;

fn register_shortcut(app: &tauri::App) {
    app.global_shortcut()
        .register("Shift+CmdOrCtrl+V", || {
            // 显示窗口
            show_window();
        })
        .unwrap();
}
```

#### 3.3.4 技术难度
- 难度等级：⭐⭐⭐ (3/5)
- 主要挑战：
  1. macOS 需要请求辅助功能权限
  2. 快捷键冲突检测
  3. Wayland 支持可能受限

#### 3.3.5 潜在风险
- ⚠️ macOS 权限请求可能被用户拒绝
- ⚠️ 快捷键可能与其他应用冲突
- ⚠️ Linux Wayland 下可能无法注册全局快捷键

#### 3.3.6 解决方案
- 提供清晰的权限请求说明
- 支持自定义快捷键
- 提供快捷键冲突检测和提示
- Wayland 下提供替代方案（如系统托盘菜单）

#### 3.3.7 可行性结论
✅ **可行**，但需要处理权限和冲突问题。

// 从富文本中提取纯文本
fn extract_plain_text(clipboard_data: &ClipboardData) -> String {
    match clipboard_data {
        ClipboardData::Html(html) => {
            // 使用 html2text 或正则表达式去除 HTML 标签
            strip_html_tags(html)
        }
        ClipboardData::Rtf(rtf) => {
            // 解析 RTF 格式，提取纯文本
            parse_rtf_to_text(rtf)

        }
    }
}

### 3.7 数据持久化与管理

#### 3.7.1 功能描述
本地存储历史记录，支持增删改查，应用重启后恢复。

#### 3.7.2 技术实现方案

**方案一：SQLite**
- 优势：轻量、快速、支持复杂查询
- 库：`rusqlite` (Rust) 或 `better-sqlite3` (Node.js)
- 适用性：⭐⭐⭐⭐⭐ (5/5) **推荐**

**方案二：文件系统 + JSON**
- 优势：简单直接
- 劣势：性能较差，不支持复杂查询
- 适用性：⭐⭐ (2/5)

**方案三：IndexedDB**
- 优势：浏览器原生支持
- 劣势：API 复杂，性能一般
- 适用性：⭐⭐⭐ (3/5)

#### 3.7.3 数据库设计

**表结构**：
```sql
CREATE TABLE clipboard_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT NOT NULL,  -- 'text', 'image', 'file'
    content TEXT,                -- 文本内容或文件路径
    content_blob BLOB,           -- 图片二进制数据
    plain_text TEXT,             -- 纯文本版本
    source_app TEXT,             -- 来源应用
    created_at INTEGER NOT NULL, -- 时间戳
    is_favorite INTEGER DEFAULT 0
);

CREATE INDEX idx_created_at ON clipboard_items(created_at DESC);
```

#### 3.7.4 存储策略
- 文本：直接存储到数据库
- 图片：保存原图到文件系统（如 `~/.clipboard-manager/images/`），数据库只存储文件路径和元信息
- 文件：只存储路径
- 缩略图：为图片生成小尺寸缩略图用于预览（如 200x200），原图按需加载

#### 3.7.5 技术难度
- 难度等级：⭐⭐ (2/5)
- 主要挑战：大量图片的存储优化

#### 3.7.6 可行性结论
✅ **可行**，SQLite 是成熟的解决方案。

## 4. 性能评估

### 4.1 性能指标

| 指标 | 目标值 | 预估值 | 可行性 |
|------|--------|--------|--------|
| 快捷键响应时间 | < 100ms | 50-80ms | ✅ 可达成 |
| 界面渲染时间 | < 200ms | 100-150ms | ✅ 可达成 |
| 记录切换延迟 | < 50ms | 20-30ms | ✅ 可达成 |
| 粘贴操作延迟 | < 100ms | 50-80ms | ✅ 可达成 |
| 内存占用（空闲） | < 100MB | 40-60MB (Tauri) | ✅ 可达成 |
| CPU 占用（空闲） | < 1% | 0.1-0.5% | ✅ 可达成 |

### 4.2 性能优化策略

**启动优化**：
- 延迟加载非核心功能
- 预编译资源
- 使用 Tauri 的轻量级架构

**运行时优化**：
- 虚拟滚动减少 DOM 节点
- 图片懒加载和缓存
- 防抖粘贴板监听（200ms 间隔）
- 使用 Web Worker 处理缩略图生成
- 只在可见区域渲染图片，离屏图片卸载

**内存优化**：
- 限制内存中的记录数量（如 100 条）
- 图片记录单独限制（如最近 50 条）
- 超大图片（如 >10MB）提示用户是否记录
- 定期清理过期缓存和临时文件
- 使用缩略图预览，原图按需加载到内存

### 4.3 性能风险
- ⚠️ 大量图片可能导致内存占用过高
- ⚠️ 频繁的粘贴板监听可能影响 CPU
- ⚠️ 数据库查询可能成为瓶颈

### 4.4 性能结论
✅ **可达成**，使用 Tauri 可以满足所有性能指标。

## 11. 图片处理方案详解

### 11.1 设计原则
**核心原则**：保持图片原始质量，不进行任何压缩或降质处理。

### 11.2 存储方案

### 11.3 内存管理策略

#### 11.3.1 分层加载
1. **列表视图**：只加载缩略图（200x200）
2. **预览模式**：按需加载原图
3. **粘贴操作**：直接读取原图文件

#### 11.3.2 缓存策略
- 使用 LRU（Least Recently Used）缓存
- 内存缓存限制：最多 20 张原图
- 缩略图可以缓存更多（如 100 张）
- 超出限制自动释放最久未使用的图片

#### 11.3.3 懒加载
```typescript
// React 示例
const ImageCard = ({ item }) => {
  const [isVisible, setIsVisible] = useState(false);
  const imgRef = useRef(null);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => setIsVisible(entry.isIntersecting),
      { threshold: 0.1 }
    );
    
    if (imgRef.current) {
      observer.observe(imgRef.current);
    }
    
    return () => observer.disconnect();
  }, []);

  return (
    <div ref={imgRef}>
      {isVisible ? (
        <img src={item.thumbnail_path} alt="clipboard" />
      ) : (
        <div className="placeholder" />
      )}
    </div>
  );
};
```

### 11.4 数量限制策略

#### 11.4.1 图片记录限制
- 默认保留最近 **50 条**图片记录
- 文本记录可以保留更多（如 200 条）
- 用户可在设置中自定义限制

#### 11.4.2 大小限制
- 单张图片 > 10MB：弹窗询问用户是否记录
- 单张图片 > 50MB：建议用户不记录，提示使用文件管理
- 总存储空间限制：如 500MB，超出后自动清理最旧记录

#### 11.4.3 自动清理
```rust
// 清理策略
fn cleanup_old_images(max_count: usize, max_size_mb: usize) {
    let items = get_image_items_sorted_by_date();
    
    // 按数量清理
    if items.len() > max_count {
        let to_remove = &items[max_count..];
        for item in to_remove {
            delete_image_files(item);
            delete_from_db(item.id);
        }
    }
    
    // 按大小清理
    let total_size = calculate_total_size();
    if total_size > max_size_mb * 1024 * 1024 {
        // 删除最旧的图片直到满足大小限制
        cleanup_by_size(max_size_mb);
    }
}
```

### 11.5 缩略图生成

#### 11.5.1 生成时机
- 图片复制时异步生成缩略图
- 不阻塞主线程
- 使用 Web Worker 或 Rust 后台线程

#### 11.5.2 缩略图规格
- 尺寸：200x200（保持宽高比）
- 格式：JPEG（质量 85%）
- 用途：仅用于列表预览

#### 11.5.3 实现示例
```rust
use image::{imageops, DynamicImage, ImageFormat};

fn generate_thumbnail(
    original_path: &Path,
    thumbnail_path: &Path,
    max_size: u32
) -> Result<()> {
    let img = image::open(original_path)?;
    
    // 计算缩略图尺寸（保持宽高比）
    let (width, height) = img.dimensions();
    let ratio = (max_size as f32) / width.max(height) as f32;
    let new_width = (width as f32 * ratio) as u32;
    let new_height = (height as f32 * ratio) as u32;
    
    // 生成缩略图
    let thumbnail = img.resize(
        new_width,
        new_height,
        imageops::FilterType::Lanczos3  // 高质量缩放
    );
    
    // 保存为 JPEG
    thumbnail.save_with_format(thumbnail_path, ImageFormat::Jpeg)?;
    
    Ok(())
}
```
