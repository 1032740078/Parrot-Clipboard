# v0.2 种子数据契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.2 Alpha 版本** 的种子数据与测试夹具契约，用于：
- 本地开发演示主面板
- 后端 SQLite 集成测试
- 前端组件与 Store 测试
- BDD 自动化验收准备

### 1.2 设计原则

- 种子数据必须覆盖文本、图片、文件三种类型
- 种子数据应包含摘要字段与类型专属详情
- 所有图片、文件样例必须脱敏，不得包含真实用户内容
- 图片原图与缩略图需有一一对应的资产文件或可生成规则

## 3. 类型覆盖要求

### 3.1 文本记录最小覆盖

- 单行短文本
- 三行以上长文本
- 含中英文与特殊符号
- 含富文本 `rich_content`
- 纯文本粘贴回退样例

### 3.2 图片记录最小覆盖

- PNG 透明背景图
- JPEG 照片图
- 缩略图 `pending` / `ready` / `failed` 三种状态至少各 1 条

### 3.3 文件记录最小覆盖

- 单文件
- 多文件
- 包含目录
- 不同扩展名（`md`、`png`、`pdf`）

## 4. 资产文件契约

### 4.1 图片资产目录

```text
fixtures/seeds/v0.2/images/
├── screenshot-dashboard.png
├── screenshot-dashboard-thumb.png
├── whiteboard-photo.jpg
├── whiteboard-photo-thumb.jpg
└── broken-thumb.placeholder
```

### 4.2 文件路径契约

- 所有文件路径均使用测试专用根目录，例如 `/mock/workspace/...`
- 不依赖开发者本机真实用户名、桌面路径或公司目录
- 若路径仅用于展示，可不存在；若用于 E2E 粘贴验证，则必须在测试前临时创建

---

## 5. 加载约定

### 5.1 开发模式加载

建议使用环境变量约定：

```bash
CLIPBOARD_SEED_FILE=fixtures/seeds/v0.2/demo-mixed-history.json
CLIPBOARD_SEED_RESET=1
```

行为说明：
- `CLIPBOARD_SEED_FILE`：指定种子清单文件
- `CLIPBOARD_SEED_RESET=1`：启动前重建测试数据库与资源目录

### 5.2 测试模式加载

- 后端集成测试使用临时目录复制数据库与资源夹具
- 前端单测直接导入 TypeScript Fixture，不依赖真实文件系统
- E2E 预置脚本负责创建临时数据库、临时图片与临时文件目录

---

## 6. 覆盖的测试场景

| 场景 | 对应种子 |
|------|----------|
| 混合卡片渲染 | `SEED-201` |
| 去重置顶 | `SEED-202/db-duplicates.json` |
| 数据库恢复 | `SEED-202/db-basic.json` |
| 图片缩略图失败降级 | `SEED-202/db-corrupted-image.json` |
| 文件卡片摘要渲染 | `SEED-203/recordSummaries.ts` |
| 事件流处理 | `SEED-203/events.ts` |

---

## 7. 数据清理约定

- 测试结束后删除临时 SQLite 文件与图片资源目录
- 所有种子资产不应写回 Git 跟踪外的真实用户目录
- 任何失败用例留下的临时文件应由测试框架 `afterEach`/`teardown` 清理
