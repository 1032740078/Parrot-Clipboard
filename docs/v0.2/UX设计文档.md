# v0.2 UX 设计文档

## 1. 文档概述

### 1.1 版本范围

`v0.2` 的 UX 设计聚焦于：**多类型历史识别、低心智负担浏览、快速再次粘贴、稳定的持久化体验**。

### 1.2 设计原则（v0.2 强调）

- **一眼识别类型**：文本、图片、文件卡片必须在 300ms 内被识别
- **主链路优先**：打开面板后优先支持方向键选择与回车粘贴
- **渐进加载**：先展示摘要，再逐步补齐缩略图等重资源信息
- **失败可感知但不阻断**：缩略图失败、文件缺失等问题应降级显示
- **保持 v0.1 心智连续性**：保留底部弹出、横向卡片、键盘优先的主交互

---

## 2. 主面板视觉设计

### 2.1 整体视觉风格

- 维持半透明、轻磨砂、圆角大卡片风格
- 保持较低视觉噪音，让卡片内容成为主角
- 顶部与底部信息区弱化，避免抢占卡片关注度

### 2.2 信息层级

从高到低依次为：
1. 当前选中卡片
2. 卡片内容类型与预览主体
3. 时间与元信息
4. 底部快捷键提示

---

## 3. 卡片类型 UX 规范

### 3.1 文本卡片

- 主体区域优先展示可读文本
- 超长文本使用 3 行截断，避免整卡变成密集文字墙
- 字符数与行数放在底部元信息栏，减少干扰

### 3.2 图片卡片

- 缩略图区域是视觉主元素，占卡片高度 60% 左右
- 在缩略图下方显示尺寸与格式，帮助用户区分不同截图
- 缩略图未就绪时使用灰色骨架 + 图片图标，占位形态应稳定不抖动

### 3.3 文件卡片

- 使用首个文件图标和文件名建立识别锚点
- 多文件时用 `+N` 数量补充，避免完整路径挤占版面
- 含目录时给出轻量“文件夹”标记，避免误判为单文件

---

## 4. 交互反馈设计

### 4.1 操作反馈矩阵

| 操作 | 成功反馈 | 失败反馈 |
|------|----------|----------|
| 打开面板 | 面板弹出、首张卡片高亮 | 无 |
| 切换卡片 | 选中卡片高亮位移动画 | 无 |
| 原格式粘贴 | 面板关闭、记录置顶 | Toast 显示失败原因 |
| 纯文本粘贴 | 面板关闭、底部短暂提示“纯文本模式” | 对非文本禁用或提示无效 |
| 删除记录 | 卡片平滑退出，邻近卡片补位 | Toast 提示删除失败 |
| 缩略图完成 | 卡片局部淡入刷新 | 不应打断选中态 |

### 4.2 Toast 规范

- 位置：主面板底部居中上浮 12px
- 时长：成功类 `1200ms`，失败类 `2200ms`
- 文案尽量短句，例如：`已切换为纯文本粘贴`、`图片预览生成失败`

---

## 5. 动效设计

### 5.1 面板弹出/收起

- 延续 `v0.1` 自底部上浮的入场动效
- 时长建议 `180ms`
- 透明度与位移同时过渡，避免突兀

### 5.2 选中卡片切换

- 使用 `120ms` 的边框/阴影过渡
- 当前卡片放大幅度控制在 `1.02` 以内，避免跳动感过强

### 5.3 图片缩略图更新

- 从骨架态切换到真实缩略图时使用 `100ms` 淡入
- 不应触发布局重排或卡片高度变化

---

## 6. 键盘优先体验

### 6.1 键位可见性

- 底部提示栏固定展示 `Enter`、`Shift+Enter`、`Delete`、`Esc`
- 对于非文本卡片，`Shift+Enter` 提示态应表现为禁用或弱化

### 6.2 焦点管理

- 主面板打开后自动聚焦卡片列表
- 删除当前卡片后，焦点迁移到“原索引位置的新卡片”；若已无数据，则迁移到空状态容器
- 缩略图异步加载不改变当前焦点位置

---

## 7. 错误与降级体验

### 7.1 图片预览失败

- 仍保留图片卡片，不直接隐藏记录
- 替换为统一占位图标与“预览不可用”文案
- 若原图仍可读取，允许继续 `Enter` 原格式粘贴

### 7.2 文件不可访问

- 文件卡片保留，但在粘贴失败时提示“文件已移动或无权限访问”
- 不因为单条错误导致整个面板退出

### 7.3 数据库恢复

- 首次恢复失败时显示轻量错误 Toast
- 同时允许用户继续使用新的空历史，而不是卡死在启动页

---

## 8. 可用性指标

| 指标 | 目标 |
|------|------|
| 用户识别卡片类型时间 | < 300ms |
| 打开面板到可操作时间 | < 200ms |
| 图片缩略图可见时间 | < 50ms（常见截图） |
| 方向键切换反馈 | < 50ms |
| 失败提示可理解性 | 用户无需查看日志即可理解下一步操作 |

---

## 9. macOS 平台特定 UX 规范

- 继续使用系统字体 `SF Pro`
- 面板位置基于可用工作区，避免被 Dock 遮挡
- 自动粘贴失败时，文案应明确说明可手动 `Cmd+V`
- Retina 屏幕下缩略图应避免发虚，优先使用 2x 输出资源
