# v0.2 自动化测试文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.2 Alpha 版本** 的自动化测试策略，覆盖：
- SQLite 持久化与迁移
- 文本、图片、文件三类记录的监听、查询、粘贴与删除
- 图片缩略图生成与降级展示
- 主面板混合卡片浏览与纯文本粘贴交互

### 1.2 测试架构分层

| 层级 | 工具 | 目标 |
|------|------|------|
| 后端单元测试 | `cargo test` | 校验聚合、仓储、图片处理、粘贴逻辑 |
| 前端单元测试 | Vitest + Testing Library | 校验 store、组件、hook、API 封装 |
| 后端集成测试 | `cargo test` + 临时 SQLite | 校验迁移、事务、事件联动 |
| 前后端契约测试 | Vitest + Mock IPC | 校验 DTO 与错误码一致性 |
| E2E / BDD 验收 | Playwright + Tauri 驱动脚本 | 校验复制→调面板→选择→粘贴主流程 |

### 1.3 覆盖率目标

| 范围 | 目标 |
|------|------|
| 前端覆盖率 | 维持 `lines/branches/functions/statements >= 75%` |
| 后端核心模块 | 新增模块建议 `>= 80%` 语句覆盖 |
| BDD 关键场景 | Must 类用户故事全部有自动化场景映射 |

## 2. 后端单元测试（Rust / cargo test）

### 2.1 持久化与迁移

#### UT-DB-001：初始化数据库创建三张业务表

- 输入：空临时目录
- 断言：创建 `clipboard_items`、`image_assets`、`file_items`
- 断言：`PRAGMA user_version` 等于 `2`

#### UT-DB-002：重复文本写入复用原记录

- 先写入文本记录 A
- 再以相同内容捕获文本记录 A
- 断言：记录总数不变，仅 `last_used_at` 更新

#### UT-DB-003：图片记录入库后关联图片资源元数据

- 保存一张测试 PNG
- 断言：主表存在 `image` 记录
- 断言：`image_assets` 存在对应 `item_id`
- 断言：`thumbnail_state='pending'`

#### UT-DB-004：文件记录保持路径顺序

- 输入：3 个路径（含目录）
- 断言：`file_items.sort_order` 连续且与输入顺序一致

#### UT-DB-005：按类型超限时仅清理同类型最旧记录

- 构造超限文本与图片集合
- 断言：文本清理不影响图片记录
- 断言：图片清理后会返回待删除资源路径

### 2.2 领域聚合与服务

#### UT-HIST-002：命中旧文件记录时返回复用决策

## 4. 后端集成测试

### 4.1 SQLite 仓储集成

- 使用临时目录创建数据库与图片资源目录
- 运行真实迁移脚本
- 执行文本/图片/文件插入、查询、删除、清理流程
- 校验数据库与文件系统最终状态一致

### 4.2 IPC 命令集成

- `get_records`：返回混合摘要列表
- `get_record_detail`：返回按类型展开的完整详情
- `paste_record`：返回被置顶后的最新摘要
- `delete_record`：删除后可通过 `get_records` 观察到差异

### 4.3 启动恢复集成

- 第一次运行写入多类型种子数据
- 第二次运行直接读取历史
- 断言：记录数量、排序、图片缩略图路径与文件明细均可恢复

## 5. BDD / E2E 自动化

### 5.1 建议工具链

- `Playwright`：驱动前端界面与键盘交互
- `pnpm tauri dev` 或测试构建产物：启动桌面应用
- macOS 辅助脚本：注入测试用粘贴板内容（文本/图片/文件）

### 5.2 必测主链路

| 场景 | 目标 |
|------|------|
| 文本复制 → 面板 → 纯文本粘贴 | 验证 `Shift + Enter` |
| 图片复制 → 面板 → 原格式粘贴 | 验证图片恢复与缩略图刷新 |
| 文件复制 → 面板 → 原格式粘贴 | 验证文件列表恢复 |
| 应用重启 → 面板读取历史 | 验证持久化 |
| 超限清理 | 验证最旧记录与资源清理 |

## 6. 测试数据与夹具规范

### 6.1 文本夹具

- 短文本、长文本、多行文本、富文本降级文本
- 包含中英文、符号、换行、空格边界

### 6.2 图片夹具

- PNG 截图（透明背景）
- JPEG 照片（大尺寸）
- GIF 静态首帧样例

### 6.3 文件夹具

- 单文件
- 多文件
- 文件 + 目录混合集合

### 6.4 脱敏要求

- 所有测试文件必须来自 `src/__tests__/fixtures/` 或专用测试目录
- 不允许提交真实用户截图、真实工作文档、真实个人路径

## 7. CI 建议执行顺序

```bash
pnpm run lint
pnpm run type-check
pnpm run test:coverage
cargo test --manifest-path src-tauri/Cargo.toml
cargo fmt --manifest-path src-tauri/Cargo.toml --all -- --check
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings
pnpm run test:e2e
```

> `pnpm run test:e2e` 为 v0.2 新增命令约定，用于触发 Playwright/Tauri 验收。

## 8. 通过标准（Definition of Done）

- 所有 Must 类用户故事均具备至少 1 条自动化测试映射
- 前端与后端测试通过，核心路径无阻断失败
- 关键错误码有契约测试覆盖
- 持久化、图片资源清理、纯文本粘贴三类高风险能力均有集成测试
