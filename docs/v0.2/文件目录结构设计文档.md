# v0.2 文件目录结构设计文档

## 1. 文档概述

### 1.1 版本范围

本文档描述 **v0.2 Alpha 版本** 为支持图片、文件与持久化所需的目录结构设计，仅覆盖当前版本新增或调整的目录与文件职责。

### 1.2 设计原则

- 延续 `v0.1` 的前后端分层结构，不重写既有模块边界
- 新增能力优先收敛到对应领域目录，避免把逻辑堆到 `main.rs`、`App.tsx`
- 前端按“组件 / store / api / 类型 / hooks”分层
- 后端按“领域 / 仓储 / 资源 / IPC / 平台”分层

## 6. 运行时生成目录

```text
(运行时)
~/Library/Application Support/clipboard-record-manager/
├── clipboard.db
├── config.json
├── images/
│   ├── original/
│   └── thumbs/
└── logs/
```

### 6.1 约束说明

- `clipboard.db` 仅由持久化模块访问
- `images/original/` 仅存应用复制得到的图片资源，不存用户原始来源文件
- `images/thumbs/` 可被重建，删除后不影响原图粘贴能力
- 运行时目录均不纳入 Git 版本控制

## 7. 命名规范

| 范围 | 规范 |
|------|------|
| React 组件 | `PascalCase.tsx` |
| Hook | `useXxx.ts` |
| Rust 模块 | `snake_case.rs` |
| 测试文件 | `*.test.ts` / `*.test.tsx` / `*_test.rs` |
| 资源目录 | 使用小写英文，如 `images/thumbs` |

---

## 8. 与 v0.1 的主要差异

| 项目 | v0.1 | v0.2 |
|------|------|------|
| 前端卡片类型 | `TextCard` | `TextCard` + `ImageCard` + `FileCard` |
| 后端仓储 | 内存 `store.rs` | SQLite `persistence/` |
| 图片模块 | 无 | 新增 `image/` |
| 粘贴能力 | 文本 | 文本 + 图片 + 文件 |
