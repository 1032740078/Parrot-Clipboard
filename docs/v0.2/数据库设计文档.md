# v0.2 数据库设计文档

## 1. 文档概述

### 1.1 版本范围

本文档描述 **v0.2 Alpha 版本** 的数据持久化设计。与 `v0.1` 仅使用内存不同，`v0.2` 引入：
- SQLite 保存结构化元数据
- 本地文件系统保存图片原图与缩略图
- 应用重启后的历史恢复

### 1.2 设计原则

- 主数据统一落在 SQLite，图片二进制不直接存表
- 文本、图片、文件三种记录共享一张主表，类型特有数据拆分到子表
- 去重基于 `(content_type, content_hash)` 唯一约束
- 图片、文件明细通过外键级联删除，删除时尽可能保持事务一致性
- 迁移版本使用 `PRAGMA user_version` 管理，不额外增加业务外的版本表

## 2. 存储布局总览

### 2.1 运行时目录

以 macOS 应用数据目录为例：

```text
~/Library/Application Support/clipboard-record-manager/
├── clipboard.db
├── config.json
├── images/
│   ├── original/
│   └── thumbs/
└── logs/
```

### 2.2 存储分工

| 存储介质 | 内容 | 说明 |
|----------|------|------|
| `clipboard.db` | 主记录、图片元数据、文件明细 | 结构化查询与持久化 |
| `images/original/` | 图片原图 | 保证再次粘贴时有完整数据 |
| `images/thumbs/` | 图片缩略图 | 面板快速预览 |
| `config.json` | 配置项 | 不纳入本数据库设计范围 |

## 3. 数据表设计

### 3.1 `clipboard_items`（主记录表）

保存所有记录的公共字段与摘要信息。

```sql
CREATE TABLE clipboard_items (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  content_type    TEXT NOT NULL CHECK (content_type IN ('text', 'image', 'files')),
  content_hash    TEXT NOT NULL,
  text_content    TEXT,
  rich_content    TEXT,
  preview_text    TEXT NOT NULL,
  search_text     TEXT NOT NULL,
  source_app      TEXT,
  file_count      INTEGER NOT NULL DEFAULT 0,
  payload_bytes   INTEGER NOT NULL DEFAULT 0,
  created_at      INTEGER NOT NULL,
  last_used_at    INTEGER NOT NULL,
  UNIQUE(content_type, content_hash)
);
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 记录主键 |
| `content_type` | TEXT | `text` / `image` / `files` |
| `content_hash` | TEXT | 去重哈希 |
| `text_content` | TEXT | 文本正文；仅文本记录必填 |
| `rich_content` | TEXT | 富文本原始内容；纯文本记录可为空 |
| `preview_text` | TEXT | 卡片摘要文本 |
| `search_text` | TEXT | 当前版本保留搜索预处理字段，用于统一摘要查询 |
| `source_app` | TEXT | 来源应用名，可空 |
| `file_count` | INTEGER | 文件记录内路径数量，其余类型为 `0` |
| `payload_bytes` | INTEGER | 内容原始字节数估算 |
| `created_at` | INTEGER | 首次捕获时间（Unix 毫秒） |
| `last_used_at` | INTEGER | 最近被复制/粘贴后置顶时间 |

### 3.2 `image_assets`（图片资源表）

仅保存图片记录的资源元数据，`item_id` 与主表一一对应。

```sql
CREATE TABLE image_assets (
  item_id          INTEGER PRIMARY KEY,
  original_path    TEXT NOT NULL UNIQUE,
  thumbnail_path   TEXT,
  mime_type        TEXT NOT NULL,
  pixel_width      INTEGER NOT NULL,
  pixel_height     INTEGER NOT NULL,
  byte_size        INTEGER NOT NULL,
  thumbnail_state  TEXT NOT NULL CHECK (thumbnail_state IN ('pending', 'ready', 'failed')),
  created_at       INTEGER NOT NULL,
  FOREIGN KEY(item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);
```

### 3.3 `file_items`（文件明细表）

一个 `files` 记录可对应多个路径项。

```sql
CREATE TABLE file_items (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id       INTEGER NOT NULL,
  sort_order    INTEGER NOT NULL,
  path          TEXT NOT NULL,
  display_name  TEXT NOT NULL,
  entry_type    TEXT NOT NULL CHECK (entry_type IN ('file', 'directory')),
  extension     TEXT,
  created_at    INTEGER NOT NULL,
  FOREIGN KEY(item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE,
  UNIQUE(item_id, sort_order)
);
```

## 4. 索引设计

```sql
CREATE INDEX idx_clipboard_items_last_used_at
  ON clipboard_items(last_used_at DESC, id DESC);

CREATE INDEX idx_clipboard_items_content_type_last_used_at
  ON clipboard_items(content_type, last_used_at DESC, id DESC);

CREATE INDEX idx_image_assets_thumbnail_state
  ON image_assets(thumbnail_state, item_id);

CREATE INDEX idx_file_items_item_id_sort_order
  ON file_items(item_id, sort_order);
```

**设计说明**：
- 主列表查询按 `last_used_at DESC, id DESC` 排序，保证“最新在前”
- 按类型清理时使用 `(content_type, last_used_at)` 索引
- 缩略图后台任务扫描 `pending` 状态时使用 `idx_image_assets_thumbnail_state`

## 5. 数据约束与业务规则

### 5.1 记录约束

| 规则 | 说明 |
|------|------|
| 文本记录必须有 `text_content` | `text` 类型不可为空 |
| 图片记录必须在 `image_assets` 中存在一条对应记录 | 通过业务层保证 |
| 文件记录必须至少有一条 `file_items` | 通过事务插入保证 |
| 同类型同哈希仅保留一条有效记录 | 通过唯一约束与仓储更新实现 |

### 5.2 去重策略

| 类型 | 哈希来源 | 去重动作 |
|------|----------|----------|
| 文本 | 原始文本 UTF-8 字节 | 更新 `last_used_at`，不新建记录 |
| 图片 | 原图字节 | 更新 `last_used_at`，必要时刷新缩略图状态 |
| 文件 | 规范化路径列表 | 更新 `last_used_at`，不重复插入文件项 |

### 5.3 保留策略

| 类型 | 默认上限 | 清理规则 |
|------|----------|----------|
| 文本 | 200 | 删除同类型最旧记录 |
| 图片 | 50 | 删除同类型最旧记录，并清理图片资源文件 |
| 文件 | 100 | 删除同类型最旧记录 |

## 6. 典型读写流程

### 6.1 插入文本记录事务

```sql
BEGIN IMMEDIATE;

INSERT INTO clipboard_items (
  content_type,
  content_hash,
  text_content,
  rich_content,
  preview_text,
  search_text,
  source_app,
  file_count,
  payload_bytes,
  created_at,
  last_used_at
) VALUES ('text', ?, ?, ?, ?, ?, ?, 0, ?, ?, ?);

COMMIT;
```

### 6.2 插入图片记录事务

```sql
BEGIN IMMEDIATE;

INSERT INTO clipboard_items (...content_type='image'...);
INSERT INTO image_assets (
  item_id,
  original_path,
  thumbnail_path,
  mime_type,
  pixel_width,
  pixel_height,
  byte_size,
  thumbnail_state,
  created_at
) VALUES (?, ?, NULL, ?, ?, ?, ?, 'pending', ?);

COMMIT;
```

### 6.3 插入文件记录事务

```sql
BEGIN IMMEDIATE;

INSERT INTO clipboard_items (...content_type='files'...);
INSERT INTO file_items (item_id, sort_order, path, display_name, entry_type, extension, created_at)
VALUES (...), (...), (...);

COMMIT;
```

### 6.4 主面板读取摘要查询

```sql
SELECT
  i.id,
  i.content_type,
  i.preview_text,
  i.source_app,
  i.file_count,
  i.created_at,
  i.last_used_at,
  ia.thumbnail_path,
  ia.mime_type,
  ia.pixel_width,
  ia.pixel_height
FROM clipboard_items i
LEFT JOIN image_assets ia ON ia.item_id = i.id
ORDER BY i.last_used_at DESC, i.id DESC
LIMIT ?;
```

### 6.5 详情读取查询

- 文本：读取 `text_content` 与 `rich_content`
- 图片：读取 `image_assets.original_path`、`mime_type`、尺寸与大小
- 文件：读取 `file_items` 明细并按 `sort_order ASC` 返回

## 7. 清理与恢复策略

### 7.1 自动清理顺序

1. 先查出超限的同类型最旧记录 ID
2. 在事务中删除 `clipboard_items`
3. 依赖外键级联删除 `image_assets` / `file_items`
4. 事务提交后删除图片原图与缩略图文件
5. 文件删除失败时记录日志并进入后续补偿清理流程

### 7.2 启动恢复策略

- 应用启动时先校验 `clipboard.db` 是否存在，不存在则创建并执行迁移
- 若数据库可读但图片文件缺失，记录保留，图片卡片降级显示占位态
- 若单条记录解析失败，仅跳过该条，不中断其余记录加载

### 7.3 孤立资源清理

- 扫描 `images/original/` 与 `images/thumbs/`
- 若文件路径不再出现在 `image_assets` 表内，则删除该文件
- 该任务在应用启动后与删除图片记录后异步执行

## 8. 性能与可靠性目标

| 指标 | 目标 |
|------|------|
| 首次加载最近 100 条摘要 | < 80ms |
| 单条文本插入事务 | < 10ms |
| 单条图片元数据事务 | < 20ms（不含缩略图生成） |
| 图片缩略图生成 | < 50ms（常见截图尺寸） |
| 数据库损坏恢复 | 应能备份旧库并自动重建新库 |

---

## 9. 与 v0.1 的差异

| 项目 | v0.1 | v0.2 |
|------|------|------|
| 存储介质 | 纯内存 | SQLite + 文件系统 |
| 内容类型 | 文本 | 文本 + 图片 + 文件 |
| 重启恢复 | 不支持 | 支持 |
| 去重范围 | 会话内 | 跨会话 |
| 自动清理 | 固定 20 条文本 | 按类型分别清理 |
