## 3. 各批次任务详情

### 3.1 第 0 批：文档基线

**目标**：明确 v0.2 范围、领域模型、接口、数据库与验收口径。

- `TASK-00-1` 已完成：输出用户故事、DDD、数据库、API、状态码、种子数据、页面地图、UX、测试与冲刺文档

### 3.2 第 1 批：数据与配置基建

**目标**：先把可持久化的数据底座搭稳。

**完成标准**：
- `config.json` 扩展三类上限配置
- SQLite 数据库可自动迁移与重建
- 主面板可读取最近 100 条混合摘要
- 超限清理与孤立资源清理可独立运行

### 3.3 第 2 批：多类型采集与图片处理

**目标**：让监听层真正接住图片与文件，并把图片资源保存完整。

**完成标准**：
- 监听到图片后可保存原图并异步生成缩略图
- 监听到文件/目录后可保存路径集合
- 文本、图片、文件统一走去重与置顶策略

### 3.4 第 3 批：粘贴与 IPC 扩展

**目标**：打通“历史记录再次粘贴”与“前端读写接口”。

**完成标准**：
- `paste_record` 支持三种类型
- `get_record_detail` 可读取完整详情
- `clipboard:record-updated` 可驱动缩略图刷新与记录置顶

### 3.5 第 4 批：主面板 UI 扩展

**目标**：让主面板从单一文本列表升级为混合记录浏览器。

**完成标准**：
- 新增图片卡片、文件卡片、骨架态
- 保留 v0.1 键盘优先体验
- 非文本卡片的纯文本操作有明确禁用反馈

### 3.6 第 5 批：自动化测试与验收

**目标**：确保新增加的持久化与多类型能力可回归、可验收。

**完成标准**：
- Must 级用户故事有自动化测试映射
- E2E 能覆盖文本、图片、文件三条主链路
- 关键性能指标达标

## 4. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 图片格式兼容性不足 | 图片记录与粘贴失败 | 先聚焦 PNG/JPEG/GIF，其他格式记录日志降级 |
| SQLite 与文件系统状态不一致 | 历史残缺或资源泄漏 | 建立事务 + 启动补偿清理 |
| 自动化桌面测试不稳定 | CI 波动 | 将高频单测与少量关键 E2E 分层执行 |
| 纯文本模式边界多 | 富文本剥离异常 | 优先对 HTML/RTF 建立回归用例 |

---

## 5. 当前建议执行顺序

1. 先完成第 1 批的数据底座
2. 再实现第 2 批的采集与图片处理
3. 随后打通第 3 批粘贴与 IPC
4. 最后完善第 4、5 批 UI 与自动化验收
