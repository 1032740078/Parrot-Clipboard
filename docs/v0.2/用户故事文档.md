# v0.2 用户故事文档

## 1. 文档概述

### 1.1 版本范围

本文档仅描述 **v0.2 Alpha 版本** 的用户故事，不包含 `v0.3+` 的搜索、设置页、系统托盘、自启动与多平台能力。

**v0.2 核心目标**：
- 支持文本、图片、文件/目录三种粘贴板记录类型
- 引入 SQLite 持久化，支持应用重启后恢复历史记录
- 支持图片缩略图展示、文件摘要展示与按需加载原始内容
- 补充纯文本粘贴模式（`Shift + Enter`）
- 建立面向后续版本可扩展的数据、接口与测试基线

### 1.2 角色定义

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 内容重用型用户 | 高频复制文本、截图、文件的办公用户 | 快速找回并再次粘贴 |
| 视觉素材型用户 | 经常复制图片、设计稿截图的用户 | 快速预览缩略图并恢复原图 |
| 文件分发型用户 | 经常复制文件路径、文件夹的用户 | 清晰识别文件集合并快速再次粘贴 |

## 3. 用户故事详细说明

### E01 - 多类型内容采集

#### US-01-01：自动记录图片复制内容

**作为** 视觉素材型用户，**我希望** 在复制图片后自动生成一条图片历史记录，**以便** 我稍后可以从面板中再次选择并粘贴这张图片。

**验收标准**：
- 当系统粘贴板内容为图片时，应用在监听周期内自动生成一条 `image` 类型记录
- 记录生成后可在面板中看到缩略图占位，缩略图生成完成后自动刷新
- 记录保存原图，不因缩略图生成而丢失原始分辨率与 MIME 信息

#### US-01-02：自动记录文件或目录复制内容

**作为** 文件分发型用户，**我希望** 复制文件或目录后自动生成文件卡片，**以便** 我可以再次粘贴同一组文件。

**验收标准**：
- 当系统粘贴板为文件/目录列表时，应用生成 `files` 类型记录
- 卡片至少展示首个文件名、总数量、是否包含目录
- 多个文件保持原始顺序，后续粘贴时顺序不丢失

#### US-01-03：重复内容复用旧记录并置顶

**作为** 内容重用型用户，**我希望** 当我再次复制历史中已存在的内容时复用旧记录，**以便** 历史列表不被重复项淹没。

**验收标准**：
- 文本、图片、文件三种类型均按对应哈希或规范化内容进行精确去重
- 命中已有记录时不创建新 ID，而是复用旧记录并移动到最新位置
- 复用记录后更新时间戳与最近使用信息，前端卡片顺序同步更新

### E02 - 历史持久化与恢复

#### US-02-01：应用重启后恢复历史记录

**作为** 内容重用型用户，**我希望** 关闭并重新打开应用后仍能看到历史记录，**以便** 我不需要重新复制常用内容。

**验收标准**：
- 文本记录保存在 SQLite 中，应用重启后可完整读取
- 图片记录的元数据保存在 SQLite 中，原图/缩略图保存在应用数据目录中
- 文件记录的路径列表保存在 SQLite 中，应用重启后卡片摘要保持一致

#### US-02-02：资源缺失时可降级展示

**作为** 用户，**我希望** 即使缩略图文件丢失或数据库部分损坏，面板也能尽量展示可恢复信息，**以便** 我不会因为单条坏数据影响整体使用。

**验收标准**：
- 缩略图丢失时图片卡片显示占位态，而不是导致面板崩溃
- 单条坏记录读取失败时跳过该记录并记录日志，其他记录正常展示
- 数据损坏不会阻塞应用启动；必要时可自动重建缩略图或清理坏记录

### E03 - 混合记录浏览体验

#### US-03-01：在同一主面板中浏览混合记录

**作为** 用户，**我希望** 在一个统一的主面板中看到文本、图片与文件记录，**以便** 我能用一致的方式浏览最近复制的内容。

**验收标准**：
- 主面板按时间倒序展示混合卡片，最新记录位于最左侧
- 左右方向键在不同类型卡片之间连续切换，无需区分类型
- 空状态、加载态、图片占位态均有明确视觉反馈

#### US-03-02：快速识别图片记录

**作为** 视觉素材型用户，**我希望** 图片卡片能优先展示缩略图与尺寸信息，**以便** 我迅速识别我要粘贴的图片。

**验收标准**：
- 图片卡片优先展示 200×200 缩略图或等比裁切后的缩略图区域
- 卡片底部展示图片尺寸、MIME 类型或大小摘要
- 缩略图未就绪时显示骨架或占位图标，不展示破图

#### US-03-03：快速识别文件记录

**作为** 文件分发型用户，**我希望** 文件卡片明确显示文件名、数量和目录标记，**以便** 我快速区分不同文件集合。

**验收标准**：
- 单文件记录展示文件名与扩展名图标
- 多文件记录展示首项名称与 “+N” 数量摘要
- 含目录时显示目录标识，避免误认为单文件

### E04 - 多模式粘贴执行

#### US-04-01：原格式粘贴三种记录类型

**作为** 用户，**我希望** 选中任意记录后按 `Enter` 即可恢复并粘贴原始内容，**以便** 保持与原复制对象一致的效果。

**验收标准**：
- 文本记录按原始文本/富文本能力写回系统粘贴板并执行粘贴
- 图片记录写回完整图片数据并执行粘贴
- 文件记录按系统文件列表格式写回并执行粘贴

#### US-04-02：纯文本模式粘贴文本记录

**作为** 内容重用型用户，**我希望** 在粘贴文本时使用 `Shift + Enter` 去掉富文本格式，**以便** 只粘贴纯文本内容。

**验收标准**：
- 纯文本模式仅对 `text` 类型记录可用
- 粘贴后保留换行与可见字符，不保留 HTML/RTF 样式
- 当用户对图片或文件记录触发该模式时，前端禁用该操作，后端返回明确错误码兜底

### E05 - 存储治理与清理

#### US-05-01：超过上限后自动清理最旧记录

**作为** 用户，**我希望** 历史记录超过容量时自动清理最旧记录，**以便** 应用始终保持可控存储占用。

**验收标准**：
- 文本默认最多保留 200 条、图片 50 条、文件 100 条
- 超出上限时按类型清理最旧记录，不影响其他类型的最新记录
- 被清理的图片记录会同步删除原图与缩略图文件

#### US-05-02：删除记录时同步释放资源

**作为** 用户，**我希望** 删除单条记录后，其关联的数据库数据与文件资源都被清理，**以便** 不留下不可见垃圾数据。

**验收标准**：
- 删除文本记录时移除主表记录
- 删除图片记录时移除主表、图片表、原图文件与缩略图文件
- 删除文件记录时移除主表与文件明细表，不误删用户原始文件

## 5. 排除范围说明

以下能力不纳入 **v0.2**：
- 搜索与筛选界面
- 设置页面与自定义快捷键
- 系统托盘与开机自启动
- Windows / Linux 平台适配
- 云同步、标签、收藏、固定置顶
- 图片编辑、文件预览器、历史分组
