# v0.2 状态码契约文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.2 Alpha 版本** 的统一错误码与错误处理约定，用于前后端 IPC 契约、自动化测试与日志分析。

### 1.2 错误对象格式

```json
{
  "code": "DB_ERROR",
  "message": "Failed to open sqlite database"
}
```

### 1.3 错误传递机制

```text
Rust AppError
  -> serde 序列化
  -> Tauri rejected Promise
  -> 前端统一 errorHandler
  -> Toast / 静默日志 / 状态回滚
```

---

## 2. 错误码清单

| 错误码 | 语义 | 典型来源 |
|--------|------|----------|
| `RECORD_NOT_FOUND` | 记录不存在 | `get_record_detail`、`delete_record`、`paste_record` |
| `INVALID_PARAM` | 参数非法或模式不支持 | `get_records`、`paste_record` |
| `CLIPBOARD_READ_ERROR` | 读取系统粘贴板失败 | 监听服务 |
| `CLIPBOARD_WRITE_ERROR` | 写入系统粘贴板失败 | 粘贴服务 |
| `KEY_SIM_ERROR` | 模拟按键失败 | 粘贴服务 |
| `DB_ERROR` | SQLite 打开、迁移、事务失败 | 持久化层 |
| `IMAGE_PROCESS_ERROR` | 图片保存、解码、缩略图生成失败 | 图片处理模块 |
| `FILE_ACCESS_ERROR` | 文件路径解析、权限或 IO 失败 | 文件记录与资源清理 |
| `WINDOW_ERROR` | 窗口显示/隐藏失败 | 面板管理 |
| `INTERNAL` | 未预期异常 | 全局兜底 |

---

## 3. 错误码详细说明

### SC-001：`RECORD_NOT_FOUND`

| 属性 | 值 |
|------|-----|
| 错误码 | `RECORD_NOT_FOUND` |
| 语义 | 请求的记录 ID 不存在 |
| 处理建议 | 删除时可静默；粘贴和详情读取时给出提示 |

**示例**：

```json
{ "code": "RECORD_NOT_FOUND", "message": "Record with id 42 not found" }
```

### SC-002：`INVALID_PARAM`

| 属性 | 值 |
|------|-----|
| 错误码 | `INVALID_PARAM` |
| 语义 | 参数缺失、越界或不支持 |
| 典型场景 | `limit<=0`、`limit>500`、对图片使用 `plain_text` |

### SC-003：`CLIPBOARD_READ_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `CLIPBOARD_READ_ERROR` |
| 语义 | 系统粘贴板读取失败 |
| 处理建议 | 记录日志，可短暂重试，不直接打断前端 |

### SC-004：`CLIPBOARD_WRITE_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `CLIPBOARD_WRITE_ERROR` |
| 语义 | 系统粘贴板写入失败 |
| 处理建议 | Toast 提示“写入粘贴板失败，请重试” |

### SC-005：`KEY_SIM_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `KEY_SIM_ERROR` |
| 语义 | 自动粘贴按键模拟失败 |
| 处理建议 | 告知用户内容已写入粘贴板，可手动 `Cmd+V` |

### SC-006：`DB_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `DB_ERROR` |
| 语义 | 数据库打开、迁移或事务失败 |
| 处理建议 | 启动阶段尝试备份损坏数据库并重建；运行阶段提示用户重试 |

### SC-007：`IMAGE_PROCESS_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `IMAGE_PROCESS_ERROR` |
| 语义 | 图片保存、读取或缩略图生成失败 |
| 处理建议 | 卡片降级为占位态；若影响粘贴则提示失败 |

### SC-008：`FILE_ACCESS_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `FILE_ACCESS_ERROR` |
| 语义 | 文件路径不存在、权限不足或 IO 失败 |
| 处理建议 | 记录失败详情并提示用户原文件可能已移动或无权限 |

### SC-009：`WINDOW_ERROR`

| 属性 | 值 |
|------|-----|
| 错误码 | `WINDOW_ERROR` |
| 语义 | 面板显示/隐藏失败 |
| 处理建议 | 记录日志，允许用户再次触发快捷键重试 |

### SC-010：`INTERNAL`

| 属性 | 值 |
|------|-----|
| 错误码 | `INTERNAL` |
| 语义 | 未分类异常 |
| 处理建议 | 统一兜底提示并记录完整日志 |

## 5. 前端统一处理约定

| 错误码 | 前端策略 |
|--------|----------|
| `RECORD_NOT_FOUND` | 同步移除本地卡片并提示“记录已不存在” |
| `INVALID_PARAM` | 视为调用方错误，记录日志，不向用户暴露实现细节 |
| `CLIPBOARD_READ_ERROR` | 静默记录日志，状态栏可显示监听异常 |
| `CLIPBOARD_WRITE_ERROR` | Toast 错误提示，面板保持打开 |
| `KEY_SIM_ERROR` | Toast 提示“已写入粘贴板，请手动粘贴” |
| `DB_ERROR` | Toast 提示“历史记录读取失败，请重启应用” |
| `IMAGE_PROCESS_ERROR` | 图片卡片显示占位态；必要时提示原图不可用 |
| `FILE_ACCESS_ERROR` | 提示“文件已移动或无权限访问” |
| `WINDOW_ERROR` | 控制台日志 + 允许用户重试 |
| `INTERNAL` | 统一兜底提示 |

---

## 6. 成功响应约定

- `get_records`：始终返回数组，空列表返回 `[]`
- `delete_record` / `hide_panel`：成功时返回 `null` / `void`
- `paste_record`：成功时返回 `PasteResult`
- `get_monitoring_status`：成功时返回对象 `{ monitoring: boolean }`
