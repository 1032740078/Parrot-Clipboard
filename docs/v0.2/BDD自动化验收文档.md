# v0.2 BDD 自动化验收文档

## 1. 文档概述

### 1.1 版本范围

本文档定义 **v0.2 Alpha 版本** 的 BDD 自动化验收场景，仅覆盖：
- 文本、图片、文件/目录三种记录类型
- 数据持久化与重启恢复
- 图片缩略图生成与延迟刷新
- `Enter` 原格式粘贴、`Shift + Enter` 纯文本粘贴
- 类型级别的容量清理与资源回收

### 1.2 BDD 编写规范

- `Feature` 表示用户可感知能力
- `Scenario` 对应一条自动化验收场景
- 每条场景必须能映射到至少一个自动化脚本
- 断言以用户结果为主，必要时补充数据库与文件系统校验

## 2. Feature：多类型粘贴板监听（Clipboard Monitoring）

### 场景 BDD-01-01：复制图片后自动生成图片记录

```gherkin
Given 应用正在后台监听粘贴板
And 当前历史列表中没有该图片记录
When 用户复制一张 PNG 图片
Then 历史列表新增 1 条 image 类型记录
And 图片卡片先显示缩略图占位态
And 缩略图生成完成后自动刷新为可预览状态
```

### 场景 BDD-01-02：复制多个文件后生成文件记录

```gherkin
Given 应用正在后台监听粘贴板
When 用户复制 3 个文件和 1 个目录
Then 历史列表新增 1 条 files 类型记录
And 卡片展示首个文件名与总数量 4
And 该记录保存全部路径且顺序与复制时一致
```

### 场景 BDD-01-03：重复复制旧文本记录时复用原记录

```gherkin
Given 历史中已存在文本记录 "日报模板"
And 该记录当前不是最新一条
When 用户再次复制相同文本 "日报模板"
Then 历史总数保持不变
And 原记录被移动到最新位置
And 该记录的最近使用时间被更新
```

## 3. Feature：持久化与恢复（Persistence & Recovery）

### 场景 BDD-02-01：应用重启后恢复混合历史记录

```gherkin
Given 历史中已有 2 条文本、1 条图片、1 条文件记录
When 用户退出应用并重新启动
Then 主面板首次打开时能看到这 4 条记录
And 顺序仍按最近使用时间倒序排列
And 图片卡片可继续显示已有缩略图或占位态
```

### 场景 BDD-02-02：缩略图文件缺失时降级显示

```gherkin
Given 数据库中存在一条图片记录
And 该记录的原图文件存在但缩略图文件已被删除
When 用户打开主面板
Then 该图片记录仍显示在列表中
And 卡片以占位态展示而不是报错崩溃
And 应用记录一次资源缺失日志
```

## 4. Feature：主面板混合浏览（Mixed Browsing）

### 场景 BDD-03-01：左右方向键在混合卡片中连续切换

```gherkin
Given 主面板中依次有文本、图片、文件三种记录卡片
And 当前选中最左侧文本卡片
When 用户按下右方向键两次
Then 选中状态依次移动到图片卡片和文件卡片
And 不因卡片类型不同而中断切换
```

### 场景 BDD-03-02：图片缩略图按需刷新不打断导航

```gherkin
Given 主面板中存在一张缩略图尚未完成的图片卡片
When 用户持续使用方向键切换卡片
And 后台缩略图在此期间生成完成
Then 当前选中状态保持不变
And 该图片卡片仅更新自身展示内容
```

### 场景 BDD-03-03：无记录时展示空状态

```gherkin
Given 当前数据库中没有任何历史记录
When 用户调出主面板
Then 主面板显示空状态视图
And 提示用户先复制文本、图片或文件
And 不显示卡片列表容器
```

## 5. Feature：粘贴执行（Paste Operation）

### 场景 BDD-04-01：文本记录按原格式粘贴

```gherkin
Given 主面板选中一条文本记录
When 用户按下 Enter
Then 应用将该文本写回系统粘贴板
And 自动向目标应用发送粘贴指令
And 面板关闭
And 被粘贴的记录被置顶
```

### 场景 BDD-04-02：文本记录按纯文本模式粘贴

```gherkin
Given 主面板选中一条带富文本来源的文本记录
When 用户按下 Shift + Enter
Then 应用仅写入纯文本内容到系统粘贴板
And 富文本样式不会被带入目标应用
And 面板关闭
```

### 场景 BDD-04-03：图片记录按原格式粘贴

```gherkin
Given 主面板选中一条图片记录
When 用户按下 Enter
Then 应用读取该记录对应的原图文件
And 将完整图片数据写回系统粘贴板
And 目标应用接收到原始图片内容
```

### 场景 BDD-04-04：文件记录按原格式粘贴

```gherkin
Given 主面板选中一条包含多个文件路径的记录
When 用户按下 Enter
Then 应用将全部文件路径以系统文件列表格式写回粘贴板
And 目标应用接收到与原复制顺序一致的文件集合
```

### 场景 BDD-04-05：对图片记录触发纯文本粘贴被阻止

```gherkin
Given 主面板选中一条图片记录
When 用户尝试执行 Shift + Enter
Then 前端不应发起 paste_record plain_text 调用
And 若后端收到该调用则返回 INVALID_PARAM
And 主面板保持打开以提示用户选择有效操作
```

## 6. Feature：存储治理（Retention & Cleanup）

### 场景 BDD-05-01：文本记录超限自动淘汰最旧记录

```gherkin
Given 文本记录数量已达到上限 200 条
When 用户再次复制一条新的文本内容
Then 历史中仍只保留 200 条文本记录
And 最旧的一条文本记录被自动移除
And 其他类型记录不受影响
```

### 场景 BDD-05-02：图片记录超限时同步清理资源文件

```gherkin
Given 图片记录数量已达到上限 50 条
When 用户再次复制一张新图片
Then 最旧的图片记录被清理
And 其数据库记录被删除
And 其原图文件与缩略图文件也被删除
```

### 场景 BDD-05-03：手动删除图片记录时清理关联资源

```gherkin
Given 主面板选中一条图片记录
When 用户按下 Delete
Then 该记录从卡片列表中移除
And 图片元数据与图片资源文件被清理
And 不会删除用户原始来源文件
```

## 7. 非功能需求验收场景（Performance / Reliability）

### 场景 BDD-NFR-01：主面板冷启动加载最近 100 条摘要

```gherkin
Given 数据库中存在 100 条混合历史记录
When 用户首次调出主面板
Then 记录摘要加载时间应小于 80ms
And 面板应先展示可交互骨架或首屏内容
```

### 场景 BDD-NFR-02：常见截图缩略图生成耗时

```gherkin
Given 用户复制一张常见屏幕截图
When 应用后台生成缩略图
Then 缩略图生成时间应小于 50ms
And 不阻塞主面板的方向键导航
```

### 场景 BDD-NFR-03：数据库损坏时应用可恢复启动

```gherkin
Given 当前数据库文件已损坏
When 用户启动应用
Then 应用不会崩溃退出
And 会备份损坏数据库并重建空数据库
And 主面板仍可正常打开
```

## 8. 验收通过标准（DoD）

- `US-01` 至 `US-05` 的 Must 级故事均有 BDD 场景覆盖
- 文本、图片、文件三条主流程至少各有 1 条全链路自动化脚本
- 持久化恢复、容量清理、资源回收至少各有 1 条自动化验收
- 所有场景在 CI 的桌面测试环境中可重复执行
